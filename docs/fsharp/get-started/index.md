Существует несколько способов, чтобы приступить к работе с F #.  У нас есть несколько статей, написанных для предоставления руководства для каждого основного способа.  В следующей таблице можно использовать для помощи в принятии решения.

| ОС | Предпочтение Visual Studio | Предпочтение кода Visual Studio | Предпочтение командной строки |
| -- |------------------------|--------------------------|-----------------------------|-------------------------|
| Windows | [Приступая к работе с Visual Studio](get-started-visual-studio.md) | [Приступая к работе с VSCode и Ionide](get-started-vscode.md) | [Начало работы с .NET Core CLI](get-started-command-line.md) |
| macOS | [Начало работы с VS для Mac](get-started-with-visual-studio-for-mac.md) | [Приступая к работе с VSCode и Ionide](get-started-vscode.md) | [Начало работы с .NET Core CLI](get-started-command-line.md) |
| Linux | Н/Д | [Приступая к работе с VSCode и Ionide](get-started-vscode.md) | [Начало работы с .NET Core CLI](get-started-command-line.md) |

Как правило нет нет специального способа, чтобы приступить к работе которой лучше, чем остальные.  Рекомендуется попробовать все способы использования F # на компьютере для просмотра, что вам понравилось лучше!
