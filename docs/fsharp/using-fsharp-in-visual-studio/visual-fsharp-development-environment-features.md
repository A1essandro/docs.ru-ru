# <a name="visual-f-development-environment-features"></a>Компоненты среды разработки Visual F#

> [!NOTE]
Данная статья не в курсе последних Visual Studio.  Он будет обновляться.

Этот раздел содержит сведения о том, какие возможности Visual Studio 2012 поддерживаются в языке F #.

## <a name="project-features"></a>Функции проекта
В следующей таблице перечислены шаблоны, которые доступны для использования в проектах F #. Сведения о шаблонах проектов и элементов см. в разделе [NIB Создание проектов из шаблонов](https://msdn.microsoft.com/library/7c36d86a-6b79-4480-8228-0f925f1204b2).

|Тип шаблона|Описание|Поддерживаемые шаблоны|
|-------------|-----------|-------------------|
|Шаблоны проектов|Типы проектов, доступные в **новый проект** диалоговое окно.|<ul><li>Приложения F #<br /></li><li>Библиотеки F #<br /></li><li>Учебник по F #<br /></li><li>F # переносимой библиотеки<br /></li><ul/>|
|Шаблоны элементов|Типы файлов, доступные в **Добавление нового элемента** диалоговое окно.|<ul><li>Исходный файл F # (.fs)<br /></li><li>Скрипт F # (.fsx)<br /></li><li>Файл подписи F # (.fsi)<br /></li><li>Файл конфигурации (.config)<br /></li><li>Подключения к базе данных SQL (поставщик типов LINQ to SQL)<br /></li><li>Подключение базы данных SQL (поставщик LINQ to Entities типа)<br /></li><li>Подключение к службе OData (поставщик LINQ типа)<br /></li><li>Подключение к службе WSDL (поставщик типов)<br /></li><li>XML-файла (.xml)<br /></li><li>Текстовый файл<br /></li><ul/>|
Чтобы создать приложение, которое может работать как отдельный исполняемый файл, выберите тип проекта приложения F #. Чтобы создать библиотеку (то есть, управляемая сборка или. DLL-файл) для использования для настольных систем Windows выберите библиотеки F #. Для создания переносимой библиотеки, который можно использовать на всех поддерживаемых платформах нажмите переносимой библиотеки F #. Проекты переносимой библиотеки F # ссылку на версию FSharp.Core.dll, который подходит для создания библиотеки F #, можно использовать с приложениями, работающими на платформах, таких как приложения для магазина Windows, .NET Framework 4.5, Xamarin.iOS и Xamarin.Android.

Дополнительные сведения о шаблонах элементов для доступа к данным см. в разделе [поставщиков типов](../tutorials/type-providers/index.md).

В следующей таблице перечислены функции свойств проекта, поддерживаются и не поддерживаются в языке F #. Дополнительные сведения см. в разделе [Настройка проектов](configuring-projects.md) и [введение в конструктор проектов](https://msdn.microsoft.com/library/898dd854-c98d-430c-ba1b-a913ce3c73d7).

|Параметр проекта|Поддерживается в языке F #?|Примечания|
|---------------|----------------|-----|
|Файлы ресурсов|Да||
|Сборки, отладки и настройки ссылок|Да||
|Настройка для различных версий|Да||
|Значок и манифест|Нет|Доступны через параметры командной строки компилятора.|
|Клиент службы ASP.NET|Нет||
|ClickOnce|Нет|Используйте проект клиента на другом языке .NET Framework, если это применимо.|
|Строгое именование|Нет|Доступны через параметры командной строки компилятора.|
|Публикация сборок и управление версиями|Нет||
|Анализ кода|Нет|Средства анализа кода можно запускать вручную или как часть команды после построения.|
|Безопасность (изменение уровней доверия)|Нет||

## <a name="code-and-text-editor-features"></a>Кода и текстового редактора компонентов
В языке F # поддерживаются следующие функции Visual Studiocode и текстовые редакторы. Общие сведения о редактировании кода в Visual Studio, а также функции текстового редактора см. в разделе [написание кода в редакторе кода и текста](/visualstudio/ide/writing-code-in-the-code-and-text-editor).

|Функция|Описание|Поддерживается в языке F #?|
|-------|-----------|----------------|
|Автоматически комментария|Можно добавить и удалить комментарии разделы кода.|Да|
|Автоматическое форматирование|Переформатирует кода со стандартным отступом и стилем.|Нет|
|Закладки|Позволяет сохранять расположения в редакторе.|Да|
|Изменение отступа|Отступы или изменяет отступ выбранных строк.|Да|
|[Поиск и замена текста](/visualstudio/ide/finding-and-replacing-text)|Позволяет выполнять поиск в файле, проекте или решении, а также позволяет изменять текст.|Да|
|Переход к определению .NET Framework API|Когда курсор наведен на .NET Framework API, показан код, созданный из метаданных платформы .NET Framework.|Нет|
|Переход к определению API-Интерфейс|Когда курсор наведен на определенной сущности программы, который определен, перемещает курсор в место в своем коде, в котором определен сущности.|Да|
|Переход к строке|Позволяет перейти к определенной строке в файле по номеру строки.|Да|
|Панели навигации в верхней части файла|Позволяет переходить в различные расположения в коде, например, имя функции.|Да|
|Структура. В разделе [структурирование](/visualstudio/ide/outlining).|Позволяет сворачивать части кода, чтобы обеспечить более компактное представление.|Да|
|Преобразование в знаки табуляции|Преобразует пробелы в знаки табуляции.|Да|
|Раскраска типов|Отображает определенные имена типов особым цветом.|Да|
|Быстрый поиск. Быстрый поиск, поиск и замена окна в разделе.|Позволяет выполнять поиск в файл или проект.|Да|

## <a name="intellisense-features"></a>Возможности IntelliSense
В следующей таблице приведены функции IntelliSense поддерживаются и не поддерживаются в языке F #. Общие сведения о технологии IntelliSense см. в разделе [использование IntelliSense](/visualstudio/ide/using-intellisense).

|Функция|Описание|Поддерживается в языке F #?|
|-------|-----------|----------------|
|Автоматическая реализация интерфейсов|Создает заглушки кода для методов интерфейса.|Нет|
|Фрагменты кода|Внедряет код из библиотеки общих конструкций кода в разделы.|Нет|
|Завершить слово|Сохраняет типизацию, завершения слов и имен при вводе.|Да|
|Режим предварительного использования|Во включенном состоянии вызывает завершение слов для выбора первого совпадения при вводе вместо ожидания выбора пользователя или нажатия **CTRL + пробел**.|Нет|
|Создание элементов кода|Позволяет создавать код заглушки для различных конструкций.|Нет|
|Список членов|При вводе оператор доступа к членам (.), показаны члены по типу.|Да|
|И директивами Open|Организует пространств имен, используемых **с помощью** операторы в C# или **откройте** директивы в языке F #.|Нет|
|Сведения о параметрах|Показывает полезные сведения о параметрах при вводе вызова функции.|Да|
|Краткие сведения|Отображает полное объявление любого идентификатора в коде.|Да|
Рефакторинг кода F # не поддерживается в Visual Studio 2012.

## <a name="debugging-features"></a>Средства отладки
В следующей таблице перечислены функции, доступные при отладке кода F #. Общие сведения об отладчике Visual Studio см. в разделе [отладки в Visual Studio](https://msdn.microsoft.com/library/sc65sadd.aspx).

|Функция|Описание|Поддерживается в языке F #?|
|-------|-----------|----------------|
|Видимые - окно|Показывает автоматические или временные переменные.|Нет|
|Точки останова|Позволяет приостанавливать выполнение кода в определенных точках во время отладки.|Да|
|Условные точки останова|Включает точки останова, которые тестируют условие, определяющее, следует ли приостановить выполнение.|Да|
|Изменить и продолжить|Для кода позволяет изменять и компилировать при отладке выполняемой программы без остановки и перезапуска отладчика.|Нет|
|Вычислитель выражений|Вычисляет и выполняет код во время выполнения.|Нет, но в C# можно использовать средство оценки выражений, несмотря на то, что необходимо использовать синтаксис C#.|
|Отладка с ведением журнала|Позволяет выполнять шаги с заходом ранее выполняемый код.|Да|
|Локальные - окно|Отображает локально определенные значения и переменные.|Да|
|Выполнение до текущей позиции|Позволяет выполнять код до достижения строки, в которой находится курсор.|Да|
|Шаг с заходом|Позволяет продолжить выполнение и перейти к вызову функции.|Да|
|Шаг с обходом|Позволяет продолжить выполнение в текущем кадре стека и перейти на любой вызов функции.|Да|
