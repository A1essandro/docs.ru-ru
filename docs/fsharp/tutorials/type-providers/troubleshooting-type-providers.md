# <a name="troubleshooting-type-providers"></a>Устранение неполадок поставщиков типов

В этом разделе описаны и представлены возможные решения проблемы, которые можно встретить при использовании поставщиков типов.


## <a name="possible-problems-with-type-providers"></a>Возможные проблемы с помощью поставщиков типов
Если возникла проблема при работе с поставщиками типов, можно просмотреть в нижеследующей таблице приведены наиболее распространенные решения.



|Проблема|Предлагаемые действия|
|-------|-----------------|
|**Изменения схемы**. Тип поставщики работают лучше всего, если стабильна схемы источника данных. Если добавить столбец или таблицу данных или выполните еще одно изменение в эту схему, поставщик типов не распознает эти изменения.|Очистить или перестройте проект. Чтобы очистить проект, выберите **построения**, **Очистить** *ProjectName* в строке меню. Чтобы перестроить проект, выберите **построения**, **Перестроить** *ProjectName* в строке меню. Эти действия сбрасывает все данные состояния типа поставщика и блокируется для подключения к источнику данных и получения сведений о Обновленная схема.|
|**Сбой подключения**. Неверный URL-адрес или строка соединения, сеть не работает или недоступности источника данных или службы.|Для веб-службы или службы OData можно попробовать URL-адреса в Internet Explorer, чтобы проверить правильность URL-адреса и служба доступна. Для строки подключения к базе данных, можно использовать средства подключения данных в **обозревателя серверов** для проверки, является ли строка подключения действительна, и база данных доступна. После восстановления подключения следует очистить или перестройте проект, чтобы поставщик типа будет подключиться к сети.|
|**Учетные данные не является допустимым**. Необходимо иметь допустимые разрешения для данных источника или веб-службы.|Для подключения к SQL имя пользователя и пароль, указанные в файле подключения строку или конфигурации должно быть допустимым для базы данных. Если вы используете проверку подлинности Windows, необходимо иметь доступ к базе данных. Администратор базы данных можно определить разрешения, которые необходимы для доступа для каждой базы данных и каждый элемент в базе данных.<br /><br />Для веб-службы или службы данных необходимо иметь соответствующие учетные данные. Большинство поставщиков типов предоставляют объект DataContext, который содержит свойство учетные данные, которые можно задать с помощью соответствующего имени пользователя и ключ доступа.|
|**Путь не является допустимым**. Недопустимый путь к файлу.|Убедитесь, что путь указан правильно и что файл существует. Кроме того необходимо соответствующим образом Квота любой backlashes в пути или использовать буквальная строка или строка тройных кавычках.|
