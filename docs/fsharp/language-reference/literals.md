## <a name="remarks"></a>Примечания
Строки в Юникоде может содержать явные кодировки, которые можно задать с помощью `\u` следуют 16-разрядное шестнадцатеричное или кодировки UTF-32, можно указать с помощью `\U` следуют 32-разрядных шестнадцатеричный код, который представляет строку в Юникоде суррогатная пара.

Начиная с версии 3.1 языка F #, можно использовать `+` входа для объединения строковых литералов. Можно также использовать побитового или (`|||`) оператор для объединения, перечисления флагов. Например следующий код является допустимым в F # 3.1:

```fsharp
[<Literal>]
let Literal1 = "a" + "b"

[<Literal>]
let FileLocation =   __SOURCE_DIRECTORY__ + "/" + __SOURCE_FILE__

[<Literal>]
let Literal2 = 1 ||| 64

[<Literal>]
let Literal3 = System.IO.FileAccess.Read ||| System.IO.FileAccess.Write
```

Использование других побитовых операторов не разрешена.


## <a name="named-literals"></a>Именованных литералов
Значения, которые должны быть константами могут быть помечены с помощью [литерала](https://msdn.microsoft.com/library/465f36ce-d146-41c0-b425-679c509cd285) атрибута. Этот атрибут приводит к сбою значение будет компилироваться как константа.

В выражениях сопоставления шаблонов идентификаторы которых начинаются с символов нижнего регистра всегда обрабатываются как переменные для привязки, а не как литералы, поэтому следует использовать прописные буквы при определении литералов.

## <a name="integers-in-other-bases"></a>Целые числа в другую систему исчисления

32-разрядных целых чисел со знаком может также быть задан шестнадцатеричные, восьмеричные и двоичные `0x`, `0o` или `0b` префикса соответственно.

```fsharp
let Numbers = (0x9F, 0o77, 0b1010)
// Result: Numbers : int * int * int = (159, 63, 10)
```

## <a name="underscores-in-numeric-literals"></a>Символ подчеркивания в числовые литералы

Начиная с версии 4.1 F #, можно разделить цифр с символа подчеркивания (`_`).

```fsharp
let DeadBeef = 0xDEAD_BEEF

let DeadBeefAsBits = 0b1101_1110_1010_1101_1011_1110_1110_1111

let ExampleSSN = 123_456_7890
```
