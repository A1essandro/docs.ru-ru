## <a name="identity-objects"></a><span data-ttu-id="2225a-109">Объекты Identity</span><span class="sxs-lookup"><span data-stu-id="2225a-109">Identity Objects</span></span>

например Kerberos V5 или пользовательское значение.</span><span class="sxs-lookup"><span data-stu-id="2225a-112">The name can either be a user's name or the name of a Windows account, while the ## <a name="principal-objects"></a><span data-ttu-id="2225a-119">Объекты Principal</span><span class="sxs-lookup"><span data-stu-id="2225a-119">Principal Objects</span></span>  
 <span data-ttu-id="2225a-120">Объект Principal, представляющий контекст безопасности, в котором работает код.</span><span class="sxs-lookup"><span data-stu-id="2225a-120">The principal object represents the security context under which code is running.</span></span> <span data-ttu-id="2225a-121">Приложения, реализующие безопасность на основе ролей, предоставляют права на основании роли, связанной с объектом Principal.</span><span class="sxs-lookup"><span data-stu-id="2225a-121">Applications that implement role-based security grant rights based on the role associated with a principal object.</span></span> <span data-ttu-id="2225a-122">Как и объекты удостоверений, платформа .NET Framework предоставляет <xref:System.Security.Principal.GenericPrincipal> объекта и <xref:System.Security.Principal.WindowsPrincipal> объекта.</span><span class="sxs-lookup"><span data-stu-id="2225a-122">Similar to identity objects, the .NET Framework provides a <xref:System.Security.Principal.GenericPrincipal> object and a <xref:System.Security.Principal.WindowsPrincipal> object.</span></span> <span data-ttu-id="2225a-123">Можно также определить собственные пользовательские классы Principal.</span><span class="sxs-lookup"><span data-stu-id="2225a-123">You can also define your own custom principal classes.</span></span>  
  
 <span data-ttu-id="2225a-124"><xref:System.Security.Principal.IPrincipal> Интерфейс определяет свойство для доступа к связанным **удостоверение** так же как и метод для определения, определяется ли пользователь **основной** элементом объекта конкретной роли.</span><span class="sxs-lookup"><span data-stu-id="2225a-124">The <xref:System.Security.Principal.IPrincipal> interface defines a property for accessing an associated **Identity** object as well as a method for determining whether the user identified by the **Principal** object is a member of a given role.</span></span> <span data-ttu-id="2225a-125">Все классы **Principal** реализуют интерфейс **IPrincipal**, а также любые необходимые дополнительные свойства и методы.</span><span class="sxs-lookup"><span data-stu-id="2225a-125">All **Principal** classes implement the **IPrincipal** interface as well as any additional properties and methods that are necessary.</span></span> <span data-ttu-id="2225a-126">Например, среда CLR предоставляет класс **WindowsPrincipal**, который обеспечивает дополнительные функциональные возможности для сопоставления членства в группе Windows NT или Windows 2000 с ролями.</span><span class="sxs-lookup"><span data-stu-id="2225a-126">For example, the common language runtime provides the **WindowsPrincipal** class, which implements additional functionality for mapping Windows NT or Windows 2000 group membership to roles.</span></span>  
  
 <span data-ttu-id="2225a-127">Объект **основной** объект привязан к контексту вызова (<xref:System.Runtime.Remoting.Messaging.CallContext>) объекта в домене приложения (<xref:System.AppDomain>).</span><span class="sxs-lookup"><span data-stu-id="2225a-127">A **Principal** object is bound to a call context (<xref:System.Runtime.Remoting.Messaging.CallContext>) object within an application domain (<xref:System.AppDomain>).</span></span> <span data-ttu-id="2225a-128">Контекст вызова по умолчанию всегда создается с каждым новым **AppDomain**, поэтому всегда доступен контекст вызова для принятия объекта **Principal** .</span><span class="sxs-lookup"><span data-stu-id="2225a-128">A default call context is always created with each new **AppDomain**, so there is always a call context available to accept the **Principal** object.</span></span> <span data-ttu-id="2225a-129">При создании нового потока для него также создается объект **CallContext**.</span><span class="sxs-lookup"><span data-stu-id="2225a-129">When a new thread is created, a **CallContext** object is also created for the thread.</span></span> <span data-ttu-id="2225a-130">Ссылка на объект **Principal** автоматически копируется из создаваемого потока в объект **CallContext** нового потока.</span><span class="sxs-lookup"><span data-stu-id="2225a-130">The **Principal** object reference is automatically copied from the creating thread to the new thread's **CallContext**.</span></span> <span data-ttu-id="2225a-131">Если среда выполнения не может определить, какой объект **Principal** принадлежит создателю потока, она применяет политику по умолчанию для создания объектов **Principal** и **Identity**.</span><span class="sxs-lookup"><span data-stu-id="2225a-131">If the runtime cannot determine which **Principal** object belongs to the creator of the thread, it follows the default policy for **Principal** and **Identity** object creation.</span></span>  
  
 <span data-ttu-id="2225a-132">Настраиваемая политика для домена приложения определяет правила выбора типа объекта **Principal** для связи с новым доменом приложения.</span><span class="sxs-lookup"><span data-stu-id="2225a-132">A configurable application domain-specific policy defines the rules for deciding what type of **Principal** object to associate with a new application domain.</span></span> <span data-ttu-id="2225a-133">Если позволяет политика безопасности, среда выполнения может создать объекты **Principal** и **Identity**, которые отражают токен операционной системы, связанный с текущим потоком выполнения.</span><span class="sxs-lookup"><span data-stu-id="2225a-133">Where security policy permits, the runtime can create **Principal** and **Identity** objects that reflect the operating system token associated with the current thread of execution.</span></span> <span data-ttu-id="2225a-134">По умолчанию среда выполнения использует объекты **Principal** и **Identity**, представляющие пользователей, не прошедших проверку подлинности.</span><span class="sxs-lookup"><span data-stu-id="2225a-134">By default, the runtime uses **Principal** and **Identity** objects that represent unauthenticated users.</span></span> <span data-ttu-id="2225a-135">Среда выполнения не создаст такие объекты **Principal** и **Identity**, пока код не попытается получить к ним доступ.</span><span class="sxs-lookup"><span data-stu-id="2225a-135">The runtime does not create these default **Principal** and **Identity** objects until the code attempts to access them.</span></span>  
  
 <span data-ttu-id="2225a-136">Доверенный код, создающий домен приложения, может установить политику домена приложения, которая управляет процессом создания объектов по умолчанию **Principal** и **Identity**.</span><span class="sxs-lookup"><span data-stu-id="2225a-136">Trusted code that creates an application domain can set the application domain policy that controls construction of the default **Principal** and **Identity** objects.</span></span> <span data-ttu-id="2225a-137">Такая политика домена приложения применяется ко всем выполняемым потокам в данном домене приложения.</span><span class="sxs-lookup"><span data-stu-id="2225a-137">This application domain-specific policy applies to all execution threads in that application domain.</span></span> <span data-ttu-id="2225a-138">Неуправляемый доверенный хост имеет возможность устанавливать эту политику, но должен иметь управляемый код, который устанавливает политику <xref:System.Security.Permissions.SecurityPermission?displayProperty=nameWithType> для управления политикой домена.</span><span class="sxs-lookup"><span data-stu-id="2225a-138">An unmanaged, trusted host inherently has the ability to set this policy, but managed code that sets this policy must have the <xref:System.Security.Permissions.SecurityPermission?displayProperty=nameWithType> for controlling domain policy.</span></span>  
  
 <span data-ttu-id="2225a-139">При передаче объекта **Principal** через домены приложений, но в рамках одного процесса (и, следовательно, на том же компьютере), инфраструктура удаленного взаимодействия копирует ссылку на объект **Principal**, связанный с контекстом вызывающей функции, в вызываемую функцию.</span><span class="sxs-lookup"><span data-stu-id="2225a-139">When transmitting a **Principal** object across application domains but within the same process (and therefore on the same computer), the remoting infrastructure copies a reference to the **Principal** object associated with the caller's context to the callee's context.</span></span>
