# Базовые классы для реализации абстракций
Строго говоря класс становится базового класса, если другой класс является производным от него. В этом разделе, базовый класс является предназначен главным образом для предоставления общих абстракции или для других классов, чтобы повторно использовать некоторые наследование по умолчанию реализация хотя класс. Базовые классы обычно располагаются в середине иерархии наследования, между абстракцию в корне иерархии и несколько пользовательских реализаций внизу.  
  
 Они служат реализации вспомогательные методы для реализации абстракций. Например, одним из Framework абстракции для упорядоченных наборов элементов является <xref:System.Collections.Generic.IList%601> интерфейса. Реализация <xref:System.Collections.Generic.IList%601> является нетривиальной задачей, и поэтому платформа предоставляет несколько базовых классов, таких как <xref:System.Collections.ObjectModel.Collection%601> и <xref:System.Collections.ObjectModel.KeyedCollection%602>, используемые в качестве вспомогательные методы для реализации пользовательских коллекций.  
  
 Базовые классы, обычно не подходит для использования в качестве абстракции сами по себе, так как они обычно содержат слишком много реализации. Например `Collection<T>` базовый класс содержит множество связанных с тем, что он реализует неуниверсальный реализации `IList` интерфейса \(Чтобы интегрировать неуниверсальных коллекций\) и о том, что он является коллекцией элементов, хранящихся в памяти в одном из его полей.  
  
 Как отмечалось ранее базовые классы могут реализовывать бесценным справки для пользователей, которым необходим для реализации абстракций, но в то же время может быть значительно ответственности. Они добавляют контактной зоны и увеличить глубину иерархии наследования и так что концептуально усложнить платформы. Таким образом базовые классы можно использовать только в том случае, если они предоставляют значительные усовершенствования для пользователей платформы. Их следует избегать, если они предоставляют значение только для разработчиков платформы, в котором вариантов делегирование внутренней реализации вместо наследования от базового класса следует настоятельно рекомендуется.  
  
 **✓ Рассмотрите ВОЗМОЖНОСТЬ** внесения базовых классов абстрактный, даже если они не содержат абстрактных членов. Это явно сообщает пользователям, что класс предназначен только для наследования.  
  
 **✓ Рассмотрите ВОЗМОЖНОСТЬ** размещать базовые классы в отдельном пространстве имен из типов основным сценарием. По определению базовые классы предназначены для сложных расширенных сценариев и поэтому не интересны большинству пользователей.  
  
 **ИЗБЕЖАТЬ X** имен базовых классов с суффиксом «Базовый», если класс предназначен для использования в открытых интерфейсах API.
