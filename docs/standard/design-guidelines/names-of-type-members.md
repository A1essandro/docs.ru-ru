# <a name="names-of-type-members"></a>Имена членов типа
Типы состоят из элементов: методы, свойства, события, конструкторы и поля. В следующих разделах описаны соглашения об именах членов типа.  
  
## <a name="names-of-methods"></a>Имена методов  
 Поскольку методы являются средствами, выполнив действия, согласно правилам разработки необходимо, имена методов глаголы или фразы команды. Приведенных ниже рекомендаций также служит для различения имена методов и введите имен, которые являются фраз существительное или прилагательное.  
  
 **✓ СДЕЛАТЬ** глаголы или фразы глагол имен методов.  
  
```  
public class String {  
    public int CompareTo(...);  
    public string[] Split(...);  
    public string Trim();  
}  
```  
  
## <a name="names-of-properties"></a>Имена свойств  
 В отличие от других членов свойства должны быть даны субстантивное словосочетание или прилагательных имена. Это, так как свойство ссылается на данные и соответствует имени свойства. PascalCasing всегда используется в именах свойств.  
  
 **✓ СДЕЛАТЬ** имя свойства с помощью существительное, субстантивное словосочетание или прилагательное.  
  
 **X не** имеют свойства, которые соответствуют имени методы «Get», как показано в следующем примере:  
  
 `public string TextWriter { get {...} set {...} }`  
 `public string GetTextWriter(int value) { ... }`  
  
 Этот шаблон обычно означает, что свойства должен быть метод.  
  
 **✓ СДЕЛАТЬ** имя свойства коллекции с во множественном числе фразу, описывающие элементы в коллекции, вместо того чтобы использовать существительные фразу, за которым следует «Список» или «Коллекция».  
  
 **СДЕЛАТЬ ✓** именах логических свойств голосами фразы (`CanSeek` вместо `CantSeek`). При необходимости задать префикс логических свойств с «Is», «может» или «Имеет,» но только в том случае, когда он добавляет значение.  
  
 **✓ Попробуйте** присваивать свойству совпадает с именем его тип.  
  
 Например, следующее свойство правильно возвращает и задает значение перечисления с именем `Color`, поэтому свойство называется `Color`:  
  
```  
public enum Color {...}  
public class Control {  
    public Color Color { get {...} set {...} }  
}  
```  
  
## <a name="names-of-events"></a>Имена событий  
 События всегда относятся к один ситуацию, или один возникшей какие-либо действия. Таким образом как и для методов, событий именуются с командами и дать команда используется для указания времени, когда событие вызывается.  
  
 **✓ СДЕЛАТЬ** имя события с командой или командной фразой.  
  
 Примеры включают `Clicked`, `Painting`, `DroppedDown`, и т. д.  
  
 **✓ СДЕЛАТЬ** Присвойте имена событий с понятием до и после, используя отсутствуют, а времен в прошлом.  
  
 Например, закрыть событие, которое возникает перед закрытием окна будет вызываться `Closing`, и может быть вызвана, возникает после закрытия окна `Closed`.  
  
 **X не** использовать «До» или «После» префиксы или postfixes для указания до и после события. Использование присутствует и времен ранее, как описано выше.  
  
 **✓ СДЕЛАТЬ** имя с суффиксом «EventHandler» обработчиков событий (делегатов, используемые в качестве типов событий), как показано в следующем примере:  
  
 `public delegate void ClickedEventHandler(object sender, ClickedEventArgs e);`  
  
 **СДЕЛАТЬ ✓** используйте два параметра с именем `sender` и `e` в обработчиках событий.  
  
 Параметр sender представляет объект, создавший событие. Параметр sender обычно имеет тип `object`, даже если можно применять к более конкретному типу.  
  
 **✓ СДЕЛАТЬ** имя события, классы аргументов с суффиксом «EventArgs».  
  
## <a name="names-of-fields"></a>Имена полей  
 Статические поля открытые и защищенные применяются правила именования полей. Внутренних и закрытых полей не затрагиваются в инструкции и открытый или защищенный экземпляр поля не допускаются [рекомендации по разработке членов](../../../docs/standard/design-guidelines/member.md).  
  
 **✓ СДЕЛАТЬ** использовать PascalCasing в именах полей.  
  
 **✓ СДЕЛАТЬ** имя поля, с помощью существительное, субстантивное словосочетание или прилагательное.  
  
 **X не** использовать префикс для имен полей.  
  
 Например не используйте «g_» или «s_» для указания статического поля.
