# Незапечатанные классы
Запечатанные классы не может наследоваться, и они препятствуют расширяемости. В противоположность этому классы, которые могут наследоваться от называются незапечатанные классы.  
  
 **✓ Рассмотрите ВОЗМОЖНОСТЬ** использование незапечатанные классы с не добавлены виртуальных и защищенных членов, как отличный способ недорогого еще очень полезного расширения в платформу.  
  
 Разработчики часто хотят наследовать незапечатанные классы таким образом, чтобы добавить членов удобства, например пользовательские конструкторы, новые методы или перегрузки метода. Например  `System.Messaging.MessageQueue` не запечатан и таким образом позволяет пользователям создавать пользовательские очереди по умолчанию используется путь к определенной очереди или добавление пользовательских методов, которые упрощают API для конкретных сценариев.  
  
 По умолчанию в языках программирования наиболее являются незапечатанных классов, и это рекомендуемое по умолчанию для большинства классов платформы. Расширяемость, обеспечиваемая незапечатанных типов является очень полезного пользователями framework и довольно легко создать из\-за сравнительно мало тестов затраты, связанные с незапечатанных типов.
