# События и обратные вызовы
Обратные вызовы являются точки расширения, обеспечивающие framework обратный вызов пользовательского кода в делегате. Обычно эти делегаты передаются framework через параметр метода.  
  
 События представляют собой особый случай обратных вызовов, поддерживает удобный и согласованные синтаксис для задания делегата \(обработчик событий\). Кроме того завершение операторов и конструкторы Visual Studio предоставить справку в с помощью интерфейсов API на основе событий. \(См. раздел [Разработка событий](../../../docs/standard/design-guidelines/event.md)\).  
  
 **✓ Рассмотрите ВОЗМОЖНОСТЬ** использования обратных вызовов, чтобы разрешить пользователям пользовательский код мог быть выполнен .NET framework.  
  
 **✓ Рассмотрите ВОЗМОЖНОСТЬ** с помощью событий, чтобы разрешить пользователям настраивать поведение платформы без необходимости понимания объектно ориентированного проектирования.  
  
 **✓ сделать** предпочтут события простых обратных вызовов, так как они знакомы более широкому кругу разработчиков и интегрированы с Visual Studio завершения операторов.  
  
 **ИЗБЕЖАТЬ X** использование обратных вызовов в API, чувствительных к производительности.  
  
 **✓ сделать** использовать новый `Func<...>`, `Action<...>`, или `Expression<...>` типов вместо пользовательских делегатов, при определении интерфейсов API с обратными вызовами.  
  
 `Func<...>` и `Action<...>` представляют универсальных методов\-делегатов.`Expression<...>` представляет определения функций, которые скомпилированы и впоследствии вызван во время выполнения, но может также быть сериализованы и передается удаленному процессу.  
  
 **✓ сделать** измерять и понимать последствия для производительности с помощью `Expression<...>`, вместо `Func<...>` и `Action<...>` делегатов.  
  
 `Expression<...>` типы, в большинстве случаев логически эквивалентно `Func<...>` и `Action<...>` делегатов. Основное различие между ними является, что делегаты предназначены для использования в сценариях локального процесса; выражения, предназначены для случаев, когда он является полезным и невозможно вычислить значение выражения в удаленном процессе или компьютере.  
  
 **✓ сделать** понять, что, вызывая делегата, вы выполняете произвольный код и, возможно, безопасность, правильность и совместимость.
