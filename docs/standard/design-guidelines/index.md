# Рекомендации по проектированию Framework
Этот раздел содержит рекомендации по разработке библиотек, расширяющих и взаимодействовать с .NET Framework. Цель — помочь разработчикам библиотек обеспечить путем предоставления унифицированной модели программирования, не зависящей от языка программирования, который используется для разработки API согласованность и простота использования. Корпорация Майкрософт рекомендует следовать этим рекомендациям по проектированию при разработке классов и компонентов, расширяющих возможности .NET Framework. Несогласованные библиотеки конструктора отрицательно влияет на производительность разработчика и не рекомендует внедрения.  
  
 Рекомендации систематизированы как простые рекомендации, префикс с условиями `Do`, `Consider`, `Avoid`, и `Do not`. Эти инструкции предназначены для облегчения проектировщикам библиотек классов понимания компромиссов между разными решениями. Могут возникнуть ситуации, где хорошо спроектированной библиотеки требуется нарушение этим правилам. Такие случаи должны быть редкими, и важно, что у вас есть ясная и существенная причина для такого решения.  
  
 Эти рекомендации выдержки из книги книги *Framework рекомендации по проектированию: условные обозначения, стили и шаблоны для повторного использования библиотеки .NET, второе издание*, Krzysztof Cwalina, Брэд Абрамс.
