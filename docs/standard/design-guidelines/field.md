# Разработка полей
Принцип инкапсуляции является одним из наиболее важные понятия в объектно ориентированного проектирования. Этот принцип утверждает, что данные, хранящиеся в объекте должен быть доступен только для этого объекта.  
  
 Удобный способ интерпретации принцип будет сказать, что тип должны разрабатываться, чтобы изменения поля этого типа \(изменения имени или типа\) может выполняться без нарушения кода, кроме членов типа. Такая интерпретация немедленно подразумевает все поля должны быть закрытый.  
  
 Мы исключите констант и статических полей только для чтения это строгое ограничение, потому, что такие поля, по определению, почти никогда не требуется изменение.  
  
 **X не** предоставляют открытые или защищенные поля экземпляра.  
  
 Необходимо предоставить свойства для доступа к полям вместо внесения их в открытый или защищенный.  
  
 **✓ сделать** используйте поля констант, которые никогда не изменится.  
  
 Компилятор горит значения таких полей непосредственно в вызывающем коде. Таким образом const значения никогда не могут изменяться без риска нарушения совместимости.  
  
 **✓ сделать** использовать открытые статические `readonly` поля для предварительно определенных экземпляров объекта.  
  
 Если существуют предварительно определенные экземпляры типа, их можно объявите как открытые только для чтения статические поля самого типа.  
  
 **X не** назначайте экземпляры изменяемых типов для `readonly` поля.  
  
 Изменяемый тип — это тип с экземплярами, которые могут быть изменены после их создания. Массивы, большинства коллекций и потоки являются изменяющимися типами, но <xref:System.Int32?displayProperty=fullName>, <xref:System.Uri?displayProperty=fullName>, и <xref:System.String?displayProperty=fullName> все неизменны. Модификатор доступа только для чтения на поля ссылочного типа предотвращает экземпляра, хранящегося в поле замену, но не запрещает данных экземпляра в поле изменения путем вызова участников, изменение экземпляра.
