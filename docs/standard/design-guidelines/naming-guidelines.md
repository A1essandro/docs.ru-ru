# Правила именования
Следующие согласованный набор соглашений об именовании в разработке структуры могут быть основных вклад в удобство использования платформы. Он позволяет использовать многие разработчики широко отдельных проектов framework. Помимо согласованности формы имена элементов framework должны быть понятными и необходимо передать функции каждого элемента.  
  
 Целью этой главы является предоставление согласованный набор соглашений об именовании, создаются имена, имеющие смысл для разработчиков немедленно.  
  
 Несмотря на то, что внедрение эти соглашения об именовании, как рекомендации по разработке обычного кода приведет к появлению более согласованное именование по всему коду, необходимо только, чтобы применить их к API, которые происходят в открытую \(открытые или защищенные типы и члены и явно реализованных интерфейсов\).
