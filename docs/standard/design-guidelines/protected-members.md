# Защищенные члены
Защищенные члены сами по себе не предоставляют все расширения, но могут сделать более мощные расширения через подклассы. Они могут использоваться для предоставления Дополнительные параметры настройки без усложнения основных открытый интерфейс без необходимости.  
  
 Конструкторы Framework нужно Будьте осторожны с защищенные члены, так как имя «protected» можно предоставить ложное чувство защищенности. Любой пользователь может подкласс Незапечатанный класс и доступ к защищенным членам, и таким образом те же защитного кодирования методы, применяемые для открытых членов применяются к защищенным членам.  
  
 **✓ Рассмотрите ВОЗМОЖНОСТЬ** использование защищенных членов для дополнительной настройки.  
  
 **✓ сделать** обрабатывать защищенные члены незапечатанных классов как открытые с целью анализа безопасности, документацию и совместимости.  
  
 Любой пользователь может наследовать от класса и обращаться к защищенным членам.
