# Операторы равенства
В этом разделе обсуждаются перегрузки операторов равенства и ссылается на `operator==` и `operator!=` как операторы равенства.  
  
 **X не** перегружать операторы равенства и отсутствуют в другом.  
  
 **✓ сделать** убедитесь, что <xref:System.Object.Equals%2A?displayProperty=fullName> и операторов равенства имеют точно такой же семантикой и аналогичные характеристики производительности.  
  
 Это часто означает, что `Object.Equals` должен быть переопределен перегруженного операторы равенства.  
  
 **ИЗБЕЖАТЬ X** создание исключений из операторов равенства.  
  
 Например, возвращает значение false, если один из аргументов имеет значение null, а не вызывает `NullReferenceException`.  
  
## Операторы равенства в типах значений  
 **✓ сделать** перегружать операторы равенства для типов значений, если равенство является значимым.  
  
 В большинстве языков программирования, является реализация по умолчанию не `operator==` для типов значений.  
  
## Операторы равенства для ссылочных типов  
 **ИЗБЕЖАТЬ X** перегрузка операторов равенства на изменяемые ссылочные типы.  
  
 Многие языки имеют операторы встроенных равенства для ссылочных типов. Встроенные операторы обычно реализуют равенство ссылок и многие разработчики удивлен, когда поведение по умолчанию изменяется на равенство значений.  
  
 Эта проблема уменьшается для неизменяемого ссылочных типов, поскольку неизменность затруднит Обратите внимание на различие между равенство ссылок и равенство значений.  
  
 **ИЗБЕЖАТЬ X** перегрузка операторов равенства для ссылочных типов, если реализация будет значительно медленнее, чем равенство ссылок.
