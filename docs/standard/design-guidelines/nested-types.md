# Вложенные типы
Вложенный тип — это тип, определенный в области другого типа, который называется включающего типа. Вложенный тип имеет доступ ко всем членам включающего его типа. Например он имеет доступ к закрытым полям, определенных в включающего типа и полей, определенных в всех предков этого типа защищены.  
  
 Как правило вложенные типы следует использовать с осторожностью. Для этого есть несколько причин. Некоторые разработчики не полностью ознакомиться с основными понятиями. Эти разработчики например, возможно, проблемы с синтаксисом объявления переменных вложенных типов. Вложенные типы также очень жестко увязан с их заключенных типов и таким образом не подходят для универсальных типов.  
  
 Вложенные типы лучше всего подходят для моделирования детали реализации, их заключенных типов. Конечному пользователю не придется объявлять переменные вложенного типа и почти никогда не должны явно создать экземпляр вложенные типы. Например перечислитель коллекции может быть вложенным типом коллекции. Перечислители обычно создаются с содержащим их типом, а поскольку многие языки поддерживают оператор foreach, перечислитель переменные редко должны быть объявлены конечным пользователем.  
  
 **✓ сделать** использовать вложенные типы, когда отношения между вложенным типом и его внешним типом таким образом, что желательна семантика доступности членов.  
  
 **X не** используйте открытые вложенные типы как логическое группирование построения; используйте для этого пространства имен.  
  
 **ИЗБЕЖАТЬ X** открытый доступ, вложенные типы. Единственным исключением является, если переменные вложенного типа должны быть объявлены только в редких случаях, таких как работа с подклассами и другими расширенными сценариями.  
  
 **X не** использовать вложенные типы, если тип может ссылаться на за пределами содержащего типа.  
  
 Например передается методу, определенного в классе перечисления не должна определяться как вложенный тип класса.  
  
 **X не** использовать вложенные типы, если они должны создаваться в клиентском коде.  Если тип имеет открытый конструктор, он, вероятно, не должен быть вложенным.  
  
 Если тип может быть создан, видимо, для указания типа имеет место в структуре самостоятельно \(можно создать его, работать с ним и уничтожать без помощи внешнего типа постоянно\) и поэтому не должен быть вложенным. Внутренние типы не должен широко использоваться за пределами внешнего типа без связи ответственности для внешнего типа.  
  
 **X не** определить вложенный тип в качестве члена интерфейса. Многие языки не поддерживают такой конструкции.
