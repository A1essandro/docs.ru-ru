# Имена классов, структур и интерфейсов
Применить правилам именования, выполните для именования общий тип.  
  
 **✓ сделать** имя классы и структуры существительные и субстантивные словосочетания, с помощью PascalCasing.  
  
 Это отличает имена типов из методов, которые именуются с помощью команды фраз.  
  
 **✓ сделать** name интерфейсов прилагательных фраз или иногда с существительные и субстантивные словосочетания.  
  
 Существительные и субстантивные словосочетания должен использоваться редко, и они могут указывать, что тип должен быть абстрактным классом и не интерфейсом.  
  
 **X не** использовать в именах префикс \(например, «C»\).  
  
 **✓ Рассмотрите ВОЗМОЖНОСТЬ** конечное имя производных классов с именем базового класса.  
  
 Это очень удобно для чтения и четко рассказывает о связи. Примеры кода,: `ArgumentOutOfRangeException`, которого представляет собой из `Exception`, и `SerializableAttribute`, который представляет собой из `Attribute`. Тем не менее важно использовать разумно этой рекомендации; например `Button` класс представляет собой из `Control` событий, несмотря на то что `Control` не отображается в его имя.  
  
 **✓ сделать** префикс имени интерфейса буквы I, чтобы указать, что тип является интерфейсом.  
  
 Например `IComponent` \(описательное существительное\), `ICustomAttributeProvider` \(субстантивное словосочетание\) и `IPersistable` \(прилагательное\) являются именами соответствующий интерфейс. Как избежать сокращения в именах других типов.  
  
 **✓ сделать** убедитесь, что имена отличаются только по «I» префикс в имени интерфейса, при определении пары класс — интерфейс, где класс является стандартной реализацией интерфейса.  
  
## Имена параметров универсального типа  
 Универсальные шаблоны были добавлены в .NET Framework 2.0. Функция появился новый тип идентификатора *параметр типа*.  
  
 **✓ сделать** имя описательные имена параметров универсального типа, если имя не полностью интуитивно понятными и описательное имя могут быть.  
  
 **✓ Рассмотрите ВОЗМОЖНОСТЬ** с использованием `T` имени типа параметра для типов с помощью одного параметра типа одной буквы.  
  
```  
public int IComparer<T> { ... }  
public delegate bool Predicate<T>(T item);  
public struct Nullable<T> where T:struct { ... }  
```  
  
 **✓ сделать** префикс описательным именам параметров типа с `T`.  
  
```  
public interface ISessionChannel<TSession> where TSession : ISession{  
    TSession Session { get; }  
}  
```  
  
 **✓ Рассмотрите ВОЗМОЖНОСТЬ** указывать ограничения накладываемые на параметр типа имени параметра.  
  
 Например, ограничение параметра `ISession` может называться `TSession`.

|`System.Delegate`|**✓ сделать** добавить суффикс «EventHandler» имена делегатов, которые используются в событиях.<br /><br /> **✓ сделать** добавляется суффикс «Обратный вызов» имена делегатов, отличные от тех, которые используются в качестве обработчиков событий.<br /><br /> **X не** Добавление суффикса «Delegate» в делегат.|  
|`System.EventArgs`|**✓ сделать** добавляется суффикс «EventArgs».|  
|`System.Enum`|**X не** производными от этого класса; ключевое слово поддерживается в используемом языке вместо; например, в C\# ключевое слово enum.<br /><br /> **X не** добавляется суффикс «Enum» или «Флагом».|

## Именование перечислений  
 Имена типов перечисления \(также называемые перечислений\) в общем должны следовать стандартным правилам именования типа \(PascalCasing, и т. д.\). Однако существуют дополнительные правила, которые применяются к перечислений.  
  
 **✓ сделать** использовать для перечисления имя единственного типа, если его значения являются битовыми полями.  
  
 **✓ сделать** в качестве значений, также называемых перечисление флагов используется имя во множественном числе типа перечисления с битовыми полями.  
  
 **X не** использовать в именах типов enum суффикс «Enum».  
  
 **X не** использовать «Пометить» или «Флаги» суффиксов в перечислении имена типов.  
  
 **X не** использовать префикс в именах значений перечислений \(например, «ad» для перечислений ADO.\), «rtf» для перечислений форматированного текста и т. д.
