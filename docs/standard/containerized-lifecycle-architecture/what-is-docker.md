<span data-ttu-id="e0a93-126">На рисунке 1-3 показано сравнение между виртуальными машинами и Docker контейнеров.</span><span class="sxs-lookup"><span data-stu-id="e0a93-126">Figure 1-3 shows a comparison between VMs and Docker containers.</span></span>

<span data-ttu-id="e0a93-127">Поскольку контейнеры требует гораздо меньше ресурсов (например, они не требуют полной ОС), их можно легко развернуть и их быстрого запуска.</span><span class="sxs-lookup"><span data-stu-id="e0a93-127">Because containers require far fewer resources (for example, they do not need a full OS), they are easy to deploy and they start fast.</span></span> <span data-ttu-id="e0a93-128">Это позволяет иметь более высокой плотности, это означает, что дополнительные службы можно запускать на одинаковые единицы оборудования, тем самым снижая затраты.</span><span class="sxs-lookup"><span data-stu-id="e0a93-128">This makes it possible for you to have higher density, meaning that you can run more services on the same hardware unit, thereby reducing costs.</span></span>

<span data-ttu-id="e0a93-129">В качестве побочного эффекта, работающих на одном ядре вы изоляция меньше чем виртуальные машины.</span><span class="sxs-lookup"><span data-stu-id="e0a93-129">As a side effect of running on the same kernel, you achieve less isolation than VMs.</span></span>

<span data-ttu-id="e0a93-130">Основная цель изображения —, поэтому среда (зависимостей) же в разных развертываниях.</span><span class="sxs-lookup"><span data-stu-id="e0a93-130">The main goal of an image is that it makes the environment (dependencies) the same across different deployments.</span></span> <span data-ttu-id="e0a93-131">Это означает, что можно отлаживать его на компьютере и затем развернуть ее на другой компьютер с той же среде гарантируется.</span><span class="sxs-lookup"><span data-stu-id="e0a93-131">This means that you can debug it on your machine and then deploy it to another machine with the same environment guaranteed.</span></span>

<span data-ttu-id="e0a93-132">Образ контейнера — способ упаковки приложения или службы и развернуть ее в надежном и воспроизводимый режиме.</span><span class="sxs-lookup"><span data-stu-id="e0a93-132">A container image is a way to package an app or service and deploy it in a reliable and reproducible manner.</span></span> <span data-ttu-id="e0a93-133">В этом отношении Docker не только технологии, а также принципы и процесс.</span><span class="sxs-lookup"><span data-stu-id="e0a93-133">In this respect, Docker is not only a technology, it's also a philosophy and a process.</span></span>

<span data-ttu-id="e0a93-134">При использовании Docker, не услышите разработчики сказать, «Он работает на компьютере, почему бы не в рабочей среде?»</span><span class="sxs-lookup"><span data-stu-id="e0a93-134">When using Docker, you will not hear developers say, "It works on my machine, why not in production?"</span></span> <span data-ttu-id="e0a93-135">Можно просто говорят, «Она выполняется в Docker,» поскольку Docker упакованного приложения может выполняться на любой поддерживаемый среде Docker, и он будет работают так, как он был предназначен для на все назначения развертывания (разработки, контроля Качества, промежуточного хранения, производства и т. д.).</span><span class="sxs-lookup"><span data-stu-id="e0a93-135">They can simply say, "It runs on Docker," because the packaged Docker application can be run on any supported Docker environment, and it will run the way it was intended to on all deployment destinations (Dev, QA, staging, production, etc.).</span></span>
