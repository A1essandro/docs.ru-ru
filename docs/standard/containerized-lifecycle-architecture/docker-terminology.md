# <a name="docker-terminology"></a><span data-ttu-id="02961-103">Терминология Docker</span><span class="sxs-lookup"><span data-stu-id="02961-103">Docker terminology</span></span>

<span data-ttu-id="02961-104">В этом разделе приводятся термины и определения, с которыми необходимо ознакомиться с прежде чем исследовать вопросы глубже Docker (для определения, см. в разделе расширенной [Глоссарий](https://docs.docker.com/glossary/) предоставляемые Docker в <https://docs.docker.com/glossary/>:</span><span class="sxs-lookup"><span data-stu-id="02961-104">This section lists terms and definitions with which you should become familiar with before delving deeper into Docker (for further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker at <https://docs.docker.com/glossary/>:</span></span>

-   <span data-ttu-id="02961-105">**Образ контейнера** пакет, содержащий все зависимости и сведения, необходимые для создания контейнера.</span><span class="sxs-lookup"><span data-stu-id="02961-105">**Container image** A package with all of the dependencies and information needed to create a container.</span></span> <span data-ttu-id="02961-106">Образ включает в себя все зависимости (например, платформы), а также развертывание и настройку для использования среды выполнения контейнера.</span><span class="sxs-lookup"><span data-stu-id="02961-106">An image includes all of the dependencies (such as frameworks) plus deployment and configuration to be used by a container runtime.</span></span> <span data-ttu-id="02961-107">Как правило изображение является производным от нескольких базовых образов, слои располагаются один поверх другого для формирования файловой системы контейнера.</span><span class="sxs-lookup"><span data-stu-id="02961-107">Usually, an image derives from multiple base images that are layers stacked one atop the other to form the container's file system.</span></span> <span data-ttu-id="02961-108">Образ остается неизменным после ее создания.</span><span class="sxs-lookup"><span data-stu-id="02961-108">An image is immutable after it has been created.</span></span>

-   <span data-ttu-id="02961-109">**Контейнер** экземпляр образа Docker.</span><span class="sxs-lookup"><span data-stu-id="02961-109">**Container** An instance of a Docker image.</span></span> <span data-ttu-id="02961-110">Контейнер представляет среду выполнения для одного приложения, процесса или службы.</span><span class="sxs-lookup"><span data-stu-id="02961-110">A container represents a runtime for a single application, process, or service.</span></span> <span data-ttu-id="02961-111">Он состоит из содержимого образа Docker, среды выполнения и стандартного набора инструкций.</span><span class="sxs-lookup"><span data-stu-id="02961-111">It consists of the contents of a Docker image, a runtime environment, and a standard set of instructions.</span></span> <span data-ttu-id="02961-112">При масштабировании службы вы создаете несколько экземпляров контейнера из одного образа.</span><span class="sxs-lookup"><span data-stu-id="02961-112">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="02961-113">Или пакетное задание можно создать несколько контейнеров из одного образа, передавая разные параметры каждому экземпляру.</span><span class="sxs-lookup"><span data-stu-id="02961-113">Or, a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

-   <span data-ttu-id="02961-114">**Тег** знак или метка, которую можно применить к изображениям, чтобы можно было идентифицировать разные образы или версии одного образа (в зависимости от номера версии или целевой среде).</span><span class="sxs-lookup"><span data-stu-id="02961-114">**Tag** A mark or label that you can apply to images so that different images or versions of the same image (depending on the version number or the destination environment) can be identified.</span></span>

-   <span data-ttu-id="02961-115">**Dockerfile** текстовый файл, который содержит инструкции по сборке образа Docker.</span><span class="sxs-lookup"><span data-stu-id="02961-115">**Dockerfile** A text file that contains instructions for how to build a Docker image.</span></span>

-   <span data-ttu-id="02961-116">**Построение** действие по созданию образа контейнера на основе сведений и контекста, предоставленных его Dockerfile, а также дополнительные файлы в папке, где создается образ.</span><span class="sxs-lookup"><span data-stu-id="02961-116">**Build** The action of building a container image based on the information and context provided by its Dockerfile as well as additional files in the folder where the image is built.</span></span> <span data-ttu-id="02961-117">Сборка образа с помощью команды docker build.</span><span class="sxs-lookup"><span data-stu-id="02961-117">You can build images by using the Docker docker build command.</span></span>

-   <span data-ttu-id="02961-118">**Репозитории (также называется)** коллекция связанных образов Docker, помеченная тегом, указывающим на версию образа.</span><span class="sxs-lookup"><span data-stu-id="02961-118">**Repository (also known as repo)** A collection of related Docker images labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="02961-119">Некоторые репозитории содержат несколько вариантов одного образа, таких как образ, содержащий изображение, содержащее только среды выполнения (меньший объем), (больше), пакеты SDK и т. д.</span><span class="sxs-lookup"><span data-stu-id="02961-119">Some repositories contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), and so on.</span></span> <span data-ttu-id="02961-120">Эти варианты можно пометить тегами.</span><span class="sxs-lookup"><span data-stu-id="02961-120">Those variants can be marked with tags.</span></span> <span data-ttu-id="02961-121">Один репозиторий может содержать варианты платформ, например образ Linux и образ Windows.</span><span class="sxs-lookup"><span data-stu-id="02961-121">A single repository can contain platform variants, such as a Linux image and a Windows image.</span></span>

-   <span data-ttu-id="02961-122">**Реестр** служба, которая предоставляет доступ к репозиториям.</span><span class="sxs-lookup"><span data-stu-id="02961-122">**Registry** A service that provides access to repositories.</span></span> <span data-ttu-id="02961-123">Реестр по умолчанию для большинства общедоступных образов — [Центр Docker](https://hub.docker.com/) (принадлежащий Docker как организации).</span><span class="sxs-lookup"><span data-stu-id="02961-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="02961-124">Реестр обычно содержит репозитории нескольких команд.</span><span class="sxs-lookup"><span data-stu-id="02961-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="02961-125">Компании часто используют частные реестры для хранения и управления образами, которые они создали.</span><span class="sxs-lookup"><span data-stu-id="02961-125">Companies often have private registries to store and manage images that they've created.</span></span> <span data-ttu-id="02961-126">*Реестр контейнеров Azure* приведен еще один пример.</span><span class="sxs-lookup"><span data-stu-id="02961-126">*Azure Container Registry* is another example.</span></span>

-   <span data-ttu-id="02961-127">**Docker Hub** — общедоступный реестр для загрузки образов и работы с ними.</span><span class="sxs-lookup"><span data-stu-id="02961-127">**Docker Hub** A public registry to upload images and work with them.</span></span> <span data-ttu-id="02961-128">Центр Docker обеспечивает размещение образов Docker и интеграцию с GitHub и Bitbucket, предоставляет общедоступные или частные реестры, триггеры сборки и веб-перехватчики.</span><span class="sxs-lookup"><span data-stu-id="02961-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

-   <span data-ttu-id="02961-129">**Реестр контейнеров Azure** общих ресурсов для работы с образами Docker и его компонентов в Azure.</span><span class="sxs-lookup"><span data-stu-id="02961-129">**Azure Container Registry** A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="02961-130">Он предоставляет реестр, близкий к вашим развертываниям в Azure, так что вы можете контролировать доступ и использовать группы и разрешения в Azure Active Directory.</span><span class="sxs-lookup"><span data-stu-id="02961-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

-   <span data-ttu-id="02961-131">**Доверенный реестр docker (DTR)** — Служба реестра Docker (из Docker), которую можно установить в локальной, чтобы он находится в центре обработки данных и сети организации.</span><span class="sxs-lookup"><span data-stu-id="02961-131">**Docker Trusted Registry (DTR)** A Docker registry service (from Docker) that you can install on-premises so that it resides within the organization's datacenter and network.</span></span> <span data-ttu-id="02961-132">Его удобно использовать для частных образов, которыми необходимо управлять внутри компании.</span><span class="sxs-lookup"><span data-stu-id="02961-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="02961-133">Доверенный реестр Docker входит в Центр данных Docker.</span><span class="sxs-lookup"><span data-stu-id="02961-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="02961-134">Дополнительные сведения см. в статье [ https://docs.docker.com/docker-trusted-registry/overview/ ](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="02961-134">For more information, go to [https://docs.docker.com/docker-trusted-registry/overview/](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

-   <span data-ttu-id="02961-135">**Docker Community Edition (CE)** средства разработки для Windows и Mac OS для создания, запуска и тестирования контейнеров локально.</span><span class="sxs-lookup"><span data-stu-id="02961-135">**Docker Community Edition (CE)** Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="02961-136">Docker CE для Windows предоставляет среды разработки для контейнеров Linux и Windows.</span><span class="sxs-lookup"><span data-stu-id="02961-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="02961-137">Узел Docker в Linux на Windows базируется на [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) виртуальной Машины.</span><span class="sxs-lookup"><span data-stu-id="02961-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) VM.</span></span> <span data-ttu-id="02961-138">Узел для контейнеров Windows базируется непосредственно на Windows.</span><span class="sxs-lookup"><span data-stu-id="02961-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="02961-139">Docker CE для Mac основана на платформе Apple Hypervisor и [гипервизоре xhyve](https://github.com/mist64/xhyve), он предоставляет виртуальная машина узла Docker в Linux на Mac OS X. Docker CE для Windows, а также для Mac заменил Docker Toolbox, который был на базе Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="02961-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host VM on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

-   <span data-ttu-id="02961-140">**Docker Enterprise Edition (EE)** версию средства Docker для Linux и Windows для разработки корпоративного уровня.</span><span class="sxs-lookup"><span data-stu-id="02961-140">**Docker Enterprise Edition (EE)** An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

-   <span data-ttu-id="02961-141">**Compose** средство командной строки и формат файлов YAML с метаданными для определения и запуска многоконтейнерного приложения.</span><span class="sxs-lookup"><span data-stu-id="02961-141">**Compose** A command-line tool and YAML file format with metadata for defining and running multicontainer applications.</span></span> <span data-ttu-id="02961-142">Вы определяете одно приложение на основе нескольких образов с помощью одного или нескольких файлов .yml, которые могут изменять значения в зависимости от среды.</span><span class="sxs-lookup"><span data-stu-id="02961-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="02961-143">После создания определения, можно развернуть всего многоконтейнерного приложения с помощью одной команды (docker-compose up), создает один контейнер на образ на узле Docker.</span><span class="sxs-lookup"><span data-stu-id="02961-143">After you have created the definitions, you can deploy the entire multicontainer application by using a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

-   <span data-ttu-id="02961-144">**Кластер** коллекция узлов Docker, представленная как если бы они были единому виртуальному узлу Docker, чтобы приложение можно масштабировать до нескольких экземпляров служб, распределены по нескольким узлам кластера.</span><span class="sxs-lookup"><span data-stu-id="02961-144">**Cluster** A collection of Docker hosts exposed as if they were a single virtual Docker host so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="02961-145">Кластеры Docker можно создать с помощью Docker Swarm, Mesosphere DC/OS, Kubernetes и Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="02961-145">You can create Docker clusters by using Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="02961-146">(В Docker Swarm кластер называется *рой*.)</span><span class="sxs-lookup"><span data-stu-id="02961-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

-   <span data-ttu-id="02961-147">**Orchestrator** — инструмент, упрощающий управление кластерами и узлами Docker.</span><span class="sxs-lookup"><span data-stu-id="02961-147">**Orchestrator** A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="02961-148">Использование оркестраторов, можно управлять их образы, контейнерами и узлами через интерфейс командной строки или графический пользовательский интерфейс.</span><span class="sxs-lookup"><span data-stu-id="02961-148">Using orchestrators, you can manage their images, containers, and hosts through a CLI or a graphical user interface.</span></span> <span data-ttu-id="02961-149">Вы можете управлять соединениями контейнеров, конфигурациями, балансировкой нагрузки, обнаружением служб, высоким уровнем доступности, конфигурацией узлов Docker и многим другим.</span><span class="sxs-lookup"><span data-stu-id="02961-149">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="02961-150">Оркестратор используется для выполнения, распределения, масштабирования и восстановления рабочих нагрузок в коллекции узлов.</span><span class="sxs-lookup"><span data-stu-id="02961-150">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="02961-151">Как правило, оркестраторы — это те же продукты, которые обеспечивают кластерную инфраструктуру, например Mesosphere DC/OS, Kubernetes, Docker Swarm и Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="02961-151">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
