# <a name="reader-writer-locks"></a><span data-ttu-id="6bbfa-102">Блокировки чтения и записи</span><span class="sxs-lookup"><span data-stu-id="6bbfa-102">Reader-Writer Locks</span></span>
<span data-ttu-id="6bbfa-103">Класс <xref:System.Threading.ReaderWriterLockSlim> позволяет нескольким потокам одновременно считывать данные из одного ресурса, но требует предоставить потоку монопольную блокировку для записи в этот ресурс.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-103">The <xref:System.Threading.ReaderWriterLockSlim> class enables multiple threads to read a resource concurrently, but requires a thread to wait for an exclusive lock in order to write to the resource.</span></span>  
  
 <span data-ttu-id="6bbfa-104">Вы можете применить <xref:System.Threading.ReaderWriterLockSlim> в приложении для координированной синхронизации доступа к общему ресурсу между потоками.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-104">You might use a <xref:System.Threading.ReaderWriterLockSlim> in your application to provide cooperative synchronization among threads that access a shared resource.</span></span> <span data-ttu-id="6bbfa-105">Блокировки выполняются непосредственно в <xref:System.Threading.ReaderWriterLockSlim>.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-105">Locks are taken on the <xref:System.Threading.ReaderWriterLockSlim> itself.</span></span>  
  
 <span data-ttu-id="6bbfa-106">Как и для любого другого механизма синхронизации, нужно гарантировать невозможность обойти блокировку, организованную в <xref:System.Threading.ReaderWriterLockSlim>.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-106">As with any thread synchronization mechanism, you must ensure that no threads bypass the locking that is provided by <xref:System.Threading.ReaderWriterLockSlim>.</span></span> <span data-ttu-id="6bbfa-107">Для этого можно, например, разработать класс, который инкапсулирует общий ресурс.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-107">One way to ensure this is to design a class that encapsulates the shared resource.</span></span> <span data-ttu-id="6bbfa-108">Этот класс будет предоставлять элементы, которые получают доступ к закрытому общему ресурсу и используют закрытый <xref:System.Threading.ReaderWriterLockSlim> для синхронизации.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-108">This class would provide members that access the private shared resource and that use a private <xref:System.Threading.ReaderWriterLockSlim> for synchronization.</span></span> <span data-ttu-id="6bbfa-109">В качестве примера изучите код класса <xref:System.Threading.ReaderWriterLockSlim>.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-109">For an example, see the code example for the <xref:System.Threading.ReaderWriterLockSlim> class.</span></span> <span data-ttu-id="6bbfa-110"><xref:System.Threading.ReaderWriterLockSlim> достаточно эффективен и может применяться для синхронизации отдельных объектов.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-110"><xref:System.Threading.ReaderWriterLockSlim> is efficient enough to be used to synchronize individual objects.</span></span>  
  
 <span data-ttu-id="6bbfa-111">Оптимизируйте приложение так, чтобы снизить продолжительность операций чтения и записи.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-111">Structure your application to minimize the duration of read and write operations.</span></span> <span data-ttu-id="6bbfa-112">Длительные операции записи напрямую ухудшают пропускную способность, поскольку блокировки записи являются взаимоисключающими.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-112">Long write operations affect throughput directly because the write lock is exclusive.</span></span> <span data-ttu-id="6bbfa-113">Длительные операции чтения блокируют потоки, ожидающие записи. Если же появится хотя бы один поток, ожидающий записи, то заблокируются и те потоки, которые запрашивают доступ на чтение.</span><span class="sxs-lookup"><span data-stu-id="6bbfa-113">Long read operations block waiting writers, and if at least one thread is waiting for write access, threads that request read access will be blocked as well.</span></span>
