<span data-ttu-id="8e4e0-105">Библиотеки классов воплощают понятие [общей библиотеки](http://en.wikipedia.org/wiki/Library_%28computing%29#Shared_libraries) в .NET.</span><span class="sxs-lookup"><span data-stu-id="8e4e0-105">Class libraries are the [shared library](http://en.wikipedia.org/wiki/Library_%28computing%29#Shared_libraries) concept for .NET.</span></span> <span data-ttu-id="8e4e0-106">Они позволяют помещать полезные функции в модули, которые могут использоваться разными приложениями.</span><span class="sxs-lookup"><span data-stu-id="8e4e0-106">They enable you to componentize useful functionality into modules that can be used by multiple applications.</span></span> <span data-ttu-id="8e4e0-107">Они также могут использоваться для подключения функций, которые не были нужны или не были известны во время запуска приложения.</span><span class="sxs-lookup"><span data-stu-id="8e4e0-107">They can also be used as a means of loading functionality that is not needed or not known at application startup.</span></span> <span data-ttu-id="8e4e0-108">Библиотеки классов описываются в [формате файла сборки .NET](assembly-format.md).</span><span class="sxs-lookup"><span data-stu-id="8e4e0-108">Class libraries are described using the [.NET Assembly file format](assembly-format.md).</span></span>

*   <span data-ttu-id="8e4e0-111">**Переносимые** библиотеки классов имеют доступ к подмножеству API и могут использоваться приложениями и библиотеками, предназначенными для нескольких платформ.</span><span class="sxs-lookup"><span data-stu-id="8e4e0-111">**Portable** class libraries have access to a subset of APIs, and can be used by apps and libraries that target multiple platforms.</span></span>

## <a name="platform-specific-class-libraries"></a><span data-ttu-id="8e4e0-113">Специфические для платформы библиотеки классов</span><span class="sxs-lookup"><span data-stu-id="8e4e0-113">Platform-specific Class Libraries</span></span>

<span data-ttu-id="8e4e0-114">Специфические для платформы библиотеки привязаны к одной реализации .NET (например, .NET Framework в Windows) и поэтому могут иметь множество зависимостей в конкретной среде выполнения.</span><span class="sxs-lookup"><span data-stu-id="8e4e0-114">Platform-specific libraries are bound to a single .NET implementation (for example, .NET Framework on Windows) and can therefore take significant dependencies on a known execution environment.</span></span> <span data-ttu-id="8e4e0-115">Такая среда предоставляет известный набор API (API-интерфейсы .NET и ОС), а также поддерживает и предоставляет данные о состоянии (например, реестр Windows).</span><span class="sxs-lookup"><span data-stu-id="8e4e0-115">Such an environment will expose a known set of APIs (.NET and OS APIs) and will maintain and expose expected state (for example, Windows registry).</span></span>

<span data-ttu-id="8e4e0-118">Специфические для платформы библиотеки были основным типом библиотек классов в платформе .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="8e4e0-118">Platform-specific libraries have been the primary class library type for the .NET Framework.</span></span> <span data-ttu-id="8e4e0-119">Даже после появления других реализаций .NET специфические для платформы библиотеки остались доминирующим типом библиотек.</span><span class="sxs-lookup"><span data-stu-id="8e4e0-119">Even as other .NET implementations emerged, platform-specific libraries remained the dominant library type.</span></span>
