<span data-ttu-id="96bfb-103"><a name="top"></a> Поиск с возвратом происходит, если шаблон регулярного выражения содержит переменные [квантификаторы](../../../docs/standard/base-types/quantifiers-in-regular-expressions.md) или [конструкции изменения](../../../docs/standard/base-types/alternation-constructs-in-regular-expressions.md), и обработчик регулярных выражений возвращается в предыдущее сохраненное состояние, чтобы продолжить поиск совпадения.</span><span class="sxs-lookup"><span data-stu-id="96bfb-103"><a name="top"></a> Backtracking occurs when a regular expression pattern contains optional [quantifiers](../../../docs/standard/base-types/quantifiers-in-regular-expressions.md) or [alternation constructs](../../../docs/standard/base-types/alternation-constructs-in-regular-expressions.md), and the regular expression engine returns to a previous saved state to continue its search for a match.</span></span> <span data-ttu-id="96bfb-104">В поиске с возвратом заключена сила регулярных выражений. Благодаря ему выражения могут быть мощными и гибкими, а также совпадать со сложными шаблонами.</span><span class="sxs-lookup"><span data-stu-id="96bfb-104">Backtracking is central to the power of regular expressions; it makes it possible for expressions to be powerful and flexible, and to match very complex patterns.</span></span> <span data-ttu-id="96bfb-105">С другой стороны, эти возможности дорого обходятся.</span><span class="sxs-lookup"><span data-stu-id="96bfb-105">At the same time, this power comes at a cost.</span></span> <span data-ttu-id="96bfb-106">Часто именно поиск с возвратом существенно снижает производительность обработчика регулярных выражений.</span><span class="sxs-lookup"><span data-stu-id="96bfb-106">Backtracking is often the single most important factor that affects the performance of the regular expression engine.</span></span> <span data-ttu-id="96bfb-107">К счастью, разработчик может управлять работой обработчика регулярных выражений и тем, как он использует поиск с возвратом.</span><span class="sxs-lookup"><span data-stu-id="96bfb-107">Fortunately, the developer has control over the behavior of the regular expression engine and how it uses backtracking.</span></span> <span data-ttu-id="96bfb-108">В этом разделе описано, как функционирует поиск с возвратом, и как им можно управлять.</span><span class="sxs-lookup"><span data-stu-id="96bfb-108">This topic explains how backtracking works and how it can be controlled.</span></span>

## <a name="linear-comparison-without-backtracking"></a><span data-ttu-id="96bfb-115">Линейное сравнение без поиска с возвратом</span><span class="sxs-lookup"><span data-stu-id="96bfb-115">Linear Comparison Without Backtracking</span></span>  
 <span data-ttu-id="96bfb-116">Если шаблон регулярного выражения не содержит переменных квантификаторов или конструкций изменения, обработчик регулярных выражений работает за линейное время.</span><span class="sxs-lookup"><span data-stu-id="96bfb-116">If a regular expression pattern has no optional quantifiers or alternation constructs, the regular expression engine executes in linear time.</span></span> <span data-ttu-id="96bfb-117">Это значит, что когда обработчик регулярных выражений находит совпадение с первым языковым элементом шаблона в тексте входной строки, он сопоставляет следующий языковой элемент шаблона со следующим символом или группой символов входной строки.</span><span class="sxs-lookup"><span data-stu-id="96bfb-117">That is, after the regular expression engine matches the first language element in the pattern with text in the input string, it tries to match the next language element in the pattern with the next character or group of characters in the input string.</span></span> <span data-ttu-id="96bfb-118">Так продолжается, пока поиск совпадения не заканчивается успешно или неуспешно.</span><span class="sxs-lookup"><span data-stu-id="96bfb-118">This continues until the match either succeeds or fails.</span></span> <span data-ttu-id="96bfb-119">В обоих случаях обработчик регулярных выражений продвигается вперед, обрабатывая по одному символу входной строки.</span><span class="sxs-lookup"><span data-stu-id="96bfb-119">In either case, the regular expression engine advances by one character at a time in the input string.</span></span>
