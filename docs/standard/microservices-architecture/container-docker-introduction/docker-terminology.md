# <a name="docker-terminology"></a><span data-ttu-id="73a9d-103">Терминология Docker</span><span class="sxs-lookup"><span data-stu-id="73a9d-103">Docker terminology</span></span>

<span data-ttu-id="73a9d-104">В этом разделе приводятся термины и определения, необходимые при работе с инструментом Docker.</span><span class="sxs-lookup"><span data-stu-id="73a9d-104">This section lists terms and definitions you should be familiar with before getting deeper into Docker.</span></span> <span data-ttu-id="73a9d-105">С развернутыми определениями можно ознакомиться в [глоссарии](https://docs.docker.com/glossary/) инструмента Docker.</span><span class="sxs-lookup"><span data-stu-id="73a9d-105">For further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker .</span></span>

<span data-ttu-id="73a9d-106">**Образ контейнера** — пакет со всеми зависимостями и сведениями, необходимыми для создания контейнера.</span><span class="sxs-lookup"><span data-stu-id="73a9d-106">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="73a9d-107">Образ включает в себя все зависимости (например, платформы), а также конфигурацию развертывания и выполнения для среды выполнения контейнера.</span><span class="sxs-lookup"><span data-stu-id="73a9d-107">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="73a9d-108">Как правило, образ создается на основе нескольких базовых образов, наслоенных друг на друга и образующих файловую систему контейнера.</span><span class="sxs-lookup"><span data-stu-id="73a9d-108">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="73a9d-109">После создания образ остается неизменным.</span><span class="sxs-lookup"><span data-stu-id="73a9d-109">An image is immutable once it has been created.</span></span>

<span data-ttu-id="73a9d-110">**Контейнер** — экземпляр образа Docker.</span><span class="sxs-lookup"><span data-stu-id="73a9d-110">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="73a9d-111">Контейнер отвечает за выполнение одного приложения, процесса или службы.</span><span class="sxs-lookup"><span data-stu-id="73a9d-111">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="73a9d-112">Он состоит из содержимого образа Docker, среды выполнения и стандартного набора инструкций.</span><span class="sxs-lookup"><span data-stu-id="73a9d-112">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="73a9d-113">При масштабировании службы вы создаете несколько экземпляров контейнера из одного образа.</span><span class="sxs-lookup"><span data-stu-id="73a9d-113">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="73a9d-114">Или пакетное задание может создать несколько контейнеров из одного образа, передавая разные параметры каждому экземпляру.</span><span class="sxs-lookup"><span data-stu-id="73a9d-114">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="73a9d-115">**Тег** — метка для образа, которая позволяет различать разные образы или версии одного образа (в зависимости от номера версии или целевой среды).</span><span class="sxs-lookup"><span data-stu-id="73a9d-115">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="73a9d-116">**Dockerfile** — текстовый файл, содержащий инструкции по сборке образа Docker.</span><span class="sxs-lookup"><span data-stu-id="73a9d-116">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="73a9d-117">**Сборка** — действие по созданию образа контейнера на основе сведений и контекста, предоставленных файлом Dockerfile, а также дополнительных файлов в папке, где создается образ.</span><span class="sxs-lookup"><span data-stu-id="73a9d-117">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="73a9d-118">Сборка образа выполняется с помощью команды docker build.</span><span class="sxs-lookup"><span data-stu-id="73a9d-118">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="73a9d-119">**Репозиторий** — коллекция связанных образов Docker, помеченная тегом, указывающим на версию образа.</span><span class="sxs-lookup"><span data-stu-id="73a9d-119">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="73a9d-120">Некоторые репозитории содержат несколько вариантов одного образа, например образ с пакетом средств разработки (больший объем), образ только со средой выполнения (меньший объем) и т. д. Эти варианты можно пометить тегами.</span><span class="sxs-lookup"><span data-stu-id="73a9d-120">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="73a9d-121">Один репозиторий может содержать варианты платформ, например образ Linux и образ Windows.</span><span class="sxs-lookup"><span data-stu-id="73a9d-121">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="73a9d-122">**Реестр** — служба, предоставляющая доступ к репозиториям.</span><span class="sxs-lookup"><span data-stu-id="73a9d-122">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="73a9d-123">Реестр по умолчанию для большинства общедоступных образов — [Центр Docker](https://hub.docker.com/) (принадлежащий Docker как организации).</span><span class="sxs-lookup"><span data-stu-id="73a9d-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="73a9d-124">Реестр обычно содержит репозитории нескольких команд.</span><span class="sxs-lookup"><span data-stu-id="73a9d-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="73a9d-125">Компании часто используют частные реестры для хранения и контроля созданных образов.</span><span class="sxs-lookup"><span data-stu-id="73a9d-125">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="73a9d-126">Еще один пример — реестр контейнеров Azure.</span><span class="sxs-lookup"><span data-stu-id="73a9d-126">Azure Container Registry is another example.</span></span>

<span data-ttu-id="73a9d-127">**Центр Docker** — общедоступный реестр для загрузки образов и работы с ними.</span><span class="sxs-lookup"><span data-stu-id="73a9d-127">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="73a9d-128">Центр Docker обеспечивает размещение образов Docker и интеграцию с GitHub и Bitbucket, предоставляет общедоступные или частные реестры, триггеры сборки и веб-перехватчики.</span><span class="sxs-lookup"><span data-stu-id="73a9d-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="73a9d-129">**Реестр контейнеров Azure** — общедоступный ресурс для работы с образами Docker и их компонентами в Azure.</span><span class="sxs-lookup"><span data-stu-id="73a9d-129">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="73a9d-130">Он предоставляет реестр, близкий к вашим развертываниям в Azure, так что вы можете контролировать доступ и использовать группы и разрешения в Azure Active Directory.</span><span class="sxs-lookup"><span data-stu-id="73a9d-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="73a9d-131">**Доверенный реестр Docker (DTR)**  — служба реестра Docker (из Docker), которую можно установить локально, чтобы она находилась в центре данных и сети организации.</span><span class="sxs-lookup"><span data-stu-id="73a9d-131">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="73a9d-132">Его удобно использовать для частных образов, которыми необходимо управлять внутри компании.</span><span class="sxs-lookup"><span data-stu-id="73a9d-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="73a9d-133">Доверенный реестр Docker входит в Центр данных Docker.</span><span class="sxs-lookup"><span data-stu-id="73a9d-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="73a9d-134">Дополнительные сведения см. в разделе [Доверенный реестр Docker (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span><span class="sxs-lookup"><span data-stu-id="73a9d-134">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="73a9d-135">**Docker Community Edition (CE)**  — инструменты разработки в среде Windows и macOS для локальной сборки, выполнения и тестирования контейнеров.</span><span class="sxs-lookup"><span data-stu-id="73a9d-135">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="73a9d-136">Docker CE для Windows предоставляет среды разработки для контейнеров Linux и Windows.</span><span class="sxs-lookup"><span data-stu-id="73a9d-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="73a9d-137">Узел Linux Docker на Windows базируется на виртуальной машине [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx).</span><span class="sxs-lookup"><span data-stu-id="73a9d-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="73a9d-138">Узел для контейнеров Windows базируется непосредственно на Windows.</span><span class="sxs-lookup"><span data-stu-id="73a9d-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="73a9d-139">Docker CE для Mac базируется на платформе Apple Hypervisor и [гипервизоре xhyve](https://github.com/mist64/xhyve), который предоставляет узлу Linux Docker виртуальную машину в Mac OS X. Docker CE для Windows и для Mac заменил Docker Toolbox на базе Oracle VirtualBox.</span><span class="sxs-lookup"><span data-stu-id="73a9d-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="73a9d-140">**Docker Enterprise Edition (EE)**  — корпоративная версия инструментов Docker для разработки в среде Windows и Linux.</span><span class="sxs-lookup"><span data-stu-id="73a9d-140">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="73a9d-141">**Compose** — программа командной строки и формат файлов YAML с метаданными для определения и выполнения многоконтейнерных приложений.</span><span class="sxs-lookup"><span data-stu-id="73a9d-141">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="73a9d-142">Вы определяете одно приложение на основе нескольких образов с помощью одного или нескольких файлов .yml, которые могут изменять значения в зависимости от среды.</span><span class="sxs-lookup"><span data-stu-id="73a9d-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="73a9d-143">Создав определения, вы можете развернуть все многоконтейнерное приложение с помощью одной команды (docker-compose up), которая создает один контейнер на образ на узле Docker.</span><span class="sxs-lookup"><span data-stu-id="73a9d-143">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="73a9d-144">**Кластер** — коллекция узлов Docker, представленная в виде единого виртуального узла Docker, чтобы можно было масштабировать приложение в нескольких экземплярах служб, распределенных по нескольким узлам кластера.</span><span class="sxs-lookup"><span data-stu-id="73a9d-144">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="73a9d-145">Кластеры Docker можно создавать с помощью Docker Swarm, Mesosphere DC/OS, Kubernetes и Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="73a9d-145">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="73a9d-146">(В Docker Swarm кластер называется *рой*.)</span><span class="sxs-lookup"><span data-stu-id="73a9d-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

<span data-ttu-id="73a9d-147">**Оркестратор** — инструмент, упрощающий управление кластерами и узлами Docker.</span><span class="sxs-lookup"><span data-stu-id="73a9d-147">**Orchestrator**: A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="73a9d-148">С помощью оркестраторов можно управлять образами, контейнерами и узлами через интерфейс командной строки (CLI) или графический интерфейс.</span><span class="sxs-lookup"><span data-stu-id="73a9d-148">Orchestrators enable you to manage their images, containers, and hosts through a command line interface (CLI) or a graphical UI.</span></span> <span data-ttu-id="73a9d-149">Вы можете управлять соединениями контейнеров, конфигурациями, балансировкой нагрузки, обнаружением служб, высоким уровнем доступности, конфигурацией узлов Docker и многим другим.</span><span class="sxs-lookup"><span data-stu-id="73a9d-149">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="73a9d-150">Оркестратор используется для выполнения, распределения, масштабирования и восстановления рабочих нагрузок в коллекции узлов.</span><span class="sxs-lookup"><span data-stu-id="73a9d-150">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="73a9d-151">Как правило, оркестраторы — это те же продукты, которые обеспечивают кластерную инфраструктуру, например Mesosphere DC/OS, Kubernetes, Docker Swarm и Azure Service Fabric.</span><span class="sxs-lookup"><span data-stu-id="73a9d-151">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
