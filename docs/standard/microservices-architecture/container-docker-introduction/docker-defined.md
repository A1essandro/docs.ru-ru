<span data-ttu-id="13424-116">В Windows или macOS [Docker Community Edition (CE)](https://www.docker.com/community-edition) размещает контейнеры в среде разработки и предоставляет дополнительные средства для разработчиков.</span><span class="sxs-lookup"><span data-stu-id="13424-116">On Windows or macOS, [Docker Community Edition (CE)](https://www.docker.com/community-edition) hosts containers in a development environment and provides additional developer tools.</span></span> <span data-ttu-id="13424-117">[Docker Enterprise Edition (EE)](https://www.docker.com/enterprise-edition) используется ИТ-командами, которые создают, поставляют и запускают крупные критически важные бизнес-приложения.</span><span class="sxs-lookup"><span data-stu-id="13424-117">[Docker Enterprise Edition (EE)](https://www.docker.com/enterprise-edition) is used by IT teams who build, ship, and run large business-critical applications.</span></span> <span data-ttu-id="13424-118">~Оба продукта устанавливают необходимую виртуальную машину (узел Docker) для размещения контейнеров.~</span><span class="sxs-lookup"><span data-stu-id="13424-118">~Both products install the necessary VM (the Docker host) to host the containers.~</span></span> 

<span data-ttu-id="13424-119">[Контейнеры Windows](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview) работают с двумя типами среды выполнения:</span><span class="sxs-lookup"><span data-stu-id="13424-119">[Windows Containers](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview) work with two types of runtimes:</span></span>

-   <span data-ttu-id="13424-120">Контейнеры Windows Server изолируют приложение с помощью технологии изоляции процесса и пространства имен.</span><span class="sxs-lookup"><span data-stu-id="13424-120">Windows Server Containers provide application isolation through process and namespace isolation technology.</span></span> <span data-ttu-id="13424-121">Контейнер Windows Server использует ядро совместно с узлом контейнеров и всеми остальными контейнерами на узле.</span><span class="sxs-lookup"><span data-stu-id="13424-121">A Windows Server Container shares a kernel with the container host and with all containers running on the host.</span></span>

-   <span data-ttu-id="13424-122">Контейнеры Hyper-V увеличивают изоляцию, обеспеченную контейнерами Windows Server, запуская каждый контейнер в оптимизированной виртуальной машине.</span><span class="sxs-lookup"><span data-stu-id="13424-122">Hyper-V Containers expand on the isolation provided by Windows Server Containers by running each container in a highly optimized virtual machine.</span></span> <span data-ttu-id="13424-123">В этой конфигурации ядро узла контейнера не используется совместно с контейнерами Hyper-V, что обеспечивает более высокую изоляцию.</span><span class="sxs-lookup"><span data-stu-id="13424-123">In this configuration, the kernel of the container host is not shared with the Hyper-V Containers, providing better isolation.</span></span> <span data-ttu-id="13424-124">Благодаря контейнерам Hyper-V недоверенные и *опасные приложения с несколькими клиентами* могут выполняться на одном и том же узле.</span><span class="sxs-lookup"><span data-stu-id="13424-124">Hyper-V Containers allow untrusted and *hostile multi-tenant* applications to run on the same host.</span></span> <span data-ttu-id="13424-125">Контейнеры Hyper-V запускаются медленнее и отличаются меньшей плотностью по сравнению с контейнерами Windows Server.</span><span class="sxs-lookup"><span data-stu-id="13424-125">Hyper-V Containers have a bit less efficiency in startup times and density than Windows Server Containers.</span></span>

<span data-ttu-id="13424-126">Образы для этих контейнеров создаются и работают одинаково.</span><span class="sxs-lookup"><span data-stu-id="13424-126">The images for these containers are created and function the same way.</span></span> <span data-ttu-id="13424-127">Отличается способ создания контейнера.</span><span class="sxs-lookup"><span data-stu-id="13424-127">They differ in how the container is created.</span></span> <span data-ttu-id="13424-128">Дополнительные сведения см. в разделе [Контейнеры Hyper-V](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview).</span><span class="sxs-lookup"><span data-stu-id="13424-128">For details, see [Hyper-V Containers](https://msdn.microsoft.com/en-us/virtualization/windowscontainers/about/about_overview).</span></span>
