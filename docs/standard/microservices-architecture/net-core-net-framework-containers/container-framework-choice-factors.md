# <a name="decision-table-net-frameworks-to-use-for-docker"></a>Таблица для принятия решений. Использование платформ .NET для Docker

Ниже приводятся сводные сведения о том, когда следует использовать .NET Framework или .NET Core, а также контейнеры Windows или Linux. Помните, что для контейнеров Linux требуются узлы Docker на основе Linux (виртуальные машины или серверы), а для контейнеров Windows — узлы Docker на основе Windows Server (виртуальные машины или серверы).

На принятие решения влияет ряд особенностей приложения. При принятии решения следует взвесить важность этих особенностей.

> [!IMPORTANT]
> На компьютерах разработки выполняется один узел Docker: Linux или Windows. Все связанные микрослужбы, которые должны запускаться и тестироваться вместе в рамках одного решения, должны выполняться на одной платформе контейнеров.

* В качестве архитектуры приложения выбраны **микрослужбы в контейнерах**.
    - В качестве реализации .NET следует выбрать *.NET Core*.
    - В качестве платформы контейнеров можно выбрать *контейнеры Linux* или *контейнеры Windows*.
* В качестве архитектуры приложения выбрано **монолитное приложение**.
    - В качестве реализации .NET можно выбрать *.NET Core* или *.NET Framework*.
    - Если вы выбрали *.NET Core*, в качестве платформы контейнеров можно выбрать *контейнеры Linux* или *контейнеры Windows*.
    - Если вы выбрали *.NET Framework*, в качестве платформы контейнеров следует выбрать *контейнеры Windows*.
* Вы разрабатываете **новое приложение на основе контейнеров (с нуля)**.
    - В качестве реализации .NET следует выбрать *.NET Core*.
    - В качестве платформы контейнеров можно выбрать *контейнеры Linux* или *контейнеры Windows*.
* Вы **переносите имеющееся (старое) приложение Windows Server в контейнеры**.
    - В качестве реализации .NET следует выбрать *.NET Framework* в соответствии с зависимостью от платформы.
    - В качестве платформы контейнеров следует выбрать *контейнеры Windows* из-за зависимости от .NET Framework.
* Целью разработки приложения является достижение **высочайшей производительности и масштабируемости**.
    - В качестве реализации .NET следует выбрать *.NET Core*.
    - В качестве платформы контейнеров можно выбрать *контейнеры Linux* или *контейнеры Windows*.
* Приложение было создано с помощью **ASP.NET Core**.
    - В качестве реализации .NET следует выбрать *.NET Core*.
    - Вы можете использовать реализацию *.NET Framework*, если имеются другие зависимости от платформы.
    - Если вы выбрали *.NET Core*, в качестве платформы контейнеров можно выбрать *контейнеры Linux* или *контейнеры Windows*.
    - Если вы выбрали *.NET Framework*, в качестве платформы контейнеров следует выбрать *контейнеры Windows*.
* Приложение было создано с помощью **ASP.NET 4 (MVC 5, Web API 2 и Web Forms)**.
    - В качестве реализации .NET следует выбрать *.NET Framework* в соответствии с зависимостью от платформы.
    - В качестве платформы контейнеров следует выбрать *контейнеры Windows* из-за зависимости от .NET Framework.
* В приложении используются **службы SignalR**.
    - В качестве реализации .NET можно выбрать *.NET Framework* или *.NET Core 2.1 (после выпуска) или более поздней версии*.
    - В качестве платформы контейнеров следует выбрать *контейнеры Windows*, если вы выбрали реализацию SignalR в .NET Framework.
    - В качестве платформы контейнеров можно выбрать контейнеры Linux или контейнеры Windows, если вы выбрали реализацию SignalR в .NET Core 2.1 или более поздней версии (после выпуска).  
    - Если **службы SignalR** выполняются в *.NET Core*, можно использовать *контейнеры Linux или Windows*.
* В приложении используются **WCF, WF и другие устаревшие платформы**.
    - В качестве реализации .NET следует выбрать *.NET Framework* или *.NET Core (планируемый будущий выпуск)*.
    - В качестве платформы контейнеров следует выбрать *контейнеры Windows* из-за зависимости от .NET Framework.
* В приложении предусмотрено **использование служб Azure**.
    - В качестве реализации .NET следует выбрать *.NET Framework* или *.NET Core (в конечном итоге все службы Azure будут предоставлять клиентские пакеты SDK для .NET Core)*.
    - В качестве платформы контейнеров следует выбрать *контейнеры Windows*, если вы используете клиентские интерфейсы API платформы .NET Framework.
    - Если вы используете клиентские интерфейсы API, доступные для *.NET Core*, можно выбрать *контейнеры Linux или Windows*.
