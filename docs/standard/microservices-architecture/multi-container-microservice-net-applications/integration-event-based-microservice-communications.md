## <a name="integration-events"></a><span data-ttu-id="61d08-130">События интеграции</span><span class="sxs-lookup"><span data-stu-id="61d08-130">Integration events</span></span>

<span data-ttu-id="61d08-131">События интеграции используются для синхронизации состояния домена между несколькими микрослужбами или внешними системами.</span><span class="sxs-lookup"><span data-stu-id="61d08-131">Integration events are used for bringing domain state in sync across multiple microservices or external systems.</span></span> <span data-ttu-id="61d08-132">Это достигается путем публикации событий интеграции за границы микрослужбы.</span><span class="sxs-lookup"><span data-stu-id="61d08-132">This is done by publishing integration events outside the microservice.</span></span> <span data-ttu-id="61d08-133">При публикации события в нескольких микрослужбах-получателях (в тех микрослужбах, которые подписаны на события интеграции), соответствующий обработчик событий в каждой микрослужбе-получателе обрабатывает событие.</span><span class="sxs-lookup"><span data-stu-id="61d08-133">When an event is published to multiple receiver microservices (to as many microservices as are subscribed to the integration event), the appropriate event handler in each receiver microservice handles the event.</span></span>

### <a name="defining-an-event-bus-interface"></a><span data-ttu-id="61d08-169">Определение интерфейса шины событий</span><span class="sxs-lookup"><span data-stu-id="61d08-169">Defining an event bus interface</span></span>

<span data-ttu-id="61d08-170">Давайте начнем с реализации кода для интерфейса шины событий и возможных реализаций для целей исследования.</span><span class="sxs-lookup"><span data-stu-id="61d08-170">Let’s start with some implementation code for the event bus interface and possible implementations for exploration purposes.</span></span> <span data-ttu-id="61d08-171">Интерфейс должен быть универсальным и простым, как и следующий интерфейс.</span><span class="sxs-lookup"><span data-stu-id="61d08-171">The interface should be generic and straightforward, as in the following interface.</span></span>

```csharp
public interface IEventBus
{
    void Publish(IntegrationEvent @event);

    void Subscribe<T, TH>()
        where T : IntegrationEvent
        where TH : IIntegrationEventHandler<T>;

    void SubscribeDynamic<TH>(string eventName)
        where TH : IDynamicIntegrationEventHandler;

    void UnsubscribeDynamic<TH>(string eventName)
        where TH : IDynamicIntegrationEventHandler;

    void Unsubscribe<T, TH>()
        where TH : IIntegrationEventHandler<T>
        where T : IntegrationEvent;
}
```

<span data-ttu-id="61d08-172">Метод `Publish` достаточно прост.</span><span class="sxs-lookup"><span data-stu-id="61d08-172">The `Publish` method is straightforward.</span></span> <span data-ttu-id="61d08-173">Шина событий передает полученное событие интеграции всем микрослужбам и даже внешним приложениям, которые подписаны на это событие.</span><span class="sxs-lookup"><span data-stu-id="61d08-173">The event bus will broadcast the integration event passed to it to any microservice, or even an external application, subscribed to that event.</span></span> <span data-ttu-id="61d08-174">Этот метод используется микрослужбой, которая публикует событие.</span><span class="sxs-lookup"><span data-stu-id="61d08-174">This method is used by the microservice that is publishing the event.</span></span>

<span data-ttu-id="61d08-175">Методы `Subscribe` (возможны различные реализации в зависимости от аргументов) используются микрослужбами, которые хотят получать события.</span><span class="sxs-lookup"><span data-stu-id="61d08-175">The `Subscribe` methods (you can have several implementations depending on the arguments) are used by the microservices that want to receive events.</span></span> <span data-ttu-id="61d08-176">У этого метода есть два аргумента.</span><span class="sxs-lookup"><span data-stu-id="61d08-176">This method has two arguments.</span></span> <span data-ttu-id="61d08-177">Первый — это событие интеграции, на которое он подписан (`IntegrationEvent`).</span><span class="sxs-lookup"><span data-stu-id="61d08-177">The first is the integration event to subscribe to (`IntegrationEvent`).</span></span> <span data-ttu-id="61d08-178">Второй аргумент — обработчик события интеграции (или метод обратного вызова) с именем `IIntegrationEventHandler<T>`, который будет выполнен, когда микрослужба-получатель получит сообщение об этом событии интеграции.</span><span class="sxs-lookup"><span data-stu-id="61d08-178">The second argument is the integration event handler (or callback method), named `IIntegrationEventHandler<T>`, to be executed when the receiver microservice gets that integration event message.</span></span>
