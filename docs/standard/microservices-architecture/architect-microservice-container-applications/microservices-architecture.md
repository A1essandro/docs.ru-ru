# <a name="microservices-architecture"></a><span data-ttu-id="05a42-104">Архитектура микрослужб</span><span class="sxs-lookup"><span data-stu-id="05a42-104">Microservices architecture</span></span>

<span data-ttu-id="05a42-105">Как названия, архитектура микрослужбами — это подход к созданию серверного приложения как ряд небольших служб.</span><span class="sxs-lookup"><span data-stu-id="05a42-105">As the name implies, a microservices architecture is an approach to building a server application as a set of small services.</span></span> <span data-ttu-id="05a42-106">Каждая служба выполняется в собственном процессе и взаимодействует с другими процессами с помощью протоколов, таких как HTTP или HTTPS, WebSockets, или [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span><span class="sxs-lookup"><span data-stu-id="05a42-106">Each service runs in its own process and communicates with other processes using protocols such as HTTP/HTTPS, WebSockets, or [AMQP](https://en.wikipedia.org/wiki/Advanced_Message_Queuing_Protocol).</span></span> <span data-ttu-id="05a42-107">Каждый микрослужбу реализует конкретный домен начала до конца или возможность бизнеса в определенные границы контекста и каждая должно быть разработано внутри автономно и быть развернут независимо друг от друга.</span><span class="sxs-lookup"><span data-stu-id="05a42-107">Each microservice implements a specific end-to-end domain or business capability within a certain context boundary, and each must be developed autonomously and be deployable independently.</span></span> <span data-ttu-id="05a42-108">Наконец каждый микрослужбу должен принадлежать его модели данных связанных доменах и логику домена (независимости и управления данными, децентрализованной) на основе разные технологии хранения данных (SQL, NoSQL) и языков программирования.</span><span class="sxs-lookup"><span data-stu-id="05a42-108">Finally, each microservice should own its related domain data model and domain logic (sovereignty and decentralized data management) based on different data storage technologies (SQL, NoSQL) and different programming languages.</span></span>

<span data-ttu-id="05a42-109">Выберите размер должен быть микрослужбу?</span><span class="sxs-lookup"><span data-stu-id="05a42-109">What size should a microservice be?</span></span> <span data-ttu-id="05a42-110">При разработке микрослужбу, размер не должен быть важным моментом.</span><span class="sxs-lookup"><span data-stu-id="05a42-110">When developing a microservice, size should not be the important point.</span></span> <span data-ttu-id="05a42-111">Вместо этого следует важно Создание слабо сочетании служб, их необходимо автономность разработки, развертывания и масштабирования для каждой службы.</span><span class="sxs-lookup"><span data-stu-id="05a42-111">Instead, the important point should be to create loosely coupled services so you have autonomy of development, deployment, and scale, for each service.</span></span> <span data-ttu-id="05a42-112">Конечно по определения и проектирование микрослужбами, следует попытаться сделать их как можно меньше, пока у вас слишком много прямой зависимости с другими микрослужбами.</span><span class="sxs-lookup"><span data-stu-id="05a42-112">Of course, when identifying and designing microservices, you should try to make them as small as possible as long as you do not have too many direct dependencies with other microservices.</span></span> <span data-ttu-id="05a42-113">Более важно, чем размер микрослужбу внутренней связности, должны быть установлены и его независимости от других служб.</span><span class="sxs-lookup"><span data-stu-id="05a42-113">More important than the size of the microservice is the internal cohesion it must have and its independence from other services.</span></span>

<span data-ttu-id="05a42-114">Почему архитектура микрослужбами?</span><span class="sxs-lookup"><span data-stu-id="05a42-114">Why a microservices architecture?</span></span> <span data-ttu-id="05a42-115">Иными словами он обеспечивает гибкость долгосрочной.</span><span class="sxs-lookup"><span data-stu-id="05a42-115">In short, it provides long-term agility.</span></span> <span data-ttu-id="05a42-116">Микрослужбами включить повышения удобства в сложных большой и масштабируемых системах, предоставляя возможность создавать приложения на основе многих независимо друг от друга развертываемых служб, каждый из которых содержит детальные и автономных жизненные циклы.</span><span class="sxs-lookup"><span data-stu-id="05a42-116">Microservices enable better maintainability in complex, large, and highly-scalable systems by letting you create applications based on many independently deployable services that each have granular and autonomous lifecycles.</span></span>

<span data-ttu-id="05a42-117">В качестве дополнительного преимущества микрослужбами можно масштабировать независимо друг от друга.</span><span class="sxs-lookup"><span data-stu-id="05a42-117">As an additional benefit, microservices can scale out independently.</span></span> <span data-ttu-id="05a42-118">Вместо одного монолитные приложения, которое необходимо масштабировать как единое целое, вместо этого можно масштабировать конкретного микрослужбами.</span><span class="sxs-lookup"><span data-stu-id="05a42-118">Instead of having a single monolithic application that you must scale out as a unit, you can instead scale out specific microservices.</span></span> <span data-ttu-id="05a42-119">Таким образом, можно масштабировать просто требуется больше обработки питания или сети пропускной способности для поддержки запросу, а не масштабирования других областей приложения, которые не обязательно должны быть масштабирован функциональной области.</span><span class="sxs-lookup"><span data-stu-id="05a42-119">That way, you can scale just the functional area that needs more processing power or network bandwidth to support demand, rather than scaling out other areas of the application that do not need to be scaled.</span></span> <span data-ttu-id="05a42-120">Это означает сократить расходы, так как требуется меньшее число устройств.</span><span class="sxs-lookup"><span data-stu-id="05a42-120">That means cost savings because you need less hardware.</span></span>

<span data-ttu-id="05a42-124">Внедрение таких возможностей детально микрослужбами-приложений включает непрерывной интеграции и непрерывной поставки и рекомендации.</span><span class="sxs-lookup"><span data-stu-id="05a42-124">Architecting fine-grained microservices-based applications enables continuous integration and continuous delivery practices.</span></span> <span data-ttu-id="05a42-125">Она также повышает доставки новых функций в приложение.</span><span class="sxs-lookup"><span data-stu-id="05a42-125">It also accelerates delivery of new functions into the application.</span></span> <span data-ttu-id="05a42-126">Детально настроенные композиции приложений также позволяет выполнять и тестировать микрослужбами в изоляции и их автономно развиваться, сохраняя очистить контракты между ними.</span><span class="sxs-lookup"><span data-stu-id="05a42-126">Fine-grained composition of applications also allows you to run and test microservices in isolation, and to evolve them autonomously while maintaining clear contracts between them.</span></span> <span data-ttu-id="05a42-127">До тех пор, пока не изменять контрактов или интерфейсы, можно изменить внутренней реализации любой микрослужбу или добавлять новые функциональные возможности, не нарушая работу других микрослужбами.</span><span class="sxs-lookup"><span data-stu-id="05a42-127">As long as you do not change the interfaces or contracts, you can change the internal implementation of any microservice or add new functionality without breaking other microservices.</span></span>

<span data-ttu-id="05a42-128">Ниже приведены важные аспекты, чтобы включить успеха в внедрением в производственной среде с системой микрослужбами.</span><span class="sxs-lookup"><span data-stu-id="05a42-128">The following are important aspects to enable success in going into production with a microservices-based system:</span></span>

-   <span data-ttu-id="05a42-129">Проверяет мониторинга и работоспособности служб и инфраструктуры.</span><span class="sxs-lookup"><span data-stu-id="05a42-129">Monitoring and health checks of the services and infrastructure.</span></span>

-   <span data-ttu-id="05a42-130">Масштабируемой инфраструктуры для служб (то есть, облака и orchestrators).</span><span class="sxs-lookup"><span data-stu-id="05a42-130">Scalable infrastructure for the services (that is, cloud and orchestrators).</span></span>

-   <span data-ttu-id="05a42-131">Обеспечение безопасности и реализации на нескольких уровнях: проверка подлинности, авторизации, секреты управления, безопасного взаимодействия, и т. д.</span><span class="sxs-lookup"><span data-stu-id="05a42-131">Security design and implementation at multiple levels: authentication, authorization, secrets management, secure communication, etc.</span></span>

-   <span data-ttu-id="05a42-132">Доставка быстрой, обычно с разными командами, сосредоточившись на различных микрослужбами.</span><span class="sxs-lookup"><span data-stu-id="05a42-132">Rapid application delivery, usually with different teams focusing on different microservices.</span></span>

-   <span data-ttu-id="05a42-133">DevOps и CI/CD методики и инфраструктуры.</span><span class="sxs-lookup"><span data-stu-id="05a42-133">DevOps and CI/CD practices and infrastructure.</span></span>

<span data-ttu-id="05a42-134">Из них только первые три охваченных или представленные в данном руководстве.</span><span class="sxs-lookup"><span data-stu-id="05a42-134">Of these, only the first three are covered or introduced in this guide.</span></span> <span data-ttu-id="05a42-135">Последние два пункта, связанных с жизненным циклом приложения, рассматриваются в дополнительной [помещать в контейнеры Docker жизненного цикла приложения с помощью платформы Майкрософт и средств](https://aka.ms/dockerlifecycleebook) электронных книг.</span><span class="sxs-lookup"><span data-stu-id="05a42-135">The last two points, which are related to application lifecycle, are covered in the additional [Containerized Docker Application Lifecycle with Microsoft Platform and Tools](https://aka.ms/dockerlifecycleebook) e-book.</span></span>
