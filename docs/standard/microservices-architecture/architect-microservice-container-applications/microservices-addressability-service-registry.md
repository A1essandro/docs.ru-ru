<span data-ttu-id="043a5-122">Обратите внимание, что некоторые сходства, между реестр службы и шаблон шлюза API, который помогает решить эту проблему также.</span><span class="sxs-lookup"><span data-stu-id="043a5-122">Note that there is certain overlap between the service registry and the API gateway pattern, which helps solve this problem as well.</span></span> <span data-ttu-id="043a5-123">Например [прокси-сервера службы структуры обратного](https://docs.microsoft.com/azure/service-fabric/service-fabric-reverseproxy) — это тип реализации API шлюз, основанный на службы Fabrice именования и помогает устранить разрешения адреса для внутренних служб.</span><span class="sxs-lookup"><span data-stu-id="043a5-123">For example, the [Service Fabric Reverse Proxy](https://docs.microsoft.com/azure/service-fabric/service-fabric-reverseproxy) is a type of implementation of an API Gateway that is based on the Service Fabrice Naming Service and that helps resolve address resolution to the internal services.</span></span>
