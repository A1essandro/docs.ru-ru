# <a name="service-oriented-architecture"></a><span data-ttu-id="e0435-103">Сервисноориентированная архитектура</span><span class="sxs-lookup"><span data-stu-id="e0435-103">Service-oriented architecture</span></span>

<span data-ttu-id="e0435-104">Термин "сервисноориентированная архитектура" (SOA) перегружен значениями и для разных людей означает разные понятия.</span><span class="sxs-lookup"><span data-stu-id="e0435-104">Service-oriented architecture (SOA) was an overused term and has meant different things to different people.</span></span> <span data-ttu-id="e0435-105">Но в качестве общего знаменателя термин SOA подразумевает структурирование приложения путем разделения его на несколько служб (наиболее часто HTTP-службы), которые можно классифицировать различными способами, например как подсистемы или уровни.</span><span class="sxs-lookup"><span data-stu-id="e0435-105">But as a common denominator, SOA means that you structure your application by decomposing it into multiple services (most commonly as HTTP services) that can be classified as different types like subsystems or tiers.</span></span>

<span data-ttu-id="e0435-106">Теперь эти службы можно развернуть как контейнеры Docker, которые помогают решать проблемы развертывания, так как в образ контейнера включены все зависимости.</span><span class="sxs-lookup"><span data-stu-id="e0435-106">Those services can now be deployed as Docker containers, which solves deployment issues, because all the dependencies are included in the container image.</span></span> <span data-ttu-id="e0435-107">Однако, если вы используете отдельные узлы Docker, то при необходимости масштабирования приложений SOA вы столкнетесь с проблемами масштабируемости и доступности.</span><span class="sxs-lookup"><span data-stu-id="e0435-107">However, when you need to scale up SOA applications, you might have scalability and availability challenges if you're deploying based on single Docker hosts.</span></span> <span data-ttu-id="e0435-108">Именно для решения таких проблем предназначено программное обеспечение для кластеризации Docker, или оркестратор, которое описано в следующих разделах, где мы расскажем о способах развертывания микрослужб.</span><span class="sxs-lookup"><span data-stu-id="e0435-108">This is where Docker clustering software or an orchestrator can help you, as explained in later sections where deployment approaches for microservices are described.</span></span>

<span data-ttu-id="e0435-109">Контейнеры Docker являются полезным (но необязательным) элементом как для традиционных архитектур, ориентированных на службы, так и более сложных архитектур с микрослужбами.</span><span class="sxs-lookup"><span data-stu-id="e0435-109">Docker containers are useful (but not required) for both traditional service-oriented architectures and the more advanced microservices architectures.</span></span>

<span data-ttu-id="e0435-110">Микрослужбы являются производными от архитектуры SOA, но эти архитектуры отличаются друг от друга.</span><span class="sxs-lookup"><span data-stu-id="e0435-110">Microservices derive from SOA, but SOA is different from microservices architecture.</span></span> <span data-ttu-id="e0435-111">Для архитектуры SOA являются типичными такие компоненты, как большие центральные брокеры и центральные оркестраторы на уровне организации, а также [сервисная шина предприятия (ESB)](https://en.wikipedia.org/wiki/Enterprise_service_bus).</span><span class="sxs-lookup"><span data-stu-id="e0435-111">Features like large central brokers, central orchestrators at the organization level, and the [Enterprise Service Bus (ESB)](https://en.wikipedia.org/wiki/Enterprise_service_bus) are typical in SOA.</span></span> <span data-ttu-id="e0435-112">Но для архитектуры микрослужб в большинстве случаев они являются признаком дурного тона.</span><span class="sxs-lookup"><span data-stu-id="e0435-112">But in most cases, these are anti-patterns in the microservice community.</span></span> <span data-ttu-id="e0435-113">Некоторые специалисты даже утверждают, что "архитектура микрослужб — это правильно реализованная архитектура SOA".</span><span class="sxs-lookup"><span data-stu-id="e0435-113">In fact, some people argue that "The microservice architecture is SOA done right."</span></span>

<span data-ttu-id="e0435-114">Это руководство посвящено описанию микрослужб, поскольку подход SOA менее требователен по сравнению с требованиями и техниками архитектуры микрослужб.</span><span class="sxs-lookup"><span data-stu-id="e0435-114">This guide focuses on microservices, because a SOA approach is less prescriptive than the requirements and techniques used in a microservice architecture.</span></span> <span data-ttu-id="e0435-115">Если вы знаете, как создать приложение на основе технологии микрослужб, то вы сможете создать и простое сервисноориентированное приложение.</span><span class="sxs-lookup"><span data-stu-id="e0435-115">If you know how to build a microservice-based application, you also know how to build a simpler service-oriented application.</span></span>
