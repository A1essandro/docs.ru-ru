# <a name="manage-net-core-installations"></a>Управление установками .NET Core

.NET Core позволяет одновременно использовать несколько версий пакета SDK и среды выполнения для удобного управления версиями при создании и запуске приложений .NET Core. Эта установка и автоматический накат версий предоставляют несомненные преимущества, но имеют один выраженный недостаток. По мере установки обновлений пакета SDK для .NET Core и среды выполнения .NET Core предыдущие версии остаются на диске. Несколько установленных версий увеличивают размер используемого места на диске с течением времени. Было выпущено более 50 обновлений пакета SDK для .NET Core. Поэтому в вашей системе может быть установлено несколько версий пакета SDK и среды выполнения, которые можно удалить.

## <a name="safe-to-remove"></a>Можно ли безопасно удалить эти версии?

[Выбор версии .NET Core](selection.md) и совместимость среды выполнения .NET Core для различных обновлений обеспечивает безопасное удаление предыдущих версий. Обновления среды выполнения .NET Core совместимы в пределах основной версии, например 1.x или 2.x. Кроме того, более поздние выпуски пакета SDK для .NET Core обычно позволяют создавать приложения, совместимые с предыдущими версиями среды выполнения.

Как правило, требуется только последняя версия пакета SDK и последняя версия исправлений для сред выполнения, необходимых для вашего приложения. К тем случаям, в которых сохраняются старые версии пакетов SDK или сред выполнения, относится поддержка приложений на основе project.json.  Если у приложения есть конкретные причины, по котором оно должно использовать ранние версии пакета SDK или среды выполнения, вы можете безопасно удалить старые версии.

Методы удаления .NET Core зависят от платформы и немного различаются в зависимости от версии .NET Core. Сведения об удалении версий .NET Core, которые больше не требуются, см. в разделе [Как удалить среду выполнения .NET Core и пакет SDK](remove-runtime-sdk-versions.md) в [документации по .NET Core](../index.md).
