 <span data-ttu-id="5d4d7-233">Арифметические операторы ([+](../../../csharp/language-reference/operators/addition-operator.md), [-](../../../csharp/language-reference/operators/subtraction-operator.md), [\*](../../../csharp/language-reference/operators/multiplication-operator.md), [/](../../../csharp/language-reference/operators/division-operator.md)) могут возвращать результаты, выходящие за пределы диапазона возможных значений для используемого числового типа.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-233">The arithmetic operators ([+](../../../csharp/language-reference/operators/addition-operator.md), [-](../../../csharp/language-reference/operators/subtraction-operator.md), [\*](../../../csharp/language-reference/operators/multiplication-operator.md), [/](../../../csharp/language-reference/operators/division-operator.md)) can produce results that are outside the range of possible values for the numeric type involved.</span></span> <span data-ttu-id="5d4d7-234">Подробнее см. в разделе, посвященном конкретному оператору, однако в общем случае применяются следующие правила.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-234">You should refer to the section on a particular operator for details, but in general:</span></span>  
  
- <span data-ttu-id="5d4d7-235">Переполнение при целочисленных арифметических операциях либо вызывает исключение <xref:System.OverflowException> или отбрасывает старшие биты результата.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-235">Integer arithmetic overflow either throws an <xref:System.OverflowException> or discards the most significant bits of the result.</span></span> <span data-ttu-id="5d4d7-236">Деление целого числа на ноль всегда вызывает исключение <xref:System.DivideByZeroException>.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-236">Integer division by zero always throws a <xref:System.DivideByZeroException>.</span></span>  

   <span data-ttu-id="5d4d7-237">Что именно произойдет при возникновении переполнения для целочисленного значения, зависит от контекста выполнения, который может быть [проверяемым или непроверяемым](../../../csharp/language-reference/keywords/checked-and-unchecked.md).</span><span class="sxs-lookup"><span data-stu-id="5d4d7-237">When integer overflow occurs, what happens depends on the execution context, which can be [checked or unchecked](../../../csharp/language-reference/keywords/checked-and-unchecked.md).</span></span> <span data-ttu-id="5d4d7-238">В проверяемом контексте возникает исключение <xref:System.OverflowException>.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-238">In a checked context, an <xref:System.OverflowException> is thrown.</span></span> <span data-ttu-id="5d4d7-239">В непроверяемом контексте старшие биты результата отбрасываются и выполнение продолжается.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-239">In an unchecked context, the most significant bits of the result are discarded and execution continues.</span></span> <span data-ttu-id="5d4d7-240">Таким образом, C# позволяет сделать выбор между обработкой и игнорированием переполнения.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-240">Thus, C# gives you the choice of handling or ignoring overflow.</span></span> <span data-ttu-id="5d4d7-241">По умолчанию арифметические операции выполняются в *непроверяемом* контексте.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-241">By default, arithmetic operations occur in an *unchecked* context.</span></span> 

   <span data-ttu-id="5d4d7-242">Кроме арифметических операторов, переполнение могут вызвать приведения целочисленного типа к другому целочисленному типу, например приведение [long](../../../csharp/language-reference/keywords/long.md) к [int](../../../csharp/language-reference/keywords/int.md). Для таких ситуаций также применяются проверяемый и непроверяемый контексты.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-242">In addition to the arithmetic operations, integral-type to integral-type casts can cause overflow (such as when you cast a [long](../../../csharp/language-reference/keywords/long.md) to an [int](../../../csharp/language-reference/keywords/int.md)), and are subject to checked or unchecked execution.</span></span> <span data-ttu-id="5d4d7-243">Однако битовые операторы и операторы сдвига никогда не вызывают переполнение.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-243">However, bitwise operators and shift operators never cause overflow.</span></span>  
   
-   <span data-ttu-id="5d4d7-244">Переполнение или деление на ноль при арифметических операциях с плавающей запятой никогда не вызывает исключение, так как типы с плавающей запятой — а поэтому и правила для представления бесконечности и значения NaN (не число) — основаны на стандарте IEEE 754.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-244">Floating-point arithmetic overflow or division by zero never throws an exception, because floating-point types are based on IEEE 754 and so have provisions for representing infinity and NaN (Not a Number).</span></span>  
  
-   <span data-ttu-id="5d4d7-245">Переполнение при [десятичной](../../../csharp/language-reference/keywords/decimal.md) арифметической операции всегда вызывает исключение <xref:System.OverflowException>.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-245">[Decimal](../../../csharp/language-reference/keywords/decimal.md) arithmetic overflow always throws an <xref:System.OverflowException>.</span></span> <span data-ttu-id="5d4d7-246">Деление десятичного числа на ноль всегда вызывает исключение <xref:System.DivideByZeroException>.</span><span class="sxs-lookup"><span data-stu-id="5d4d7-246">Decimal division by zero always throws a <xref:System.DivideByZeroException>.</span></span>
