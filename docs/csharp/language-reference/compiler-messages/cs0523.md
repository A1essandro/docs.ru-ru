<span data-ttu-id="ea6c6-103">Член структуры "поле структуры2" типа "структура1" приводит к циклу в этом макете структуры</span><span class="sxs-lookup"><span data-stu-id="ea6c6-103">Struct member 'struct2 field' of type 'struct1' causes a cycle in the struct layout</span></span>  
  
 <span data-ttu-id="ea6c6-104">Определения двух структур содержат рекурсивные ссылки.</span><span class="sxs-lookup"><span data-stu-id="ea6c6-104">The definitions of two structs include recursive references.</span></span> <span data-ttu-id="ea6c6-105">Измените определения [структур](../../../csharp/language-reference/keywords/struct.md) таким образом, чтобы в каждом из них не задавалось определение самой структуры на основе другой.</span><span class="sxs-lookup"><span data-stu-id="ea6c6-105">Change the [struct](../../../csharp/language-reference/keywords/struct.md) definitions such that each does not define itself on the other.</span></span> <span data-ttu-id="ea6c6-106">Это ограничение относится только к структурам, поскольку они являются типами значений.</span><span class="sxs-lookup"><span data-stu-id="ea6c6-106">This limitation applies only to structs, since structs are value types.</span></span> <span data-ttu-id="ea6c6-107">При использовании рекурсивных ссылок объявите типы в качестве классов.</span><span class="sxs-lookup"><span data-stu-id="ea6c6-107">If using recursive references, declare your types as classes.</span></span>  
  
 <span data-ttu-id="ea6c6-108">В следующем примере возникает ошибка CS0523:</span><span class="sxs-lookup"><span data-stu-id="ea6c6-108">The following sample generates CS0523:</span></span>  
  
```csharp

{  
   public RecursiveLayoutStruct2 field;  
}

{  
   public RecursiveLayoutStruct1 field;   // CS0523  
}  
```
