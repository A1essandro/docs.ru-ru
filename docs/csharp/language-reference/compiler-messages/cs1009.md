# <a name="compiler-error-cs1009"></a><span data-ttu-id="5744d-102">Ошибка компилятора CS1009</span><span class="sxs-lookup"><span data-stu-id="5744d-102">Compiler Error CS1009</span></span>
<span data-ttu-id="5744d-103">Нераспознанная escape-последовательность</span><span class="sxs-lookup"><span data-stu-id="5744d-103">Unrecognized escape sequence</span></span>  
  
 <span data-ttu-id="5744d-104">Непредвиденный символ после обратной косой черты (\\) в [строке](../../../csharp/language-reference/keywords/string.md).</span><span class="sxs-lookup"><span data-stu-id="5744d-104">An unexpected character follows a backslash (\\) in a [string](../../../csharp/language-reference/keywords/string.md).</span></span> <span data-ttu-id="5744d-105">Компилятор ожидает один из допустимых escape-символов.</span><span class="sxs-lookup"><span data-stu-id="5744d-105">The compiler expects one of the valid escape characters.</span></span> <span data-ttu-id="5744d-106">Дополнительные сведения см. в разделе [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span><span class="sxs-lookup"><span data-stu-id="5744d-106">For more information, see [Character Escapes](../../../standard/base-types/character-escapes-in-regular-expressions.md).</span></span>  
  
 <span data-ttu-id="5744d-107">Следующий пример приводит к возникновению ошибки CS1009.</span><span class="sxs-lookup"><span data-stu-id="5744d-107">The following sample generates CS1009.</span></span>  
  
```csharp  
// CS1009-a.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string a = "\m";     
      // Try the following line instead.  
      // string a = "\t";  
   }  
}  
```  
  
 <span data-ttu-id="5744d-108">Наиболее распространенной причиной этой ошибки является использование обратной косой черты в имени файла, как показано в следующем примере.</span><span class="sxs-lookup"><span data-stu-id="5744d-108">A common cause of this error is using the backslash character in a file name, as the following example shows.</span></span>  
  
```csharp  
string filename = "c:\myFolder\myFile.txt";  
```  
  
 <span data-ttu-id="5744d-109">Для устранения этой ошибки воспользуйтесь оператором "\\\\" или строковым литералом @-quoted, как показано в следующем примере.</span><span class="sxs-lookup"><span data-stu-id="5744d-109">To resolve this error, use "\\\\" or the @-quoted string literal, as the following example shows.</span></span>  
  
```csharp  
// CS1009-b.cs  
class MyClass  
{  
   static void Main()  
   {  
      // The following line causes CS1009.  
      string filename = "c:\myFolder\myFile.txt";     
      // Try one of the following lines instead.  
      // string filename = "c:\\myFolder\\myFile.txt";  
      // string filename = @"c:\myFolder\myFile.txt";  
   }  
}  
```
