# <a name="compiler-warning-level-1-cs1956"></a><span data-ttu-id="5e112-102">Предупреждение компилятора (уровень 1) CS1956</span><span class="sxs-lookup"><span data-stu-id="5e112-102">Compiler Warning (level 1) CS1956</span></span>
<span data-ttu-id="5e112-103">Член "name" реализует имя члена интерфейса "name" в типе "type".</span><span class="sxs-lookup"><span data-stu-id="5e112-103">Member 'name' implements interface member 'name' in type 'type'.</span></span> <span data-ttu-id="5e112-104">Во время выполнения для члена интерфейса возникает несколько соответствий.</span><span class="sxs-lookup"><span data-stu-id="5e112-104">There are multiple matches for the interface member at run-time.</span></span> <span data-ttu-id="5e112-105">Реализация зависит от того, какой метод будет вызван.</span><span class="sxs-lookup"><span data-stu-id="5e112-105">It is implementation dependent which method will be called.</span></span>  
  
 <span data-ttu-id="5e112-106">Это предупреждение может возникать, если два метода интерфейса отличаются только тем, как помечен определенный параметр: `ref` или `out`.</span><span class="sxs-lookup"><span data-stu-id="5e112-106">This warning can be generated when two interface methods are differentiated only by whether a particular parameter is marked with `ref` or with `out`.</span></span> <span data-ttu-id="5e112-107">Рекомендуется изменить код, чтобы предотвратить появление этого предупреждения, поскольку нельзя однозначно определить, какой метод будет вызван во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="5e112-107">It is best to change your code to avoid this warning because it is not obvious or guaranteed which method is called at runtime.</span></span>  
  
 <span data-ttu-id="5e112-108">Хотя язык C# различает параметры с модификаторами `out` and `ref`, в среде CLR они считаются одинаковыми.</span><span class="sxs-lookup"><span data-stu-id="5e112-108">Although C# distinguishes between `out` and `ref`, the CLR sees them as the same.</span></span> <span data-ttu-id="5e112-109">CLR выбирает метод реализации случайным образом.</span><span class="sxs-lookup"><span data-stu-id="5e112-109">When deciding which method implements the interface, the CLR just picks one.</span></span>  
  
### <a name="to-avoid-this-warning"></a><span data-ttu-id="5e112-110">Устранение этого предупреждения</span><span class="sxs-lookup"><span data-stu-id="5e112-110">To avoid this warning</span></span>

## <a name="example"></a><span data-ttu-id="5e112-113">Пример</span><span class="sxs-lookup"><span data-stu-id="5e112-113">Example</span></span>  
 <span data-ttu-id="5e112-114">В следующем коде создается предупреждение CS1956, поскольку два метода `Test` класса `Base` отличаются только модификаторами `ref`/`out` первого параметра.</span><span class="sxs-lookup"><span data-stu-id="5e112-114">The following code generates CS1956 because the two `Test` methods in `Base` differ only by the `ref`/`out` modifier on the first parameter.</span></span>  
  
```csharp  
// cs1956.cs  
class Base<T, S>  
{  
    // This is the method that should be called.  
    public virtual int Test(out T x) // CS1956  
    {  
        x = default(T);  
        return 0;  
    }  
  
    // This is the "last" method and is the one that ends up being called  
    public virtual int Test(ref S x)  
    {  
        return 1;  
    }  
}  
  
interface IFace  
{  
    int Test(out int x);  
}  
  
class Derived : Base<int, int>, IFace  
{  
    static int Main()  
    {  
        IFace x = new Derived();  
        int y;  
        return x.Test(out y);  
    }  
}  
```
