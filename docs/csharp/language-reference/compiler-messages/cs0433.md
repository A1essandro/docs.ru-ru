 <span data-ttu-id="a0d9a-104">Две разных сборки, на которые ссылается ваше приложение, содержат одинаковые пространство имен и тип, в результате чего возникает неоднозначность.</span><span class="sxs-lookup"><span data-stu-id="a0d9a-104">Two different assemblies referenced in your application contain the same namespace and type, which produces ambiguity.</span></span>

## <a name="example"></a><span data-ttu-id="a0d9a-106">Пример</span><span class="sxs-lookup"><span data-stu-id="a0d9a-106">Example</span></span>

## <a name="example"></a><span data-ttu-id="a0d9a-110">Пример</span><span class="sxs-lookup"><span data-stu-id="a0d9a-110">Example</span></span>  
 <span data-ttu-id="a0d9a-111">В следующем примере возникает ошибка CS0433.</span><span class="sxs-lookup"><span data-stu-id="a0d9a-111">The following example generates CS0433.</span></span>  
  
```csharp  
// CS0433_3.cs  
// compile with: /reference:cs0433_1.dll /reference:cs0433_2.dll  
using TypeBindConflicts;  
public class Test   
{  
   public static void Main()   
   {  
      AggPubImpAggPubImp n6 = new AggPubImpAggPubImp();   // CS0433  
   }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="a0d9a-112">Пример</span><span class="sxs-lookup"><span data-stu-id="a0d9a-112">Example</span></span>  
 <span data-ttu-id="a0d9a-113">В следующем примере показано, как можно использовать функцию псевдонима параметра компилятора **/reference** для устранения возникшей ошибки CS0433.</span><span class="sxs-lookup"><span data-stu-id="a0d9a-113">The following example shows how you can use the alias feature of the **/reference** compiler option to resolve this CS0433 error.</span></span>

   }
