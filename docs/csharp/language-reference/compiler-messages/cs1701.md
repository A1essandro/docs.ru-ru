<span data-ttu-id="f3738-103">Если ссылка на сборку "Имя сборки 1" соответствует "Имя сборки 2", может понадобиться предоставить политику времени выполнения</span><span class="sxs-lookup"><span data-stu-id="f3738-103">Assuming assembly reference "Assembly Name #1" matches "Assembly Name #2", you may need to supply runtime policy</span></span>  
  
 <span data-ttu-id="f3738-104">Две сборки имеют разные номера выпуска и (или) версии.</span><span class="sxs-lookup"><span data-stu-id="f3738-104">The two assemblies differ in release and/or version number.</span></span> <span data-ttu-id="f3738-105">Для унификации необходимо указать директивы в CONFIG-файле приложения и предоставить правильное строгое имя сборки, как показано в следующем примере кода.</span><span class="sxs-lookup"><span data-stu-id="f3738-105">For unification to occur, you must specify directives in the application's .config file, and you must provide the correct strong name of an assembly, as demonstrated in the following example code.</span></span>  
  
## <a name="example"></a><span data-ttu-id="f3738-106">Пример</span><span class="sxs-lookup"><span data-stu-id="f3738-106">Example</span></span>  
 <span data-ttu-id="f3738-107">В следующем примере с несколькими файлами ссылка на сборку задается с использованием двух разных внешних псевдонимов.</span><span class="sxs-lookup"><span data-stu-id="f3738-107">The following multifile sample references an assembly using two different external aliases.</span></span> <span data-ttu-id="f3738-108">В первом примере выполняется построение старой версии кода, в результате чего создается сборка CS1701_d.</span><span class="sxs-lookup"><span data-stu-id="f3738-108">This first sample builds the older version of the code that creates assembly CS1701_d.</span></span>  
  
```csharp

// compile with: /target:library /out:cs1701_d.dll /keyfile:mykey.snk  
using System.Reflection;  
[assembly:AssemblyVersion("1.0")]  
public class A {  
   public void M1() {}  
}  
  
public class C1 {}  
```  
  
## <a name="example"></a><span data-ttu-id="f3738-109">Пример</span><span class="sxs-lookup"><span data-stu-id="f3738-109">Example</span></span>  
 <span data-ttu-id="f3738-110">Этот код создает более новую версию сборки CS1701_d.</span><span class="sxs-lookup"><span data-stu-id="f3738-110">This is the code that creates the newer version of assembly CS1701_d.</span></span> <span data-ttu-id="f3738-111">Обратите внимание, что компиляция осуществляется в каталог, отличный от каталога старой версии, поскольку выходные файлы имеют одинаковые имена.</span><span class="sxs-lookup"><span data-stu-id="f3738-111">Note that it compiles into a different directory than the older version, necessary since the output files have the same names.</span></span>

```csharp  
using System.Reflection;  
[assembly:AssemblyVersion("2.0")]  
public class A {  
   public void M2() {}  
   public void M1() {}  
}  
  
public class C2 {}  
public class C1 {}  
```  
  
## <a name="example"></a><span data-ttu-id="f3738-112">Пример</span><span class="sxs-lookup"><span data-stu-id="f3738-112">Example</span></span>  
 <span data-ttu-id="f3738-113">В этом примере задаются внешние псевдонимы A1 и A2.</span><span class="sxs-lookup"><span data-stu-id="f3738-113">This sample sets up the external aliases A1 and A2.</span></span>  
  
```csharp  
// CS1701_c.cs  
// compile with: /target:library /reference:A2=c:\\cs1701_d.dll /reference:A1=cs1701_d.dll  
  
extern alias A1;  
extern alias A2;  
// using System;  
using a1 = A1::A;  
using a2 = A2::A;  
  
public class Ref {  
   public static a1 A1() { return new a1(); }  
   public static a2 A2() { return new a2(); }  
  
   public static A1::C1 M1() { return new A1::C1(); }  
   public static A2::C2 M2() { return new A2::C2(); }  
}  
```
