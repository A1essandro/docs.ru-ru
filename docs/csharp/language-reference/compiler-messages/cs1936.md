 <span data-ttu-id="53d9c-105">Для отправки запроса к исходному типу этот тип должен реализовывать стандартные методы операторов запросов, которые вызываются в запросе.</span><span class="sxs-lookup"><span data-stu-id="53d9c-105">In order to query a source type, that type must implement the standard query operator methods that you are invoking in the query.</span></span> <span data-ttu-id="53d9c-106">Реализация может иметь вид членов класса или методов расширения, которые вводятся в область действия с помощью соответствующей директивы `using`.</span><span class="sxs-lookup"><span data-stu-id="53d9c-106">The implementation can be either in the form of class members or extension methods that are brought into scope with the appropriate `using` directive.</span></span>

## <a name="example"></a><span data-ttu-id="53d9c-110">Пример</span><span class="sxs-lookup"><span data-stu-id="53d9c-110">Example</span></span>  
 <span data-ttu-id="53d9c-111">Следующий пример приводит к возникновению ошибки CS1936.</span><span class="sxs-lookup"><span data-stu-id="53d9c-111">The following example produces CS1936:</span></span>  
  
```csharp  
// cs1936.cs  
using System.Collections;  
using System.Linq;  
class Test  
{  
    static int Main()  
    {  
        object obj;  
        IEnumerable e = from x in obj // CS1936  
                        select x;  
        return 0;  
    }  
}  
```  
  
 <span data-ttu-id="53d9c-112">Эта ошибка обычно возникает, если случайно запрашивается объект какого-то типа вместо коллекции таких объектов.</span><span class="sxs-lookup"><span data-stu-id="53d9c-112">This error typically occurs when you accidentally try to query an object of some type instead of a collection of those objects.</span></span>
