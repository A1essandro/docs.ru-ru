<span data-ttu-id="55367-103">Метод доступа "метод доступа" не может реализовать член интерфейса "член" для типа "тип".</span><span class="sxs-lookup"><span data-stu-id="55367-103">Accessor 'accessor' cannot implement interface member 'member' for type 'type'.</span></span> <span data-ttu-id="55367-104">Используйте явную реализацию интерфейса.</span><span class="sxs-lookup"><span data-stu-id="55367-104">Use an explicit interface implementation.</span></span>  
  
 <span data-ttu-id="55367-105">Предположение: эта ошибка может возникать при реализации интерфейса, который содержит имена методов, конфликтующие с автоматически создаваемыми для свойства или события методами.</span><span class="sxs-lookup"><span data-stu-id="55367-105">Suggested: This error can occur when implementing an interface that contains method names which conflict with the auto-generated methods associated with a property or event.</span></span> <span data-ttu-id="55367-106">Методы get и set для свойств создаются с именами get_property и set_property. Методам add и remove для событий при создании присваиваются имена add_event и remove_event.</span><span class="sxs-lookup"><span data-stu-id="55367-106">The get/set methods for properties are generated as get_property and set_property, and the add/remove methods for events are generated as add_event and remove_event.</span></span> <span data-ttu-id="55367-107">Если интерфейс содержит любой из этих методов, возникает конфликт.</span><span class="sxs-lookup"><span data-stu-id="55367-107">If an interface contains either of these methods, a conflict occurs.</span></span> <span data-ttu-id="55367-108">Чтобы избежать этой ошибки, реализуйте методы с помощью явной реализации интерфейса.</span><span class="sxs-lookup"><span data-stu-id="55367-108">To avoid this error, implement the methods using an explicit interface implementation.</span></span> <span data-ttu-id="55367-109">Для этого задайте функцию следующим образом:</span><span class="sxs-lookup"><span data-stu-id="55367-109">To do this, specify the function as:</span></span>

```csharp  
Interface.get_property() { /* */ }

```  
  
## <a name="example"></a><span data-ttu-id="55367-110">Пример</span><span class="sxs-lookup"><span data-stu-id="55367-110">Example</span></span>  
 <span data-ttu-id="55367-111">В следующем примере возникает ошибка CS0686:</span><span class="sxs-lookup"><span data-stu-id="55367-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="55367-112">Пример</span><span class="sxs-lookup"><span data-stu-id="55367-112">Example</span></span>  
 <span data-ttu-id="55367-113">Эта ошибка также может возникнуть при объявлении событий.</span><span class="sxs-lookup"><span data-stu-id="55367-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="55367-114">Конструктор события автоматически создает методы `add_``event` и `remove_``event`, которые могут конфликтовать с одноименными методами интерфейса, как показано в следующем примере:</span><span class="sxs-lookup"><span data-stu-id="55367-114">The event construct automatically generates the `add_``event` and `remove_``event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
