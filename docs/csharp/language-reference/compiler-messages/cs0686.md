## <a name="example"></a><span data-ttu-id="10148-110">Пример</span><span class="sxs-lookup"><span data-stu-id="10148-110">Example</span></span>  
 <span data-ttu-id="10148-111">В следующем примере возникает ошибка CS0686:</span><span class="sxs-lookup"><span data-stu-id="10148-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="10148-112">Пример</span><span class="sxs-lookup"><span data-stu-id="10148-112">Example</span></span>  
 <span data-ttu-id="10148-113">Эта ошибка также может возникнуть при объявлении событий.</span><span class="sxs-lookup"><span data-stu-id="10148-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="10148-114">Конструктор события автоматически создает методы `add_``event` и `remove_``event`, которые могут конфликтовать с одноименными методами интерфейса, как показано в следующем примере:</span><span class="sxs-lookup"><span data-stu-id="10148-114">The event construct automatically generates the `add_``event` and `remove_``event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }

// Correct (using explicit interface implementation):

{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}

```
