# <a name="compiler-error-cs0173"></a><span data-ttu-id="5b727-102">Ошибка компилятора CS0173</span><span class="sxs-lookup"><span data-stu-id="5b727-102">Compiler Error CS0173</span></span>
<span data-ttu-id="5b727-103">Невозможно определить тип условного выражения, поскольку неявного преобразования между "class1" и "class2" не существует</span><span class="sxs-lookup"><span data-stu-id="5b727-103">Type of conditional expression cannot be determined because there is no implicit conversion between 'class1' and 'class2'</span></span>  
  
 <span data-ttu-id="5b727-104">Преобразование между классами является целесообразным при работе объектов различных классов с одним кодом.</span><span class="sxs-lookup"><span data-stu-id="5b727-104">Conversions between classes are useful when you want objects of different classes to work with the same code.</span></span> <span data-ttu-id="5b727-105">Однако два класса, действующие совместно, не могут поддерживать взаимные и избыточные преобразования, но могут поддерживать неявные преобразования.</span><span class="sxs-lookup"><span data-stu-id="5b727-105">However, two classes that work together cannot have mutual and redundant conversions, or no implicit conversions.</span></span> <span data-ttu-id="5b727-106">Типы `class1` и `class2` определяются независимо, и наиболее общий тип выбирается в качестве типа условного выражения.</span><span class="sxs-lookup"><span data-stu-id="5b727-106">The types of `class1` and `class2` are determined independently, and the more general type is selected as the type of the conditional expression.</span></span> <span data-ttu-id="5b727-107">Дополнительные сведения об определении типов см. на странице [Условный оператор не выполняет неявное приведение](https://stackoverflow.com/questions/2215745/conditional-operator-cannot-cast-implicitly/2215959#2215959).</span><span class="sxs-lookup"><span data-stu-id="5b727-107">For more information about how types are determined, see [Conditional operator cannot cast implicitly](https://stackoverflow.com/questions/2215745/conditional-operator-cannot-cast-implicitly/2215959#2215959).</span></span>  
  
 <span data-ttu-id="5b727-108">Для исправления ошибки CS0173 убедитесь в наличии только одного неявного преобразования между `class1` и `class2` независимо от его направления и класса, в котором оно выполняется.</span><span class="sxs-lookup"><span data-stu-id="5b727-108">To resolve CS0173, verify that there is one and only one implicit conversion between `class1` and `class2`, regardless of which direction the conversion is in and regardless of which class the conversion is in.</span></span> <span data-ttu-id="5b727-109">Дополнительные сведения см. в разделах [Таблица неявных числовых преобразований](../../../csharp/language-reference/keywords/implicit-numeric-conversions-table.md) и [Операторы преобразования](../../../csharp/programming-guide/statements-expressions-operators/conversion-operators.md).</span><span class="sxs-lookup"><span data-stu-id="5b727-109">For more information, see [Implicit Numeric Conversions Table](../../../csharp/language-reference/keywords/implicit-numeric-conversions-table.md) and [Conversion Operators](../../../csharp/programming-guide/statements-expressions-operators/conversion-operators.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="5b727-110">Пример</span><span class="sxs-lookup"><span data-stu-id="5b727-110">Example</span></span>  
 <span data-ttu-id="5b727-111">Следующий пример приводит к возникновению ошибки CS0173:</span><span class="sxs-lookup"><span data-stu-id="5b727-111">The following sample generates CS0173:</span></span>  
  
```csharp  
// CS0173.cs  
public class C {}  
  
public class A   
{  
    //// The following code defines an implicit conversion operator from    
    //// type C to type A.  
    //public static implicit operator A(C c)  
    //{  
    //    A a = new A();  
    //    a = c;  
    //    return a;  
    //}  
}  
  
public class MyClass  
{  
    public static void F(bool b)  
    {  
        A a = new A();  
        C c = new C();  
  
        // The following line causes CS0173 because there is no implicit  
        // conversion from a to c or from c to a.  
        object o = b ? a : c;  
  
        // To resolve the error, you can cast a and c.  
        //object o = b ? (object)a : (object)c;  
  
        // Alternatively, you can add a conversion operator from class C to  
        // class A, or from class A to class C, but not both.  
    }  
  
   public static void Main()  
   {  
      F(true);  
   }  
}  
```
