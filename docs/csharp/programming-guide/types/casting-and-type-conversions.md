<span data-ttu-id="602dc-103">Поскольку код C# является статически типизированным во время компиляции, после объявления переменной ее нельзя объявить повторно или использовать для хранения значений другого типа, если этот тип не преобразуется в тип переменной.</span><span class="sxs-lookup"><span data-stu-id="602dc-103">Because C# is statically-typed at compile time, after a variable is declared, it cannot be declared again or used to store values of another type unless that type is convertible to the variable's type.</span></span> <span data-ttu-id="602dc-104">Например, отсутствует преобразование из целого числа в произвольную строку.</span><span class="sxs-lookup"><span data-stu-id="602dc-104">For example, there is no conversion from an integer to any arbitrary string.</span></span> <span data-ttu-id="602dc-105">Поэтому после объявления `i` как целого числа нельзя назначить ей строку "Hello", как показано в следующем коде.</span><span class="sxs-lookup"><span data-stu-id="602dc-105">Therefore, after you declare `i` as an integer, you cannot assign the string "Hello" to it, as is shown in the following code.</span></span>


-   <span data-ttu-id="602dc-111">**Неявные преобразования**: никакой специальный синтаксис не требуется, поскольку преобразование является строго типизированным и данные не будут потеряны.</span><span class="sxs-lookup"><span data-stu-id="602dc-111">**Implicit conversions**: No special syntax is required because the conversion is type safe and no data will be lost.</span></span> <span data-ttu-id="602dc-112">Примеры включают преобразования из меньших в большие целочисленные типы и преобразования из производных классов в базовые классы.</span><span class="sxs-lookup"><span data-stu-id="602dc-112">Examples include conversions from smaller to larger integral types, and conversions from derived classes to base classes.</span></span>
