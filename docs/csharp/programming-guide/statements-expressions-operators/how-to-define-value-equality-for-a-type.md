# <a name="how-to-define-value-equality-for-a-type-c-programming-guide"></a><span data-ttu-id="64ae1-102">Практическое руководство. Определение равенства значений для типа (Руководство по программированию на C#)</span><span class="sxs-lookup"><span data-stu-id="64ae1-102">How to: Define Value Equality for a Type (C# Programming Guide)</span></span>
<span data-ttu-id="64ae1-103">При определении класса или структуры необходимо решить, имеет ли смысл создавать пользовательское определение равенства значений (или эквивалентности) для этого типа.</span><span class="sxs-lookup"><span data-stu-id="64ae1-103">When you define a class or struct, you decide whether it makes sense to create a custom definition of value equality (or equivalence) for the type.</span></span> <span data-ttu-id="64ae1-104">Обычно правила определения равенства реализуются, если объекты этого типа будут добавляться в коллекции или если они в первую очередь предназначены для хранения набора полей или свойств.</span><span class="sxs-lookup"><span data-stu-id="64ae1-104">Typically, you implement value equality when objects of the type are expected to be added to a collection of some sort, or when their primary purpose is to store a set of fields or properties.</span></span> <span data-ttu-id="64ae1-105">В основу определения равенства значений можно положить сравнение всех полей и свойств в типе или только их части.</span><span class="sxs-lookup"><span data-stu-id="64ae1-105">You can base your definition of value equality on a comparison of all the fields and properties in the type, or you can base the definition on a subset.</span></span> <span data-ttu-id="64ae1-106">Но в любом случае (как для классов, так и для структур) реализация должна соответствовать следующим пяти гарантиям равенства.</span><span class="sxs-lookup"><span data-stu-id="64ae1-106">But in either case, and in both classes and structs, your implementation should follow the five guarantees of equivalence:</span></span>  
  
1.  <span data-ttu-id="64ae1-107">x.`Equals`(x) возвращает значение `true.`. Это называется свойством рефлексивности.</span><span class="sxs-lookup"><span data-stu-id="64ae1-107">x.`Equals`(x) returns `true.` This is called the reflexive property.</span></span>  
  
2.  <span data-ttu-id="64ae1-108">x.`Equals`(y) возвращает то же значение, что и y.`Equals`(x).</span><span class="sxs-lookup"><span data-stu-id="64ae1-108">x.`Equals`(y) returns the same value as y.`Equals`(x).</span></span> <span data-ttu-id="64ae1-109">Это называется свойством симметрии.</span><span class="sxs-lookup"><span data-stu-id="64ae1-109">This is called the symmetric property.</span></span>  
  
3.  <span data-ttu-id="64ae1-110">Если (x.`Equals`(y) && y.`Equals`(z)) возвращает `true`, то x.`Equals`(z) возвращает `true`.</span><span class="sxs-lookup"><span data-stu-id="64ae1-110">if (x.`Equals`(y) && y.`Equals`(z)) returns `true`, then x.`Equals`(z) returns `true`.</span></span> <span data-ttu-id="64ae1-111">Это называется свойством транзитивности.</span><span class="sxs-lookup"><span data-stu-id="64ae1-111">This is called the transitive property.</span></span>  
  
4.  <span data-ttu-id="64ae1-112">Последовательные вызовы x.`Equals`(y) возвращают одно то же значение до тех пор, пока объекты, на которые ссылаются x и y, не будут изменены.</span><span class="sxs-lookup"><span data-stu-id="64ae1-112">Successive invocations of x.`Equals`(y) return the same value as long as the objects referenced by x and y are not modified.</span></span>  
  
5.  <span data-ttu-id="64ae1-113">x.`Equals`(null) возвращает значение `false`.</span><span class="sxs-lookup"><span data-stu-id="64ae1-113">x.`Equals`(null) returns `false`.</span></span> <span data-ttu-id="64ae1-114">Однако null.Equals(null) вызывает исключение. Эта инструкция не удовлетворяет приведенному выше правилу номер 2.</span><span class="sxs-lookup"><span data-stu-id="64ae1-114">However, null.Equals(null) throws an exception; it does not obey rule number two above.</span></span>  
  
 <span data-ttu-id="64ae1-115">Любая определяемая вами структура имеет заданную по умолчанию реализацию равенства значений, которая наследуется от переопределения <xref:System.ValueType?displayProperty=nameWithType> метода <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="64ae1-115">Any struct that you define already has a default implementation of value equality that it inherits from the <xref:System.ValueType?displayProperty=nameWithType> override of the <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="64ae1-116">Эта реализация использует отражение для проверки всех полей и свойств в типе.</span><span class="sxs-lookup"><span data-stu-id="64ae1-116">This implementation uses reflection to examine all the fields and properties in the type.</span></span> <span data-ttu-id="64ae1-117">Хотя эта реализация возвращает верный результат, она отличается невысокой скоростью по сравнению с пользовательской реализацией, которую можно написать специально для конкретного типа.</span><span class="sxs-lookup"><span data-stu-id="64ae1-117">Although this implementation produces correct results, it is relatively slow compared to a custom implementation that you write specifically for the type.</span></span>  
  
 <span data-ttu-id="64ae1-118">Детали реализации равенства значений для классов и структур различаются.</span><span class="sxs-lookup"><span data-stu-id="64ae1-118">The implementation details for value equality are different for classes and structs.</span></span> <span data-ttu-id="64ae1-119">Однако для реализации равенства как для классов, так и для структур, необходимо выполнить одни и те же базовые действия.</span><span class="sxs-lookup"><span data-stu-id="64ae1-119">However, both classes and structs require the same basic steps for implementing equality:</span></span>  
  
1.  <span data-ttu-id="64ae1-120">Переопределите [виртуальный](../../../csharp/language-reference/keywords/virtual.md) метод <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="64ae1-120">Override the [virtual](../../../csharp/language-reference/keywords/virtual.md) <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> method.</span></span> <span data-ttu-id="64ae1-121">В большинстве случаев пользовательская реализация `bool Equals( object obj )` должна вызывать относящийся к конкретному типу метод `Equals`, который является реализацией интерфейса <xref:System.IEquatable%601?displayProperty=nameWithType>.</span><span class="sxs-lookup"><span data-stu-id="64ae1-121">In most cases, your implementation of `bool Equals( object obj )` should just call into the type-specific `Equals` method that is the implementation of the <xref:System.IEquatable%601?displayProperty=nameWithType> interface.</span></span> <span data-ttu-id="64ae1-122">(См. шаг 2.)</span><span class="sxs-lookup"><span data-stu-id="64ae1-122">(See step 2.)</span></span>  
  
2.  <span data-ttu-id="64ae1-123">Реализуйте интерфейс <xref:System.IEquatable%601?displayProperty=nameWithType>, предоставив метод `Equals` для конкретного типа.</span><span class="sxs-lookup"><span data-stu-id="64ae1-123">Implement the <xref:System.IEquatable%601?displayProperty=nameWithType> interface by providing a type-specific `Equals` method.</span></span> <span data-ttu-id="64ae1-124">Именно на этом этапе происходит фактическое сравнение значений.</span><span class="sxs-lookup"><span data-stu-id="64ae1-124">This is where the actual equivalence comparison is performed.</span></span> <span data-ttu-id="64ae1-125">Например, функцию равенства можно определить путем сравнения только одного из двух полей в типе.</span><span class="sxs-lookup"><span data-stu-id="64ae1-125">For example, you might decide to define equality by comparing only one or two fields in your type.</span></span> <span data-ttu-id="64ae1-126">Не вызывайте исключения из `Equals`.</span><span class="sxs-lookup"><span data-stu-id="64ae1-126">Do not throw exceptions from `Equals`.</span></span> <span data-ttu-id="64ae1-127">Только для классов: этот метод должен проверять только те поля, которые объявлены в классе.</span><span class="sxs-lookup"><span data-stu-id="64ae1-127">For classes only: This method should examine only fields that are declared in the class.</span></span> <span data-ttu-id="64ae1-128">Он должен вызывать метод `base.Equals` для проверки полей в базовом классе.</span><span class="sxs-lookup"><span data-stu-id="64ae1-128">It should call `base.Equals` to examine fields that are in the base class.</span></span> <span data-ttu-id="64ae1-129">(Не делайте этого, если тип наследуется напрямую от <xref:System.Object>, поскольку data-stu-id="64ae1-129">(Do not do this if the type inherits directly from <xref:System.Object>, because the <xref:System.Object> implementation of <xref:System.Object.Equals%28System.Object%29?displayProperty=nameWithType> performs a reference equality check.)</span></span>  
  
3.  <span data-ttu-id="64ae1-130">Рекомендуется (хотя это и не обязательно) перегрузить операторы [==](../../../csharp/language-reference/operators/equality-comparison-operator.md) и [!=](../../../csharp/language-reference/operators/not-equal-operator.md).</span><span class="sxs-lookup"><span data-stu-id="64ae1-130">Optional but recommended: Overload the [==](../../../csharp/language-reference/operators/equality-comparison-operator.md) and [!=](../../../csharp/language-reference/operators/not-equal-operator.md) operators.</span></span>  
  
4.  <span data-ttu-id="64ae1-131">Переопределите <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType> таким образом, чтобы два объекта с равными значениями создавали одинаковый хэш-код.</span><span class="sxs-lookup"><span data-stu-id="64ae1-131">Override <xref:System.Object.GetHashCode%2A?displayProperty=nameWithType> so that two objects that have value equality produce the same hash code.</span></span>  
  
5.  <span data-ttu-id="64ae1-132">Необязательно. Для поддержки определений "больше" и "меньше" реализуйте для типа интерфейс <xref:System.IComparable%601>, а также перегрузите операторы [<=](../../../csharp/language-reference/operators/less-than-equal-operator.md) и [>=](../../../csharp/language-reference/operators/greater-than-equal-operator.md).</span><span class="sxs-lookup"><span data-stu-id="64ae1-132">Optional: To support definitions for "greater than" or "less than," implement the <xref:System.IComparable%601> interface for your type, and also overload the [<=](../../../csharp/language-reference/operators/less-than-equal-operator.md) and [>=](../../../csharp/language-reference/operators/greater-than-equal-operator.md) operators.</span></span>
