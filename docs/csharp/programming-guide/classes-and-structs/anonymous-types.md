# <a name="anonymous-types-c-programming-guide"></a><span data-ttu-id="a52f3-102">Анонимные типы (Руководство по программированию в C#)</span><span class="sxs-lookup"><span data-stu-id="a52f3-102">Anonymous Types (C# Programming Guide)</span></span>
<span data-ttu-id="a52f3-103">Анонимные типы позволяют легко инкапсулировать свойства только для чтения в один объект без необходимости предварительного определения типа.</span><span class="sxs-lookup"><span data-stu-id="a52f3-103">Anonymous types provide a convenient way to encapsulate a set of read-only properties into a single object without having to explicitly define a type first.</span></span> <span data-ttu-id="a52f3-104">Имя типа создается компилятором и недоступно на уровне исходного кода.</span><span class="sxs-lookup"><span data-stu-id="a52f3-104">The type name is generated by the compiler and is not available at the source code level.</span></span> <span data-ttu-id="a52f3-105">Тип каждого свойства выводится компилятором.</span><span class="sxs-lookup"><span data-stu-id="a52f3-105">The type of each property is inferred by the compiler.</span></span>  
  
 <span data-ttu-id="a52f3-106">Анонимные типы создаются с помощью оператора [new](../../../csharp/language-reference/keywords/new.md) и инициализатора объекта.</span><span class="sxs-lookup"><span data-stu-id="a52f3-106">You create anonymous types by using the [new](../../../csharp/language-reference/keywords/new.md) operator together with an object initializer.</span></span> <span data-ttu-id="a52f3-107">Дополнительные сведения об инициализаторах объектов см. в статье [Инициализаторы объектов и коллекций](../../../csharp/programming-guide/classes-and-structs/object-and-collection-initializers.md).</span><span class="sxs-lookup"><span data-stu-id="a52f3-107">For more information about object initializers, see [Object and Collection Initializers](../../../csharp/programming-guide/classes-and-structs/object-and-collection-initializers.md).</span></span>  
  
 <span data-ttu-id="a52f3-108">В следующем примере показан анонимный тип, инициализированный с помощью двух свойств — `Amount` и `Message`.</span><span class="sxs-lookup"><span data-stu-id="a52f3-108">The following example shows an anonymous type that is initialized with two properties named `Amount` and `Message`.</span></span>  
  
```csharp

// Rest the mouse pointer over v.Amount and v.Message in the following

```  
  
 <span data-ttu-id="a52f3-109">Обычно анонимные типы используются в предложении [select](../../../csharp/language-reference/keywords/select-clause.md) выражения запроса для возврата поднабора свойств из каждого объекта в исходной последовательности.</span><span class="sxs-lookup"><span data-stu-id="a52f3-109">Anonymous types typically are used in the [select](../../../csharp/language-reference/keywords/select-clause.md) clause of a query expression to return a subset of the properties from each object in the source sequence.</span></span> <span data-ttu-id="a52f3-110">Дополнительные сведения о запросах см. в разделе [Выражения запросов LINQ](../../../csharp/programming-guide/linq-query-expressions/index.md).</span><span class="sxs-lookup"><span data-stu-id="a52f3-110">For more information about queries, see [LINQ Query Expressions](../../../csharp/programming-guide/linq-query-expressions/index.md).</span></span>  
  
 <span data-ttu-id="a52f3-111">Анонимные типы содержат один или несколько публичных свойств только для чтения.</span><span class="sxs-lookup"><span data-stu-id="a52f3-111">Anonymous types contain one or more public read-only properties.</span></span> <span data-ttu-id="a52f3-112">Другие члены класса, например методы или события, недопустимы.</span><span class="sxs-lookup"><span data-stu-id="a52f3-112">No other kinds of class members, such as methods or events, are valid.</span></span> <span data-ttu-id="a52f3-113">Выражение, которое используется для инициализации свойства, не может быть `null`, анонимной функцией или типом указателя.</span><span class="sxs-lookup"><span data-stu-id="a52f3-113">The expression that is used to initialize a property cannot be `null`, an anonymous function, or a pointer type.</span></span>  
  
 <span data-ttu-id="a52f3-114">Наиболее частый сценарий — это инициализация анонимного типа со свойствами из другого типа.</span><span class="sxs-lookup"><span data-stu-id="a52f3-114">The most common scenario is to initialize an anonymous type with properties from another type.</span></span> <span data-ttu-id="a52f3-115">В следующем примере предполагается, что существует класс с именем `Product`.</span><span class="sxs-lookup"><span data-stu-id="a52f3-115">In the following example, assume that a class exists that is named `Product`.</span></span> <span data-ttu-id="a52f3-116">Класс `Product` включает свойства `Color` и `Price`, а также другие свойства, в которых вы не заинтересованы.</span><span class="sxs-lookup"><span data-stu-id="a52f3-116">Class `Product` includes `Color` and `Price` properties, together with other properties that you are not interested in.</span></span> <span data-ttu-id="a52f3-117">Переменная `products` является коллекцией объектов `Product`.</span><span class="sxs-lookup"><span data-stu-id="a52f3-117">Variable `products` is a collection of `Product` objects.</span></span> <span data-ttu-id="a52f3-118">Объявление анонимного типа запускается с помощью ключевого слова `new`.</span><span class="sxs-lookup"><span data-stu-id="a52f3-118">The anonymous type declaration starts with the `new` keyword.</span></span> <span data-ttu-id="a52f3-119">Объявление инициализирует новый тип, который использует только два свойства из `Product`.</span><span class="sxs-lookup"><span data-stu-id="a52f3-119">The declaration initializes a new type that uses only two properties from `Product`.</span></span> <span data-ttu-id="a52f3-120">Это приводит к тому, что для возврата остается меньшее количество данных.</span><span class="sxs-lookup"><span data-stu-id="a52f3-120">This causes a smaller amount of data to be returned in the query.</span></span>  
  
 <span data-ttu-id="a52f3-121">Если имена членов в анонимном типе не указаны, компилятор присваивает членам анонимного типа одно и то же имя, поскольку для их инициализации используется свойство.</span><span class="sxs-lookup"><span data-stu-id="a52f3-121">If you do not specify member names in the anonymous type, the compiler gives the anonymous type members the same name as the property being used to initialize them.</span></span> <span data-ttu-id="a52f3-122">Необходимо указать имя для свойства, инициализируемого с помощью выражения, как показано в предыдущем примере.</span><span class="sxs-lookup"><span data-stu-id="a52f3-122">You must provide a name for a property that is being initialized with an expression, as shown in the previous example.</span></span> <span data-ttu-id="a52f3-123">В следующем примере используются имена свойств анонимного типа `Color` и `Price`.</span><span class="sxs-lookup"><span data-stu-id="a52f3-123">In the following example, the names of the properties of the anonymous type are `Color` and `Price`.</span></span>

 <span data-ttu-id="a52f3-124">Обычно, если для инициализации переменной используется анонимный тип, необходимо обозначить ее как неявно типизированную переменную с помощью функции [var](../../../csharp/language-reference/keywords/var.md).</span><span class="sxs-lookup"><span data-stu-id="a52f3-124">Typically, when you use an anonymous type to initialize a variable, you declare the variable as an implicitly typed local variable by using [var](../../../csharp/language-reference/keywords/var.md).</span></span> <span data-ttu-id="a52f3-125">Имя типа не может быть указано в объявлении переменной, так как доступ к базовому имени анонимного типа имеет только компилятор.</span><span class="sxs-lookup"><span data-stu-id="a52f3-125">The type name cannot be specified in the variable declaration because only the compiler has access to the underlying name of the anonymous type.</span></span> <span data-ttu-id="a52f3-126">Дополнительные сведения о `var` см. в разделе [Неявно типизированные локальные переменные](../../../csharp/programming-guide/classes-and-structs/implicitly-typed-local-variables.md).</span><span class="sxs-lookup"><span data-stu-id="a52f3-126">For more information about `var`, see [Implicitly Typed Local Variables](../../../csharp/programming-guide/classes-and-structs/implicitly-typed-local-variables.md).</span></span>

## <a name="remarks"></a><span data-ttu-id="a52f3-128">Примечания</span><span class="sxs-lookup"><span data-stu-id="a52f3-128">Remarks</span></span>  
 <span data-ttu-id="a52f3-129">Анонимные типы являются типами [class](../../../csharp/language-reference/keywords/class.md), прямыми производными от типа [object](../../../csharp/language-reference/keywords/object.md), и не могут быть приведены ни к какому иному типу, кроме [object](../../../csharp/language-reference/keywords/object.md).</span><span class="sxs-lookup"><span data-stu-id="a52f3-129">Anonymous types are [class](../../../csharp/language-reference/keywords/class.md) types that derive directly from [object](../../../csharp/language-reference/keywords/object.md), and that cannot be cast to any type except [object](../../../csharp/language-reference/keywords/object.md).</span></span> <span data-ttu-id="a52f3-130">Компилятор назначает имя для каждого анонимного типа, несмотря на то что для вашего приложения он недоступен.</span><span class="sxs-lookup"><span data-stu-id="a52f3-130">The compiler provides a name for each anonymous type, although your application cannot access it.</span></span> <span data-ttu-id="a52f3-131">С точки зрения среды CLR анонимный тип не отличается от остальных типов ссылки.</span><span class="sxs-lookup"><span data-stu-id="a52f3-131">From the perspective of the common language runtime, an anonymous type is no different from any other reference type.</span></span>  
  
 <span data-ttu-id="a52f3-132">Если два или несколько инициализаторов анонимных объектов в сборке указывают на последовательность свойств, идущих в том же порядке и имеющих те же типы и имена, компилятор обрабатывает объекты как экземпляры одного типа.</span><span class="sxs-lookup"><span data-stu-id="a52f3-132">If two or more anonymous object initializers in an assembly specify a sequence of properties that are in the same order and that have the same names and types, the compiler treats the objects as instances of the same type.</span></span> <span data-ttu-id="a52f3-133">Они используют одни и те же сведения типа, созданные компилятором.</span><span class="sxs-lookup"><span data-stu-id="a52f3-133">They share the same compiler-generated type information.</span></span>  
  
 <span data-ttu-id="a52f3-134">К анонимному типу нельзя отнести никакое поле, свойство, событие или тип возвращаемого значения метода.</span><span class="sxs-lookup"><span data-stu-id="a52f3-134">You cannot declare a field, a property, an event, or the return type of a method as having an anonymous type.</span></span> <span data-ttu-id="a52f3-135">Точно так же нельзя отнести к анонимному типу ни один формальный параметр метода, свойство, конструктор или индексатор.</span><span class="sxs-lookup"><span data-stu-id="a52f3-135">Similarly, you cannot declare a formal parameter of a method, property, constructor, or indexer as having an anonymous type.</span></span> <span data-ttu-id="a52f3-136">Для передачи анонимного типа или коллекции, содержащей анонимные типы, как аргумента для метода можно использовать этот параметр в качестве объекта типа.</span><span class="sxs-lookup"><span data-stu-id="a52f3-136">To pass an anonymous type, or a collection that contains anonymous types, as an argument to a method, you can declare the parameter as type object.</span></span> <span data-ttu-id="a52f3-137">Однако подобное решение противоречит цели строгого ввода.</span><span class="sxs-lookup"><span data-stu-id="a52f3-137">However, doing this defeats the purpose of strong typing.</span></span> <span data-ttu-id="a52f3-138">Если вам нужно сохранить результаты запроса или передать их за пределы метода, используйте вместо анонимного типа структуру или класс, названные обычным образом.</span><span class="sxs-lookup"><span data-stu-id="a52f3-138">If you must store query results or pass them outside the method boundary, consider using an ordinary named struct or class instead of an anonymous type.</span></span>  
  
 <span data-ttu-id="a52f3-139">Так как методы <xref:System.Object.Equals%2A> и <xref:System.Object.GetHashCode%2A> в анонимных типах определяются с точки зрения методов свойств `Equals` и `GetHashCode`, два экземпляра одного и того же анонимного типа равны, только если равны их свойства.</span><span class="sxs-lookup"><span data-stu-id="a52f3-139">Because the <xref:System.Object.Equals%2A> and <xref:System.Object.GetHashCode%2A> methods on anonymous types are defined in terms of the `Equals` and `GetHashCode` methods of the properties, two instances of the same anonymous type are equal only if all their properties are equal.</span></span>
