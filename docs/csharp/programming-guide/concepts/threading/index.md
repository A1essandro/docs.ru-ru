# <a name="threading-c"></a>Работа с потоками (C#)
Потоки позволяют программе C# выполнять параллельную обработку, которая может выполнять несколько операций одновременно. Например работа с потоками можно использовать для наблюдения за входные данные пользователя, выполнения фоновых задач и обработки одновременных потоков ввода.  
  
 Потоки имеют следующие свойства:  
  
-   Потоки позволяют программе выполнять параллельную обработку.  
  
-   Платформа .NET Framework <xref:System.Threading> имен упрощает использование потоков.  
  
-   Потоки совместно используют ресурсы приложения. Дополнительные сведения см. в разделе [использование потоков и потоки](https://msdn.microsoft.com/library/e1dx6b2h).  
  
 По умолчанию программы на C# имеет один поток. Тем не менее вспомогательные потоки можно создать и использовать для выполнения кода параллельно с основным потоком. Эти потоки часто называются *рабочих потоков*.  
  
 Рабочие потоки могут использоваться для выполнения трудоемких или срочных задач без прерывания основного потока. Например рабочие потоки часто используются в серверных приложениях для выполнения входящих запросов, не дожидаясь завершения предыдущего запроса. Рабочие потоки также используются для выполнения задач «фон» в настольных приложениях, чтобы основной поток--дисков элементы пользовательского интерфейса — остается реагировать на действия пользователя.  
  
 Работа с потоками позволяет решить проблемы с пропускной способностью и скорость реагирования, но также могут вызвать проблемы, общий доступ к ресурсам, например взаимоблокировки и состояния гонки. Несколько потоков лучше всего подходят для задач, которые используют различные ресурсы, такие как дескрипторы файлов и сетевые подключения. Назначение нескольких потоков для одного ресурса может вызвать проблемы синхронизации и частую блокировку во время ожидания других потоков потоков согласуется с помощью нескольких потоков.  
  
 Обычно является использование рабочих потоков для выполнения трудоемких или срочные задачи, которые не требуется большое количество ресурсов, используемых другими потоками. Естественно некоторые ресурсы в приложении могут быть доступны из нескольких потоков. В этих случаях <xref:System.Threading> пространство имен предоставляет классы для синхронизации потоков. Эти классы содержат <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, и <xref:System.Threading.ManualResetEvent>.  
  
 Некоторые или все эти классы можно использовать для синхронизации действий нескольких потоков, но некоторые поддержки потоков поддерживается в языке C#. Например [оператора Lock](../../../../csharp/language-reference/keywords/lock-statement.md) предоставляет возможности синхронизации за счет неявного использования <xref:System.Threading.Monitor>.  
  
> [!NOTE]
>  Начиная с версии [!INCLUDE[net_v40_long](~/includes/net-v40-long-md.md)], многопоточного программирования упрощено с <xref:System.Threading.Tasks.Parallel?displayProperty=fullName> и <xref:System.Threading.Tasks.Task?displayProperty=fullName> классы, [Parallel LINQ (PLINQ)](https://msdn.microsoft.com/library/dd460688), новых классов параллельных коллекций в <xref:System.Collections.Concurrent?displayProperty=fullName> пространства имен и новая модель программирования, которая основана на концепции задач вместо потоков. Дополнительные сведения см. в разделе [Параллельное программирование](https://msdn.microsoft.com/library/dd460693).
