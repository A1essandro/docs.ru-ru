# <a name="threading-c"></a><span data-ttu-id="0fe53-102">Работа с потоками (C#)</span><span class="sxs-lookup"><span data-stu-id="0fe53-102">Threading (C#)</span></span>
<span data-ttu-id="0fe53-103">Потоки позволяют программе на C# осуществлять параллельную обработку, то есть вы можете выполнять несколько операций одновременно.</span><span class="sxs-lookup"><span data-stu-id="0fe53-103">Threading enables your C# program to perform concurrent processing so that you can do more than one operation at a time.</span></span> <span data-ttu-id="0fe53-104">Например, потоки можно использовать для отслеживания пользовательского ввода, выполнения фоновых задач и одновременной обработки нескольких потоков ввода.</span><span class="sxs-lookup"><span data-stu-id="0fe53-104">For example, you can use threading to monitor input from the user, perform background tasks, and handle simultaneous streams of input.</span></span>  
  
 <span data-ttu-id="0fe53-105">Потоки имеют следующие свойства.</span><span class="sxs-lookup"><span data-stu-id="0fe53-105">Threads have the following properties:</span></span>  
  
-   <span data-ttu-id="0fe53-106">Потоки позволяют программе выполнять параллельную обработку.</span><span class="sxs-lookup"><span data-stu-id="0fe53-106">Threads enable your program to perform concurrent processing.</span></span>  
  
-   <span data-ttu-id="0fe53-107">Пространство имен <xref:System.Threading> платформы .NET Framework упрощает использование потоков.</span><span class="sxs-lookup"><span data-stu-id="0fe53-107">The .NET Framework <xref:System.Threading> namespace makes using threads easier.</span></span>  
  
-   <span data-ttu-id="0fe53-108">Потоки используют одни и те же ресурсы приложения.</span><span class="sxs-lookup"><span data-stu-id="0fe53-108">Threads share the application's resources.</span></span> <span data-ttu-id="0fe53-109">Дополнительные сведения см. в разделе [Использование потоков и работа с потоками](../../../../../docs/standard/threading/using-threads-and-threading.md).</span><span class="sxs-lookup"><span data-stu-id="0fe53-109">For more information, see [Using Threads and Threading](../../../../../docs/standard/threading/using-threads-and-threading.md).</span></span>  
  
 <span data-ttu-id="0fe53-110">По умолчанию программа на языке C# имеет один поток.</span><span class="sxs-lookup"><span data-stu-id="0fe53-110">By default, a C# program has one thread.</span></span> <span data-ttu-id="0fe53-111">Но можно также создавать вспомогательные потоки и выполнять в них код параллельно основному.</span><span class="sxs-lookup"><span data-stu-id="0fe53-111">However, auxiliary threads can be created and used to execute code in parallel with the primary thread.</span></span> <span data-ttu-id="0fe53-112">Эти потоки часто называются *рабочими потоками*.</span><span class="sxs-lookup"><span data-stu-id="0fe53-112">These threads are often called *worker threads*.</span></span>  
  
 <span data-ttu-id="0fe53-113">Рабочие потоки можно использовать для выполнения трудоемких или срочных задач без прерывания основного потока.</span><span class="sxs-lookup"><span data-stu-id="0fe53-113">Worker threads can be used to perform time-consuming or time-critical tasks without tying up the primary thread.</span></span> <span data-ttu-id="0fe53-114">Рабочие потоки часто используются, например, в серверных приложениях, когда необходимо обработать входящий запрос, не дожидаясь завершения обработки предыдущего запроса.</span><span class="sxs-lookup"><span data-stu-id="0fe53-114">For example, worker threads are often used in server applications to fulfill incoming requests without waiting for the previous request to be completed.</span></span> <span data-ttu-id="0fe53-115">Кроме того, они используются для выполнения «фоновых» задач в настольных приложениях, что позволяет не занимать ресурсы основного потока, ответственного за работу пользовательского интерфейса.</span><span class="sxs-lookup"><span data-stu-id="0fe53-115">Worker threads are also used to perform "background" tasks in desktop applications so that the main thread--which drives user interface elements--remains responsive to user actions.</span></span>  
  
 <span data-ttu-id="0fe53-116">Применение потоков позволяет решить проблемы с пропускной способностью и быстротой работы системы, но оно также может привести к проблемам с совместным использованием ресурсов, например к конфликтам и взаимоблокировкам.</span><span class="sxs-lookup"><span data-stu-id="0fe53-116">Threading solves problems with throughput and responsiveness, but it can also introduce resource-sharing issues such as deadlocks and race conditions.</span></span> <span data-ttu-id="0fe53-117">Использование нескольких потоков лучше всего подходит в тех случаях, когда разные задачи используют разные ресурсы, например дескрипторы файлов и сетевые подключения.</span><span class="sxs-lookup"><span data-stu-id="0fe53-117">Multiple threads are best for tasks that require different resources such as file handles and network connections.</span></span> <span data-ttu-id="0fe53-118">Назначение нескольких потоков одному ресурсу, вероятнее всего, вызовет проблемы синхронизации и частую блокировку из-за ожидания, что сделает само использование нескольких потоков нецелесообразным.</span><span class="sxs-lookup"><span data-stu-id="0fe53-118">Assigning multiple threads to a single resource is likely to cause synchronization issues, and having threads frequently blocked when waiting for other threads defeats the purpose of using multiple threads.</span></span>  
  
 <span data-ttu-id="0fe53-119">Обычно рабочие потоки используются для выполнения трудоемких или срочных задач, которым не требуется большое количество ресурсов, занятых другими потоками.</span><span class="sxs-lookup"><span data-stu-id="0fe53-119">A common strategy is to use worker threads to perform time-consuming or time-critical tasks that do not require many of the resources used by other threads.</span></span> <span data-ttu-id="0fe53-120">Естественно, некоторые используемые программой ресурсы должны быть доступны нескольким потокам.</span><span class="sxs-lookup"><span data-stu-id="0fe53-120">Naturally, some resources in your program must be accessed by multiple threads.</span></span> <span data-ttu-id="0fe53-121">Для таких случаев в пространстве имен <xref:System.Threading> доступны классы по синхронизации потоков.</span><span class="sxs-lookup"><span data-stu-id="0fe53-121">For these cases, the <xref:System.Threading> namespace provides classes for synchronizing threads.</span></span> <span data-ttu-id="0fe53-122">К таким классам относятся <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent> и <xref:System.Threading.ManualResetEvent>.</span><span class="sxs-lookup"><span data-stu-id="0fe53-122">These classes include <xref:System.Threading.Mutex>, <xref:System.Threading.Monitor>, <xref:System.Threading.Interlocked>, <xref:System.Threading.AutoResetEvent>, and <xref:System.Threading.ManualResetEvent>.</span></span>  
  
 <span data-ttu-id="0fe53-123">Все классы или некоторые из них можно использовать для синхронизации работы нескольких потоков, но некоторые функции для потоков поддерживаются также и языком C#.</span><span class="sxs-lookup"><span data-stu-id="0fe53-123">You can use some or all these classes to synchronize the activities of multiple threads, but some support for threading is supported by the C# language.</span></span> <span data-ttu-id="0fe53-124">Например [оператор lock](../../../../csharp/language-reference/keywords/lock-statement.md) дает возможность синхронизации за счет неявного использования класса <xref:System.Threading.Monitor>.</span><span class="sxs-lookup"><span data-stu-id="0fe53-124">For example, the [Lock Statement](../../../../csharp/language-reference/keywords/lock-statement.md) provides synchronization features through implicit use of <xref:System.Threading.Monitor>.</span></span>
