# <a name="the-history-of-c"></a>Журнал C# #

Языка как выглядит в его раннее деятельности? И как его развивался лет с момента?

## <a name="c-version-10"></a>В C# версии 1.0

Когда вернитесь и посмотрите, C# версии 1.0 выглядела гораздо Java. Как [частью его задач проектирования указанных выше ECMA](http://feeldotneteasy.blogspot.com/2011/01/c-design-goals.html), его поиск для «простой современный, общие цели объектно ориентированный язык.»  Во время поиска как Java предназначен его достичь этих ранних целей разработки.

Однако если взглянуть на C# 1.0 теперь, необходимо найти самостоятельно немного Диззи. Оно не было возможности встроенных async и некоторые функциональные возможности называется вокруг универсальные шаблоны, которые мы используем должное. Как фактически оно не было универсальных шаблонов вообще.  И [LINQ](../linq/index.md)? Не доступно еще. Что потребовало бы некоторые годы.

В C# версии 1.0 выглядел очищенный возможностей по сравнению с сегодняшнего дня. Вы увидите самостоятельно написания подробного кода. Однако, необходимо запустить где-либо. В C# версии 1.0 была достойная альтернатива Java на платформе Windows.

## <a name="c-version-20"></a>В C# версии 2.0

Интересной начнут вещей. Давайте рассмотрим некоторые основные возможности языка C# 2.0, выпущена в 2005 г., вместе с Visual Studio 2005:

- [Универсальные шаблоны](../programming-guide/generics/index.md)
- [Разделяемые типы](../programming-guide/classes-and-structs/partial-classes-and-methods.md#partial-classes)
- [Анонимные методы](../programming-guide/statements-expressions-operators/anonymous-methods.md)
- [Типы, допускающие значения NULL](../programming-guide/nullable-types/index.md)
- [Итераторы](../programming-guide/concepts/iterators.md)
- [Ковариация и контрвариация](../programming-guide/concepts/covariance-contravariance/index.md)

Хотя C# может быть запущена в режиме довольно универсального языка объектно (ОО), в C# версии 2.0 изменены, нужно быстро. Если они есть свои футов в них, они пришли после сложностями некоторые серьезные разработчика. И они стала после их себя.

При использовании универсальных шаблонов у вас есть типы и методы, которые можно использовать для произвольного типа, сохраняя безопасность типов. Таким образом, например, для которых <xref:System.Collections.Generic.List%601> позволяет использовать `List<string>` или `List<int>` и выполнять тип безопасном операций над этими строками или целыми числами, пока выполнять через них итерацию. Это лучше, чем создание `ListInt` наследники или приведение `Object` для каждой операции.

C# версии 2.0 переведена в режим итераторов. Чтобы поместить четко, это дает возможность перебора элементов в `List` (или другие перечислимые типы) с `foreach` цикла. Наличие имени первого класса частью языка значительно улучшены удобочитаемость языка и пользователей возможность делать выводы о коде.

А еще, C# продолжает прослушивание немного успевает Java. Java уже выпустила версий, включающих универсальные шаблоны и итераторы. Но, скоро будет изменять как языки продолжает развивать друг от друга.

Эта версия помечен значительное изменение рост C#. Доказал как действительно подойдут язык программирования C#. Давайте рассмотрим некоторые основные возможности в этой версии:

- [Автоматически реализуемых свойств](../programming-guide/classes-and-structs/auto-implemented-properties.md)
- [Анонимные типы](../programming-guide/classes-and-structs/anonymous-types.md)
- [Выражения запросов](../linq/query-expression-basics.md)
- [Лямбда-выражения](https://www.daedtech.com/introduction-to-c-lambda-expressions/)
- [Деревья выражений](https://blogs.msdn.microsoft.com/charlie/2008/01/31/expression-tree-basics/)
- [Методы расширения](https://www.codeproject.com/Tips/709310/Extension-Method-In-Csharp)

В ретроспективе многие из этих средств достигла неизбежно и неотделимыми. Все они взаимодействуют Стратегическое планирование. Обычно думал, что C# версии Убойное функция была выражение запроса, также известные как встроенных в язык запросы (LINQ).

Более огромным представление проверяет выражение tress, лямбда-выражения и анонимные типы как основу, на котором создается LINQ. Однако в любом случае C# 3.0 представляются революционную концепции. Чтобы обеспечить фундамент для включения C# в гибридной объектно ориентированного начатая C# 3.0 / функционального языка.

В частности теперь может написать SQL-стиль декларативных запросов для выполнения операций с коллекциями, помимо прочего. Вместо написания `for` цикл для вычисления среднего значения списка целых чисел, может теперь вам, как просто как `list.Average()`. Сочетание выражения запросов и методы расширения стало выглядеть так, будто список целых чисел потребовал гораздо быстрее.

Затраченное время действительно Возьмитесь за и интегрировать концепцию для пользователей, но постепенно работали. А теперь несколько лет, код гораздо более четкими, простая и функциональных.

## <a name="c-version-40"></a>В C# версии 4.0

В C# версии 4.0 бы затруднительно живущих нововведений состояние версии 3.0. В версии 3.0 C# перешла язык надежно помещает от тени Java и в объект. Быстро стали элегантное языка.

Следующая версия представляют ряд интересных новых функций.

- [Динамическая привязка](../language-reference/keywords/dynamic.md)
- [Именованные и необязательные аргументы](../programming-guide/classes-and-structs/named-and-optional-arguments.md)
- [Универсальные параметры ковариантного и контравариантного](../../standard/generics/covariance-and-contravariance.md)
- [Типы внедренных сборок взаимодействия](https://stackoverflow.com/questions/20514240/whats-the-difference-setting-embed-interop-types-true-and-false-in-visual-studi)

Типы внедренных сборок взаимодействия облегчено неудобной развертывания. Универсальный ковариантности и контрвариантности предоставляют вам больше возможностей для использования универсальных шаблонов, но все они несколько науки и, вероятно, наиболее оценили авторами framework и библиотеки. Именованные и необязательные параметры позволяют избежать многих перегрузки метода и предоставить удобства. Однако ни один из этих функций точно парадигма изменение.

Основные функции был появлением `dynamic` ключевое слово. `dynamic` Ключевое слово введен в C# версии 4.0 возможность переопределения компилятор на ввода во время компиляции. Используя ключевое слово dynamic, можно создать конструкции аналогично динамического типа языков, таких как JavaScript. Можно создать `dynamic x = "a string"` и добавьте шесть, оставив его во время выполнения для сортировки, что должно происходить рядом.

Это дает возможность для ошибки, но отличный питания в рамках языка.
