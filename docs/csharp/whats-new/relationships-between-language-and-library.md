# <a name="relationships-between-language-features-and-library-types"></a>Связи между языковым компонентам и библиотеки типов

Определение языка C# требует стандартной библиотеки для определенных типов, а также некоторые доступные члены этих типов. Компилятор создает код, который использует эти необходимые типы и члены для реализации многих функций на другом языке. При необходимости, существуют пакеты NuGet, которые содержат типы, необходимые для более новые версии языка при написании кода для сред, где эти типы или члены имеют еще не развернут.

Эта зависимость от функциональных возможностей стандартной библиотеки был частью языка C# с самой первой версии. В эту версию включены примеры:

* <xref:System.Exception>-используется для всех исключений, создаваемый компилятором.
* <xref:System.String>-C# `string` тип является синонимом для <xref:System.String>.
* <xref:System.Int32>— синоним `int`.

Эта первая версия была простой: компилятора и стандартной библиотеки поставляемом вместе друг с другом, и только одна версия.

Последующие версии C# иногда добавлены новые типы или члены зависимостям. Примеры: <xref:System.Runtime.CompilerServices.INotifyCompletion>, <xref:System.Runtime.CompilerServices.CallerFilePathAttribute> и <xref:System.Runtime.CompilerServices.CallerMemberNameAttribute>. C# 7.0 продолжается это путем добавления зависимость на <xref:System.ValueTuple> реализовать [кортежей](../tuples.md) функции языка.

Чтобы уменьшить контактную зону, типов и членов соответствует стандартной библиотеки работает группа разработки языка. Соответствие чистой структуре, где новых возможностей библиотеки включены без проблем в язык равномерно этой цели. Будет существовать новые функции в будущих версиях C#, требующих новых типов и членов в стандартную библиотеку. Важно понять, как управлять эти зависимости свою работу.

## <a name="managing-your-dependencies"></a>Управление зависимостями

Средства компилятора C# теперь лишается цикла выпуска библиотек .NET на поддерживаемых платформах. В действительности различных библиотеках .NET содержат различные жизненного цикла: .NET Framework в Windows является relesed как Центр обновления Windows, .NET Core поставляется на отдельное расписание и Xamarin версии библиотеки отгрузки обновлений с помощью средства Xamarin для каждой целевой платформы.

Большую часть времени, можно не заметить, эти изменения. Однако при работе с более новой версии, языка, который не требуются возможности, но в библиотеках .NET на этой платформе будет ссылаться на пакеты NuGet, чтобы добавить эти новые типы.
Как платформы вашего приложения поддерживает обновляются с помощью новых установок framework можно удалить лишние ссылки.

Такое разделение означает, что даже в том случае, если вы используете машины, которые могут не иметь соответствующего framework можно использовать новые возможности языка.
