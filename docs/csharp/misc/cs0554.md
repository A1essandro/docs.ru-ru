# <a name="compiler-error-cs0554"></a><span data-ttu-id="05470-102">Ошибка компилятора CS0554</span><span class="sxs-lookup"><span data-stu-id="05470-102">Compiler Error CS0554</span></span>
<span data-ttu-id="05470-103">"подпрограмма_преобразования": определенное пользователем преобразование в производный класс/из производного класса</span><span class="sxs-lookup"><span data-stu-id="05470-103">'conversion routine' : user defined conversion to/from derived class</span></span>  
  
 <span data-ttu-id="05470-104">Пользовательские преобразования в значения производного класса не допускаются; в операторе подобного типа нет необходимости.</span><span class="sxs-lookup"><span data-stu-id="05470-104">User-defined conversions to values of a derived class are not allowed; you do not need such an operator.</span></span>  
  
 <span data-ttu-id="05470-105">Дополнительные сведения о преобразованиях, определяемых пользователем, см. в главе 6 спецификации языка C#.</span><span class="sxs-lookup"><span data-stu-id="05470-105">See chapter 6 in the C# language specification for more information on user-defined conversions.</span></span>  
  
 <span data-ttu-id="05470-106">В следующем примере возникает ошибка CS0554:</span><span class="sxs-lookup"><span data-stu-id="05470-106">The following sample generates CS0554:</span></span>  
  
```  
// CS0554.cs  
namespace x  
{  
   public class ii  
   {  
      // delete the conversion routine to resolve CS0554  
      public static implicit operator ii(a aa) // CS0554  
      {  
         return new ii();  
      }  
   }  
  
   public class a : ii  
   {  
      public static void Main()  
      {  
      }  
   }  
}  
```
