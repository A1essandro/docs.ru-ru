 <span data-ttu-id="af57d-104">Если используется параметр [/doc](../../csharp/language-reference/compiler-options/doc-compiler-option.md), все комментарии в исходном коде должны быть в формате XML.</span><span class="sxs-lookup"><span data-stu-id="af57d-104">When using [/doc](../../csharp/language-reference/compiler-options/doc-compiler-option.md), any comments in the source code must be in XML.</span></span> <span data-ttu-id="af57d-105">Любая ошибка разметки XML вызывает предупреждение CS1570.</span><span class="sxs-lookup"><span data-stu-id="af57d-105">Any error with your XML markup will generate CS1570.</span></span> <span data-ttu-id="af57d-106">Пример:</span><span class="sxs-lookup"><span data-stu-id="af57d-106">For example:</span></span>  
  
-   <span data-ttu-id="af57d-107">Если передается строка, **cref**, например в [ \<исключение >](../../csharp/programming-guide/xmldoc/exception.md) тег, строка должна быть заключена в двойные кавычки.</span><span class="sxs-lookup"><span data-stu-id="af57d-107">If you are passing a string to a **cref**, such as in an [\<exception>](../../csharp/programming-guide/xmldoc/exception.md) tag, the string must be enclosed in double quotation marks.</span></span>  
  
-   <span data-ttu-id="af57d-108">При использовании тега, таких как [ \<seealso >](../../csharp/programming-guide/xmldoc/seealso.md), который не имеет закрывающего тега, необходимо указать косую черту перед закрывающей угловой скобкой.</span><span class="sxs-lookup"><span data-stu-id="af57d-108">If you are using a tag, such as [\<seealso>](../../csharp/programming-guide/xmldoc/seealso.md), which does not have a closing tag, you must specify a forward slash before the closing angle bracket.</span></span>  
  
-   <span data-ttu-id="af57d-109">Если в тексте описания необходимо использовать символ "больше" или "меньше", следует представить эти символы как **&gt;** или **&lt;**.</span><span class="sxs-lookup"><span data-stu-id="af57d-109">If you need to use a greater-than or less-than symbol in the text of description, you need to represent them with **&gt;** or **&lt;**.</span></span>  
  
-   <span data-ttu-id="af57d-110">Атрибут file или path в [ \<включают >](../../csharp/programming-guide/xmldoc/include.md) тег отсутствует или имеет неправильный формат.</span><span class="sxs-lookup"><span data-stu-id="af57d-110">The file or path attribute on an [\<include>](../../csharp/programming-guide/xmldoc/include.md) tag was missing or improperly formed.</span></span>
