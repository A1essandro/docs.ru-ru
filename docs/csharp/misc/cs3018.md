# <a name="compiler-warning-level-1-cs3018"></a><span data-ttu-id="d8acb-102">Предупреждение компилятора (уровень 1) CS3018</span><span class="sxs-lookup"><span data-stu-id="d8acb-102">Compiler Warning (level 1) CS3018</span></span>
<span data-ttu-id="d8acb-103">"тип" не может быть помечен как совместимый с CLS, поскольку является членом типа "тип", не совместимого с CLS</span><span class="sxs-lookup"><span data-stu-id="d8acb-103">'type' cannot be marked as CLS-Compliant because it is a member of non CLS-compliant type 'type'</span></span>  
  
 <span data-ttu-id="d8acb-104">Это предупреждение возникает, когда вложенный класс, атрибуту CLSCompliant которого присвоено значение `true` , объявляется как член класса, который объявлен с атрибутом CLSCompliant, равным `false`.</span><span class="sxs-lookup"><span data-stu-id="d8acb-104">This warning occurs if a nested class with the CLSCompliant attribute set to `true` is declared as a member of a class declared with the CLSCompliant attribute set to `false`.</span></span> <span data-ttu-id="d8acb-105">Это недопустимо, так как вложенный класс не может быть совместимым с CLS, если он является членом внешнего класса, несовместимого с CLS.</span><span class="sxs-lookup"><span data-stu-id="d8acb-105">This is not allowed, since a nested class cannot be CLS-compliant if it is a member of an outer class that is not CLS-compliant.</span></span> <span data-ttu-id="d8acb-106">Для устранения этого предупреждения удалите атрибут CLSCompliant из вложенного класса либо измените его значение с `true` на `false`.</span><span class="sxs-lookup"><span data-stu-id="d8acb-106">To resolve this warning, remove the CLSCompliant attribute from the nested class, or change it from `true` to `false`.</span></span> <span data-ttu-id="d8acb-107">Дополнительные сведения о соответствии спецификации CLS см. в разделах [Создание кода, совместимого с CLS](http://msdn.microsoft.com/library/4c705105-69a2-4e5e-b24e-0633bc32c7f3) и [Независимость от языка и независимые от языка компоненты](../../../docs/standard/language-independence-and-language-independent-components.md).</span><span class="sxs-lookup"><span data-stu-id="d8acb-107">For more information on CLS Compliance, see [Writing CLS-Compliant Code](http://msdn.microsoft.com/library/4c705105-69a2-4e5e-b24e-0633bc32c7f3) and [Language Independence and Language-Independent Components](../../../docs/standard/language-independence-and-language-independent-components.md).</span></span>  
  
## <a name="example"></a><span data-ttu-id="d8acb-108">Пример</span><span class="sxs-lookup"><span data-stu-id="d8acb-108">Example</span></span>  
 <span data-ttu-id="d8acb-109">Следующий пример приводит к возникновению предупреждения CS3018:</span><span class="sxs-lookup"><span data-stu-id="d8acb-109">The following sample generates CS3018.</span></span>  
  
```csharp  
// CS3018.cs  
// compile with: /target:library  
using System;  
  
[assembly: CLSCompliant(true)]  
[CLSCompliant(false)]  
public class Outer  
{  
   [CLSCompliant(true)]   // CS3018  
   public class Nested {}  
  
   // OK  
   public class Nested2 {}  
  
   [CLSCompliant(false)]  
   public class Nested3 {}  
}  
```
