# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="d81f1-102">Предупреждение компилятора (уровень 3) CS0414</span><span class="sxs-lookup"><span data-stu-id="d81f1-102">Compiler Warning (level 3) CS0414</span></span>
<span data-ttu-id="d81f1-103">Частное поле field назначено, но его значение не используется</span><span class="sxs-lookup"><span data-stu-id="d81f1-103">The private field 'field' is assigned but its value is never used</span></span>  
  
 <span data-ttu-id="d81f1-104">Это предупреждение может возникнуть в нескольких сценариях, в которых компилятор может проверить, что на переменную не указана ссылка:</span><span class="sxs-lookup"><span data-stu-id="d81f1-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>  
  
-   <span data-ttu-id="d81f1-105">Частному полю назначено значение константы, но оно впоследствии не считывается.</span><span class="sxs-lookup"><span data-stu-id="d81f1-105">A private field is assigned a constant value but is never subsequently read.</span></span> <span data-ttu-id="d81f1-106">Ненужное назначение может снизить производительность.</span><span class="sxs-lookup"><span data-stu-id="d81f1-106">The unnecessary assignment could effect performance.</span></span> <span data-ttu-id="d81f1-107">Рекомендуется удалить поле.</span><span class="sxs-lookup"><span data-stu-id="d81f1-107">Consider removing the field.</span></span>  
  
-   <span data-ttu-id="d81f1-108">Частному или внутреннему статическому полю назначено значение константы только в инициализаторе.</span><span class="sxs-lookup"><span data-stu-id="d81f1-108">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="d81f1-109">Рассмотрите возможность смены поля на константу.</span><span class="sxs-lookup"><span data-stu-id="d81f1-109">Consider changing the field to a const.</span></span>  
  
-   <span data-ttu-id="d81f1-110">Частному или внутреннему полю назначены значения констант, которые используются только в блоках, исключенных директивами #ifdef.</span><span class="sxs-lookup"><span data-stu-id="d81f1-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="d81f1-111">Рекомендуется поместить поле в блок #ifdef.</span><span class="sxs-lookup"><span data-stu-id="d81f1-111">Consider putting the field inside the #ifdef block.</span></span>  
  
-   <span data-ttu-id="d81f1-112">Частному или внутреннему полю назначены значения констант в нескольких расположениях, но доступ к ним отсутствует.</span><span class="sxs-lookup"><span data-stu-id="d81f1-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="d81f1-113">Если вам не нужно это поле, рекомендуется удалить его.</span><span class="sxs-lookup"><span data-stu-id="d81f1-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="d81f1-114">В противном случае используйте его соответствующим образом.</span><span class="sxs-lookup"><span data-stu-id="d81f1-114">Otherwise, use it in some appropriate way.</span></span>  
  
 <span data-ttu-id="d81f1-115">В других ситуациях или в случаях, когда предлагаемый обходной путь недопустим, используйте директиву #pragma 0414.</span><span class="sxs-lookup"><span data-stu-id="d81f1-115">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>  
  
 <span data-ttu-id="d81f1-116">В следующем примере представлена ситуация, в которой возникнет ошибка CS0414:</span><span class="sxs-lookup"><span data-stu-id="d81f1-116">The following sample shows one way in which CS0414 will be generated:</span></span>  
  
```csharp  
// CS0414  
// compile with: /W3  
class C  
{  
   private int i = 1;  // CS0414  
  
   public static void Main()  
   { }  
}  
```  
  
 <span data-ttu-id="d81f1-117">**Примечание.** Если переменная `i` объявлена как `protected or public`, ошибка не возникнет, так как компилятор не знает, может ли производный класс использовать ее, или создать экземпляры класса и сослаться на переменную может другой клиентский код</span><span class="sxs-lookup"><span data-stu-id="d81f1-117">**Note** If the variable `i` is declared as `protected or public`, no error will be generated because the compiler cannot know whether a derived class might use it or some other client code might instantiate the class and reference the variable</span></span>
