## <a name="example"></a><span data-ttu-id="35b06-105">Пример</span><span class="sxs-lookup"><span data-stu-id="35b06-105">Example</span></span>  
 <span data-ttu-id="35b06-106">В следующем примере возникает ошибка CS0135:</span><span class="sxs-lookup"><span data-stu-id="35b06-106">The following sample generates CS0135:</span></span>  
  
```csharp
// CS0135.cs  
public class MyClass2  
{  
   public static int i = 0;  
  
   public static void Main()  
   {  
      {  
         int i = 4;  
         i++;  
      }  
      i = 0;   // CS0135  
   }  
}  
```

 <span data-ttu-id="35b06-108">Для каждого вхождения данного идентификатора в качестве простого имени в выражении или деклараторе в пространстве объявлений локальной переменной (§3.3), непосредственно включающем это вхождение, каждое следующее вхождение того же идентификатора в качестве простого имени в выражении или деклараторе должно ссылаться на ту же сущность.</span><span class="sxs-lookup"><span data-stu-id="35b06-108">For each occurrence of a given identifier as a simple-name in an expression or declarator, within the local variable declaration space (§3.3) immediately enclosing that occurrence, every other occurrence of the same identifier as a simple-name in an expression or declarator must refer to the same entity.</span></span> <span data-ttu-id="35b06-109">Это правило позволяет гарантировать, что внутри определенного блока, блока switch, операторов for, foreach и using или анонимной функции имя всегда имеет одинаковое значение.</span><span class="sxs-lookup"><span data-stu-id="35b06-109">This rule ensures that the meaning of a name is always the same within a given block, switch block, for-, foreach- or using-statement, or anonymous function.</span></span>
