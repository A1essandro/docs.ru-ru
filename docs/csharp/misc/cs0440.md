# <a name="compiler-warning-level-2-cs0440"></a><span data-ttu-id="1856e-102">Предупреждение компилятора (уровень 2) CS0440</span><span class="sxs-lookup"><span data-stu-id="1856e-102">Compiler Warning (level 2) CS0440</span></span>
<span data-ttu-id="1856e-103">Определение псевдонима с именем "global" не рекомендуется из-за того, что "global::" всегда указывает на глобальное пространство имен и не является псевдонимом.</span><span class="sxs-lookup"><span data-stu-id="1856e-103">Defining an alias named 'global' is ill-advised since 'global::' always references the global namespace and not an alias</span></span>  
  
 <span data-ttu-id="1856e-104">Это предупреждение возникает при определении псевдонима с именем global.</span><span class="sxs-lookup"><span data-stu-id="1856e-104">This warning is issued when you define an alias named global.</span></span>  
  
## <a name="example"></a><span data-ttu-id="1856e-105">Пример</span><span class="sxs-lookup"><span data-stu-id="1856e-105">Example</span></span>  
 <span data-ttu-id="1856e-106">В следующем примере возникает ошибка CS0440:</span><span class="sxs-lookup"><span data-stu-id="1856e-106">The following example generates CS0440:</span></span>  
  
```csharp  
// CS0440.cs  
// Compile with: /W:1  
  
using global = MyClass;   // CS0440  
class MyClass  
{  
    static void Main()  
    {  
        // Note how global refers to the global namespace  
        // even though it is redefined above.  
        global::System.Console.WriteLine();  
    }  
}  
```
