 <span data-ttu-id="c77fb-104">Излишне одновременно помечать метод как [абстрактный](../../csharp/language-reference/keywords/abstract.md) и как [виртуальный](../../csharp/language-reference/keywords/virtual.md) , потому что подразумевается, что **абстрактный** метод является **виртуальным**.</span><span class="sxs-lookup"><span data-stu-id="c77fb-104">It is redundant to mark a member method as both [abstract](../../csharp/language-reference/keywords/abstract.md) and [virtual](../../csharp/language-reference/keywords/virtual.md) because **abstract** implies **virtual**.</span></span>

```  
   abstract public class clx  
   {  
      abstract virtual public void f();   // CS0503  
   }

```
