# <a name="compiler-error-cs0503"></a><span data-ttu-id="0f3d0-102">Ошибка компилятора CS0503</span><span class="sxs-lookup"><span data-stu-id="0f3d0-102">Compiler Error CS0503</span></span>
<span data-ttu-id="0f3d0-103">Абстрактный метод "метод" не может быть помечен как виртуальный</span><span class="sxs-lookup"><span data-stu-id="0f3d0-103">The abstract method 'method' cannot be marked virtual</span></span>  
  
 <span data-ttu-id="0f3d0-104">Излишне одновременно помечать метод как [абстрактный](../../csharp/language-reference/keywords/abstract.md) и как [виртуальный](../../csharp/language-reference/keywords/virtual.md) , потому что подразумевается, что **абстрактный** метод является **виртуальным**.</span><span class="sxs-lookup"><span data-stu-id="0f3d0-104">It is redundant to mark a member method as both [abstract](../../csharp/language-reference/keywords/abstract.md) and [virtual](../../csharp/language-reference/keywords/virtual.md) because **abstract** implies **virtual**.</span></span>  
  
 <span data-ttu-id="0f3d0-105">Следующий пример приводит к возникновению ошибки CS0503:</span><span class="sxs-lookup"><span data-stu-id="0f3d0-105">The following sample generates CS0503:</span></span>  
  
```csharp

{  
   abstract public class clx  
   {  
      abstract virtual public void f();   // CS0503  
   }  
}  
```
