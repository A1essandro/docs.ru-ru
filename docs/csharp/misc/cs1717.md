# <a name="compiler-warning-level-3-cs1717"></a><span data-ttu-id="002cf-102">Предупреждение компилятора (уровень 3) CS1717</span><span class="sxs-lookup"><span data-stu-id="002cf-102">Compiler Warning (level 3) CS1717</span></span>
<span data-ttu-id="002cf-103">Проведено присвоение той же переменной; действительно выполнить такое назначение, а не иное?</span><span class="sxs-lookup"><span data-stu-id="002cf-103">Assignment made to same variable; did you mean to assign something else?</span></span>  
  
 <span data-ttu-id="002cf-104">Это предупреждение возникает при назначении переменной самой себе, например `a = a`.</span><span class="sxs-lookup"><span data-stu-id="002cf-104">This warning occurs when you assign a variable to itself, such as `a = a`.</span></span>  
  
 <span data-ttu-id="002cf-105">Это предупреждение создается в результате нескольких распространенных ошибок.</span><span class="sxs-lookup"><span data-stu-id="002cf-105">Several common mistakes can generate this warning:</span></span>  
  
-   <span data-ttu-id="002cf-106">Указание `a = a` в качестве условия инструкции **if** , например `if (a = a)`.</span><span class="sxs-lookup"><span data-stu-id="002cf-106">Writing `a = a` as the condition of an **if** statement, such as `if (a = a)`.</span></span> <span data-ttu-id="002cf-107">Возможно, имелось в виду условие `if (a == a)`, которое всегда имеет значение true, так что это можно написать более сжато как `if (true)`.</span><span class="sxs-lookup"><span data-stu-id="002cf-107">You probably meant to say `if (a == a)`, which is always true, so you could write this more concisely as `if (true)`.</span></span>  
  
-   <span data-ttu-id="002cf-108">Опечатки.</span><span class="sxs-lookup"><span data-stu-id="002cf-108">Mistyping.</span></span> <span data-ttu-id="002cf-109">Возможно, имелось в виду `a = b`.</span><span class="sxs-lookup"><span data-stu-id="002cf-109">You probably meant to say `a = b`.</span></span>  
  
-   <span data-ttu-id="002cf-110">В конструкторе, где параметр имеет то же имя, что и поле, не используется ключевое слово **this** : возможно, имелось в виду `this.a = a`.</span><span class="sxs-lookup"><span data-stu-id="002cf-110">In a constructor where the parameter has the same name as the field, not using the **this** keyword: you probably meant to say `this.a = a`.</span></span>
