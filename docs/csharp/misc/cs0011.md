 <span data-ttu-id="9247b-104">Класс, который был импортирован из файла с помощью **/reference**, является производным от не найденного класса или реализует не найденный интерфейс.</span><span class="sxs-lookup"><span data-stu-id="9247b-104">A class that was imported from a file with **/reference**, is derived from a class or implements an interface that is not found.</span></span> <span data-ttu-id="9247b-105">Это может произойти, если требуемая библиотека DLL не включена в компиляцию с помощью **/reference**.</span><span class="sxs-lookup"><span data-stu-id="9247b-105">This can occur if a required DLL is not also included in the compilation with **/reference**.</span></span>

## <a name="example"></a><span data-ttu-id="9247b-107">Пример</span><span class="sxs-lookup"><span data-stu-id="9247b-107">Example</span></span>  
  
```csharp  
// CS0011_1.cs  
// compile with: /target:library  
  
public class Outer   
{  
   public class B { }  
}  
```  
  
## <a name="example"></a><span data-ttu-id="9247b-108">Пример</span><span class="sxs-lookup"><span data-stu-id="9247b-108">Example</span></span>  
 <span data-ttu-id="9247b-109">Второй файл создает библиотеку DLL, которая определяет класс `C` , производный от класса `B` , созданного в предыдущем примере.</span><span class="sxs-lookup"><span data-stu-id="9247b-109">The second file creates a DLL that defines a class `C` that is derived from the class `B` that was created in the previous example.</span></span>  
  
```  
// CS0011_2.cs  
// compile with: /target:library /reference:CS0011_1.dll  
// post-build command: del /f CS0011_1.dll  
public class C : Outer.B {}  
```  
  
## <a name="example"></a><span data-ttu-id="9247b-110">Пример</span><span class="sxs-lookup"><span data-stu-id="9247b-110">Example</span></span>  
 <span data-ttu-id="9247b-111">Третий файл заменяет библиотеку DLL, созданную на первом шаге, и пропускает определение внутреннего класса `B`.</span><span class="sxs-lookup"><span data-stu-id="9247b-111">The third file replaces the DLL created by the first step, and omits the definition of the inner class `B`.</span></span>  
  
```

```  
  
## <a name="example"></a><span data-ttu-id="9247b-112">Пример</span><span class="sxs-lookup"><span data-stu-id="9247b-112">Example</span></span>  
 <span data-ttu-id="9247b-113">Наконец, четвертый файл ссылается на класс `C` , определенный во втором примере, являющийся производным от класса `B`, который в данный момент отсутствует.</span><span class="sxs-lookup"><span data-stu-id="9247b-113">Finally, the fourth file references the class `C` defined in the second example, which is derived from class `B`, and which is now missing.</span></span>
