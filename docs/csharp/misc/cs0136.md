# <a name="compiler-error-cs0136"></a><span data-ttu-id="fc44a-102">Ошибка компилятора CS0136</span><span class="sxs-lookup"><span data-stu-id="fc44a-102">Compiler Error CS0136</span></span>
<span data-ttu-id="fc44a-103">Невозможно объявить локальную переменную с именем var в этой области видимости, т. к. она придаст другое значение var, которое уже используется в области видимости "родительская или текущая/дочерняя" для обозначения чего-то другого</span><span class="sxs-lookup"><span data-stu-id="fc44a-103">A local variable named 'var' cannot be declared in this scope because it would give a different meaning to 'var', which is already used in a 'parent or current/child' scope to denote something else</span></span>  
  
 <span data-ttu-id="fc44a-104">Объявление переменной скрывает другое объявление, которое в противном случае присутствовало бы в области.</span><span class="sxs-lookup"><span data-stu-id="fc44a-104">A variable declaration hides another declaration that would otherwise be in scope.</span></span> <span data-ttu-id="fc44a-105">Переименуйте переменную, объявленную в строке, которая создала ошибку CS0136.</span><span class="sxs-lookup"><span data-stu-id="fc44a-105">Rename the variable that is declared on the line that generated CS0136.</span></span>  
  
## <a name="example"></a><span data-ttu-id="fc44a-106">Пример</span><span class="sxs-lookup"><span data-stu-id="fc44a-106">Example</span></span>  
 <span data-ttu-id="fc44a-107">Следующий пример приводит к возникновению ошибки CS0136:</span><span class="sxs-lookup"><span data-stu-id="fc44a-107">The following sample generates CS0136:</span></span>  
  
```csharp
// CS0136.cs  
namespace MyNamespace  
{  
   public class MyClass  
   {  
      public static void Main()  
      {  
         int i = 0;  
         {  
            char i = 'a';   // CS0136, hides int i  
         }  
         i++;  
      }  
   }  
}  
```

 <span data-ttu-id="fc44a-109">Для каждого вхождения данного идентификатора в качестве простого имени в выражении или деклараторе в пространстве объявлений локальной переменной (§3.3), непосредственно включающем это вхождение, каждое следующее вхождение того же идентификатора в качестве простого имени в выражении или деклараторе должно ссылаться на ту же сущность.</span><span class="sxs-lookup"><span data-stu-id="fc44a-109">For each occurrence of a given identifier as a simple-name in an expression or declarator, within the local variable declaration space (§3.3) immediately enclosing that occurrence, every other occurrence of the same identifier as a simple-name in an expression or declarator must refer to the same entity.</span></span> <span data-ttu-id="fc44a-110">Это правило позволяет гарантировать, что внутри определенного блока, блока switch, операторов for, foreach и using или анонимной функции имя всегда имеет одинаковое значение.</span><span class="sxs-lookup"><span data-stu-id="fc44a-110">This rule ensures that the meaning of a name is always the same within a given block, switch block, for-, foreach- or using-statement, or anonymous function.</span></span>
