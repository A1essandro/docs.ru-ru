<span data-ttu-id="d4f76-103">Квалификатор псевдонима пространства имен "::" всегда разрешается в тип или пространство имен, что в данном случае недопустимо.</span><span class="sxs-lookup"><span data-stu-id="d4f76-103">The namespace alias qualifier '::' always resolves to a type or namespace so is illegal here.</span></span> <span data-ttu-id="d4f76-104">Рассмотрите возможность использования "." вместо "::".</span><span class="sxs-lookup"><span data-stu-id="d4f76-104">Consider using '.' instead.</span></span>  
  
 <span data-ttu-id="d4f76-105">Эта ошибка возникает, если нечто, интерпретируемое средством синтаксического анализа как тип, используется в неподходящем месте.</span><span class="sxs-lookup"><span data-stu-id="d4f76-105">This error occurs if you used something which the parser interpreted as a type in an unexpected place.</span></span> <span data-ttu-id="d4f76-106">Имя типа или пространства имен допустимо только в выражении доступа к членам, с использованием оператора доступа к членам (**.**).</span><span class="sxs-lookup"><span data-stu-id="d4f76-106">A type or namespace name is valid only in a member access expression, using the member access (**.**) operator.</span></span> <span data-ttu-id="d4f76-107">Это может произойти при использовании оператора глобальной области (::) в другом контексте.</span><span class="sxs-lookup"><span data-stu-id="d4f76-107">This could occur if you used the global scope operator (::) in another context.</span></span>  
  
## <a name="example"></a><span data-ttu-id="d4f76-108">Пример</span><span class="sxs-lookup"><span data-stu-id="d4f76-108">Example</span></span>  
 <span data-ttu-id="d4f76-109">В следующем примере происходит ошибка CS0687.</span><span class="sxs-lookup"><span data-stu-id="d4f76-109">The following sample generates CS0687:</span></span>  
  
```  
// CS0687.cs  
  
using M = Test;  
using System;  
  
public class Test   
{  
    public static int x = 77;  
  
    public static void Main()   
    {  
        Console.WriteLine(M::x); // CS0687  
        // To resolve use the following line instead:  
        // Console.WriteLine(M.x);  
    }  
}  
```
