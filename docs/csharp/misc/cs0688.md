# <a name="compiler-warning-level-1-cs0688"></a><span data-ttu-id="52fde-102">Предупреждение компилятора (уровень 1) CS0688</span><span class="sxs-lookup"><span data-stu-id="52fde-102">Compiler Warning (level 1) CS0688</span></span>
<span data-ttu-id="52fde-103">Метод "метод1" содержит запрос компоновки, но переопределяет или реализует метод "метод2", который не имеет запрос компоновки.</span><span class="sxs-lookup"><span data-stu-id="52fde-103">'method1' has a link demand, but overrides or implements 'method2' which does not have a link demand.</span></span> <span data-ttu-id="52fde-104">Может возникнуть нарушение защиты.</span><span class="sxs-lookup"><span data-stu-id="52fde-104">A security hole may exist.</span></span>  
  
 <span data-ttu-id="52fde-105">Запрос компоновки, установленный в методе производного класса, можно легко обойти, вызвав метод базового класса.</span><span class="sxs-lookup"><span data-stu-id="52fde-105">The link demand set up on the derived class method can easily be circumvented by calling the base class method.</span></span> <span data-ttu-id="52fde-106">Чтобы закрыть эту брешь в системе безопасности, метод базового класса также должен использовать запрос компоновки.</span><span class="sxs-lookup"><span data-stu-id="52fde-106">To close the security hole, the base class method needs to also use the link demand.</span></span> <span data-ttu-id="52fde-107">Дополнительные сведения см. в разделе [требовать vs. LinkDemand](http://msdn.microsoft.com/en-us/1ab877f2-70f4-4e0d-8116-943999dfe8f5).</span><span class="sxs-lookup"><span data-stu-id="52fde-107">For more information, see [Demand vs. LinkDemand](http://msdn.microsoft.com/en-us/1ab877f2-70f4-4e0d-8116-943999dfe8f5).</span></span>  
  
## <a name="example"></a><span data-ttu-id="52fde-108">Пример</span><span class="sxs-lookup"><span data-stu-id="52fde-108">Example</span></span>  
 <span data-ttu-id="52fde-109">В следующем примере возникает ошибка CS0688.</span><span class="sxs-lookup"><span data-stu-id="52fde-109">The following sample generates CS0688.</span></span> <span data-ttu-id="52fde-110">Чтобы устранить это предупреждение без изменения базового класса, удалите атрибут безопасности из переопределяющего метода.</span><span class="sxs-lookup"><span data-stu-id="52fde-110">To resolve the warning without modifying the base class, remove the security attribute from the overriding method.</span></span> <span data-ttu-id="52fde-111">Это не решит проблему безопасности.</span><span class="sxs-lookup"><span data-stu-id="52fde-111">This will not solve the security problem.</span></span>  
  
```  
// CS0688.cs  
// compile with: /W:1  
using System;  
using System.Security.Permissions;  
  
class Base   
{  
    //Uncomment the following line to close the security hole  
    //[FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")]  
    public virtual void DoScaryFileStuff()  
    {  
    }  
}  
  
class Derived: Base  
{  
    [FileIOPermission(SecurityAction.LinkDemand, All=@"C:\\")] // CS0688  
    public override void DoScaryFileStuff()  
    {  
    }  
    static void Main()  
    {  
    }  
}  
```
