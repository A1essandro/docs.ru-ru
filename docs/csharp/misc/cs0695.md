# <a name="compiler-error-cs0695"></a><span data-ttu-id="ab8a1-102">Ошибка компилятора CS0695</span><span class="sxs-lookup"><span data-stu-id="ab8a1-102">Compiler Error CS0695</span></span>
<span data-ttu-id="ab8a1-103">"универсальный тип" не в состоянии реализовать ни "универсальный интерфейс", ни "универсальный интерфейс", так как они могут быть идентичными для некоторых подстановок параметров типа</span><span class="sxs-lookup"><span data-stu-id="ab8a1-103">'generic type' cannot implement both 'generic interface' and 'generic interface' because they may unify for some type parameter substitutions</span></span>  
  
 <span data-ttu-id="ab8a1-104">Эта ошибка возникает, если универсальный класс реализует несколько параметризаций одного универсального интерфейса и существует замена параметров типа, которая приведет к тому, что интерфейсы станут идентичными.</span><span class="sxs-lookup"><span data-stu-id="ab8a1-104">This error occurs when a generic class implements more than one parameterization of the same generic interface, and there exists a type parameter substitution which would make the two interfaces identical.</span></span> <span data-ttu-id="ab8a1-105">Чтобы избежать этой ошибки, реализуйте только один из интерфейсов или замените параметры типа, чтобы избежать конфликта.</span><span class="sxs-lookup"><span data-stu-id="ab8a1-105">To avoid this error, implement only one of the interfaces, or change the type parameters to avoid the conflict.</span></span>  
  
 <span data-ttu-id="ab8a1-106">Следующий пример приводит к возникновению ошибки CS0695:</span><span class="sxs-lookup"><span data-stu-id="ab8a1-106">The following sample generates CS0695:</span></span>  
  
```csharp  
// CS0695.cs  
// compile with: /target:library  
  
interface I<T>  
{  
}  
  
class G<T1, T2> : I<T1>, I<T2>  // CS0695  
{  
}  
```
