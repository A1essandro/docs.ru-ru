# <a name="compiler-error-cs0473"></a><span data-ttu-id="48e4f-102">Ошибка компилятора CS0473</span><span class="sxs-lookup"><span data-stu-id="48e4f-102">Compiler Error CS0473</span></span>
<span data-ttu-id="48e4f-103">Реализация явного интерфейса "имя_метода" соответствует более чем одному членом интерфейса.</span><span class="sxs-lookup"><span data-stu-id="48e4f-103">Explicit interface implementation 'method name' matches more than one interface member.</span></span> <span data-ttu-id="48e4f-104">Фактический выбор конкретного члена интерфейса зависит от реализации.</span><span class="sxs-lookup"><span data-stu-id="48e4f-104">Which interface member is actually chosen is implementation-dependent.</span></span> <span data-ttu-id="48e4f-105">Попробуйте вместо явной реализации использовать неявную реализацию интерфейса.</span><span class="sxs-lookup"><span data-stu-id="48e4f-105">Consider using a non-explicit implementation instead.</span></span>  
  
 <span data-ttu-id="48e4f-106">В некоторых случаях универсальный метод может получить такую же сигнатуру, что и неуниверсальный метод.</span><span class="sxs-lookup"><span data-stu-id="48e4f-106">In some cases a generic method might acquire the same signature as a non-generic method.</span></span> <span data-ttu-id="48e4f-107">Проблема заключается в том, что система метаданных Common Language Infrastructure (CLI) никаким способом не может однозначно установить, какой метод привязан к какому слоту.</span><span class="sxs-lookup"><span data-stu-id="48e4f-107">The problem is that there is no way in the common language infrastructure (CLI) metadata system to unambiguously state which method binds to which slot.</span></span> <span data-ttu-id="48e4f-108">Это определение делается на усмотрение CLI.</span><span class="sxs-lookup"><span data-stu-id="48e4f-108">It is up to the CLI to make that determination.</span></span>  
  
> [!NOTE]

## <a name="to-correct-this-error"></a><span data-ttu-id="48e4f-110">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="48e4f-110">To correct this error</span></span>  
  
1.  <span data-ttu-id="48e4f-111">Удалите явную реализацию и оставьте только неявную реализацию `public int TestMethod(int)` в обоих методах интерфейса.</span><span class="sxs-lookup"><span data-stu-id="48e4f-111">Eliminate the explicit implementation and just have the implicit implementation `public int TestMethod(int)` implement both of the interface methods.</span></span>  
  
## <a name="example"></a><span data-ttu-id="48e4f-112">Пример</span><span class="sxs-lookup"><span data-stu-id="48e4f-112">Example</span></span>  
 <span data-ttu-id="48e4f-113">Следующий код приводит к возникновению ошибки CS0473:</span><span class="sxs-lookup"><span data-stu-id="48e4f-113">The following code generates CS0473:</span></span>  
  
```csharp  
// cs0473.cs  
public interface ITest<T>  
{  
    int TestMethod(int i);  
    int TestMethod(T i);  
}  
  
public class ImplementingClass : ITest<int>  
{  
    int ITest<int>.TestMethod(int i) // CS0473  
    {  
        return i + 1;  
    }  
  
    public int TestMethod(int i)  
    {  
        return i - 1;  
    }  
}  
  
class T  
{  
    static int Main()  
    {  
        ImplementingClass a = new ImplementingClass();  
        if (a.TestMethod(0) != -1)  
            return -1;  
  
        ITest<int> i_a = a;  
        System.Console.WriteLine(i_a.TestMethod(0).ToString());  
        if (i_a.TestMethod(0) != 1)  
            return -1;  
  
        return 0;  
    }  
}  
```
