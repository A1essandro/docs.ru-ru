 <span data-ttu-id="cc646-104">Существует конфликт имен между параметром типа универсального метода и локальной переменной в методе или один из параметров метода.</span><span class="sxs-lookup"><span data-stu-id="cc646-104">There is a name conflict between the type parameter of a generic method and a local variable in the method or one of the method's parameters.</span></span> <span data-ttu-id="cc646-105">Чтобы избежать этой ошибки, переименуйте все конфликтующие параметры и локальные переменные.</span><span class="sxs-lookup"><span data-stu-id="cc646-105">To avoid this error, rename any conflicting parameters or local variables.</span></span>  
  
## <a name="example"></a><span data-ttu-id="cc646-106">Пример</span><span class="sxs-lookup"><span data-stu-id="cc646-106">Example</span></span>  
 <span data-ttu-id="cc646-107">Следующий пример приводит к возникновению ошибки CS0412:</span><span class="sxs-lookup"><span data-stu-id="cc646-107">The following sample generates CS0412:</span></span>  
  
```csharp  
// CS0412.cs  
using System;  
  
class C  
{  
    // Parameter name is the same as method type parameter name  
    public void G<T>(int T)  // CS0412  
    {  
    }  
    public void F<T>()  
    {  
        // Method local variable name is the same as method type  
        // parameter name  
        double T = 0.0;  // CS0412  
        Console.WriteLine(T);  
    }  
  
    public static void Main()  
    {  
    }  
}  
```
