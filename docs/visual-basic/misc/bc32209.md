# <a name="project-ltprojectnamegt-requires-a-reference-to-version-ltversionnumber1gt-of-assembly-ltassemblynamegt-but-references-version-ltversionnumber2gt-of-assembly-ltassemblynamegt-visual-basic-error"></a><span data-ttu-id="9ca1d-102">Проект "&lt;имя_проекта&gt;«требует ссылку на версию»&lt;номер_версии1&gt;«из сборки»&lt;assemblyname&gt;", но ссылается на версию "&lt; номер_версии2&gt;«из сборки»&lt;assemblyname&gt;"(ошибка Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="9ca1d-102">Project '&lt;projectname&gt;' requires a reference to version '&lt;versionnumber1&gt;' of assembly '&lt;assemblyname&gt;', but references version '&lt;versionnumber2&gt;' of assembly '&lt;assemblyname&gt;' (Visual Basic Error)</span></span>
<span data-ttu-id="9ca1d-103">Проект делает косвенную ссылку на сборку, которая определена в каком-либо месте, но проект также имеет прямую ссылку на более позднюю версию этой сборки.</span><span class="sxs-lookup"><span data-stu-id="9ca1d-103">A project makes an indirect reference to an assembly that is defined elsewhere, but the project also has a direct reference to a later version of that assembly.</span></span>  
  
 <span data-ttu-id="9ca1d-104">Если компилятор разрешил вашему коду использовать косвенную ссылку, возможно, вы не сможете получить доступ к типам и программным элементам, которые были определены определен в более поздней версии, но не в более ранней версии.</span><span class="sxs-lookup"><span data-stu-id="9ca1d-104">If the compiler allowed your code to use the indirect reference, you might not be able to access types and programming elements that are defined in the later version but not in the earlier version.</span></span>
