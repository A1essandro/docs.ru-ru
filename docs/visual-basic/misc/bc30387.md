# <a name="class-ltclassnamegt-must-declare-a-sub-new-because-its-base-class-ltclassname2gt-does-not-have-an-accessible-sub-new-that-can-be-called-with-no-arguments"></a><span data-ttu-id="17ebd-102">Класс&lt;classname&gt;"должен объявить «Sub New», так как его базовый класс&lt;имя_класса2&gt;" не имеет доступного «Sub New», который может быть вызван без аргументов</span><span class="sxs-lookup"><span data-stu-id="17ebd-102">Class '&lt;classname&gt;' must declare a 'Sub New' because its base class '&lt;classname2&gt;' does not have an accessible 'Sub New' that can be called with no arguments</span></span>
<span data-ttu-id="17ebd-103">Производный класс не объявляет конструктор и Visual Basic не может создать его, так как отсутствует, он может вызвать конструктор базового класса.</span><span class="sxs-lookup"><span data-stu-id="17ebd-103">A derived class does not declare a constructor, and Visual Basic cannot generate one because there is no base class constructor it can call.</span></span>  
  
 <span data-ttu-id="17ebd-104">Если производный класс не объявляет конструктор, Visual Basic пытается создать неявный конструктор без параметров, который вызывает `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="17ebd-104">When a derived class does not declare a constructor, Visual Basic attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="17ebd-105">Если нет доступного конструктора в базовом классе, которые могут вызываться без аргументов или если имеется более одного, Visual Basic не удается создать неявный конструктор.</span><span class="sxs-lookup"><span data-stu-id="17ebd-105">If there is no accessible constructor in the base class that can be called without arguments, or if there is more than one, Visual Basic cannot generate an implicit constructor.</span></span>  
  
 <span data-ttu-id="17ebd-106">**Идентификатор ошибки:** BC30387</span><span class="sxs-lookup"><span data-stu-id="17ebd-106">**Error ID:** BC30387</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="17ebd-107">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="17ebd-107">To correct this error</span></span>  
  
1.  <span data-ttu-id="17ebd-108">Объявите и реализуйте по крайней мере один конструктор `Sub New` где-нибудь в производном классе.</span><span class="sxs-lookup"><span data-stu-id="17ebd-108">Declare and implement at least one `Sub New` constructor somewhere in the derived class.</span></span>  
  
2.  <span data-ttu-id="17ebd-109">Добавьте вызов конструктора базового класса `MyBase.New()`в качестве первой строки каждого `Sub New`.</span><span class="sxs-lookup"><span data-stu-id="17ebd-109">Add a call to a base class constructor, `MyBase.New()`, as the first line of every `Sub New`.</span></span>
