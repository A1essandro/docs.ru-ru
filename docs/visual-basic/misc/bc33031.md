# <a name="conversion-operators-cannot-convert-from-a-derived-type"></a><span data-ttu-id="a0608-102">Операторы преобразования не могут выполнять преобразование из производного типа.</span><span class="sxs-lookup"><span data-stu-id="a0608-102">Conversion operators cannot convert from a derived type</span></span>
<span data-ttu-id="a0608-103">Оператор преобразования объявлен с параметром типа, производным от типа возврата.</span><span class="sxs-lookup"><span data-stu-id="a0608-103">A conversion operator is declared with a parameter type that derives from the return type.</span></span>  
  
 <span data-ttu-id="a0608-104">Во время компиляции [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] считает, что существует предопределенное преобразование из любого ссылочного типа в любой тип в его иерархии наследования, то есть в любой тип, от которого он является производным или который является производным от него.</span><span class="sxs-lookup"><span data-stu-id="a0608-104">At compile time, [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] considers a predefined conversion to exist from any reference type to any type in its inheritance hierarchy, that is, any type from which it derives or which derives from it.</span></span> <span data-ttu-id="a0608-105">Такое преобразование может вызвать сбой во время выполнения, однако компилятор не может предсказать результаты выполнения, поэтому он позволяет компилировать такие преобразования.</span><span class="sxs-lookup"><span data-stu-id="a0608-105">Such a conversion might fail at run time, but the compiler cannot predict run-time results, so it allows any such conversion to compile.</span></span>  
  
 <span data-ttu-id="a0608-106">Так как компилятор считает, что это преобразование уже определено, он не позволяет переопределить его.</span><span class="sxs-lookup"><span data-stu-id="a0608-106">Because the compiler considers this conversion to be already defined, it does not allow you to redefine it.</span></span>
