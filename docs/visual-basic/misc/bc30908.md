# <a name="type-typename-cannot-inherit-from-a-type-nested-within-it"></a><span data-ttu-id="f320b-102">Тип "\<typename >" не может наследовать от вложенного в него типа</span><span class="sxs-lookup"><span data-stu-id="f320b-102">Type '\<typename>' cannot inherit from a type nested within it</span></span>
<span data-ttu-id="f320b-103">Определение класса или интерфейса включает инструкцию [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) , указывающую вложенный в него тип.</span><span class="sxs-lookup"><span data-stu-id="f320b-103">A class or interface definition includes an [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) that specifies a type nested within it.</span></span>  
  
 <span data-ttu-id="f320b-104">Наследование должно быть линейным, не циклическим.</span><span class="sxs-lookup"><span data-stu-id="f320b-104">Inheritance must be linear, not circular.</span></span> <span data-ttu-id="f320b-105">Тип не может наследовать от типа, который наследует от него.</span><span class="sxs-lookup"><span data-stu-id="f320b-105">A type cannot inherit from a type that inherits from it.</span></span>  
  
 <span data-ttu-id="f320b-106">Связанное ограничение — тип не может наследовать от типа, который еще не определен.</span><span class="sxs-lookup"><span data-stu-id="f320b-106">A related restriction is that a type cannot inherit from a type that is not yet defined.</span></span> <span data-ttu-id="f320b-107">Наследование включает возможность повторного использования элементов базового класса, а для этого необходимо, чтобы эти элементы были определены.</span><span class="sxs-lookup"><span data-stu-id="f320b-107">Inheritance involves the ability to reuse members of the base class, which in turn requires that these members be defined.</span></span> <span data-ttu-id="f320b-108">Таким образом Visual Basic не удается скомпилировать код, как в следующем примере.</span><span class="sxs-lookup"><span data-stu-id="f320b-108">Therefore, Visual Basic cannot compile code such as the following example.</span></span>  
  
```  
Public Class outerClass  
    ' The following statement is INVALID because innerClass is not defined.  
    Inherits innerClass  
    Public Class innerClass  
        Public Sub doSomething()  
        End Sub  
    End Class  
End Class  
```
