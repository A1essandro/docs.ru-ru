# <a name="data-types-of-the-type-parameters-in-extension-method-39ltmethodnamegt39-defined-in-39lttypenamegt39-cannot-be-inferred-from-these-arguments-because-more-than-one-type-is-possible"></a><span data-ttu-id="e1baa-102">Типы данных параметров типа в методе расширения &#39; &lt;имя_метода&gt; &#39; определенные в &#39; &lt;typename&gt; &#39; не могут выводиться из этих аргументов, так как несколько возможен тип</span><span class="sxs-lookup"><span data-stu-id="e1baa-102">Data type(s) of the type parameter(s) in extension method &#39;&lt;methodname&gt;&#39; defined in &#39;&lt;typename&gt;&#39; cannot be inferred from these arguments because more than one type is possible</span></span>
<span data-ttu-id="e1baa-103">Типы данных параметров типа в методе расширения "\<имя_метода >" определен в "\<typename >" не могут выводиться из этих аргументов, так как возможны несколько типов.</span><span class="sxs-lookup"><span data-stu-id="e1baa-103">Data type(s) of the type parameter(s) in extension method '\<methodname>' defined in '\<typename>' cannot be inferred from these arguments because more than one type is possible.</span></span> <span data-ttu-id="e1baa-104">Эту ошибку может исправить явное указание типов данных.</span><span class="sxs-lookup"><span data-stu-id="e1baa-104">Specifying the data type(s) explicitly might correct this error.</span></span>  
  
 <span data-ttu-id="e1baa-105">Предпринята попытка использования вывода типов для определения типа (или типов) параметра (или параметров) типа в вызове универсального метода расширения.</span><span class="sxs-lookup"><span data-stu-id="e1baa-105">An attempt has been made to use type inference to determine the type (or types) of the type parameter (or parameters) in a call to a generic extension method.</span></span> <span data-ttu-id="e1baa-106">Компилятор обнаружил несколько возможных типов данных для одного или нескольких параметров типа и сообщает об ошибке.</span><span class="sxs-lookup"><span data-stu-id="e1baa-106">The compiler finds more than one possible data type for one or more of the type parameters, and it reports this error.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="e1baa-107">Когда указание аргументов является обязательным (например, в операторах выражений запросов), это сообщение об ошибке появляется без второй фразы.</span><span class="sxs-lookup"><span data-stu-id="e1baa-107">When specifying arguments is not an option (for example, for query operators in query expressions), the error message appears without the second sentence.</span></span>  
  
 <span data-ttu-id="e1baa-108">Эта ошибка демонстрируется в приведенном ниже коде.</span><span class="sxs-lookup"><span data-stu-id="e1baa-108">The following code demonstrates the error.</span></span>  
  
```vb  
Option Strict Off  
Imports System.Runtime.CompilerServices  
Module Module1  
    Sub Main()  
  
        Dim caller As New Class1  
        '' Not valid.  
        'caller.targetExtension(1, "2")  
  
    End Sub  
  
    <Extension()> _  
    Sub targetExtension(Of T)(ByVal p0 As Class1, ByVal p1 As T, ByVal p2 As T)  
    End Sub  
  
    Class Class1  
    End Class  
  
End Module  
```
