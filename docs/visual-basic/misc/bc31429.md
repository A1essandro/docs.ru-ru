# <a name="ltelementnamegt-is-ambiguous-because-multiple-kinds-of-members-with-this-name-exist-in-lttypegt-lttypenamegt"></a><span data-ttu-id="dc9f1-102">"&lt;elementname&gt;" является неоднозначным, поскольку несколько видов членов с таким именем существует в &lt;тип&gt; "&lt;typename&gt;"</span><span class="sxs-lookup"><span data-stu-id="dc9f1-102">'&lt;elementname&gt;' is ambiguous because multiple kinds of members with this name exist in &lt;type&gt; '&lt;typename&gt;'</span></span>
<span data-ttu-id="dc9f1-103">Выражение обращается к программному элементу, определенному в классе, структуре, модуле или интерфейсе, содержащем несколько членов с тем же именем.</span><span class="sxs-lookup"><span data-stu-id="dc9f1-103">An expression accesses a programming element defined in a class, structure, module, or interface that contains more than one member with the same name.</span></span>  
  
 <span data-ttu-id="dc9f1-104">Наиболее вероятной причиной этой ошибки является *учет регистра*.</span><span class="sxs-lookup"><span data-stu-id="dc9f1-104">The most likely cause of this error is *case sensitivity*.</span></span> <span data-ttu-id="dc9f1-105">В именах Visual Basic регистр не учитывается, что означает, что в разных местах в коде эти имена можно указывать в разном регистре.</span><span class="sxs-lookup"><span data-stu-id="dc9f1-105">Visual Basic names are case-insensitive, which means you can capitalize them differently at different places in your code.</span></span> <span data-ttu-id="dc9f1-106">Например, если определить переменную с именем `XYZ` и позже обратиться к ней как к `xyz`, компилятор сочтет эти два имени равнозначными.</span><span class="sxs-lookup"><span data-stu-id="dc9f1-106">For example, if you define a variable with the name `XYZ` and later access it as `xyz`, the compiler considers the two names to be equivalent.</span></span>  
  
 <span data-ttu-id="dc9f1-107">Тем не менее другие языки, такие как [ C# ](../../csharp/index.md) и [Visual C++](/cpp/index), чувствительны к регистру.</span><span class="sxs-lookup"><span data-stu-id="dc9f1-107">However, other languages, such as [C#](../../csharp/index.md) and [Visual C++](/cpp/index), are case-sensitive.</span></span> <span data-ttu-id="dc9f1-108">В таких языках `XYZ` и `xyz` не считаются одинаковыми.</span><span class="sxs-lookup"><span data-stu-id="dc9f1-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="dc9f1-109">Следовательно, в классе, написанном на таком языке, должна быть задана переменная с именем `XYZ` и свойство с именем `xyz`.</span><span class="sxs-lookup"><span data-stu-id="dc9f1-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="dc9f1-110">Среда CLR сохраняет учет регистра в сборках.</span><span class="sxs-lookup"><span data-stu-id="dc9f1-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="dc9f1-111">Однако если приложение Visual Basic обращается к сборке с именами `XYZ` и `xyz`, они отображаются как одно и то же имя.</span><span class="sxs-lookup"><span data-stu-id="dc9f1-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>  
  
 <span data-ttu-id="dc9f1-112">**Идентификатор ошибки:** BC31429</span><span class="sxs-lookup"><span data-stu-id="dc9f1-112">**Error ID:** BC31429</span></span>
