<span data-ttu-id="cefce-103">Выражение обращается к программному элементу, определенному в классе, структуре, модуле или интерфейсе, содержащем несколько членов с тем же именем.</span><span class="sxs-lookup"><span data-stu-id="cefce-103">An expression accesses a programming element defined in a class, structure, module, or interface that contains more than one member with the same name.</span></span>

 <span data-ttu-id="cefce-107">Однако другие языки, такие как [C#](../../csharp/csharp.md) и [Visual C++](https://docs.microsoft.com/cpp/top/visual-cpp-in-visual-studio-2015), зависят от регистра.</span><span class="sxs-lookup"><span data-stu-id="cefce-107">However, other languages, such as [C#](../../csharp/csharp.md) and [Visual C++](https://docs.microsoft.com/cpp/top/visual-cpp-in-visual-studio-2015), are case-sensitive.</span></span> <span data-ttu-id="cefce-108">В таких языках `XYZ` и `xyz` не считаются одинаковыми.</span><span class="sxs-lookup"><span data-stu-id="cefce-108">In such a language, `XYZ` and `xyz` are not considered to be the same name.</span></span> <span data-ttu-id="cefce-109">Следовательно, в классе, написанном на таком языке, должна быть задана переменная с именем `XYZ` и свойство с именем `xyz`.</span><span class="sxs-lookup"><span data-stu-id="cefce-109">Therefore, a class written in such a language could define a variable named `XYZ` and a property named `xyz`.</span></span> <span data-ttu-id="cefce-110">Среда CLR сохраняет учет регистра в сборках.</span><span class="sxs-lookup"><span data-stu-id="cefce-110">The common language runtime (CLR) preserves case sensitivity in assemblies.</span></span> <span data-ttu-id="cefce-111">Однако если приложение Visual Basic обращается к сборке с именами `XYZ` и `xyz`, они отображаются как одно и то же имя.</span><span class="sxs-lookup"><span data-stu-id="cefce-111">However, if a Visual Basic application accesses an assembly with names `XYZ` and `xyz`, they appear as the same name.</span></span>
