# <a name="class-39ltclassname1gt39-must-declare-a-39sub-new39-because-its-base-class-39ltclassname2gt39-has-more-than-one-accessible-39sub-new39-that-can-be-called-with-no-arguments"></a><span data-ttu-id="e34e0-102">Класс&lt;classname1&gt;"должен объявлять Sub New, поскольку его базовый класс&lt;classname2&gt;" имеет более одного доступного Sub New, который может вызываться без аргументов</span><span class="sxs-lookup"><span data-stu-id="e34e0-102">Class &#39;&lt;classname1&gt;&#39; must declare a &#39;Sub New&#39; because its base class &#39;&lt;classname2&gt;&#39; has more than one accessible &#39;Sub New&#39; that can be called with no arguments</span></span>
<span data-ttu-id="e34e0-103">Производный класс не объявляет конструктор, и [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] не может создать одно, так как невозможно определить, какой конструктор базового класса для вызова.</span><span class="sxs-lookup"><span data-stu-id="e34e0-103">A derived class does not declare a constructor, and [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] cannot generate one because it cannot determine which base class constructor to call.</span></span>  
  
 <span data-ttu-id="e34e0-104">Если производный класс не объявляет конструктор, [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] пытается создать неявный конструктор без параметров, который вызывает `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="e34e0-104">When a derived class does not declare a constructor, [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="e34e0-105">Если нет доступного конструктора в базовом классе, который может вызываться без аргументов или если имеется более одного [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] не удается создать неявный конструктор.</span><span class="sxs-lookup"><span data-stu-id="e34e0-105">If there is no accessible constructor in the base class that can be called without arguments, or if there is more than one, [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] cannot generate an implicit constructor.</span></span>  
  
 <span data-ttu-id="e34e0-106">Эта ситуация может возникнуть, например, если один конструктор базового класса содержит один аргумент `Optional` , а другой — один аргумент `ParamArray` .</span><span class="sxs-lookup"><span data-stu-id="e34e0-106">This situation can arise, for example, if one base class constructor has a single `Optional` argument and another has a single `ParamArray` argument.</span></span> <span data-ttu-id="e34e0-107">Каждый из них можно вызвать без аргументов.</span><span class="sxs-lookup"><span data-stu-id="e34e0-107">Each of these can be called with no arguments.</span></span>
