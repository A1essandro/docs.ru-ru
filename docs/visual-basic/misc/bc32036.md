# <a name="class-39ltclassname1gt39-must-declare-a-39sub-new39-because-its-base-class-39ltclassname2gt39-has-more-than-one-accessible-39sub-new39-that-can-be-called-with-no-arguments"></a><span data-ttu-id="bcbcf-102">Класс &#39; &lt;имя_класса1&gt;&#39; необходимо объявить &#39; Конструктор Sub New &#39; так как его базовый класс &#39; &lt;имя_класса2&gt;&#39; имеет более чем один доступный &#39; Конструктор Sub New &#39; может вызываться без аргументов</span><span class="sxs-lookup"><span data-stu-id="bcbcf-102">Class &#39;&lt;classname1&gt;&#39; must declare a &#39;Sub New&#39; because its base class &#39;&lt;classname2&gt;&#39; has more than one accessible &#39;Sub New&#39; that can be called with no arguments</span></span>
<span data-ttu-id="bcbcf-103">Производный класс не объявляет конструктор, и [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] не может создать его, так как он не может определить, какой конструктор базового класса следует вызвать.</span><span class="sxs-lookup"><span data-stu-id="bcbcf-103">A derived class does not declare a constructor, and [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] cannot generate one because it cannot determine which base class constructor to call.</span></span>  
  
 <span data-ttu-id="bcbcf-104">Когда производный класс не объявляет конструктор, [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] пытается создать неявный конструктор без параметров, который вызывает `MyBase.New()`.</span><span class="sxs-lookup"><span data-stu-id="bcbcf-104">When a derived class does not declare a constructor, [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] attempts to generate an implicit parameterless constructor that calls `MyBase.New()`.</span></span> <span data-ttu-id="bcbcf-105">Если нет доступного конструктора в базовом классе, который можно вызвать без аргументов, или если есть несколько конструкторов, [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] не может создать неявный конструктор.</span><span class="sxs-lookup"><span data-stu-id="bcbcf-105">If there is no accessible constructor in the base class that can be called without arguments, or if there is more than one, [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] cannot generate an implicit constructor.</span></span>  
  
 <span data-ttu-id="bcbcf-106">Эта ситуация может возникнуть, например, если один конструктор базового класса содержит один аргумент `Optional` , а другой — один аргумент `ParamArray` .</span><span class="sxs-lookup"><span data-stu-id="bcbcf-106">This situation can arise, for example, if one base class constructor has a single `Optional` argument and another has a single `ParamArray` argument.</span></span> <span data-ttu-id="bcbcf-107">Каждый из них можно вызвать без аргументов.</span><span class="sxs-lookup"><span data-stu-id="bcbcf-107">Each of these can be called with no arguments.</span></span>
