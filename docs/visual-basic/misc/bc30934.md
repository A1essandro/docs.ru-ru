# <a name="conversion-from-39lttype1gt39-to-39lttype2gt39-cannot-occur-in-a-constant-expression-used-as-an-argument-to-an-attribute"></a><span data-ttu-id="83ed2-102">Преобразование из "&lt;type1&gt;«to»&lt;type2&gt;" не может встречаться в константном выражении, используемом в качестве аргумента атрибута</span><span class="sxs-lookup"><span data-stu-id="83ed2-102">Conversion from &#39;&lt;type1&gt;&#39; to &#39;&lt;type2&gt;&#39; cannot occur in a constant expression used as an argument to an attribute</span></span>
<span data-ttu-id="83ed2-103">Вычисляет выражение, используемое для аргумента атрибута с типом данных, отличающимся от соответствующего параметра атрибута и [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] не допускает преобразования требуемый тип для аргументов атрибута.</span><span class="sxs-lookup"><span data-stu-id="83ed2-103">An expression used for an attribute argument evaluates to a data type different from that of the corresponding attribute parameter, and [!INCLUDE[vbprvb](../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)] does not allow the required type conversion for attribute arguments.</span></span>  
  
 <span data-ttu-id="83ed2-104">Атрибут содержит метаданные для элемента, к которому он применяется, и компилятор должен быть способен создать все метаданные во время компиляции.</span><span class="sxs-lookup"><span data-stu-id="83ed2-104">An attribute provides metadata for the element it is applied to, and the compiler must be able to construct all the metadata at compile time.</span></span> <span data-ttu-id="83ed2-105">По этой причине каждый атрибут должен использовать значения, которые постоянны во время компиляции. Таким образом, каждый аргумент атрибута должен принимать постоянное значение во время компиляции.</span><span class="sxs-lookup"><span data-stu-id="83ed2-105">For this reason, every attribute must use values that are constant at compile time, and therefore every attribute argument must evaluate to a compile-time constant value.</span></span>  
  
 <span data-ttu-id="83ed2-106">Некоторые преобразования типов не создают значений, которые являются постоянными во время компиляции.</span><span class="sxs-lookup"><span data-stu-id="83ed2-106">Certain type conversions cannot produce values that are constant at compile time.</span></span> <span data-ttu-id="83ed2-107">Например, преобразование `String` в `Double` или в `Date` зависит от языковых стандартов, используемых во время выполнения.</span><span class="sxs-lookup"><span data-stu-id="83ed2-107">For example, converting a `String` to a `Double` or a `Date` depends on the locale setting at run time.</span></span> <span data-ttu-id="83ed2-108">Другие преобразования, например из массива производного типа в массив типа `Object`, порождают различные ошибки, не допускающие их применения к аргументам атрибутов во время компиляции.</span><span class="sxs-lookup"><span data-stu-id="83ed2-108">Other conversions, such as an array of a derived type to an array of `Object`, present a variety of problems that do not permit the compiler to allow them on attribute arguments.</span></span>  
  
 <span data-ttu-id="83ed2-109">**Идентификатор ошибки:** BC30934</span><span class="sxs-lookup"><span data-stu-id="83ed2-109">**Error ID:** BC30934</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="83ed2-110">Исправление ошибки</span><span class="sxs-lookup"><span data-stu-id="83ed2-110">To correct this error</span></span>  
  
-   <span data-ttu-id="83ed2-111">Используйте выражение, результатом которого является тот же тип данных, что и у соответствующего параметра, который определяется атрибутом.</span><span class="sxs-lookup"><span data-stu-id="83ed2-111">Use an expression that evaluates to the same data type as the corresponding parameter, as defined by the attribute.</span></span>
