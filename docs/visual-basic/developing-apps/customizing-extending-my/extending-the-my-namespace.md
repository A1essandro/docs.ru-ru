-   <span data-ttu-id="1dcfd-163">**Содержит только логику расширения.**</span><span class="sxs-lookup"><span data-stu-id="1dcfd-163">**Include only the extension logic.**</span></span> <span data-ttu-id="1dcfd-164">Логика `My` расширение пространства имен должно содержать только код, необходимый для предоставления требуемых функций в `My` пространства имен.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-164">The logic included in the `My` namespace extension should include only the code that is needed to expose the required functionality in the `My` namespace.</span></span> <span data-ttu-id="1dcfd-165">Так как расширение будет присутствовать в проектах пользователей в виде исходного кода, обновление компонента расширения влечет за собой высокую стоимость обслуживания и по возможности следует избегать.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-165">Because your extension will reside in user projects as source code, updating the extension component incurs a high maintenance cost and should be avoided if possible.</span></span>

-   <span data-ttu-id="1dcfd-170">**Локализация кода расширения.**</span><span class="sxs-lookup"><span data-stu-id="1dcfd-170">**Isolate the extension code.**</span></span> <span data-ttu-id="1dcfd-171">Поместив данный код в одном файле расширение делает простым развертыванием в качестве шаблона элемента Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-171">Placing the code in a single file makes your extension easily deployable as a Visual Studio item template.</span></span> <span data-ttu-id="1dcfd-172">Дополнительные сведения см. в разделе «Упаковка и развертывание расширения» далее в этом разделе.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-172">For more information, see "Packaging and Deploying Extensions" later in this topic.</span></span> <span data-ttu-id="1dcfd-173">Размещение всех `My` кода расширения пространства имен в один файл или отдельную папку в проекте также поможет пользователям найти `My` расширение пространства имен.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-173">Placing all the `My` namespace extension code in a single file or a separate folder in a project will also help users locate the `My` namespace extension.</span></span>  
  
##  <a name="designing"></a> <span data-ttu-id="1dcfd-174">Разработка библиотек классов для My</span><span class="sxs-lookup"><span data-stu-id="1dcfd-174">Designing Class Libraries for My</span></span>  
 <span data-ttu-id="1dcfd-175">Как и в случае с большинством объектных моделей, одни шаблоны разработки хорошо работают в `My` пространства имен, а другие нет.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-175">As is the case with most object models, some design patterns work well in the `My` namespace and others do not.</span></span> <span data-ttu-id="1dcfd-176">При разработке расширения для `My` пространства имен, необходимо учитывать следующие принципы:</span><span class="sxs-lookup"><span data-stu-id="1dcfd-176">When designing an extension to the `My` namespace, consider the following principles:</span></span>  
  
-   <span data-ttu-id="1dcfd-177">**Методы без сохранения состояния.**</span><span class="sxs-lookup"><span data-stu-id="1dcfd-177">**Stateless methods.**</span></span> <span data-ttu-id="1dcfd-178">Методы в `My` пространства имен следует создать полноценное решение для определенной задачи.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-178">Methods in the `My` namespace should provide a complete solution to a specific task.</span></span> <span data-ttu-id="1dcfd-179">Убедитесь, что значения параметров, которые передаются в метод предоставляет все входные данные, необходимые для выполнения конкретной задачи.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-179">Ensure that the parameter values that are passed to the method provide all the input required to complete the particular task.</span></span> <span data-ttu-id="1dcfd-180">Избегайте создания методов, которые зависят от предыдущего состояния, таких как открытые подключения к ресурсам.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-180">Avoid creating methods that rely on prior state, such as open connections to resources.</span></span>  
  
-   <span data-ttu-id="1dcfd-181">**Глобальные экземпляры.**</span><span class="sxs-lookup"><span data-stu-id="1dcfd-181">**Global instances.**</span></span> <span data-ttu-id="1dcfd-182">Единственное состояние, которое хранится в `My` пространством имен является глобальным для проекта.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-182">The only state that is maintained in the `My` namespace is global to the project.</span></span> <span data-ttu-id="1dcfd-183">Например `My.Application.Info` инкапсулирует состояние, общее для всего приложения.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-183">For example, `My.Application.Info` encapsulates state that is shared throughout the application.</span></span>  
  
-   <span data-ttu-id="1dcfd-184">**Простые типы параметров.**</span><span class="sxs-lookup"><span data-stu-id="1dcfd-184">**Simple parameter types.**</span></span> <span data-ttu-id="1dcfd-185">Простоты, чтобы избежать сложных типов параметров.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-185">Keep things simple by avoiding complex parameter types.</span></span> <span data-ttu-id="1dcfd-186">Создайте методы, которые либо принимают без параметра ввода, либо принимают простых типов входных данных, такие как строки, типы-примитивы и т. д.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-186">Instead, create methods that either take no parameter input or that take simple input types such as strings, primitive types, and so on.</span></span>  
  
-   <span data-ttu-id="1dcfd-187">**Фабричные методы.**</span><span class="sxs-lookup"><span data-stu-id="1dcfd-187">**Factory methods.**</span></span> <span data-ttu-id="1dcfd-188">Некоторые типы трудно создать экземпляр.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-188">Some types are necessarily difficult to instantiate.</span></span> <span data-ttu-id="1dcfd-189">Предоставление фабричных методов в качестве расширений для `My` пространство имен позволяет упростить обнаружение и использование типов, которые попадают в эту категорию.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-189">Providing factory methods as extensions to the `My` namespace enables you to more easily discover and consume types that fall into this category.</span></span> <span data-ttu-id="1dcfd-190">Пример фабричный метод, которое работает хорошо — `My.Computer.FileSystem.OpenTextFileReader`.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-190">An example of a factory method that works well is `My.Computer.FileSystem.OpenTextFileReader`.</span></span> <span data-ttu-id="1dcfd-191">Существует несколько типов потоков в .NET Framework.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-191">There are several stream types available in the .NET Framework.</span></span> <span data-ttu-id="1dcfd-192">В частности, задавая текстовые файлы `OpenTextFileReader` помогает пользователю понять, какой поток следует использовать.</span><span class="sxs-lookup"><span data-stu-id="1dcfd-192">By specifying text files specifically, the `OpenTextFileReader` helps the user understand which stream to use.</span></span>
