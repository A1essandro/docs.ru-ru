# <a name="adding-printable-reports-to-visual-studio-applications"></a>Добавление печатаемых отчетов в приложения Visual Studio
Visual Studio поддерживает широкий спектр решений для добавления отчетов в приложения Visual Basic с богатыми составления отчетов. Можно создавать и добавлять отчеты с помощью элементов управления ReportViewer, Crystal Reports или служб SQL Server Reporting Services.  

  
## <a name="overview-of-microsoft-reporting-technology-in-visual-basic-applications"></a>Общие сведения о технологии в приложениях Visual Basic отчетов Майкрософт  
 Для использования технологии в приложении Microsoft reporting, выберите из следующих подходов:  
  
-   Добавьте один или несколько экземпляров элемента управления ReportViewer в приложение Visual Basic Windows.  
  
-   Интеграция SQL Server Reporting Services программным способом с помощью обращений к серверу веб-службы отчетов.  
  
-   Используйте элемент управления ReportViewer и Microsoft SQL Server 2005 Reporting Services вместе с помощью элемента управления как средство просмотра отчетов и сервер отчетов в качестве обработчика отчетов. (Обратите внимание, что необходимо использовать версию SQL Server 2005 Reporting Services, если вы хотите совместно использовать сервер отчетов и элемента управления ReportViewer).  
  
## <a name="using-reportviewer-controls"></a>С помощью элементов управления ReportViewer  
 Для внедрения функций отчета в приложение Visual Basic Windows проще всего добавить элемент управления ReportViewer на форму в приложении. Элемент управления непосредственно приложению возможности обработки отчетов и предоставляет встроенный конструктор отчетов, чтобы создавать отчеты, используя данные из любого объекта данных ADO.NET. Полнофункциональный API обеспечивает программный доступ к элементу управления и отчеты, чтобы можно было настроить во время выполнения.  
  
 Элемент ReportViewer предоставляет встроенные отчета возможности обработки и просмотра в элементе управления данными отдельный, свободно распространяемый. Выберите элементы управления ReportViewer, если требуются следующие функциональные возможности отчетов:  
  
-   Обработки отчетов в клиентское приложение. Обработанный отчет отображается в области просмотра, предоставляемые элементом управления.  
  
-   Привязка данных к таблицам данных ADO.NET. Можно создавать отчеты, которые используют <xref:System.Data.DataTable> экземпляров, переданных в элемент управления. Кроме того, можно также привязать напрямую к бизнес-объектам.  
  
-   Распространяемые элементы управления, которые могут включать в приложения.  
  
-   Функции среды выполнения, такие как навигация по страницам, печать, поиск и форматы экспорта. Панель инструментов ReportViewer обеспечивает поддержку для этих операций.  
  
 Чтобы использовать элемент управления ReportViewer, можно перетащить его из **данных** разделе панели элементов Visual Studio на форму в приложении Visual Basic Windows.  
  
### <a name="creating-reports-in-visual-studio-for-reportviewer-controls"></a>Создание отчетов в Visual Studio для элементов управления ReportViewer  
 Чтобы создать отчет, выполняемый в ReportViewer, добавьте **отчетов** шаблона в проект. Visual Studio создает файл определения клиентского отчета (с расширением RDLC), добавляет файл в проект и открывает встроенный конструктор отчетов в рабочей области Visual Studio.  
  
 Конструктор отчетов среды Visual Studio интегрируется с **источников данных** окна. При перетаскивании поля из **источников данных** окно в отчет конструктор отчетов копирует метаданные об источнике данных в файле определения отчета. Эти метаданные используются элементом управления ReportViewer для автоматического формирования кода привязки данных.  
  
 Конструктор отчетов Visual Studio не поддерживает функции предварительного просмотра отчета. Предварительный просмотр отчета, запустите приложение и просмотреть отчет, внедренный в него.  
  
|Добавление функциональных возможностей простого отчета в приложение|  
|---|    
|1.  Перетащите элемент управления ReportViewer из **данных** вкладке **элементов** на форму.<br />2.  В меню **Проект** выберите пункт **Добавить новый элемент**. В **Добавление нового элемента** выберите **отчетов** значок и нажмите кнопку **добавить**.<br />     Открывается конструктор отчетов в среде разработки, и файл отчета (RDLC) добавляется в проект.<br />3.  Перетащите элементы отчета из **элементов** на макет отчета и Упорядочьте их, как требуется.<br />4.  Перетащить поля из **источников данных** окна на элементы отчета в макете отчета.|  
  
## <a name="using-reporting-services-in-visual-basic-applications"></a>С помощью служб Reporting Services в приложениях Visual Basic  
 Службы Reporting Services — серверные технологии создания отчетов, который входит в состав SQL Server. Службы Reporting Services включают дополнительные возможности, которые не находятся в элементах управления ReportViewer. Выберите службы Reporting Services, если требуется любая из следующих компонентов:  
  
-   Масштабное развертывание и обработка на сервере отчетов, которая обеспечивает улучшенную производительность для сложных или долго выполняющихся отчетов, а также для действий с отчетами большого объема.  
  
-   Интегрированная обработка данных и отчета, поддержку пользовательских элементов управления отчетами и широкие возможности визуализации выходных форматов.  
  
-   Обработка отчетов, можно указать точно в том случае, когда отчеты запускаются по расписанию.  
  
-   Распространение отчетов на основе подписки по электронной почте или общие папки.  
  
-   Автоматизированная система отчетности, бизнес-пользователей позволяют создавать отчеты, при необходимости.  
  
-   Управляемые данными подписки, направляющие настраиваемый вывод отчета динамическому списку получателей.  
  
-   Пользовательские расширения для обработки данных, доставки отчетов, нестандартной проверки подлинности и подготовки отчета к просмотру.  
  
 Сервер отчетов реализован как веб-службу. Код приложения должен включать вызовы веб-службы для доступа к отчетам и другие метаданные. Веб-служба предоставляет полный программный доступ к экземпляру сервера отчетов.  
  
 Поскольку службы Reporting Services веб-технологий создания отчетов, средством просмотра по умолчанию отображаются отчеты, подготавливаются к просмотру в формате HTML. Если вы не хотите использовать HTML в качестве формата представления отчетов по умолчанию, будет необходимо написать собственное средство просмотра отчетов для вашего приложения.  
  
### <a name="creating-reports-in-visual-studio-for-reporting-services"></a>Создание отчетов в Visual Studio для служб Reporting Services  
 Для создания отчетов, которые выполняются на сервере отчетов, создании определения отчета (RDL-файл) файлов в Visual Studio с помощью Business Intelligence Development Studio, входящий в состав SQL Server 2005.  
  
 Business Intelligence Development Studio добавляет шаблоны проектов, которые относятся к компонентам SQL Server. Для создания отчетов, можно выбрать из **проект сервера отчетов** или **мастер проекта сервера отчетов** шаблонов. Можно указать соединения с источниками данных и запросы к различным типам источников данных, включая SQL Server, Oracle, службы Analysis Services, XML и SQL Server Integration Services. Объект **данных** вкладке **макета** вкладке и **предварительной версии** вкладки позволяют определить данные, создать макет отчета и просмотреть отчет в той же рабочей области.  
  
 Определения отчетов, создаваемых для элемента управления или сервер отчетов можно было использовать повторно из этих технологий.  
  
|Чтобы создать отчет, который выполняется на сервере отчетов|  
|---|    
|1.  На **файл** меню, выберите **New**.<br />     Откроется диалоговое окно **Новый проект** .<br />2.  В **типы проектов** панели щелкните **проекты бизнес-аналитики**.<br />3.  В области «Шаблоны» выберите **проект сервера отчетов** или **мастер проекта сервера отчетов**.|  
  
## <a name="using-reportviewer-controls-and-sql-server-reporting-services-together"></a>С помощью элементов управления ReportViewer и служб отчетов SQL Server  
 Элементы управления ReportViewer и служб SQL Server 2005 Reporting Services могут использоваться совместно в одном приложении.  
  
-   Элемент управления ReportViewer предоставляет средство просмотра, которое используется для отображения отчетов в приложении.  
  
-   Службы Reporting Services предоставляет отчеты и выполняет всю обработку на удаленном сервере.  
  
 Элемент управления ReportViewer можно настроить для отображения отчетов, которые хранятся и обрабатываются на удаленном сервере отчетов служб Reporting Services. Такая конфигурация называется *режиме удаленной обработки*. В режиме удаленной обработки элемент управления запрашивает отчет, хранящийся на удаленном сервере отчетов. Сервер отчетов выполняет все обработки отчета, обработки данных и подготовки отчета к просмотру. Готово, готовый для просмотра отчет возвращается в элемент управления и отображается в области просмотра.  
  
 Отчеты, выполняемые в поддержке сервера отчетов Дополнительные форматы экспорта, имеют различную реализацию параметризации отчетов, использовать типы источников данных, которые поддерживаются сервером отчетов и предоставляются через модель авторизации на основе ролей на на сервере отчетов.  
  
 Для использования режима удаленной обработки, укажите URL-адрес и путь к серверу отчетов при настройке элемента управления ReportViewer.
