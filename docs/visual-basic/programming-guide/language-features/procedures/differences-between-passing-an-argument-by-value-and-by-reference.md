# <a name="differences-between-passing-an-argument-by-value-and-by-reference-visual-basic"></a><span data-ttu-id="3319d-102">Различия между передачей аргумента по значению и по ссылке (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="3319d-102">Differences Between Passing an Argument By Value and By Reference (Visual Basic)</span></span>
<span data-ttu-id="3319d-103">Если передать один или несколько аргументов в процедуру, каждый аргумент соответствует базовому элементу программирования в вызывающем коде.</span><span class="sxs-lookup"><span data-stu-id="3319d-103">When you pass one or more arguments to a procedure, each argument corresponds to an underlying programming element in the calling code.</span></span> <span data-ttu-id="3319d-104">Можно передать значение этого базового элемента или ссылку на него.</span><span class="sxs-lookup"><span data-stu-id="3319d-104">You can pass either the value of this underlying element, or a reference to it.</span></span> <span data-ttu-id="3319d-105">Это называется *механизм передачи*.</span><span class="sxs-lookup"><span data-stu-id="3319d-105">This is known as the *passing mechanism*.</span></span>  
  
## <a name="passing-by-value"></a><span data-ttu-id="3319d-106">передача по значению</span><span class="sxs-lookup"><span data-stu-id="3319d-106">Passing by Value</span></span>  
 <span data-ttu-id="3319d-107">При передаче аргумента *по значению* , указав [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) ключевое слово для соответствующего параметра в определении процедуры.</span><span class="sxs-lookup"><span data-stu-id="3319d-107">You pass an argument *by value* by specifying the [ByVal](../../../../visual-basic/language-reference/modifiers/byval.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="3319d-108">При использовании этого механизма передачи, Visual Basic копирует значение из базового элемента программирования в локальную переменную в процедуре.</span><span class="sxs-lookup"><span data-stu-id="3319d-108">When you use this passing mechanism, Visual Basic copies the value of the underlying programming element into a local variable in the procedure.</span></span> <span data-ttu-id="3319d-109">Код процедуры не имеет доступа к базовому элементу в вызывающем коде.</span><span class="sxs-lookup"><span data-stu-id="3319d-109">The procedure code does not have any access to the underlying element in the calling code.</span></span>  
  
## <a name="passing-by-reference"></a><span data-ttu-id="3319d-110">Передача по ссылке</span><span class="sxs-lookup"><span data-stu-id="3319d-110">Passing by Reference</span></span>  
 <span data-ttu-id="3319d-111">При передаче аргумента *по ссылке* , указав [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) ключевое слово для соответствующего параметра в определении процедуры.</span><span class="sxs-lookup"><span data-stu-id="3319d-111">You pass an argument *by reference* by specifying the [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md) keyword for the corresponding parameter in the procedure definition.</span></span> <span data-ttu-id="3319d-112">При использовании этого механизма передачи, Visual Basic предоставляет процедуре прямую ссылку на основной элемент программирования в вызывающем коде.</span><span class="sxs-lookup"><span data-stu-id="3319d-112">When you use this passing mechanism, Visual Basic gives the procedure a direct reference to the underlying programming element in the calling code.</span></span>  
  
## <a name="passing-mechanism-and-element-type"></a><span data-ttu-id="3319d-113">Механизм передачи и тип элемента</span><span class="sxs-lookup"><span data-stu-id="3319d-113">Passing Mechanism and Element Type</span></span>  
 <span data-ttu-id="3319d-114">Выбор механизма передачи отличается от классификации базовый тип элемента.</span><span class="sxs-lookup"><span data-stu-id="3319d-114">The choice of passing mechanism is not the same as the classification of the underlying element type.</span></span> <span data-ttu-id="3319d-115">Передача по значению или по ссылке ссылается на Visual Basic предоставляет коду процедуры.</span><span class="sxs-lookup"><span data-stu-id="3319d-115">Passing by value or by reference refers to what Visual Basic supplies to the procedure code.</span></span> <span data-ttu-id="3319d-116">Тип значения или ссылочный тип ссылается на способ хранения элемента программирования в памяти.</span><span class="sxs-lookup"><span data-stu-id="3319d-116">A value type or reference type refers to how a programming element is stored in memory.</span></span>  
  
 <span data-ttu-id="3319d-117">Однако механизм передачи и тип элемента взаимосвязаны.</span><span class="sxs-lookup"><span data-stu-id="3319d-117">However, the passing mechanism and element type are interrelated.</span></span> <span data-ttu-id="3319d-118">Значение ссылочного типа — указатель на данные в другом месте в памяти.</span><span class="sxs-lookup"><span data-stu-id="3319d-118">The value of a reference type is a pointer to the data elsewhere in memory.</span></span> <span data-ttu-id="3319d-119">Это означает, что при передаче ссылочного типа по значению, код процедуры имеет указатель на базовый элемент данных, несмотря на то, что он не может получить доступ к самому базовому элементу.</span><span class="sxs-lookup"><span data-stu-id="3319d-119">This means that when you pass a reference type by value, the procedure code has a pointer to the underlying element's data, even though it cannot access the underlying element itself.</span></span> <span data-ttu-id="3319d-120">Например если элемент является переменной массива, код процедуры не имеет доступа на саму переменную, но он имеет доступ к членам массива.</span><span class="sxs-lookup"><span data-stu-id="3319d-120">For example, if the element is an array variable, the procedure code does not have access to the variable itself, but it can access the array members.</span></span>
