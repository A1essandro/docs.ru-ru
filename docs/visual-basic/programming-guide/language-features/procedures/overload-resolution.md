# <a name="overload-resolution-visual-basic"></a><span data-ttu-id="8291b-102">Разрешение перегрузки (Visual Basic)</span><span class="sxs-lookup"><span data-stu-id="8291b-102">Overload Resolution (Visual Basic)</span></span>
<span data-ttu-id="8291b-103">Когда компилятор Visual Basic обнаруживает вызов процедуры, которая определена в нескольких перегруженных версиях, компилятор должен определить, какую из перегрузок для вызова.</span><span class="sxs-lookup"><span data-stu-id="8291b-103">When the Visual Basic compiler encounters a call to a procedure that is defined in several overloaded versions, the compiler must decide which of the overloads to call.</span></span> <span data-ttu-id="8291b-104">Это достигается путем выполнения следующих шагов:</span><span class="sxs-lookup"><span data-stu-id="8291b-104">It does this by performing the following steps:</span></span>  
  
1.  <span data-ttu-id="8291b-105">**Специальные возможности.**</span><span class="sxs-lookup"><span data-stu-id="8291b-105">**Accessibility.**</span></span> <span data-ttu-id="8291b-106">Это исключает ни одной перегрузке с уровнем доступа, который запрещает вызов этого метода вызывающий код.</span><span class="sxs-lookup"><span data-stu-id="8291b-106">It eliminates any overload with an access level that prevents the calling code from calling it.</span></span>  
  
2.  <span data-ttu-id="8291b-107">**Число параметров.**</span><span class="sxs-lookup"><span data-stu-id="8291b-107">**Number of Parameters.**</span></span> <span data-ttu-id="8291b-108">Это исключает ни одной перегрузке, определяющий разное число параметров, чем задано в вызове.</span><span class="sxs-lookup"><span data-stu-id="8291b-108">It eliminates any overload that defines a different number of parameters than are supplied in the call.</span></span>  
  
3.  <span data-ttu-id="8291b-109">**Типы данных параметров.**</span><span class="sxs-lookup"><span data-stu-id="8291b-109">**Parameter Data Types.**</span></span> <span data-ttu-id="8291b-110">Компилятор отдает предпочтение методам экземпляра по сравнению с методами расширения.</span><span class="sxs-lookup"><span data-stu-id="8291b-110">The compiler gives instance methods preference over extension methods.</span></span> <span data-ttu-id="8291b-111">Если найдено любой метод экземпляра, для которого требуется только расширяющие преобразования в соответствии с вызова процедуры, удаляются все методы расширения, и компилятор продолжает выполняться с использованием только методы экземпляров.</span><span class="sxs-lookup"><span data-stu-id="8291b-111">If any instance method is found that requires only widening conversions to match the procedure call, all extension methods are dropped and the compiler continues with only the instance method candidates.</span></span> <span data-ttu-id="8291b-112">Если такого метода экземпляр не найден, он продолжает с экземпляром и методы расширения.</span><span class="sxs-lookup"><span data-stu-id="8291b-112">If no such instance method is found, it continues with both instance and extension methods.</span></span>  
  
     <span data-ttu-id="8291b-113">На этом шаге он исключает ни одной перегрузке, для которого не может преобразовать типы данных аргументов вызова для типов параметров, определенных в перегрузке.</span><span class="sxs-lookup"><span data-stu-id="8291b-113">In this step, it eliminates any overload for which the data types of the calling arguments cannot be converted to the parameter types defined in the overload.</span></span>  
  
4.  <span data-ttu-id="8291b-114">**Сужающие преобразования.**</span><span class="sxs-lookup"><span data-stu-id="8291b-114">**Narrowing Conversions.**</span></span> <span data-ttu-id="8291b-115">Это устраняет любые перегрузку, которая требует сужающего преобразования из типов аргументов вызова в типы определенных параметров.</span><span class="sxs-lookup"><span data-stu-id="8291b-115">It eliminates any overload that requires a narrowing conversion from the calling argument types to the defined parameter types.</span></span> <span data-ttu-id="8291b-116">Это верно, является ли переключатель проверки типа ([оператор Option Strict](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) является `On` или `Off`.</span><span class="sxs-lookup"><span data-stu-id="8291b-116">This is true whether the type checking switch ([Option Strict Statement](../../../../visual-basic/language-reference/statements/option-strict-statement.md)) is `On` or `Off`.</span></span>  
  
5.  <span data-ttu-id="8291b-117">**Наименьшее расширение.**</span><span class="sxs-lookup"><span data-stu-id="8291b-117">**Least Widening.**</span></span> <span data-ttu-id="8291b-118">Компилятор считает, что оставшиеся перегрузки парами.</span><span class="sxs-lookup"><span data-stu-id="8291b-118">The compiler considers the remaining overloads in pairs.</span></span> <span data-ttu-id="8291b-119">Для каждой пары он сравнивает типы данных определенных параметров.</span><span class="sxs-lookup"><span data-stu-id="8291b-119">For each pair, it compares the data types of the defined parameters.</span></span> <span data-ttu-id="8291b-120">Если типы в одну из перегрузок все расширяется до соответствующих типов в другой, компилятор исключает последнюю.</span><span class="sxs-lookup"><span data-stu-id="8291b-120">If the types in one of the overloads all widen to the corresponding types in the other, the compiler eliminates the latter.</span></span> <span data-ttu-id="8291b-121">То есть остается перегрузка, которая требует наименьший объем расширения.</span><span class="sxs-lookup"><span data-stu-id="8291b-121">That is, it retains the overload that requires the least amount of widening.</span></span>  
  
6.  <span data-ttu-id="8291b-122">**Единственный вариант.**</span><span class="sxs-lookup"><span data-stu-id="8291b-122">**Single Candidate.**</span></span> <span data-ttu-id="8291b-123">Он продолжает рассматривать перегрузки парами пока только одна перегрузка остается, и он разрешает вызов этой перегрузки.</span><span class="sxs-lookup"><span data-stu-id="8291b-123">It continues considering overloads in pairs until only one overload remains, and it resolves the call to that overload.</span></span> <span data-ttu-id="8291b-124">Если компилятор не может уменьшить перегрузок единственный вариант, он создает ошибку.</span><span class="sxs-lookup"><span data-stu-id="8291b-124">If the compiler cannot reduce the overloads to a single candidate, it generates an error.</span></span>
