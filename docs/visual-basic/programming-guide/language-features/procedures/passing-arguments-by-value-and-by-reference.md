## <a name="choice-of-passing-mechanism"></a><span data-ttu-id="b08cb-113">Выбор механизма передачи</span><span class="sxs-lookup"><span data-stu-id="b08cb-113">Choice of Passing Mechanism</span></span>

-   <span data-ttu-id="b08cb-115">**Защита**.</span><span class="sxs-lookup"><span data-stu-id="b08cb-115">**Protection**.</span></span> <span data-ttu-id="b08cb-116">При выборе между двумя механизмами передачи, наиболее важный критерий является раскрытие исходных переменных для изменения.</span><span class="sxs-lookup"><span data-stu-id="b08cb-116">In choosing between the two passing mechanisms, the most important criterion is the exposure of calling variables to change.</span></span> <span data-ttu-id="b08cb-117">Преимущество передачи аргумента `ByRef` является, что процедура может возвращать значение вызывающему коду через этот аргумент.</span><span class="sxs-lookup"><span data-stu-id="b08cb-117">The advantage of passing an argument `ByRef` is that the procedure can return a value to the calling code through that argument.</span></span> <span data-ttu-id="b08cb-118">Преимущество передачи аргумента `ByVal` является, что в этом случае переменная защищена от изменений, вносимых в процедуре.</span><span class="sxs-lookup"><span data-stu-id="b08cb-118">The advantage of passing an argument `ByVal` is that it protects a variable from being changed by the procedure.</span></span>

     <span data-ttu-id="b08cb-124">Указатель на данные работает для ссылочных типов, скопированный (4 байта на 32-разрядных платформах, восемь байтов на 64-разрядных платформах).</span><span class="sxs-lookup"><span data-stu-id="b08cb-124">For reference types, only the pointer to the data is copied (four bytes on 32-bit platforms, eight bytes on 64-bit platforms).</span></span> <span data-ttu-id="b08cb-125">Таким образом, можно передавать аргументы типа `String` или `Object` по значению без ущерба для производительности.</span><span class="sxs-lookup"><span data-stu-id="b08cb-125">Therefore, you can pass arguments of type `String` or `Object` by value without harming performance.</span></span>

-   <span data-ttu-id="b08cb-138">Если процедуре необходимо изменить базовый элемент в вызывающем коде, объявите соответствующий параметр [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md).</span><span class="sxs-lookup"><span data-stu-id="b08cb-138">If the procedure has a genuine need to change the underlying element in the calling code, declare the corresponding parameter [ByRef](../../../../visual-basic/language-reference/modifiers/byref.md).</span></span>  
  
-   <span data-ttu-id="b08cb-139">Если правильное выполнение кода зависит от изменения базового элемента в вызывающем коде процедуры, объявите параметр `ByRef`.</span><span class="sxs-lookup"><span data-stu-id="b08cb-139">If the correct execution of the code depends on the procedure changing the underlying element in the calling code, declare the parameter `ByRef`.</span></span> <span data-ttu-id="b08cb-140">Если при передаче по значению или код вызова переопределяет `ByRef` механизма передачи, заключив аргумент в круглые скобки, вызов процедуры может привести к непредвиденным результатам.</span><span class="sxs-lookup"><span data-stu-id="b08cb-140">If you pass it by value, or if the calling code overrides the `ByRef` passing mechanism by enclosing the argument in parentheses, the procedure call might produce unexpected results.</span></span>
