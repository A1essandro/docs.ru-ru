## <a name="what-is-pure-functional-transformation"></a><span data-ttu-id="2046b-107">Что такое чисто функциональное преобразование?</span><span class="sxs-lookup"><span data-stu-id="2046b-107">What Is Pure Functional Transformation?</span></span>  
 <span data-ttu-id="2046b-108">В *чисто функциональное преобразование*, набор функций, называемых *чистые функции*, определяют, как преобразовать набор структурированных данных из одной формы в другую.</span><span class="sxs-lookup"><span data-stu-id="2046b-108">In *pure functional transformation*, a set of functions, called *pure functions*, define how to transform a set of structured data from its original form into another form.</span></span> <span data-ttu-id="2046b-109">Слово «чистые» указывает, что функции *композицию*, что требует, что они являются:</span><span class="sxs-lookup"><span data-stu-id="2046b-109">The word "pure" indicates that the functions are *composable*, which requires that they are:</span></span>  
  
-   <span data-ttu-id="2046b-110">*Автономные*, что свободно упорядочены и переупорядочены без вовлечения взаимозависимости с остальной частью программы.</span><span class="sxs-lookup"><span data-stu-id="2046b-110">*Self-contained*, so that they can be freely ordered and rearranged without entanglement or interdependencies with the rest of the program.</span></span> <span data-ttu-id="2046b-111">Чистые преобразования не имеют представления о своей среде и не влияют на нее.</span><span class="sxs-lookup"><span data-stu-id="2046b-111">Pure transformations have no knowledge of or effect upon their environment.</span></span> <span data-ttu-id="2046b-112">То есть, использованные при преобразовании функции не имеют *побочные эффекты*.</span><span class="sxs-lookup"><span data-stu-id="2046b-112">That is, the functions used in the transformation have no *side effects*.</span></span>  
  
-   <span data-ttu-id="2046b-113">*Без сохранения состояния*, поэтому выполнение той же функции или определенного набора функций с одинаковыми входными данными всегда приводит к одинаковым выходным данным.</span><span class="sxs-lookup"><span data-stu-id="2046b-113">*Stateless*, so that executing the same function or specific set of functions on the same input will always result in the same output.</span></span> <span data-ttu-id="2046b-114">Чистые преобразования не знают о предыдущих использованиях.</span><span class="sxs-lookup"><span data-stu-id="2046b-114">Pure transformations have no memory of their prior use.</span></span>  
  
> [!IMPORTANT]
>  <span data-ttu-id="2046b-115">До конца данного учебника термин «чистая возможность» используется в основном для указания подхода к программированию, а не специальной характеристики языка.</span><span class="sxs-lookup"><span data-stu-id="2046b-115">In the rest of this tutorial, the term "pure function" is used in a general sense to indicate a programming approach, and not a specific language feature.</span></span>  
>   
>  <span data-ttu-id="2046b-116">Обратите внимание, что чистые функции должны быть реализованы как функции в Visual Basic.</span><span class="sxs-lookup"><span data-stu-id="2046b-116">Note that pure functions must be implemented as functions in Visual Basic.</span></span>  
>   
>  <span data-ttu-id="2046b-117">Также не надо путать чистые функции с чисто виртуальными методами в C++.</span><span class="sxs-lookup"><span data-stu-id="2046b-117">Also, you should not confuse pure functions with pure virtual methods in C++.</span></span> <span data-ttu-id="2046b-118">Второй случай показывает, что содержащийся класс абстрактный и текст метода не указан.</span><span class="sxs-lookup"><span data-stu-id="2046b-118">The latter indicates that the containing class is abstract and that no method body is supplied.</span></span>
