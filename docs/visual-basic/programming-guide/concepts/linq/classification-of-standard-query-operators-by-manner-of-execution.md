## <a name="manners-of-execution"></a><span data-ttu-id="5c34e-108">Способ выполнения</span><span class="sxs-lookup"><span data-stu-id="5c34e-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="5c34e-109">Интерпретация</span><span class="sxs-lookup"><span data-stu-id="5c34e-109">Immediate</span></span>  
 <span data-ttu-id="5c34e-110">Немедленное выполнение означает, что источник данных считывается, и операция выполняется в том месте кода, где объявлен запрос.</span><span class="sxs-lookup"><span data-stu-id="5c34e-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="5c34e-111">Все стандартные операторы запросов, возвращающие один результат не перечислимый немедленно.</span><span class="sxs-lookup"><span data-stu-id="5c34e-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="5c34e-112">Отложено</span><span class="sxs-lookup"><span data-stu-id="5c34e-112">Deferred</span></span>  
 <span data-ttu-id="5c34e-113">Отложенное выполнение означает, что операция не выполняется в том месте кода где объявлен запрос.</span><span class="sxs-lookup"><span data-stu-id="5c34e-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="5c34e-114">Она выполняется только после перечисления переменной запроса, например с помощью `For Each` инструкции.</span><span class="sxs-lookup"><span data-stu-id="5c34e-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="5c34e-115">Это означает, что результаты выполнения запроса зависят от содержимого источника данных при выполнении запроса вместо того, чтобы после определения запроса.</span><span class="sxs-lookup"><span data-stu-id="5c34e-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="5c34e-116">Если переменная запроса перечисляется несколько раз, результаты могут отличаться каждый раз.</span><span class="sxs-lookup"><span data-stu-id="5c34e-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="5c34e-117">Почти все стандартные операторы запросов, возвращающая <xref:System.Collections.Generic.IEnumerable%601>или <xref:System.Linq.IOrderedEnumerable%601>выполнение отложенным способом.</xref:System.Linq.IOrderedEnumerable%601> </xref:System.Collections.Generic.IEnumerable%601></span><span class="sxs-lookup"><span data-stu-id="5c34e-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="5c34e-118">Операторы запросов, использующие отложенное выполнение может классифицироваться Кроме того, потоковые и непотоковые.</span><span class="sxs-lookup"><span data-stu-id="5c34e-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="5c34e-119">Потоковые операторы</span><span class="sxs-lookup"><span data-stu-id="5c34e-119">Streaming</span></span>  
 <span data-ttu-id="5c34e-120">Потоковые операторы не считывают все исходные данные до получения элементов.</span><span class="sxs-lookup"><span data-stu-id="5c34e-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="5c34e-121">Во время выполнения потоковый оператор осуществляет свои действия в каждый исходный элемент доступен для чтения и возвращает элемент, если это уместно.</span><span class="sxs-lookup"><span data-stu-id="5c34e-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="5c34e-122">Потоковый оператор продолжает считывание исходных элементов до результирующий элемент может быть произведено.</span><span class="sxs-lookup"><span data-stu-id="5c34e-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="5c34e-123">Это означает, что для получения одного результата элемента могут быть прочитаны более одного исходного элемента.</span><span class="sxs-lookup"><span data-stu-id="5c34e-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="5c34e-124">Непотоковые</span><span class="sxs-lookup"><span data-stu-id="5c34e-124">Non-Streaming</span></span>  
 <span data-ttu-id="5c34e-125">Непотоковые операторы должны считывать все исходные данные до получения результирующего элемента.</span><span class="sxs-lookup"><span data-stu-id="5c34e-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="5c34e-126">В эту категорию попадают операциях сортировки и группировки.</span><span class="sxs-lookup"><span data-stu-id="5c34e-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="5c34e-127">Во время выполнения непотоковые операторы запросов считывают все исходные данные, помещают их в структуру данных, выполнить операцию и выдают итоговые элементы.</span><span class="sxs-lookup"><span data-stu-id="5c34e-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
