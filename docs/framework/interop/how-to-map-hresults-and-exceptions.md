<span data-ttu-id="84c31-103">Методы COM сообщают об ошибках, возвращая значения HRESULT.  <span data-ttu-id="84c31-106">Соответствующие значения HRESULT задаются определяемыми пользователем классами исключений.</span><span class="sxs-lookup"><span data-stu-id="84c31-106">User-defined exception classes can specify whatever HRESULT is appropriate.</span></span> <span data-ttu-id="84c31-107">Эти классы исключений могут динамически изменять значение HRESULT, возвращаемое при возникновении исключения, устанавливая поле **HResult** соответствующего объекта исключения.</span><span class="sxs-lookup"><span data-stu-id="84c31-107">These exception classes can dynamically change the HRESULT to be returned when the exception is generated by setting the **HResult** field on the exception object.</span></span> <span data-ttu-id="84c31-108">Дополнительные сведения об исключении предоставляются клиенту посредством интерфейса **IErrorInfo**, который реализуется для объекта .NET в неуправляемом процессе.</span><span class="sxs-lookup"><span data-stu-id="84c31-108">Additional information about the exception is provided to the client through the **IErrorInfo** interface, which is implemented on the .NET object in the unmanaged process.</span></span>

### <a name="to-create-a-new-exception-class-and-map-it-to-an-hresult"></a><span data-ttu-id="84c31-114">Создание нового класса исключения и его сопоставление со значением HRESULT</span><span class="sxs-lookup"><span data-stu-id="84c31-114">To create a new exception class and map it to an HRESULT</span></span>  
  
1.  <span data-ttu-id="84c31-115">В следующем коде создается новый класс исключения `NoAccessException`, который сопоставляется со значением HRESULT `E_ACCESSDENIED`.</span><span class="sxs-lookup"><span data-stu-id="84c31-115">Use the following code to create a new exception class called `NoAccessException` and map it to the HRESULT `E_ACCESSDENIED`.</span></span>  
  
    ```cpp

    {  
        NoAccessException () {  
        HResult = E_ACCESSDENIED;

    }

    {  
    throw new NoAccessException();  
    }  
    ```
