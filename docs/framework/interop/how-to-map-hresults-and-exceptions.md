# <a name="how-to-map-hresults-and-exceptions"></a><span data-ttu-id="e1626-102">Практическое руководство. Сопоставление значений HRESULT и исключений</span><span class="sxs-lookup"><span data-stu-id="e1626-102">How to: Map HRESULTs and Exceptions</span></span>
<span data-ttu-id="e1626-103">Методы COM сообщают об ошибках, возвращая значения HRESULT. Методы .NET в этом случае вызывают исключения.</span><span class="sxs-lookup"><span data-stu-id="e1626-103">COM methods report errors by returning HRESULTs; .NET methods report them by throwing exceptions.</span></span> <span data-ttu-id="e1626-104">Среда выполнения обеспечивает сопоставление этих двух элементов.</span><span class="sxs-lookup"><span data-stu-id="e1626-104">The runtime handles the transition between the two.</span></span> <span data-ttu-id="e1626-105">Каждый класс платформы .NET Framework сопоставляется со значением HRESULT.</span><span class="sxs-lookup"><span data-stu-id="e1626-105">Each exception class in the .NET Framework maps to an HRESULT.</span></span>  
  
 <span data-ttu-id="e1626-106">Соответствующие значения HRESULT задаются определяемыми пользователем классами исключений.</span><span class="sxs-lookup"><span data-stu-id="e1626-106">User-defined exception classes can specify whatever HRESULT is appropriate.</span></span> <span data-ttu-id="e1626-107">Эти классы исключений могут динамически изменять значение HRESULT, возвращаемое при возникновении исключения, устанавливая поле **HResult** соответствующего объекта исключения.</span><span class="sxs-lookup"><span data-stu-id="e1626-107">These exception classes can dynamically change the HRESULT to be returned when the exception is generated by setting the **HResult** field on the exception object.</span></span> <span data-ttu-id="e1626-108">Дополнительные сведения об исключении предоставляются клиенту посредством интерфейса **IErrorInfo**, который реализуется для объекта .NET в неуправляемом процессе.</span><span class="sxs-lookup"><span data-stu-id="e1626-108">Additional information about the exception is provided to the client through the **IErrorInfo** interface, which is implemented on the .NET object in the unmanaged process.</span></span>  
  
 <span data-ttu-id="e1626-109">При создании класса, который расширяет **System.Exception**, необходимо устанавливать значение поля HRESULT во время построения.</span><span class="sxs-lookup"><span data-stu-id="e1626-109">If you create a class that extends **System.Exception**, you must set the HRESULT field during construction.</span></span> <span data-ttu-id="e1626-110">В противном случае значение HRESULT будет присваиваться базовым классом.</span><span class="sxs-lookup"><span data-stu-id="e1626-110">Otherwise, the base class assigns the HRESULT value.</span></span> <span data-ttu-id="e1626-111">Чтобы сопоставить новые классы исключений с существующим значением HRESULT, укажите значение в конструкторе исключения.</span><span class="sxs-lookup"><span data-stu-id="e1626-111">You can map new exception classes to an existing HRESULT by supplying the value in the exception's constructor.</span></span>  
  
 <span data-ttu-id="e1626-112">Обратите внимание, что среда выполнения в некоторых случаях будет игнорировать значение `HRESULT`, если в потоке присутствует `IErrorInfo`.</span><span class="sxs-lookup"><span data-stu-id="e1626-112">Note that the runtime will sometimes ignore an `HRESULT` in cases where there is an `IErrorInfo` present on the thread.</span></span>  <span data-ttu-id="e1626-113">Это возможно в тех случаях, когда `HRESULT` и `IErrorInfo` представляют разные ошибки.</span><span class="sxs-lookup"><span data-stu-id="e1626-113">This behavior can occur in cases where the `HRESULT` and the `IErrorInfo` do not represent the same error.</span></span>  
  
### <a name="to-create-a-new-exception-class-and-map-it-to-an-hresult"></a><span data-ttu-id="e1626-114">Создание нового класса исключения и его сопоставление со значением HRESULT</span><span class="sxs-lookup"><span data-stu-id="e1626-114">To create a new exception class and map it to an HRESULT</span></span>  
  
1.  <span data-ttu-id="e1626-115">В следующем коде создается новый класс исключения `NoAccessException`, который сопоставляется со значением HRESULT `E_ACCESSDENIED`.</span><span class="sxs-lookup"><span data-stu-id="e1626-115">Use the following code to create a new exception class called `NoAccessException` and map it to the HRESULT `E_ACCESSDENIED`.</span></span>  
  
    ```cpp

    {  
        NoAccessException () {  
        HResult = E_ACCESSDENIED;

    }  
    CMyClass::MethodThatThrows  
    {  
    throw new NoAccessException();  
    }  
    ```
