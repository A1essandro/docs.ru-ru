## <a name="strong-name-scenario"></a><span data-ttu-id="5b198-114">Сценарий со строгим именем</span><span class="sxs-lookup"><span data-stu-id="5b198-114">Strong Name Scenario</span></span>  
 <span data-ttu-id="5b198-115">В следующем сценарии описывается процедура подписи сборки со строгим именем и последующая ссылка на сборку с использованием этого имени.</span><span class="sxs-lookup"><span data-stu-id="5b198-115">The following scenario outlines the process of signing an assembly with a strong name and later referencing it by that name.</span></span>  
  
1.  <span data-ttu-id="5b198-116">Сборка A создается со строгим именем с помощью одного из следующих методов.</span><span class="sxs-lookup"><span data-stu-id="5b198-116">Assembly A is created with a strong name using one of the following methods:</span></span>  
  
    -   <span data-ttu-id="5b198-117">Использование среды разработки, поддерживающей создание строгих имен, например [!INCLUDE[vsprvslong](../../../includes/vsprvslong-md.md)].</span><span class="sxs-lookup"><span data-stu-id="5b198-117">Using a development environment that supports creating strong names, such as [!INCLUDE[vsprvslong](../../../includes/vsprvslong-md.md)].</span></span>  
  
    -   <span data-ttu-id="5b198-118">Создание пары ключей шифрования с использованием [Программы строгих имен (Sn.exe)](../../../docs/framework/tools/sn-exe-strong-name-tool.md) и назначение этой пары ключей сборке с использованием компилятора командной строки или [компоновщика сборок (Al.exe)](../../../docs/framework/tools/al-exe-assembly-linker.md).</span><span class="sxs-lookup"><span data-stu-id="5b198-118">Creating a cryptographic key pair using the [Strong Name tool (Sn.exe)](../../../docs/framework/tools/sn-exe-strong-name-tool.md) and assigning that key pair to the assembly using either a command-line compiler or the [Assembly Linker (Al.exe)](../../../docs/framework/tools/al-exe-assembly-linker.md).</span></span> <span data-ttu-id="5b198-119">Пакет средств разработки Windows SDK предоставляет оба инструмента: Sn.exe и Al.exe.</span><span class="sxs-lookup"><span data-stu-id="5b198-119">The Windows Software Development Kit (SDK) provides both Sn.exe and Al.exe.</span></span>  
  
2.  <span data-ttu-id="5b198-120">Среда разработки или средство подписывает хэш файла, содержащего манифест сборки, закрытым ключом разработчика.</span><span class="sxs-lookup"><span data-stu-id="5b198-120">The development environment or tool signs the hash of the file containing the assembly's manifest with the developer's private key.</span></span> <span data-ttu-id="5b198-121">Эта цифровая подпись хранится в переносимом исполняемом файле (PE), который содержит манифест сборки А.</span><span class="sxs-lookup"><span data-stu-id="5b198-121">This digital signature is stored in the portable executable (PE) file that contains Assembly A's manifest.</span></span>  
  
3.  <span data-ttu-id="5b198-122">Сборка B является потребителем сборки A. Раздел ссылок манифеста сборки B содержит токен, представляющий открытый ключ сборки А.</span><span class="sxs-lookup"><span data-stu-id="5b198-122">Assembly B is a consumer of Assembly A. The reference section of Assembly B's manifest includes a token that represents Assembly A's public key.</span></span> <span data-ttu-id="5b198-123">Токен является частью полного открытого ключа и используется вместо самого ключа в целях экономии пространства.</span><span class="sxs-lookup"><span data-stu-id="5b198-123">A token is a portion of the full public key and is used rather than the key itself to save space.</span></span>  
  
4.  <span data-ttu-id="5b198-124">Среда CLR проверяет подпись строгого имени, когда сборка помещается в глобальный кэш сборок.</span><span class="sxs-lookup"><span data-stu-id="5b198-124">The common language runtime verifies the strong name signature when the assembly is placed in the global assembly cache.</span></span> <span data-ttu-id="5b198-125">При привязке по строгому имени во время выполнения среда CLR сравнивает ключ, сохраненный в манифесте сборки B, с ключом, используемым для создания строгого имени для сборки А. Если проверки безопасности .NET Framework пройдены и привязка выполнена успешно, у сборки B есть гарантия, что биты сборки A не были изменены и действительно получены от разработчиков сборки А.</span><span class="sxs-lookup"><span data-stu-id="5b198-125">When binding by strong name at run time, the common language runtime compares the key stored in Assembly B's manifest with the key used to generate the strong name for Assembly A. If the .NET Framework security checks pass and the bind succeeds, Assembly B has a guarantee that Assembly A's bits have not been tampered with and that these bits actually come from the developers of Assembly A.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="5b198-126">В этом сценарии не рассматриваются вопросы доверия.</span><span class="sxs-lookup"><span data-stu-id="5b198-126">This scenario doesn't address trust issues.</span></span> <span data-ttu-id="5b198-127">Помимо строгого имени сборки могут содержать полные подписи кода аутентификации Microsoft Authenticode.</span><span class="sxs-lookup"><span data-stu-id="5b198-127">Assemblies can carry full Microsoft Authenticode signatures in addition to a strong name.</span></span> <span data-ttu-id="5b198-128">Подписи Authenticode включают сертификат, который устанавливает отношения доверия.</span><span class="sxs-lookup"><span data-stu-id="5b198-128">Authenticode signatures include a certificate that establishes trust.</span></span> <span data-ttu-id="5b198-129">Важно отметить, что строгие имена не требуют подписи кода таким способом.</span><span class="sxs-lookup"><span data-stu-id="5b198-129">It's important to note that strong names don't require code to be signed in this way.</span></span> <span data-ttu-id="5b198-130">Строгие имена служат только для однозначной идентификации.</span><span class="sxs-lookup"><span data-stu-id="5b198-130">Strong names only provide a unique identity.</span></span>

## <a name="bypassing-signature-verification-of-trusted-assemblies"></a><span data-ttu-id="5b198-132">Обход проверки подписей в доверенных сборках</span><span class="sxs-lookup"><span data-stu-id="5b198-132">Bypassing Signature Verification of Trusted Assemblies</span></span>  
 <span data-ttu-id="5b198-133">Начиная с [!INCLUDE[net_v35SP1_long](../../../includes/net-v35sp1-long-md.md)] подписи строгого имени не проверяются, если сборка загружается в домен приложения с полным доверием, например в домен приложения по умолчанию для зоны `MyComputer`.</span><span class="sxs-lookup"><span data-stu-id="5b198-133">Starting with the [!INCLUDE[net_v35SP1_long](../../../includes/net-v35sp1-long-md.md)], strong-name signatures are not validated when an assembly is loaded into a full-trust application domain, such as the default application domain for the `MyComputer` zone.</span></span> <span data-ttu-id="5b198-134">Это называется функцией обхода строгих имен.</span><span class="sxs-lookup"><span data-stu-id="5b198-134">This is referred to as the strong-name bypass feature.</span></span> <span data-ttu-id="5b198-135">В среде с полным доверием всегда успешно обрабатываются запросы <xref:System.Security.Permissions.StrongNameIdentityPermission> для подписанных сбором с полным доверием независимо от их подписи.</span><span class="sxs-lookup"><span data-stu-id="5b198-135">In a full-trust environment, demands for <xref:System.Security.Permissions.StrongNameIdentityPermission> always succeed for signed, full-trust assemblies, regardless of their signature.</span></span> <span data-ttu-id="5b198-136">Функция обхода строгих имен позволяет избежать ненужных затрат ресурсов для проверки подписей строгих имен в сборках с полным доверием, что позволяет быстрее загружать такие сборки.</span><span class="sxs-lookup"><span data-stu-id="5b198-136">The strong-name bypass feature avoids the unnecessary overhead of strong-name signature verification of full-trust assemblies in this situation, allowing the assemblies to load faster.</span></span>  
  
 <span data-ttu-id="5b198-137">Функция обхода применима к любой сборке, подписанной со строгим именем и имеющей следующие характеристики.</span><span class="sxs-lookup"><span data-stu-id="5b198-137">The bypass feature applies to any assembly that is signed with a strong name and that has the following characteristics:</span></span>  
  
-   <span data-ttu-id="5b198-138">Полное доверие без доказательства <xref:System.Security.Policy.StrongName> (например, при наличии доказательства зоны `MyComputer`).</span><span class="sxs-lookup"><span data-stu-id="5b198-138">Fully trusted without <xref:System.Security.Policy.StrongName> evidence (for example, has `MyComputer` zone evidence).</span></span>  
  
-   <span data-ttu-id="5b198-139">Загрузка в домен <xref:System.AppDomain> с полным доверием.</span><span class="sxs-lookup"><span data-stu-id="5b198-139">Loaded into a fully trusted <xref:System.AppDomain>.</span></span>  
  
-   <span data-ttu-id="5b198-140">Загрузка из расположения со свойством <xref:System.AppDomainSetup.ApplicationBase%2A> домена <xref:System.AppDomain>.</span><span class="sxs-lookup"><span data-stu-id="5b198-140">Loaded from a location under the <xref:System.AppDomainSetup.ApplicationBase%2A> property of that <xref:System.AppDomain>.</span></span>  
  
-   <span data-ttu-id="5b198-141">Подпись осуществлена без задержки.</span><span class="sxs-lookup"><span data-stu-id="5b198-141">Not delay-signed.</span></span>  
  
 <span data-ttu-id="5b198-142">Эту возможность можно отключить для отдельных приложений или для компьютера.</span><span class="sxs-lookup"><span data-stu-id="5b198-142">This feature can be disabled for individual applications or for a computer.</span></span> <span data-ttu-id="5b198-143">См. информацию в разделе [Практическое руководство. Отключение возможности обхода строгих имен](../../../docs/framework/app-domains/how-to-disable-the-strong-name-bypass-feature.md).</span><span class="sxs-lookup"><span data-stu-id="5b198-143">See [How to: Disable the Strong-Name Bypass Feature](../../../docs/framework/app-domains/how-to-disable-the-strong-name-bypass-feature.md).</span></span>
