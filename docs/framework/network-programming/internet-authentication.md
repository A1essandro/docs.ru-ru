# Аутентификация в Интернете
Классы <xref:System.Net> поддерживают разнообразные механизмы проверки подлинности клиента, включая стандартные методы проверки подлинности через интернет основные, digest, NTLM и negotiate, проверка подлинности Kerberos, а также пользовательские методы, которые можно создать.  
  
 Учетные данные проверки подлинности хранятся в классах <xref:System.Net.NetworkCredential> и <xref:System.Net.CredentialCache>, которые реализуют интерфейс <xref:System.Net.ICredentials>.  Если один из этих классов запрашиваются для учетных данных, он возвращает экземпляр класса **NetworkCredential**.  Процесс проверки подлинности управлять классом <xref:System.Net.AuthenticationManager> и фактический процесс проверки подлинности выполняется классом модуля проверки подлинности, который реализует интерфейс <xref:System.Net.IAuthenticationModule>.  Необходимо зарегистрировать пользовательский модуль проверки подлинности с **AuthenticationManager**, прежде чем на них можно будет использовать. модули для основного, дайджест\-проверка подлинности, NTLM и рассматриваются методы проверки подлинности Kerberos регистрируется по умолчанию.  
  
 **NetworkCredential** хранит набор учетных данных, связанных с одним интернет\-ресурс указанный универсальный код ресурса \(uri\) и возвращает их в ответ на любой вызов метода <xref:System.Net.NetworkCredential.GetCredential%2A>.  Класс **NetworkCredential** обычно используется приложениями, которые обращаются к ограниченное число ресурсов в интернете или приложениями, которые используют один и тот же набор учетных данных во всех случаях.  
  
 Класс **CredentialCache** хранит коллекцию учетных данных для различных ресурсов в интернете.  Если метод <xref:System.Net.CredentialCache.GetCredential%2A> вызова, **CredentialCache** возвращает правильный набор учетных данных, как определено в uri ресурса в интернете и схемы проверки подлинности.  Приложения, использующие различные ресурсы через интернет с разными схемами проверки подлинности с помощью класса **CredentialCache** могут использовать, так как он сохраняет все учетные данные, и обеспечивает их как требуется.  
  
 Если интернет\-ресурс запрашивает проверку подлинности, метод <xref:System.Net.WebRequest.GetResponse%2A?displayProperty=fullName> отправляет <xref:System.Net.WebRequest> к **AuthenticationManager** вместе с запросом ввода учетных данных.  После этого запрос проходит проверку подлинности по следующему процессу.  
  
1.  **AuthenticationManager** вызывает метод <xref:System.Net.IAuthenticationModule.Authenticate%2A> на каждом из зарегистрированных модулей проверки подлинности в том порядке, в котором они были зарегистрированы.  **AuthenticationManager** использует первый модуль, который не возвращает **null** для выполнения процесса проверки подлинности.  Сведения процесса различаются в зависимости от включенных типа модуля проверки подлинности.  
  
2.  При завершении процесса проверки подлинности, модули проверки подлинности возвращает <xref:System.Net.Authorization> к **WebRequest**, содержащий сведения, необходимые для доступа к данному интернет\-ресурсу.  
  
 Некоторые схемы проверки подлинности, могут проверить подлинность пользователя без предварительного выполнения запрос ресурса.  Приложение может сэкономить время, preauthenticating пользователь с ресурсом, таким образом, удаление хотя бы один круговой путь к серверу.  Или он может выполнять проверку подлинности во время запуска программы для быть более отзывчив пользователю более поздней версии.  Схема проверки подлинности, которые могут использоваться предварительная проверка подлинности задать свойство [CanPreAuthenticate](frlrfsystemnetiauthenticationmoduleclasspreauthenticatetopic) к **true**.
