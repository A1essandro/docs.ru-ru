# Одноранговые имена и идентификаторы PNRP
Имя однорангового узла представляет конечную точку для взаимодействия, которая может быть компьютером, пользователем или группой, службой, связанным с узлом, который можно разрешить по адресу IP версии 6.  Протокол PNRP предполагает статистические уникальные имя однорангового узла для создания идентификатор PNRP, который используется для определения элементов облака.  
  
## Имена однорангового узла  
 Имя однорангового узла может быть зарегистрирован как незащищённое или защищены.  Небезопасные имена только текстовые строки, которые подлежат подмена, как любое может зарегистрировать повторяющееся имя незащищённое.  Небезопасные имена с закрытых дверях используется наилучшим образом в противном случае защищенные сети.  Предоставляемые имена защищатьы и цифровой подписью с сертификатом.  Только исходного издателя будет подтвердить владение предоставленного имени.  
  
 Область облака и сочетания предоставляют достаточно безопасную среду для одноранговых узлов, участвующих в действии PNRP.  Однако используя предоставленное имя однорангового узла не предоставляет общую безопасность приложения сети.  Безопасность приложения зависит от реализации.  
  
 Предоставленные имя однорангового узла зарегистрированы их только владелец и защищатьы с шифрованием открытого ключа.  Предоставленное имя однорангового узла является именный сущностью однорангового узла, для которого существует соответствующий закрытый ключ.  Владение может доказать через аттестованный адрес однорангового узла \(CPA\), подписывание с использованием закрытого ключа.  Злонамеренный пользователь не может выковать владение имени однорангового узла без соответствующего закрытого ключа.  
  
## Идентификатор PNRP  
 ![Идентификатор PNRP](../../../docs/framework/network-programming/media/fdc9e8a0-4a1c-488d-a019-bc3a1973220c.png "fdc9e8a0\-4a1c\-488d\-a019\-bc3a1973220c")  
  
 Идентификаторы PNRP состоят из следующих:  
  
-   Биты высок\- заказа 128, называемые одноранговое идентификатор \(P2P\), хэш имени однорангового узла, присвоенный конечной точке.  Имя однорангового узла имеет следующий формат: *Authority.Classifier*.  Для безопасных имен, *nt authority* безопасное хэш хэш\-алгоритм 1 \(SHA1\) открытого ключа имени однорангового узла в шестнадцатеричных знаков.  Для незащищенных имен, *nt authority* одиночный символ "0".  *Классификатор* строка, идентифицирующая приложение.  Нет классификатор имени однорангового узла не может превышать 149 символов, в том числе признак конца `null`.  
  
-   Младшие 128 разрядов используются для обнаружения службы, созданный номер, который определяет разные экземпляры одного и того же идентификатор P2P в одном и том же облаке.  
  
 Этот идентификатор и обнаружения службы P2P сочетание позволяют нескольким идентификаторам PNRP, который необходимо зарегистрировать с одного компьютера.
