# Интерпретация сетевой трассировки
Если включена трассировка сети, можно использовать трассировку для перехвата вызовов приложение делает на другой <xref:System.Net> членов класса.  Выход из этих вызовов может быть аналогичен следующим примерам.  
  
```  
[588]   (4357)   Entering Socket#33574638::Send()  
[588]   (4387)   Exiting Socket#33574638::Send()-> 61#61  
```  
  
 В предыдущем примере \[588\] — уникальный идентификатор текущего потока.  \(4357\) и \(4387\) отметки времени, обозначающая число миллисекунд, в которых истекли с момента запуска приложения.  Данные за отметкой времени показано приложение ввода и оставить метод **Socket.Send**.  Объект при выполнении метода **Отправить** имеет 33574638 в качестве уникального идентификатора.  Трассировка оставить метода содержит возвращаемое значение \(61 в предыдущем примере\).  
  
 Трассировка сети могут захватить сетевой трафик, отправляет или получен приложением с помощью протоколов уровня приложения, например протокол передачи гипертекста \(HTTP\).  Эти данные можно записать в виде текста и при необходимости шестнадцатеричные данные.  Шестнадцатеричные данные доступны при указании **includehex** в качестве значения атрибута **tracemode**.  \(Подробные сведения об этом атрибуте см. в разделе [Практическое руководство. Настройте трассировку сети](../../../docs/framework/network-programming/how-to-configure-network-tracing.md)\). Ниже приведен пример трассировки было создано с помощью **includehex**.  
  
 `[1692]   (1142)   00000000 : 47 45 54 20 2F 77 70 61-64 2E 64 61 74 20 48 54 : GET /wpad.dat HT`  
  
 `[1692]   (1142)   00000010 : 54 50 2F 31 2E 31 0D 0A-48 6F 73 74 3A 20 69 74 : TP/1.1..Host: it`  
  
 `[1692]   (1142)   00000020 : 67 70 72 6F 78 79 0D 0A-43 6F 6E 6E 65 63 74 69 : gproxy..Connecti`  
  
 `[1692]   (1142)   00000030 : 6F 6E 3A 20 43 6C 6F 73-65 0D 0A 0D 0A     : on: Close....`  
  
 Чтобы исключить шестнадцатеричные данные, укажите **protocolonly** в качестве значения для атрибута **tracemode**.  В следующем примере показан трассировку, если **protocolonly** указано.  
  
 `[2444]   (594)   Data from ConnectStream#33574638::WriteHeaders<<GET /wpad.dat HTTP/1.1`  
  
 `Host: itgproxy`  
  
 `Connection: Close`
