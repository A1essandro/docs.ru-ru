# Группирование подключений
Группирование подключений связывает особые требования в пределах одного приложения в указанный пул соединения.  Это может быть приложением среднего уровня, который подключается к конечный сервер, именем пользователя и использует протокол проверки подлинности, поддерживают делегирование Kerberos или приложением, например среднего уровня, который предоставляет свои учетные данные, как показано в примере ниже.  Предположим, пользователь, Джо, посещения его внутреннее веб\-сайта, указывающее сведения о зарплате.  После проверки подлинности Джо, сервер приложений среднего уровня joe использует учетные данные для подключения к конечным сервер для извлечения него сведения о зарплате.  Далее Сьюзан посещает сайт и запрашивает ее сведения о зарплате.  Поскольку приложение промежуточного уровня, уже налаживало соединение, используя учетные данные Джо, конечный сервер отвечает с данными joe.  Однако если приложение будет присвоено каждый запрос, отправленный на конечный сервер в группе соединения сформированной от имени пользователя, то каждый пользователь принадлежит к отдельной группе соединения и случайным образом, не может совместно использовать данные проверки подлинности с другим пользователем.  
  
 Для присвоения запрос к определенной группе соединения необходимо присвоить свойству <xref:System.Net.WebRequest.ConnectionGroupName%2A> вашего имени <xref:System.Net.WebRequest> перед выполнением запроса.
