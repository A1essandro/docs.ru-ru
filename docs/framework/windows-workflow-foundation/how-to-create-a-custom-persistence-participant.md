# <a name="how-to-create-a-custom-persistence-participant"></a><span data-ttu-id="430e0-102">Как создать настраиваемого участника сохраняемости</span><span class="sxs-lookup"><span data-stu-id="430e0-102">How to: Create a Custom Persistence Participant</span></span>
<span data-ttu-id="430e0-103">В следующей процедуре содержаться шаги для создания участника сохраняемости.</span><span class="sxs-lookup"><span data-stu-id="430e0-103">The following procedure has steps to create a persistence participant.</span></span> <span data-ttu-id="430e0-104">В разделе [участия в сохраняемости](http://go.microsoft.com/fwlink/?LinkID=177735) образца и [расширения хранилища](../../../docs/framework/windows-workflow-foundation/store-extensibility.md) раздел, описывающий образец реализации участников сохраняемости.</span><span class="sxs-lookup"><span data-stu-id="430e0-104">See the [Participating in Persistence](http://go.microsoft.com/fwlink/?LinkID=177735) sample and [Store Extensibility](../../../docs/framework/windows-workflow-foundation/store-extensibility.md) topic for sample implementations of persistence participants.</span></span>  
  
1.  <span data-ttu-id="430e0-105">Создайте класс, который происходит от класса <xref:System.Activities.Persistence.PersistenceParticipant> или <xref:System.Activities.Persistence.PersistenceIOParticipant>.</span><span class="sxs-lookup"><span data-stu-id="430e0-105">Create a class deriving from the <xref:System.Activities.Persistence.PersistenceParticipant> or the <xref:System.Activities.Persistence.PersistenceIOParticipant> class.</span></span> <span data-ttu-id="430e0-106">Класс PersistenceIOParticipant предлагает же точки расширяемости, что и класс PersistenceParticipant является возможность участия в операциях ввода-вывода.</span><span class="sxs-lookup"><span data-stu-id="430e0-106">The PersistenceIOParticipant class offers the same extensibility points as the PersistenceParticipant class in addition to being able to participate in I/O operations.</span></span> <span data-ttu-id="430e0-107">Выполните один или несколько следующих шагов.</span><span class="sxs-lookup"><span data-stu-id="430e0-107">Follow one or more of the following steps.</span></span>  
  
2.  <span data-ttu-id="430e0-108">Выполните метод <xref:System.Activities.Persistence.PersistenceParticipant.CollectValues%2A>.</span><span class="sxs-lookup"><span data-stu-id="430e0-108">Implement the <xref:System.Activities.Persistence.PersistenceParticipant.CollectValues%2A> method.</span></span> <span data-ttu-id="430e0-109">**CollectValues** содержит два словарных параметра, один для хранения значений для чтения и записи и другой способ для сохранения значений только для записи (используется в дальнейшем в запросах).</span><span class="sxs-lookup"><span data-stu-id="430e0-109">The **CollectValues** method has two dictionary parameters, one for storing read/write values and the other one for storing write-only values (used later in queries).</span></span> <span data-ttu-id="430e0-110">В этом методе необходимо заполнить словари данными, соответствующими участнику сохраняемости.</span><span class="sxs-lookup"><span data-stu-id="430e0-110">In this method, you should populate these dictionaries with data that is specific to a persistence participant.</span></span> <span data-ttu-id="430e0-111">Каждый словарь содержит имя значения в качестве ключа и само значение в качестве объекта <xref:System.Runtime.DurableInstancing.InstanceValue>.</span><span class="sxs-lookup"><span data-stu-id="430e0-111">Each dictionary contains the name of the value as the key and the value itself as an <xref:System.Runtime.DurableInstancing.InstanceValue> object.</span></span>  
  
     <span data-ttu-id="430e0-112">Значения в словаре readWriteValues упаковываются в виде **InstanceValue** объектов.</span><span class="sxs-lookup"><span data-stu-id="430e0-112">The values in the readWriteValues dictionary are packaged as **InstanceValue** objects.</span></span> <span data-ttu-id="430e0-113">Значения в словаре только для записи упаковываются в виде **InstanceValue** объектов с заданными параметрами InstanceValueOptions.Optional и InstanceValueOption.WriteOnly.</span><span class="sxs-lookup"><span data-stu-id="430e0-113">The values in the write-only dictionary are packaged as **InstanceValue** objects with InstanceValueOptions.Optional and InstanceValueOption.WriteOnly set.</span></span> <span data-ttu-id="430e0-114">Каждый **InstanceValue** , предоставляемые **CollectValues** реализации для всех участников сохраняемости, должен иметь уникальное имя.</span><span class="sxs-lookup"><span data-stu-id="430e0-114">Each **InstanceValue** provided by the **CollectValues** implementations across all persistence participants must have a unique name.</span></span>  
  
    ```  
    protected virtual void CollectValues (out IDictionary<XName,Object> readWriteValues, out IDictionary<XName,Object> writeOnlyValues)  
    ```  
  
3.  <span data-ttu-id="430e0-115">Выполните метод <xref:System.Activities.Persistence.PersistenceParticipant.MapValues%2A>.</span><span class="sxs-lookup"><span data-stu-id="430e0-115">Implement the <xref:System.Activities.Persistence.PersistenceParticipant.MapValues%2A> method.</span></span> <span data-ttu-id="430e0-116">**MapValues** метод принимает два параметра, которые похожи на параметры, **CollectValues** метод получает.</span><span class="sxs-lookup"><span data-stu-id="430e0-116">The **MapValues** method takes two parameters that are similar to the parameters that the **CollectValues** method receives.</span></span> <span data-ttu-id="430e0-117">Все значения, собранные в **CollectValues** этап передаются посредством этих словарных параметров.</span><span class="sxs-lookup"><span data-stu-id="430e0-117">All the values collected in the **CollectValues** stage are passed through these dictionary parameters.</span></span> <span data-ttu-id="430e0-118">Новые значения, добавленные **MapValues** этапе добавляются значения только для записи.</span><span class="sxs-lookup"><span data-stu-id="430e0-118">The new values added by the **MapValues** stage are added to the write-only values.</span></span>  <span data-ttu-id="430e0-119">Доступный только на запись словарь используется для передачи данных во внешний источник данных, не связанный напрямую со значениями экземпляра.</span><span class="sxs-lookup"><span data-stu-id="430e0-119">The write-only dictionary is used to provide data to an external source not directly associated with the instance values.</span></span> <span data-ttu-id="430e0-120">Каждого значения, предоставленные реализациями **MapValues** метод для всех участников сохраняемости, должен иметь уникальное имя.</span><span class="sxs-lookup"><span data-stu-id="430e0-120">Each value provided by implementations of the **MapValues** method across all persistence participants must have a unique name.</span></span>  
  
    ```  
    protected virtual IDictionary<XName,Object> MapValues (IDictionary<XName,Object> readWriteValues,IDictionary<XName,Object> writeOnlyValues)  
    ```  
  
     <span data-ttu-id="430e0-121">Метод <xref:System.Activities.Persistence.PersistenceParticipant.MapValues%2A> предоставляет функциональность, не реализуемую <xref:System.Activities.Persistence.PersistenceParticipant.CollectValues%2A>, а именно допускает зависимость от другого значения, предоставленного другим участником сохраняемости, который еще не был обработан <xref:System.Activities.Persistence.PersistenceParticipant.CollectValues%2A>.</span><span class="sxs-lookup"><span data-stu-id="430e0-121">The <xref:System.Activities.Persistence.PersistenceParticipant.MapValues%2A> method provides functionality that <xref:System.Activities.Persistence.PersistenceParticipant.CollectValues%2A> does not, in that it allows for a dependency on another value provided by another persistence participant that hasn’t been processed by <xref:System.Activities.Persistence.PersistenceParticipant.CollectValues%2A> yet.</span></span>  
  
4.  <span data-ttu-id="430e0-122">Реализуйте **PublishValues** метод.</span><span class="sxs-lookup"><span data-stu-id="430e0-122">Implement the **PublishValues** method.</span></span> <span data-ttu-id="430e0-123">**PublishValues** метод получает словарь, содержащий все значения, загруженные из хранилища сохраняемости.</span><span class="sxs-lookup"><span data-stu-id="430e0-123">The **PublishValues** method receives a dictionary containing all the values loaded from the persistence store.</span></span>  
  
    ```  
    protected virtual void PublishValues (IDictionary<XName,Object> readWriteValues)  
    ```  
  
5.  <span data-ttu-id="430e0-124">Реализуйте **BeginOnSave** метод, если участник является участником сохраняемости ввода-вывода.</span><span class="sxs-lookup"><span data-stu-id="430e0-124">Implement the **BeginOnSave** method if the participant is a persistence I/O participant.</span></span> <span data-ttu-id="430e0-125">Метод вызывается во время создания сохранения.</span><span class="sxs-lookup"><span data-stu-id="430e0-125">This method is called during a Save operation.</span></span> <span data-ttu-id="430e0-126">В этом методе необходимо выполнить дополнительные операции ввода-вывода при сохранении экземпляров рабочих процессов.</span><span class="sxs-lookup"><span data-stu-id="430e0-126">In this method, you should perform I/O adjunct to the persisting (saving) workflow instances.</span></span>  <span data-ttu-id="430e0-127">Если узел использует транзакцию для соответствующей команды сохраняемости, та же самая транзакция используется в Transaction.Current.</span><span class="sxs-lookup"><span data-stu-id="430e0-127">If the host is using a transaction for the corresponding persistence command, the same transaction is provided in Transaction.Current.</span></span>  <span data-ttu-id="430e0-128">Помимо этого, PersistenceIOParticipants могут объявить о требовании к совместимости транзакций, в случае чего узел создаст транзакцию для сеанса сохраняемости, если такая транзакция не будет использована иначе.</span><span class="sxs-lookup"><span data-stu-id="430e0-128">Additionally, PersistenceIOParticipants may advertise a transactional consistency requirement, in which case the host creates a transaction for the persistence episode if one would not otherwise be used.</span></span>  
  
    ```  
    protected virtual IAsyncResult BeginOnSave (IDictionary<XName,Object> readWriteValues, IDictionary<XName,Object> writeOnlyValues, TimeSpan timeout, AsyncCallback callback, Object state)  
    ```  
  
6.  <span data-ttu-id="430e0-129">Реализуйте **BeginOnLoad** метод, если участник является участником сохраняемости ввода-вывода.</span><span class="sxs-lookup"><span data-stu-id="430e0-129">Implement the **BeginOnLoad** method if the participant is a persistence I/O participant.</span></span> <span data-ttu-id="430e0-130">Метод вызывается во время создания загрузки.</span><span class="sxs-lookup"><span data-stu-id="430e0-130">This method is called during a Load operation.</span></span> <span data-ttu-id="430e0-131">В этом методе необходимо выполнить дополнительные операции ввода-вывода при загрузке экземпляров рабочих процессов.</span><span class="sxs-lookup"><span data-stu-id="430e0-131">In this method, you should perform I/O adjunct to the loading of workflow instances.</span></span> <span data-ttu-id="430e0-132">Если узел использует транзакцию для соответствующей команды сохраняемости, та же самая транзакция используется в Transaction.Current.</span><span class="sxs-lookup"><span data-stu-id="430e0-132">If the host is using a transaction for the corresponding persistence command, the same transaction is provided in Transaction.Current.</span></span> <span data-ttu-id="430e0-133">Кроме того участники ввода-вывода сохраняемости могут объявить о требовании к совместимости транзакций, в случае чего узел создаст транзакцию для сеанса сохраняемости, если такая транзакция не будет использована иначе.</span><span class="sxs-lookup"><span data-stu-id="430e0-133">Additionally, Persistence I/O participants may advertise a transactional consistency requirement, in which case the host creates a transaction for the persistence episode if one would not otherwise be used.</span></span>  
  
    ```  
    protected virtual IAsyncResult BeginOnLoad (IDictionary<XName,Object> readWriteValues, TimeSpan timeout, AsyncCallback callback, Object state)  
    ```
