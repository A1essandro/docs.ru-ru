# <a name="dynamic-arguments"></a><span data-ttu-id="ce50f-102">Динамические аргументы</span><span class="sxs-lookup"><span data-stu-id="ce50f-102">Dynamic Arguments</span></span>
<span data-ttu-id="ce50f-103">В этом образце показано, как реализовать действие, аргументы которого определяются объектом-получателем, а не автором действия.</span><span class="sxs-lookup"><span data-stu-id="ce50f-103">This sample demonstrates how to implement an activity for which the arguments are defined by the activity consumer rather than the activity author.</span></span> <span data-ttu-id="ce50f-104">Это достигается переопределением способа построения метаданных действия в среде выполнения.</span><span class="sxs-lookup"><span data-stu-id="ce50f-104">It does this by overriding the way the runtime constructs the activity’s metadata.</span></span>  
  
## <a name="sample-details"></a><span data-ttu-id="ce50f-105">Подробные сведения об образце</span><span class="sxs-lookup"><span data-stu-id="ce50f-105">Sample details</span></span>  
 <span data-ttu-id="ce50f-106">Перед выполнением среда выполнения строит описание действия путем анализа открытых членов типа действия и автоматического объявления аргументов, переменных, дочерних действий и делегатов действия как частей метаданных действия.</span><span class="sxs-lookup"><span data-stu-id="ce50f-106">Prior to execution, the runtime builds a description of an activity by data-ttu-id="ce50f-107">Это делается, чтобы обеспечить правильное построение рабочего процесса, а также управление связями времени выполнения и правилами времени существования с помощью реализации.</span><span class="sxs-lookup"><span data-stu-id="ce50f-107">It does this to ensure correct construction of a workflow as well as to manage run-time relationships and lifetime rules.</span></span> <span data-ttu-id="ce50f-108">Обычно автор действия определяет аргументы действия, задавая открытые члены типа действия, производные от <xref:System.Activities.Argument>.</span><span class="sxs-lookup"><span data-stu-id="ce50f-108">Typically an activity author defines the arguments of an activity by specifying public members on the activity type that derive from <xref:System.Activities.Argument>.</span></span> <span data-ttu-id="ce50f-109">Для каждого открытого члена, производного от <xref:System.Activities.Argument>, среда data-stu-id="ce50f-109">For each public member that derives from <xref:System.Activities.Argument>, the runtime creates a <xref:System.Activities.RuntimeArgument> and binds it to the user-provided argument set on that member.</span></span> <span data-ttu-id="ce50f-110">Но в некоторых случаях получатель действия предоставляет конфигурацию, которая определяет набор аргументов.</span><span class="sxs-lookup"><span data-stu-id="ce50f-110">In some cases, however, the consumer of the activity provides some configuration that determines the set of arguments.</span></span> <span data-ttu-id="ce50f-111">Автор действия переопределяет <xref:System.Activities.Activity.CacheMetadata%2A>, чтобы настроить способ построения метаданных действия, в том числе набор аргументов, связанных с действием.</span><span class="sxs-lookup"><span data-stu-id="ce50f-111">An activity author overrides <xref:System.Activities.Activity.CacheMetadata%2A> to customize the way activity metadata is  <span data-ttu-id="ce50f-112">В этом образце показано, как динамически построить список аргументов для действия, вызывающего метод.</span><span class="sxs-lookup"><span data-stu-id="ce50f-112">This sample demonstrates how to build an argument list dynamically for an activity that invokes a method.</span></span> <span data-ttu-id="ce50f-113">Получатель действия указывает тип и имя метода, который нужно вызвать, наряду с коллекцией передаваемых этому методу аргументов.</span><span class="sxs-lookup"><span data-stu-id="ce50f-113">The activity consumer specifies the type and the method name they want to invoke along with a collection of arguments to be passed to that method.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="ce50f-114">Цель этого образца - продемонстрировать переопределение метода <xref:System.Activities.CodeActivity.CacheMetadata%2A> и использование класса <xref:System.Activities.RuntimeArgument>.</span><span class="sxs-lookup"><span data-stu-id="ce50f-114">The purpose of this sample is to demonstrate how to override <xref:System.Activities.CodeActivity.CacheMetadata%2A> and how to use <xref:System.Activities.RuntimeArgument>.</span></span> <span data-ttu-id="ce50f-115">Существует несколько проблем, связанных с разновидностями методов, которые может вызывать действие.</span><span class="sxs-lookup"><span data-stu-id="ce50f-115">There are several caveats with respect to the kinds of methods that this activity can invoke.</span></span> <span data-ttu-id="ce50f-116">Например, оно не работает с универсальными шаблонами или массивами параметров.</span><span class="sxs-lookup"><span data-stu-id="ce50f-116">For example, it does not work with generics or parameter arrays.</span></span> <span data-ttu-id="ce50f-117">Действие <xref:System.Activities.Statements.InvokeMethod>, поставляемое с .NET Framework, обрабатывает эти и другие случаи.</span><span class="sxs-lookup"><span data-stu-id="ce50f-117">The <xref:System.Activities.Statements.InvokeMethod> activity that ships in.NET Framework handles these cases and more.</span></span>  
  
 <span data-ttu-id="ce50f-118">Действие `MethodInvoke` переопределяет метод <xref:System.Activities.CodeActivity.CacheMetadata%2A> и начинает с создания класса <xref:System.Activities.RuntimeArgument> для обработки любого результата вызова метода.</span><span class="sxs-lookup"><span data-stu-id="ce50f-118">The `MethodInvoke` activity overrides <xref:System.Activities.CodeActivity.CacheMetadata%2A> and begins by creating a <xref:System.Activities.RuntimeArgument> to handle any result from the method invocation.</span></span> <span data-ttu-id="ce50f-119">Оно привязывает этот аргумент <xref:System.Activities.RuntimeArgument> к открыто задаваемому классу <xref:System.Activities.OutArgument> с именем `Result`.</span><span class="sxs-lookup"><span data-stu-id="ce50f-119">It binds this <xref:System.Activities.RuntimeArgument> to the publicly settable <xref:System.Activities.OutArgument> named `Result`.</span></span> <span data-ttu-id="ce50f-120">Если `MethodInvoke.Result` имеет значение `null`, среда выполнения автоматически заполняет его значением <xref:System.Activities.OutArgument>, которому присвоено выражение по умолчанию для данного типа.</span><span class="sxs-lookup"><span data-stu-id="ce50f-120">If `MethodInvoke.Result` is `null`, the runtime automatically populates it with an <xref:System.Activities.OutArgument> configured with the default expression for its type.</span></span> <span data-ttu-id="ce50f-121">Поэтому автору действия никогда не требуется проверять, имеет ли свойство аргумента значение `null`.</span><span class="sxs-lookup"><span data-stu-id="ce50f-121">This behavior means an activity author never has to check whether an argument property is `null`.</span></span>  
  
 <span data-ttu-id="ce50f-122">Затем переопределение метода <xref:System.Activities.CodeActivity.CacheMetadata%2A> определяет `MethodInfo`, используемый для вызова, из предоставляемых пользователем значений `MethodName` и `TargetType`.</span><span class="sxs-lookup"><span data-stu-id="ce50f-122">Next, the <xref:System.Activities.CodeActivity.CacheMetadata%2A> override determines the `MethodInfo` it uses for invocation from the user-provided `MethodName` and `TargetType`.</span></span> <span data-ttu-id="ce50f-123">Метод `DetermineMethodInfo` принимает параметр <xref:System.Activities.CodeActivityMetadata>, переданный переопределению <xref:System.Activities.CodeActivity.CacheMetadata%2A>, чтобы о любых ошибках конфигурации сообщалось как об ошибках проверки.</span><span class="sxs-lookup"><span data-stu-id="ce50f-123">The `DetermineMethodInfo` method takes the <xref:System.Activities.CodeActivityMetadata> parameter passed to the <xref:System.Activities.CodeActivity.CacheMetadata%2A> override so that any configuration errors can be reported as validation errors.</span></span> <span data-ttu-id="ce50f-124">Это выполняется с помощью вызова метода `metadata.AddValidationError`.</span><span class="sxs-lookup"><span data-stu-id="ce50f-124">This is done by calling `metadata.AddValidationError`.</span></span>  
  
 <span data-ttu-id="ce50f-125">После того, как задано значение `MethodInfo`, образец выполняет перебор параметров `MethodInfo`.</span><span class="sxs-lookup"><span data-stu-id="ce50f-125">Once the `MethodInfo` has been set, the sample iterates over the `MethodInfo` parameters.</span></span> <span data-ttu-id="ce50f-126">Для каждого параметра он создает объект <xref:System.Activities.RuntimeArgument> и привязывает его к соответствующему аргументу в предоставленной пользователем коллекции из свойства `Parameters`.</span><span class="sxs-lookup"><span data-stu-id="ce50f-126">For each parameter, it creates a <xref:System.Activities.RuntimeArgument> and binds it to the corresponding argument in the user-provided collection from the `Parameters` property.</span></span> <span data-ttu-id="ce50f-127">Наконец, коллекция объектов <xref:System.Activities.RuntimeArgument> связывается с действием посредством вызова `metadata.SetArgumentsCollection`.</span><span class="sxs-lookup"><span data-stu-id="ce50f-127">Finally, the collection of <xref:System.Activities.RuntimeArgument>s is associated with the activity by calling `metadata.SetArgumentsCollection`.</span></span>  
  
 <span data-ttu-id="ce50f-128">Обратите внимание, что разрешение аргументов может быть выполнено с использованием класса <xref:System.Activities.RuntimeArgument>, как в случае с `resultArgument`, или по предоставленному пользователем аргументу, как в случае коллекции `Parameters`.</span><span class="sxs-lookup"><span data-stu-id="ce50f-128">Note that argument resolution can be done using a <xref:System.Activities.RuntimeArgument>, as in the case of `resultArgument` or the user-provided argument, as in the case of the `Parameters` collection.</span></span>
