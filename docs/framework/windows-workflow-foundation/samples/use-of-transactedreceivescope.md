# <a name="use-of-transactedreceivescope"></a><span data-ttu-id="e5bf1-102">Использование TransactedReceiveScope</span><span class="sxs-lookup"><span data-stu-id="e5bf1-102">Use of TransactedReceiveScope</span></span>
<span data-ttu-id="e5bf1-103">В этом образце показывается передача транзакции от клиента к серверу с использованием действия <xref:System.Activities.Statements.TransactionScope> для создания новой транзакции на клиенте и действия <xref:System.ServiceModel.Activities.TransactedReceiveScope> для получения сообщения с передаваемой транзакцией и определения времени ее существования на сервере.</span><span class="sxs-lookup"><span data-stu-id="e5bf1-103">This sample shows how to flow a transaction from a client to a server using <xref:System.Activities.Statements.TransactionScope> to create a new transaction on the client and a <xref:System.ServiceModel.Activities.TransactedReceiveScope> to receive a message with a flowed transaction and scope the lifetime of the transaction on the server.</span></span> <span data-ttu-id="e5bf1-104">Образец состоит из двух проектов, исполняющих роли клиента и сервера.</span><span class="sxs-lookup"><span data-stu-id="e5bf1-104">The sample consists of two projects that fill the roles of client and server.</span></span>  
  
## <a name="client-application"></a><span data-ttu-id="e5bf1-105">Клиентское приложение</span><span class="sxs-lookup"><span data-stu-id="e5bf1-105">Client Application</span></span>  
 <span data-ttu-id="e5bf1-106">Клиентское приложение выполняет рабочий процесс, который выводит идентификатор распределенной транзакции, отсылает сообщение на сервер, передает транзакцию, получает ответ, снова выводит идентификатор распределенной транзакции и завершает свою работу.</span><span class="sxs-lookup"><span data-stu-id="e5bf1-106">The client application runs a workflow that prints the distributed transaction ID, sends a message to the server, flows the transaction, receives the reply, prints the distributed transaction ID again and completes.</span></span> <span data-ttu-id="e5bf1-107">При первоначальном выводе идентификатора распределенной транзакции это будет пустой идентификатор GUID, поскольку транзакция все еще является локальной.</span><span class="sxs-lookup"><span data-stu-id="e5bf1-107">When the distributed transaction ID is initially prints, it is an empty GUID as the transaction is still only local.</span></span>  
  
## <a name="server-application"></a><span data-ttu-id="e5bf1-108">Серверное приложение</span><span class="sxs-lookup"><span data-stu-id="e5bf1-108">Server Application</span></span>  
 <span data-ttu-id="e5bf1-109">Проект сервера схож, однако рабочий процесс размещен на узле <xref:System.ServiceModel.Activities.WorkflowServiceHost>, поскольку он должен прослушивать конечную точку на предмет получения сообщения от клиента.</span><span class="sxs-lookup"><span data-stu-id="e5bf1-109">The server project is similar, however, the workflow is hosted in <xref:System.ServiceModel.Activities.WorkflowServiceHost> because it must listen on an endpoint for the message from the client.</span></span> <span data-ttu-id="e5bf1-110">Рабочий процесс ориентирован на действие <xref:System.ServiceModel.Activities.TransactedReceiveScope>, которое получает переданную транзакцию от клиента, выводит отправленное сообщение, выводит идентификатор распределенной транзакции и отправляет ответ клиенту.</span><span class="sxs-lookup"><span data-stu-id="e5bf1-110">The workflow is centered on the <xref:System.ServiceModel.Activities.TransactedReceiveScope>, which receives the flowed transaction from the client, prints the message that was sent, prints the distributed transaction ID and sends the reply to the client.</span></span> <span data-ttu-id="e5bf1-111">Теперь идентификатор распределенной транзакции является не пустым идентификатором GUID; если в текст действия <xref:System.ServiceModel.Activities.TransactedReceiveScope> было добавлено поддерживающее транзакции действие, оно будет выполнено в переданной транзакции.</span><span class="sxs-lookup"><span data-stu-id="e5bf1-111">The distributed transaction ID is now a non-empty GUID and if a transaction-aware activity was added to the body of the <xref:System.ServiceModel.Activities.TransactedReceiveScope>, it would execute under the flowed transaction.</span></span>
