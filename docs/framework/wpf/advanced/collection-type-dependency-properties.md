 <span data-ttu-id="9dad5-105">Для свойства зависимостей в общем случае реализация шаблон следовать — определить [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] обертки свойства, где это свойство является <xref:System.Windows.DependencyProperty> идентификатор, а не поле или другой конструкции.</span><span class="sxs-lookup"><span data-stu-id="9dad5-105">For a dependency property in general, the implementation pattern that you follow is that you define a [!INCLUDE[TLA2#tla_clr](../../../../includes/tla2sharptla-clr-md.md)] property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="9dad5-106">При реализации свойства типа "коллекция" вы следуете этому же шаблону.</span><span class="sxs-lookup"><span data-stu-id="9dad5-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="9dad5-107">Однако свойство типа коллекции представляет некоторую сложность шаблону, каждый раз, когда сам является тип, который содержится в коллекции <xref:System.Windows.DependencyObject> или <xref:System.Windows.Freezable> производного класса.</span><span class="sxs-lookup"><span data-stu-id="9dad5-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

 <span data-ttu-id="9dad5-109">При создании свойства зависимостей вы не указываете значение свойства по умолчанию в качестве начального значения поля.</span><span class="sxs-lookup"><span data-stu-id="9dad5-109">When you create a dependency property, you do not specify the property default value as the initial field value.</span></span> <span data-ttu-id="9dad5-110">Вместо этого значение по умолчанию указывается через метаданные свойства зависимостей.</span><span class="sxs-lookup"><span data-stu-id="9dad5-110">Instead, you specify the default value through the dependency property metadata.</span></span> <span data-ttu-id="9dad5-111">Если свойство является ссылочным типом, значение по умолчанию, заданное в метаданных свойства зависимостей, не является значением по умолчанию для каждого экземпляра. Напротив, это значение по умолчанию, которое применяется ко всем экземплярам типа.</span><span class="sxs-lookup"><span data-stu-id="9dad5-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; instead it is a default value that applies to all instances of the type.</span></span> <span data-ttu-id="9dad5-112">Поэтому необходимо избегать использования единственной статической коллекции, определенной метаданными свойства коллекции, в качестве рабочего значения по умолчанию для вновь создаваемых экземпляров типа.</span><span class="sxs-lookup"><span data-stu-id="9dad5-112">Therefore you must be careful to not use the singular static collection defined by the collection property metadata as the working default value for newly created instances of your type.</span></span> <span data-ttu-id="9dad5-113">Вместо этого необходимо явно задать в качестве значения коллекции уникальную коллекцию (экземпляр) как часть логики конструктора класса.</span><span class="sxs-lookup"><span data-stu-id="9dad5-113">Instead, you must make sure that you deliberately set the collection value to a unique (instance) collection as part of your class constructor logic.</span></span> <span data-ttu-id="9dad5-114">В противном случае будет создан случайный одноэлементный класс.</span><span class="sxs-lookup"><span data-stu-id="9dad5-114">Otherwise you will have created an unintentional singleton class.</span></span>

## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="9dad5-129">Передача сведений об изменении значений привязки из свойств коллекции</span><span class="sxs-lookup"><span data-stu-id="9dad5-129">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="9dad5-130">Свойство коллекции, которое само является свойством зависимостей, не сообщает об изменениях своих подсвойств автоматически.</span><span class="sxs-lookup"><span data-stu-id="9dad5-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="9dad5-131">При создании привязок в коллекции это может мешать привязке сообщать об изменениях и, следовательно, нарушить некоторые сценарии привязки данных.</span><span class="sxs-lookup"><span data-stu-id="9dad5-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="9dad5-132">Тем не менее если использовать тип коллекции <xref:System.Windows.FreezableCollection%601> как типа коллекции, затем подсвойства изменения содержащихся в нем элементов в коллекции будет сообщено, и привязка работает должным образом.</span><span class="sxs-lookup"><span data-stu-id="9dad5-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>  
  
 <span data-ttu-id="9dad5-133">Чтобы включить привязку подсвойства в коллекции объекта зависимостей, создайте свойство коллекции как тип <xref:System.Windows.FreezableCollection%601>, с ограничением типа для этой коллекции любым <xref:System.Windows.DependencyObject> производного класса.</span><span class="sxs-lookup"><span data-stu-id="9dad5-133">To enable subproperty binding in a dependency object collection, create the collection property as type <xref:System.Windows.FreezableCollection%601>, with a type constraint for that collection to any <xref:System.Windows.DependencyObject> derived class.</span></span>
