# Интерфейсы API для поддержки размещения в собственном браузере WPF
Размещение приложений [!INCLUDE[TLA#tla_titlewinclient](../../../../includes/tlasharptla-titlewinclient-md.md)] в веб\-браузерах упрощается с помощью сервера активных документов \(также известного как DocObject\), зарегистрированного вне узла WPF.  [!INCLUDE[TLA2#tla_ie](../../../../includes/tla2sharptla-ie-md.md)] может прямо активироваться и встраиваться в активный документ.  Для размещения приложений XBAP и свободных XAML\-документов в браузерах Mozilla приложение [!INCLUDE[TLA#tla_titlewinclient](../../../../includes/tlasharptla-titlewinclient-md.md)] предоставляет подключаемый модуль NPAPI, который обеспечивает аналогичную среду размещения для сервера активных документов [!INCLUDE[TLA#tla_titlewinclient](../../../../includes/tlasharptla-titlewinclient-md.md)], как это делает [!INCLUDE[TLA2#tla_ie](../../../../includes/tla2sharptla-ie-md.md)].  Однако самым простым практическим способом размещения приложений XBAP и XAML\-документов в других браузерах и автономных приложениях является использование элемента управления веб\-браузера Internet Explorer.  Элемент управления веб\-браузера предоставляет сложную среду размещения сервера активных документов, а также включает собственный узел, чтобы настроить и расширить эту среду и напрямую соединиться с текущим объектом активного документа.  
  
 Сервер активных документов [!INCLUDE[TLA#tla_titlewinclient](../../../../includes/tlasharptla-titlewinclient-md.md)] реализует некоторые общие интерфейсы размещения, включая [IOleObject](http://go.microsoft.com/fwlink/?LinkId=162049), [IOleDocument](http://go.microsoft.com/fwlink/?LinkId=162050), [IOleInPlaceActiveObject](http://go.microsoft.com/fwlink/?LinkId=162051), [IPersistMoniker](http://go.microsoft.com/fwlink/?LinkId=162045), [IOleCommandTarget](http://go.microsoft.com/fwlink/?LinkId=162047).  При размещении в элементе управления веб\-браузера эти интерфейсы могут быть запросами от объектов, возвращенных свойством [IWebBrowser2::Document](http://go.microsoft.com/fwlink/?LinkId=162048).  
  
## IOleCommandTarget  
 Реализация [IOleCommandTarget](http://go.microsoft.com/fwlink/?LinkId=162047) сервером активных документов WPF поддерживает многочисленные команды из группы стандартных OLE\-команд \(включая GUID группы команд со значением null\), связанные с переходами и специфические для браузера.  Кроме этого, она распознает пользовательскую группу команд CGID\_PresentationHost.  В данный момент существует только одна команда, определенная в этой группе.  
  
```  
DEFINE_GUID(CGID_PresentationHost, 0xd0288c55, 0xd6, 0x4f5e, 0xa8, 0x51, 0x79, 0xde, 0xc5, 0x1b, 0x10, 0xec);  
enum PresentationHostCommands {   
   PHCMDID_TABINTO = 1   
};  
```  
  
 PHCMDID\_TABINTO указывает PresentationHost переключить фокус на первый или последний фокусируемый элемент из его содержимого в зависимости от состояния клавиши SHIFT.
