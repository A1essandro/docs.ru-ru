# Загрузка пакета проверки реестра имен поставщиков
В этом разделе обсуждается порядок загрузки и использования VINR в проекте.  
  
## Загрузка пакета проверки реестра имен поставщиков  
 Расширение обработчика токена Интернета VINR доступно как пакет NuGet, который добавляет необходимые сборки и ссылки в проект.  Если NuGet еще не установлен, перейдите н [nuget.org](http://nuget.org) и загрузите его.  Журнал управления версиями для расширения можно просмотреть, посетив его страницу в NuGet с [реестром имен поставщиков в NuGet, утвержденных корпорацией Майкрософт](https://nuget.org/packages/System.IdentityModel.Tokens.ValidatingIssuerNameRegistry/)  
  
#### Загрузка пакета проверки реестра имен поставщиков с помощью графического интерфейса диспетчера пакетов  
  
1.  В **обозревателе решений** Visual Studio щелкните проект правой кнопкой мыши и выберите **Управление пакетами NuGet**.  
  
2.  В поле **Управление пакетами NuGet** щелкните поле поиска и введите `ValidatingIssuerNameRegistry` и нажмите клавишу **ВВОД**.  
  
3.  В области результатов нажмите кнопку **Установить** для первого результата.  
  
4.  Пакет начнет скачиваться.  Прежде чем он будет добавлен в проект, появится диалоговое окно принятия лицензионного соглашения.  Если вы согласны с условиями лицензионного соглашения, нажмите кнопку **Принимаю**.  
  
5.  Последние сборки обработчика веб\-токенов VINR загружены и добавлены в проект.  
  
#### Загрузка пакета проверки реестра имен поставщиков с помощью консоли диспетчера пакетов  
  
1.  В Visual Studio выберите **Сервис**, **Library Package Manager** и **Package Manager Console**.  
  
2.  Откроется окно **Консоль диспетчера пакетов**.  Выберите следующий текст и нажмите клавишу **ВВОД**:  
  
    ```powershell  
    Install-Package System.IdentityModel.Tokens.ValidatingIssuerNameRegistry  
    ```  
  
3.  Последние сборки обработчика веб\-токенов VINR загружены и добавлены в проект.
