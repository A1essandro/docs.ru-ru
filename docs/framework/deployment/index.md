# Развертывание .NET Framework
Этот раздел документации по платформе .NET Framework предоставляет сведения для разработчиков, которые хотят установить платформу .NET Framework вместе со своими приложениями, и администраторов, которым требуется развертывание платформы .NET Framework по сети.  В нем также рассматриваются вопросы активации и перезапуска, связанные с развертыванием, и то, как контролировать ход установки платформы .NET Framework.  
  
## В этом подразделе  
 [Руководство по развертыванию для разработчиков](../../../docs/framework/deployment/deployment-guide-for-developers.md)  
 Описание способов установки платформы .NET Framework с приложениями на компьютеры пользователей.  
  
 [Руководство по развертыванию для администраторов](../../../docs/framework/deployment/guide-for-administrators.md)  
 Описание развертывания администратором платформы .NET Framework и системных зависимостей в сети с помощью System Center Configuration Manager \(SCCM\).  
  
 [Уменьшение числа перезагрузок при установке платформы .NET Framework 4.5](../../../docs/framework/deployment/reducing-system-restarts.md)  
 Описание диспетчера перезапуска, который предотвращает перезагрузки, когда возможно, и объяснение того, как приложения, устанавливающие платформу .NET Framework, могут его использовать.  
  
 [Практическое руководство. Получение хода выполнения установщика .NET Framework 4.5](../../../docs/framework/deployment/how-to-get-progress-from-the-dotnet-installer.md)  
 Описание того, как автоматически запускать и отслеживать процесс установки платформы .NET Framework, и в то же время отображать собственное окно выполнения установки.  
  
 [Ошибки инициализации платформы .NET Framework. Управление взаимодействием с пользователем](../../../docs/framework/deployment/initialization-errors-managing-the-user-experience.md)  
 Объясняет, что происходит, когда приложению .NET Framework требуется версия среды CLR, которая является недопустимой или не установлена на компьютере пользователя, как устранить эти ошибки и как управлять сообщениями об ошибках для пользователя.  
  
 [Практическое руководство. Отладка проблем при активации CLR](../../../docs/framework/deployment/how-to-debug-clr-activation-issues.md)  
 Объясняется, как просматривать и отлаживать журнал активации среды CLR для разрешения проблем, которые могут возникнуть при попытке запустить приложение с правильной версией среды CLR.
