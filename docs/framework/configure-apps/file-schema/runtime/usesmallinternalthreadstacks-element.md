# Элемент &lt;UseSmallInternalThreadStacks&gt;
Предлагает среде CLR уменьшить потребление памяти путем указания явных размеров стека при создании определенных потоков, используемых внутренним образом, вместо использования для этих потоков размера стека по умолчанию.  
  
## Синтаксис  
  
```  
<UseSmallInternalThreadStacks enabled="true|false" />  
```  
  
## Атрибуты и элементы  
 В следующих разделах описаны атрибуты, дочерние и родительские элементы.  
  
### Атрибуты  
  
|Атрибут|Описание|  
|-------------|--------------|  
|enabled|Обязательный атрибут.<br /><br /> Указывает, следует ли запрашивать, чтобы среда CLR использовала явные размеры стека вместо установленных по умолчанию при создании определенных потоков, используемых для внутренних целей.  Явные размеры стека меньше, чем установленный по умолчанию размер в 1 МБ.|  
  
## Атрибут enabled  
  
|Значение|Описание|  
|--------------|--------------|  
|true|Запросите явные размеры стека.|  
|false|Используйте размер стека по умолчанию.  Этот вариант используется в [!INCLUDE[net_v40_long](../../../../../includes/net-v40-long-md.md)] по умолчанию.|  
  
### Дочерние элементы  
 Нет.  
  
### Родительские элементы  
  
|Элемент|Описание|  
|-------------|--------------|  
|`configuration`|Корневой элемент в любом файле конфигурации, используемом средой CLR и приложениями платформы .NET Framework.|  
|`runtime`|Элемент, содержащий сведения о привязке сборок и сборке мусора.|  
  
## Заметки  
 Этот элемент конфигурации используется для запроса на использование ограниченной виртуальной памяти в процессе, так как явные размеры потоков, которые среда CLR использует для своих внутренних потоков, если запрос удовлетворен, меньше размера по умолчанию.  
  
> [!IMPORTANT]
>  Этот элемент конфигурации — запрос среды CLR, а не абсолютное требование.  В [!INCLUDE[net_v40_short](../../../../../includes/net-v40-short-md.md)] запрос выполняется только для архитектуры x86.  Этот элемент может быть полностью игнорирован в будущих версиях среды CLR или же заменен на явные размеры стека, которые всегда используются для выбранных внутренних потоков.  
  
 Указание данного элемента конфигурации меняет надежность для меньшего использования виртуальной памяти, если среда CLR учитывает запрос, поскольку меньшие размеры стека могут потенциально сделать переполнения стека более вероятными.  
  
## Пример  
 В следующем примере показывается, как запросить использование средой CLR явных размеров стека для определенных потоков, используемых для внутренних целей.  
  
```  
<configuration>  
   <runtime>  
      <UseSmallInternalThreadStacks enabled="true" />  
   </runtime>  
</configuration>  
```
