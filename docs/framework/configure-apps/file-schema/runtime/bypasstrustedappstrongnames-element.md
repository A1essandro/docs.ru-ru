# Элемент &lt;bypassTrustedAppStrongNames&gt;
Определяет, должна ли пропускаться проверка строгих имен сборок с полным доверием, которые загружаются в домен <xref:System.AppDomain> с полным доверием.  
  
## Синтаксис  
  
```  
<bypassTrustedAppStrongNames    
   enabled="true|false"/>  
```  
  
## Атрибуты и элементы  
 В следующих разделах описаны атрибуты, дочерние и родительские элементы.  
  
### Атрибуты  
  
|Атрибут|Описание|  
|-------------|--------------|  
|`enabled`|Обязательный атрибут.<br /><br /> Определяет, включена ли функция пропуска, позволяющая избежать проверки строгих имен сборок с полным доверием.  Если эта функция включена, правильность строгих имен при загрузке сборки не проверяется.  Значение по умолчанию — `true`.|  
  
## Атрибут enabled  
  
|Значение|Описание|  
|--------------|--------------|  
|`true`|Подписи строгого имения сборок с полным доверием не проверяются при загрузке сборок в домен <xref:System.AppDomain> с полным доверием.  Это значение по умолчанию.|  
|`false`|Подписи строгого имения сборок с полным доверием проверяются при загрузке сборок в домен <xref:System.AppDomain> с полным доверием.  Проверяется только правильность подписи строгого имени; имя не сравнивается с другим строгим именем.|  
  
### Дочерние элементы  
 Нет.  
  
### Родительские элементы  
  
|Элемент|Описание|  
|-------------|--------------|  
|`configuration`|Корневой элемент в любом файле конфигурации, используемом средой CLR и приложениями платформы .NET Framework.|  
|`runtime`|Элемент, содержащий сведения о привязке сборок и сборке мусора.|  
  
## Заметки  
 Функция пропуска строгих имен позволяет избавиться от дополнительной нагрузки, связанной с проверкой подписей строгого имени для сборок с полным доверием.  
  
 Функция пропуска применяется ко всем сборкам, подписанным строгим именем и обладающим следующими характеристиками.  
  
-   Полное доверие без свидетельства <xref:System.Security.Policy.StrongName> \(например, наличие свидетельства `MyComputer`\).  
  
-   Загрузка в домен <xref:System.AppDomain> с полным доверием.  
  
-   Загрузка из расположения, указанного в свойстве <xref:System.AppDomainSetup.ApplicationBase%2A> такого объекта <xref:System.AppDomain>.  
  
-   Нет отложенной подписи.  
  
> [!NOTE]
>  Если функция пропуска отключена для всех приложений на компьютере с помощью реестра, этот параметр не действует.  Для получения дополнительной информации см. [Практическое руководство. Отключение функции пропуска строгих имен](../../../../../docs/framework/app-domains/how-to-disable-the-strong-name-bypass-feature.md).  
  
## Пример  
 В следующем примере показано, как включить проверку подписей строгого имени сборок с полным доверием.  
  
```  
<configuration>  
   <runtime>  
      <bypassTrustedAppStrongNames enabled="false"/>  
   </runtime>  
</configuration>  
```
