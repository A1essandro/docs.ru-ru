## Заметки  
 Обработка SOAP \- это процесс преобразования сообщения из одной версии в другую.  
  
 Служба маршрутизации [!INCLUDE[indigo1](../../../../../includes/indigo1-md.md)] может преобразовывать сообщения из одного протокола в другой.  Если исходящее и входящее сообщение имеют разные версии, создается новое сообщение с правильной версией.  Преобразование сообщений из одного <xref:System.ServiceModel.Channel.MessageVersion> в другой выполняется путем составления нового сообщения [!INCLUDE[indigo2](../../../../../includes/indigo2-md.md)], содержащего текстовую часть и соответствующие заголовки из входящего сообщения [!INCLUDE[indigo2](../../../../../includes/indigo2-md.md)].  Заголовки, заданные для адресации или распознаваемые на уровне маршрутизатора, не применяются при построении нового сообщения WCF, так как эти заголовки либо имеют другую версию \(в случае заголовков адресации\), либо обрабатываются на этапе связи между клиентом и маршрутизатором.  
  
 Будет ли заголовок добавлен в исходящее сообщение, зависит от того, было ли это сообщение помечено как распознанное при прохождении уровня входящего канала.  Заголовки, которые не удалось распознать \(например, пользовательские заголовки\), не удаляются и проходят через службу маршрутизации \(копируются в исходящее сообщение\).  Текст сообщения копируется в исходящее сообщение.  Затем сообщение отправляется по исходящему каналу. К этому моменту создаются и добавляются все заголовки и другие данные конверта, относящиеся к этому протоколу связи или транспорта.  
  
 Эти шаги обработки выполняются, если определено поведение обработки SOAP.  Данное [\<soapProcessingExtension\>](../../../../../docs/framework/configure-apps/file-schema/wcf/soapprocessing.md) представляет собой поведение конечной точки, применяемое ко всем клиентским \(исходящим\) конечным точкам, когда запускается служба маршрутизации.  По умолчанию [\<маршрутизация\>](../../../../../docs/framework/configure-apps/file-schema/wcf/routing-of-servicebehavior.md) создает и прикрепляет новое поведение [\<soapProcessingExtension\>](../../../../../docs/framework/configure-apps/file-schema/wcf/soapprocessing.md), для которого `processMessages` установлено в значение `true` для каждого клиента конечной точки.  Если используется протокол, который не распознается службой маршрутизации, или требуется переопределить поведение обработки по умолчанию, можно отключить обработку SOAP для всей службы маршрутизации или для отдельных конечных точек. Чтобы отключить обработку SOAP для всей службы на всех конечных точках, задайте атрибуту `soapProcessing` поведения [\<маршрутизация\>](../../../../../docs/framework/configure-apps/file-schema/wcf/routing-of-servicebehavior.md) значение `false`.  Чтобы отключить обработку SOAP для одной конечной точки, используйте это поведение и присвойте его атрибуту `processMessages` значение `false`. Затем прикрепите это поведение к конечной точке, которая должна быть пропущена при запуске кода обработки по умолчанию. Если поведение [\<маршрутизация\>](../../../../../docs/framework/configure-apps/file-schema/wcf/routing-of-servicebehavior.md) установлено для службы маршрутизации, поведение конечной точки применяться не будет \(т. к. оно уже определено\).
