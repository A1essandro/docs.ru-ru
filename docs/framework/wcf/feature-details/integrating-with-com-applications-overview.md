# <a name="integrating-with-com-applications-overview"></a><span data-ttu-id="a0561-102">Общие сведения об интеграции с приложениями COM</span><span class="sxs-lookup"><span data-stu-id="a0561-102">Integrating with COM Applications Overview</span></span>
<span data-ttu-id="a0561-103">Windows Communication Foundation (WCF) предоставляет разработчику управляемого кода среду с широкими возможностями для создания подключаемых приложений.</span><span class="sxs-lookup"><span data-stu-id="a0561-103">Windows Communication Foundation (WCF) provides the managed code developer with a rich environment for creating connected applications.</span></span> <span data-ttu-id="a0561-104">Тем не менее если вы существенно потратились в неуправляемом коде на основе COM и переносить не требуется, можно интегрировать веб-служб WCF непосредственно в существующий код с помощью моникера службы WCF.</span><span class="sxs-lookup"><span data-stu-id="a0561-104">However, if you have a substantial investment in unmanaged COM-based code and do not want to migrate, you can still integrate WCF Web services directly into your existing code by using the WCF service moniker.</span></span> <span data-ttu-id="a0561-105">Моникер служб можно использовать в широком наборе сред разработки на базе модели COM, например в Office VBA, Visual Basic 6.0 и Visual C++ 6.0.</span><span class="sxs-lookup"><span data-stu-id="a0561-105">The service moniker can be used from a wide range of COM-based development environments, such as Office VBA, Visual Basic 6.0, or Visual C++ 6.0.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="a0561-106">Моникер служб использует коммуникационный канал WCF для всех видов связи.</span><span class="sxs-lookup"><span data-stu-id="a0561-106">The service moniker uses a WCF communication channel for all communication.</span></span> <span data-ttu-id="a0561-107">Механизмы безопасности и идентификации для этого канала отличаются от механизмов, используемых в стандартных прокси-средствах COM и DCOM.</span><span class="sxs-lookup"><span data-stu-id="a0561-107">The security and identity mechanisms for that channel differ from those used in standard COM and DCOM proxies.</span></span> <span data-ttu-id="a0561-108">Кроме того поскольку моникер служб использует коммуникационный канал WCF истечения времени ожидания по умолчанию — одна минута для всех вызовов.</span><span class="sxs-lookup"><span data-stu-id="a0561-108">In addition, because the service moniker uses a WCF communication channel the default timeout period is one minute for all calls.</span></span>  
  
 <span data-ttu-id="a0561-109">Моникер служб используется с `GetObject` функции, чтобы разработчик неуправляемого кода с помощью подхода со строгой типизацией, зависящие от COM для вызова веб-служб WCF.</span><span class="sxs-lookup"><span data-stu-id="a0561-109">The service moniker is used with the `GetObject` function to provide the unmanaged developer with a strongly-typed, COM-specific approach for calling WCF Web services.</span></span> <span data-ttu-id="a0561-110">Для этого локального, видимой для COM определение контракта службы WCF Web и привязка, которая будет использоваться.</span><span class="sxs-lookup"><span data-stu-id="a0561-110">This requires a local, COM-visible definition of the WCF Web service contract and the binding that is to be used.</span></span> <span data-ttu-id="a0561-111">Как и другие клиенты WCF моникер служб должен создать типизированный канал к службе, хотя создание этого канала происходит прозрачно для COM-программиста при первом вызове метода.</span><span class="sxs-lookup"><span data-stu-id="a0561-111">Like other WCF clients, the service moniker must construct a typed channel to the service, though this channel construction occurs transparently to the COM programmer on the first method call.</span></span>  
  
 <span data-ttu-id="a0561-112">Общие с другими клиентами WCF, при использовании моникера приложения указать адрес, привязку и контракт для взаимодействия со службой.</span><span class="sxs-lookup"><span data-stu-id="a0561-112">In common with other WCF clients, when using the moniker, applications specify the address, binding and contract to communicate with a service.</span></span> <span data-ttu-id="a0561-113">Контракт можно задать одним из следующих способов.</span><span class="sxs-lookup"><span data-stu-id="a0561-113">The contract can be specified in one of the following ways:</span></span>  
  
-   <span data-ttu-id="a0561-114">Типизированный контракт: контракт регистрируется как видимый для модели COM тип на клиентском компьютере.</span><span class="sxs-lookup"><span data-stu-id="a0561-114">Typed contract – the contract is registered as a COM visible type on the client machine.</span></span>  
  
-   <span data-ttu-id="a0561-115">Контракт WSDL: контракт предоставляется в виде документа WSDL.</span><span class="sxs-lookup"><span data-stu-id="a0561-115">WSDL contract – the contract is supplied in the form of a WSDL document.</span></span>  
  
-   <span data-ttu-id="a0561-116">Контракт MEX: контракт получается в среде выполнения из конечной точки обмена метаданными (MEX).</span><span class="sxs-lookup"><span data-stu-id="a0561-116">MEX contract – the contract is retrieved at runtime from a Metadata Exchange (MEX) endpoint.</span></span>
