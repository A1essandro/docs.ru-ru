## <a name="multiple-base-addresses"></a><span data-ttu-id="0db78-109">Несколько базовых адресов</span><span class="sxs-lookup"><span data-stu-id="0db78-109">Multiple Base Addresses</span></span>  
 <span data-ttu-id="0db78-110">Эта функция доступна только для служб WCF, размещенных в службах IIS.</span><span class="sxs-lookup"><span data-stu-id="0db78-110">This feature is only available to WCF services that are hosted under IIS.</span></span> <span data-ttu-id="0db78-111">Эта возможность отключена по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="0db78-111">This feature is not enabled by default.</span></span> <span data-ttu-id="0db78-112">Чтобы его включить, то необходимо добавить `multipleSiteBindingsEnabled` атрибут <`serviceHostingEnvironment`> элемент в файле Web.config файл и присвойте ему значение `true`, как показано в следующем примере.</span><span class="sxs-lookup"><span data-stu-id="0db78-112">To enable it you must add the `multipleSiteBindingsEnabled` attribute to the <`serviceHostingEnvironment`> element in your Web.config file and set it to `true`, as shown in the following example.</span></span>  
  
```xml

```  
  
 <span data-ttu-id="0db78-113">При размещении службы WCF в IIS, службы IIS создают один базовый адрес в зависимости от URI к виртуальному каталогу, содержащему приложение.</span><span class="sxs-lookup"><span data-stu-id="0db78-113">When hosting a WCF service under IIS, IIS creates one base address for you based on the URI to the virtual directory that contains the application.</span></span> <span data-ttu-id="0db78-114">Чтобы добавить одну или несколько привязок, можно с помощью диспетчера служб IIS добавить на веб-сайт дополнительные базовые адреса, использующие тот же протокол.</span><span class="sxs-lookup"><span data-stu-id="0db78-114">You can add additional base addresses that use the same protocol by using Internet Information Services Manager to add one or more bindings to your Web site.</span></span> <span data-ttu-id="0db78-115">Для каждой привязки укажите протокол (HTTP или HTTPS), IP-адрес, порт и имя узла.</span><span class="sxs-lookup"><span data-stu-id="0db78-115">For each binding specify a protocol (HTTP or HTTPS), an IP address, a port, and a host name.</span></span> <span data-ttu-id="0db78-116">Дополнительные сведения об использовании диспетчера служб IIS см. в разделе [диспетчера служб IIS (IIS 7)](https://go.microsoft.com/fwlink/?LinkId=164057).</span><span class="sxs-lookup"><span data-stu-id="0db78-116">For more information about using Internet Information Services Manager, see [IIS Manager (IIS 7)](https://go.microsoft.com/fwlink/?LinkId=164057).</span></span> <span data-ttu-id="0db78-117">Дополнительные сведения о добавлении привязок к сайту, см. в разделе [Создание веб-сайта (IIS 7)](https://go.microsoft.com/fwlink/?LinkId=164060)</span><span class="sxs-lookup"><span data-stu-id="0db78-117">For more information about adding bindings to a site, see [Create a Web Site (IIS 7)](https://go.microsoft.com/fwlink/?LinkId=164060)</span></span>  
  
 <span data-ttu-id="0db78-118">Указание нескольких базовых адресов для этого же сайта влияет на содержимое страницы справки WCF, Импорт схемы и сведения WSDL/MEX, сформированные службой.</span><span class="sxs-lookup"><span data-stu-id="0db78-118">Specifying multiple base addresses for the same site affects the content of the WCF Help page, importing schema, and the WSDL/MEX information generated by the service.</span></span> <span data-ttu-id="0db78-119">На странице справки WCF отображаются командной строки, используемый для создания клиента WCF, который может взаимодействовать со службой.</span><span class="sxs-lookup"><span data-stu-id="0db78-119">The WCF Help page displays the command line to use to generate a WCF client that can communicate with the service.</span></span> <span data-ttu-id="0db78-120">Эта командная строка содержит только первый адрес, указанный в привязке служб IIS для веб-сайта.</span><span class="sxs-lookup"><span data-stu-id="0db78-120">This command line contains only the first address specified in the IIS binding for the Web site.</span></span> <span data-ttu-id="0db78-121">Аналогичным образом при импорте схемы используется только первый базовый адрес, указанный в привязке служб IIS.</span><span class="sxs-lookup"><span data-stu-id="0db78-121">Similarly when importing schema, only the first base address specified in the IIS binding is used.</span></span> <span data-ttu-id="0db78-122">Данные WSDL и MEX содержит все базовые адреса, указанные в привязках служб IIS.</span><span class="sxs-lookup"><span data-stu-id="0db78-122">WSDL and MEX data contain all the base addresses specified in the IIS bindings.</span></span>
