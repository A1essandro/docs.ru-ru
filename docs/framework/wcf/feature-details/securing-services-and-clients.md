# Защита служб и клиентов
Этот раздел посвящен безопасности программирования в [!INCLUDE[indigo1](../../../../includes/indigo1-md.md)].  Обычно обеспечение безопасности включает выбор подходящей предоставляемой системой привязки, задание свойств элемента безопасности и задание свойств поведений службы, управляющих извлечением учетных данных для использования службой или клиентом.  Эти подходы охватывают требования к безопасности большинства пользователей и сценариев, как показано в разделе [Типовые сценарии безопасности](../../../../docs/framework/wcf/feature-details/common-security-scenarios.md).  Если для сценария требуются более широкие возможности, ознакомьтесь с разделом [Возможности безопасности при использовании пользовательских привязок](../../../../docs/framework/wcf/feature-details/security-capabilities-with-custom-bindings.md); если решение достаточно сложное, ознакомьтесь с разделом [Расширение системы безопасности](../../../../docs/framework/wcf/extending/extending-security.md).  Если создается система, использующая сложные утверждения \(или осуществляется взаимодействие с такой системой\), изучите разделы [Авторизация](../../../../docs/framework/wcf/feature-details/authorization-in-wcf.md).
