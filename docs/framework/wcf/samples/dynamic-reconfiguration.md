# Динамическая реконфигурация
В этом образце показана служба маршрутизации [!INCLUDE[indigo1](../../../../includes/indigo1-md.md)].Служба маршрутизации — это компонент [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)], который упрощает включение маршрутизатора на основе содержимого в приложение.В этом образце стандартный образец [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)] Calculator применяется для связи с использованием службы маршрутизацииЭтот образец показывает, как можно динамически переконфигурировать службу маршрутизации во время выполнения.  
  
> [!IMPORTANT]
>  Образцы уже могут быть установлены на компьютере.Перед продолжением проверьте следующий каталог \(по умолчанию\).  
>   
>  `<диск_установки>:\WF_WCF_Samples`  
>   
>  Если этот каталог не существует, перейдите на страницу [Образцы Windows Communication Foundation \(WCF\) и Windows Workflow Foundation \(WF\) для .NET Framework 4](http://go.microsoft.com/fwlink/?LinkId=150780), чтобы загрузить все образцы [!INCLUDE[indigo1](../../../../includes/indigo1-md.md)] и [!INCLUDE[wf1](../../../../includes/wf1-md.md)].Этот образец расположен в следующем каталоге.  
>   
>  `<диск_установки>:\WF_WCF_Samples\WCF\Basic\RoutingServices\DynamicReconfiguration`  
  
## Подробные сведения об образце  
 Для того, чтобы динамически переконфигурировать службу маршрутизации во время выполнения, данный образец запускает таймер, который каждые пять секунд создает и применяет новый объект <xref:System.ServiceModel.Routing.RoutingConfiguration>.Данная конфигурация ссылается либо на стандартную конечную точку калькулятора, либо на конечную точку округляющего калькулятора.Клиентское приложение калькулятора получает сообщения от одной службы или от другой, в зависимости от того, как настроена в данный момент служба маршрутизации.  
  
 При этом используется возможность динамической переконфигурации службы маршрутизации при помощи пользовательского поведения.Пользовательское поведение присоединяет расширение службы, которое содержит простой потоковой таймер, срабатывающий каждые пять секунд, в результате чего производится обращение к методу обратного вызова `UpdateRules`.Этот обратный вызов создает и применяет новую конфигурацию маршрутизации.В настоящем развертывании этот обратный вызов, скорее всего, будет выполнен в результате другого типа события, например уведомления о событии SQL или объявления об обнаружении WS.  
  
#### Использование этого образца  
  
1.  Откройте в среде [!INCLUDE[vs_current_long](../../../../includes/vs-current-long-md.md)] файл DynamicReconfiguration.sln.  
  
2.  Чтобы открыть окно **Обозреватель решений**, выберите пункт **Обозреватель решений** в меню **Вид**.  
  
3.  Нажмите клавишу **F5** или сочетание клавиш **CTRL\+SHIFT\+B** в среде [!INCLUDE[vs_current_short](../../../../includes/vs-current-short-md.md)].  
  
    1.  Если вы хотите, чтобы при нажатии клавиши **F5** автоматически загружались все необходимые проекты, щелкните правой кнопкой мыши решение и выберите пункт **Свойства**.Выберите узел **Запускаемый проект** в разделе **Общие свойства** в левой области.Выберите переключатель **Несколько запускаемых проектов** и задайте для всех проектов действие **Запустить**.  
  
    2.  Если проект строится с помощью клавиш **CTRL\+SHIFT\+B**, необходимо запустить следующие приложения.  
  
        1.  Клиентская часть калькулятора \(.\/CalculatorClient\/bin\/client.exe\)  
  
        2.  Служба \-калькулятора \(.\/CalculatorService\/bin\/service.exe\)  
  
        3.  Служба калькулятора с маршрутизацией \(.\/RoundingCalcService\/bin\/service.exe\)  
  
        4.  RoutingService \(.\/RoutingService\/bin\/RoutingService.exe\)  
  
4.  В консольном окне клиентской части калькулятора нажмите клавишу ВВОД для запуска клиента и вызова операций службы «Калькулятор».  
  
     Служба маршрутизации передает сообщения поочередно округляющему калькулятору и обычному калькулятору по мере того, как конфигурация маршрутизации динамически меняется каждые пять секунд.Вывод в консольном окне клиента различается в зависимости от того, какой из конечных точек служба маршрутизации сконфигурирована отправлять сообщения.  
  
5.  Последовательно нажимайте клавишу ВВОД в течение более чем пяти секунд и понаблюдайте за изменением результатов, получаемых от службы.  
  
    1.  Далее приведен вывод, получаемый, когда служба маршрутизации настроена для передачи сообщении службе округляющего калькулятора.  
  
        ```Output  
        Add(100,15.99) = 116  
        Subtract(145,76.54) = 68.5  
        Multiply(9,81.25) = 731.2  
        Divide(22,7) = 3.1  
  
        ```  
  
    2.  Далее приведен вывод, получаемый, когда служба маршрутизации настроена для передачи сообщений обычной службе калькулятора.  
  
        ```Output  
        Add(100,15.99) = 115.99  
        Subtract(145,76.54) = 68.46  
        Multiply(9,81.25) = 731.25  
        Divide(22,7) = 3.14285714285714  
  
        ```  
  
6.  Служба калькулятора и служба округляющего калькулятора также печатают журнал вызванных операций в соответствующих консольных окнах.  
  
7.  В консольном окне клиента введите команду «quit» и нажмите клавишу ВВОД для выхода.  
  
8.  Нажмите клавишу ВВОД в окнах консоли служб, чтобы завершить службы.  
  
## Сценарий  
 В этом образце показан маршрутизатор, работающий на основе содержимого и предоставляющий доступ к нескольким типам реализации служб через одну конечную точку.  
  
### Реальный сценарий  
 Компания Contoso планирует провести виртуализацию всех служб, чтобы сделать общедоступной только одну конечную точку, через которую возможен доступ к нескольким различным типам служб.В этом случае применяются функции маршрутизации на основе содержимого, чтобы определить назначение для отправки входящих запросов.
