# <a name="xaml-namespaces-for-net-framework-xaml-services"></a>Пространства имен языка XAML для служб .NET Framework XAML
Пространства имен XAML — это понятие, расширяющее определение пространства имен XML. Как и пространство имен XML, можно определить пространства имен XAML с помощью `xmlns` атрибута в разметке. Пространства имен XAML также представлены в потоке узлов XAML и других API служб XAML. В этом разделе определяет понятие пространства имен XAML, а также описывается, как можно определить пространства имен языка XAML и используются контекстами схемы XAML и других аспектов служб XAML .NET Framework.  
  
## <a name="xml-namespace-and-xaml-namespace"></a>Пространство имен XML и пространством имен XAML  
 Пространство имен XAML — специализированное пространство имен XML, так же, как XAML — это специализированная форма XML и использует его разметки базовая форма XML. В разметке объявления пространства имен XAML и его сопоставление через `xmlns` атрибута, применяемого к элементу. `xmlns` Объявление может быть сделано на один элемент, объявленный в пространстве имен XAML. Объявление пространства имен XAML, внесенные в элемент является допустимым для этого элемента, все атрибуты этого элемента и всех дочерних элементов данного элемента. Атрибуты можно использовать пространства имен языка XAML, не совпадает с элементом, атрибутом, при условии, что само имя атрибута ссылается на префикс в составе имени атрибута в разметке.  
  
 Различие пространством имен XAML и пространством имен XML, что пространство имен XML может использоваться для ссылки на схему, или может использоваться просто для различения сущностей. Для XAML типы и члены, используемые в языке XAML в конечном счете должны разрешаться в резервных типах и концепции схемы XML не применяются также к этой возможности. Пространство имен XAML содержит сведения, что контекст схемы XAML необходимо иметь доступ для выполнения такого сопоставления типов.  
  
## <a name="xaml-namespace-components"></a>Компоненты пространства имен XAML  
 Определение пространства имен XAML состоит из двух компонентов: префикса и идентификатора. Каждый из этих компонентов отсутствуют, при объявлении в разметке пространства имен XAML, или определенного в системе типов XAML.  
  
 Префикс может быть любая строка с разрешения [W3C для пространств имен в спецификации XML 1.0](http://go.microsoft.com/fwlink/?LinkID=161735) . По соглашению обычно очень короткие строки, имеют префиксы, поскольку в типичном файле разметки префикс повторяется многократно. Некоторые пространства имен языка XAML, которые предназначены для использования в нескольких реализациях XAML используются определенные традиционные префиксы. Например, пространства имен XAML языка XAML обычно сопоставляется с помощью префикса `x`. Можно определить пространство имен XAML по умолчанию, где префикс не указан в определении, но представляется как пустая строка, если определена или запросить платформы.NET Framework API служб XAML. Как правило, чтобы максимизировать объем опустить префикс пространства имен XAML по умолчанию намеренно выбирают разметки, реализующей XAML технологией, сценариев и словари.  
  
 Идентификатор может быть любая строка с разрешения [W3C для пространств имен в спецификации XML 1.0](http://go.microsoft.com/fwlink/?LinkID=161735). По соглашению идентификаторы для пространств имен XML или пространства имен языка XAML часто встречающиеся в форме URI обычно в виде абсолютного URI с квалификатором протокола. Часто как часть строки пути подразумевается сведения о версии, который определяет конкретный словарь XAML. Пространства имен языка XAML добавьте соглашение о дополнительный идентификатор за соглашение с использованием XML URI. Для пространства имен языка XAML идентификатор передает информацию, необходим контекст схемы XAML для разрешения типов, указанных в качестве элементов в списке этого пространства имен XAML, или для разрешения атрибутов в члены.  
  
 В целях передачи информации контекст схемы XAML идентификатор пространства имен XAML может быть записан в виде URI. Однако в этом случае URI также объявляется как соответствующий идентификатор в определенной сборке или списке сборок. Это можно сделать в сборках, присвоение атрибутов сборки с <xref:System.Windows.Markup.XmlnsDefinitionAttribute>. Этот метод идентификации пространства имен XAML и поддержка поведения разрешения типов на основе среды CLR в сборки с соответствующими атрибутами поддерживается контекст схемы XAML по умолчанию в службах XAML .NET Framework. Как правило это соглашение можно использовать для случаев, где контекст схемы XAML включает в себя среду CLR или зависит от контекста схемы XAML по умолчанию, что необходимо для считывания атрибутов среды CLR из сборок среды CLR.  
  
 Пространства имен языка XAML также могут идентифицироваться по соглашению, которое взаимодействует имен среды CLR и сборки, определяющей тип. Это обозначение используется в случаях, где нет <xref:System.Windows.Markup.XmlnsDefinitionAttribute> в сборок, содержащих типы существует однозначного соответствия. Это соглашение потенциально более сложными, чем соглашение с использованием URI, а также не исключает неоднозначности и дублирования, поскольку существует несколько способов создания ссылок на сборки.  
  
 Самая простая форма идентификатора, который использует соглашение о пространства имен и сборки CLR выглядит следующим образом:  
  
 `clr-namespace:`*clrnsName* `; assembly=` *assemblyShortName*  
  
 `clr-namespace:`и `; assembly=` — литеральные компоненты синтаксиса.  
  
 *clrnsName* строковое имя, которое определяет пространство имен среды CLR. Это имя строки включает символы внутренняя точка (.), определять подсказки, пространство имен среды CLR и его отношений с другими пространствами имен среды CLR.  
  
 *assemblyShortName* — это строковое имя сборки, определяющий типы, которые используются в языке XAML. Типы, которые должны осуществляться через объявленного пространства имен XAML ожидаются определен в сборке и специально объявляться в пространстве имен среды CLR, определяемый *clrnsName*. Это строковое имя обычно соответствует информации, сообщаемые <xref:System.Reflection.AssemblyName.Name%2A?displayProperty=nameWithType>.  
  
 Более полное определение соглашения о пространства имен и сборки CLR выглядит следующим образом:  
  
 `clr-namespace:`*clrnsName* `; assembly=` *assemblyName*  
  
 *assemblyName* представляет любую строку, допустимую в качестве <xref:System.Reflection.Assembly.Load%28System.String%29?displayProperty=nameWithType> ввода. Эта строка может включать язык и региональные параметры, открытый ключ или сведения о версии (определенные в справочном разделе для определения этих понятий <xref:System.Reflection.Assembly>). COFF формат и свидетельства (как использовать другие перегрузки <xref:System.Reflection.Assembly.Load%2A>) не являются значимыми для целей; загрузки сборок XAML вся информация для загрузки должна быть представлена в виде строки.  
  
 Указание открытый ключ для сборки является полезным приемом для обеспечения безопасности XAML или для устранения возможных неоднозначности, если сборки загружаются по простому имени или уже существуют в кэше или домене приложения. Дополнительные сведения см. в разделе [соображения безопасности XAML](../../../docs/framework/xaml-services/xaml-security-considerations.md).  
  
## <a name="xaml-namespace-declarations-in-the-xaml-services-api"></a>Объявления пространства имен XAML в API служб XAML  
 В API-Интерфейсе служб XAML объявление пространства имен XAML представляется <xref:System.Xaml.NamespaceDeclaration> объекта. Если объявляется пространство имен XAML в коде, вызовите <xref:System.Xaml.NamespaceDeclaration.%23ctor%28System.String%2CSystem.String%29> конструктор. `ns` И `prefix` параметры задаются в виде строк, а входные данные, предоставляемые для этих параметров соответствует определению идентификатор пространства имен XAML и префикс пространства имен XAML, как указано ранее в этом разделе.  
  
 Если сведения о пространстве имен XAML в рамках потока узлов XAML или с помощью другого доступа к системе типов XAML, просматриваемую <xref:System.Xaml.NamespaceDeclaration.Namespace%2A?displayProperty=nameWithType> сообщает идентификатор пространства имен XAML и <xref:System.Xaml.NamespaceDeclaration.Prefix%2A?displayProperty=nameWithType> сообщает префикс пространства имен XAML.  
  
 В потоке узлов XAML сведения о пространстве имен XAML может отображаться как узел XAML, предшествующего сущности, к которому он применяется. Сюда входят случаи, где сведения о пространстве имен XAML предшествует `StartObject` корневого элемента XAML. Для получения дополнительной информации см. [Understanding XAML Node Stream Structures and Concepts](../../../docs/framework/xaml-services/understanding-xaml-node-stream-structures-and-concepts.md).  
  
 Для многих сценариев, использующих API служб XAML .NET Framework по крайней мере одно объявление пространства имен XAML должен существовать, и объявление должно содержать и ссылки на сведения, которые требуются для контекста схемы XAML. Пространства имен языка XAML необходимо указать сборки для загрузки или помочь в устранении определенных типов в пространствах имен и сборок, которые уже загружены или известны контекст схемы XAML.  
  
 Чтобы создать поток узлов XAML, сведения о типе XAML должна быть доступна через контекст схемы XAML. Не удается определить сведения о типе XAML без предварительного определения соответствующего пространства имен XAML для каждого создаваемого узла. На этом этапе экземпляры типов еще не созданы, но контекст схемы XAML может потребоваться поиск сведений из определяющей сборки и резервного типа. Например, для обработки разметки `<Party><PartyFavor/></Party>`, контекста схемы XAML должен иметь возможность определить имя и тип `ContentProperty` из `Party`и поэтому также необходимо знать сведения о пространстве имен XAML для `Party` и `PartyFavor`. В случае контекст схемы XAML по умолчанию статическое отражение сообщает значительную часть информации о системе типов XAML, необходимую для формирования узлов типов XAML в потоке узлов.  
  
 Для формирования графа объектов из потока узлов XAML, для каждого префикса XAML, используемые в исходной разметке и записывается в поток узлов XAML должен существовать объявления пространства имен XAML. На этом этапе создаются экземпляры, а значение true, сопоставление типов происходит.  
  
 Если вам нужно заполнить сведения о пространстве имен XAML, в случаях, где требуется контекст схемы XAML для использования пространства имен XAML не определен в разметке, можно использовать один из способов — объявить объявления пространств имен XML в <xref:System.Xml.XmlParserContext> для <xref:System.Xml.XmlReader>. Используйте его <xref:System.Xml.XmlReader> как входные параметры для конструктора средства чтения XAML, или <xref:System.Xaml.XamlServices.Load%28System.Xml.XmlReader%29?displayProperty=nameWithType>.  
  
 Две другие API, которые важны для обработки в службах XAML .NET Framework пространства имен XAML — это атрибуты <xref:System.Windows.Markup.XmlnsDefinitionAttribute> и <xref:System.Windows.Markup.XmlnsPrefixAttribute>. Эти атрибуты применяются к сборкам. <xref:System.Windows.Markup.XmlnsDefinitionAttribute>используется контекст схемы XAML для интерпретации любого объявления пространства имен XAML, включающего в себя URI. <xref:System.Windows.Markup.XmlnsPrefixAttribute>используется средствами, выдающими XAML, чтобы определенного пространства имен XAML, которые можно сериализовать с предсказуемым префиксом. Дополнительные сведения см. в разделе [XAML-Related атрибуты среды CLR для пользовательских типов и библиотек](../../../docs/framework/xaml-services/xaml-related-clr-attributes-for-custom-types-and-libraries.md).
