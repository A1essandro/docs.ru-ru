# <a name="types-of-bitmaps"></a>Типы точечных рисунков
Битовая карта представляет собой массив битов, задающих цвет каждого пикселя в прямоугольном массиве точек. Количество битов, задающих цвет одного пикселя определяет количество цветов, которые можно назначить для этой точки. Например, если каждая точка, представленная 4 битам, затем произвольной точки можно задать один из 16 различных цветов (2 ^ 4 = 16). Ниже приведены несколько примеров число цветов, которые можно назначить для точки, представляемого заданное число бит.  
  
|Бит на пиксель|Число цветов, которые могут быть назначены пиксель|  
|--------------------|------------------------------------------------------|  
|1|2^1 = 2|  
|2|2^2 = 4|  
|4|2^4 = 16|  
|8|2^8 = 256|  
|16|2^16 = 65,536|  
|24|2^24 = 16,777,216|  
  
 Файлы, которые хранения точечных рисунков, обычно включают один или несколько блоков данных, хранящих сведения, такие как число бит на пиксель, количество точек в каждой строке и количество строк в массиве. Этот файл также может содержать таблицу цветов (иногда называется цветовой палитры). Таблица цветов сопоставляет числа в битовой карте цветов. На следующем рисунке увеличенное изображение вместе с его растрового изображения и цвет таблицы. Каждая точка представленного 4-разрядное число, поэтому существуют 2 ^ 4 = 16 цветов в таблице цветов. Каждый цвет в таблице имеет номер 24-разрядный: 8 бит для красного, 8 бит для зеленого и 8 бит для синего. Числа отображаются в форме шестнадцатеричным (основание 16): A = 10, B = 11, C = 12, D = 13, E = 14, F = 15.  
  
 ![Пример растрового изображения](../../../../docs/framework/winforms/advanced/media/aboutgdip03-art01.gif "AboutGdip03_Art01")  
  
 Посмотрите на пиксель в строке 3, столбец 5 изображения. Соответствующий номер в битовой карте — 1. Таблицы цветов число 1 соответствует красный цвет, пикселя отображается красным цветом. Все элементы в верхней строке растрового изображения равны 3. Согласно таблице цветов число 3 соответствует синему, поэтому все пиксели верхней строки изображения синие.  
  
> [!NOTE]
>  Некоторые рисунки хранятся в формате снизу вверх. числа в первой строке растрового изображения соответствуют пиксели в нижней строке изображения.  
  
 Битовая карта, содержат ссылки на таблицу цветов называется индексированными палитрой. Некоторые рисунки имеют не требуется таблица цветов. Например если растровое изображение использует 24 бита на пиксель, это растровое изображение можно хранить цвета самостоятельно, а не индексы в таблице цветов. Ниже показан точечного рисунка, который хранит непосредственно цветов (24 бита на пиксель), вместо того чтобы использовать таблицу цветов. На рисунке также увеличенное изображение. В битовой карте FFFFFF обозначает белый цвет, FF0000 обозначает красный, число 00FF00 обозначает зеленый и синий представляет 0000FF.

## <a name="graphics-file-formats"></a>Графические форматы файлов  
 Существует множество стандартных форматов сохранения растровых изображений в файлы на диске. [!INCLUDE[ndptecgdiplus](../../../../includes/ndptecgdiplus-md.md)]поддерживает форматы, описанные в следующих абзацах графических файлов.  
  
### <a name="bmp"></a>BMP  
 BMP — это стандартный формат, используемый Windows для хранения изображений, независимо от устройства или приложения. Количество битов на пиксель (1, 4, 8, 15, 24, 32 или 64), в данном файле BMP указывается в заголовке файла. Наиболее часто используются файлы BMP с 24 бита на пиксель. BMP-файлы обычно не сжимаются и, следовательно, не подходят для передачи через Интернет.  
  
### <a name="graphics-interchange-format-gif"></a>Формат GIF (Graphics Interchange Format)  
 GIF-распространенный формат для изображения, отображаемые на веб-страницах. GIF-файлы подходят для рисунков, изображения с блоками одного цвета и рисунков с четкой границей между цветами. GIF-файлы будут сжаты, но не данные теряются в процессе сжатия; распакованный изображение абсолютно совпадает с исходным. Можно назначить один цвет в формате GIF как прозрачный, чтобы изображение будет иметь цвет фона веб-страницах, отображает его. Последовательность изображений GIF могут храниться в одном файле для формирования анимированного GIF-файла. GIF могут содержать не более 8 бит на пиксель, поэтому они не более 256 цветов.  
  
### <a name="joint-photographic-experts-group-jpeg"></a>Формат JPEG (JPEG)  
 JPEG является схемой сжатия, который хорошо подходит для естественных образов, таких как отсканированные фотографии. Некоторые данные теряются в процессе сжатия, но обычно потеря остается незаметной излучения. JPEG содержат 24 бита на пиксель, поэтому они способны отображение более чем 16 миллионов цветов. Файлы JPEG не поддерживают прозрачность или анимацию.  
  
 Степень сжатия JPEG-изображения можно настраивать, но большая степень сжатия (меньшие файлы) привести к росту потерь данных. Коэффициент сжатия 20:1 часто создает образ, который излучения находит трудно отличить от оригинала. На следующем рисунке изображение в формате BMP и два изображения JPEG, которые были сжаты из изображения в формате BMP. Первый JPEG имеет коэффициент сжатия 4:1, а второй ПОЛУЧЕН с использованием коэффициента сжатия 8:1.  
  
 ![Примеры идентификатора типа файла](../../../../docs/framework/winforms/advanced/media/aboutgdip03-art03.gif "AboutGdip03_Art03")  
  
 Сжатие JPEG не подходит для рисунков, блоки сплошным цветом и четкие границы. На следующем рисунке BMP, два изображения JPEG и GIF. BMP были сжатия JPEG или GIF. Коэффициент сжатия — 4:1 для GIF, 4:1 для меньшего изображения JPEG и 8:3 для большего JPEG. Обратите внимание, что GIF сохраняет четкие границы вдоль линий, но JPEG, как правило, эти границы размываются.  
  
 ![Типы файлов](../../../../docs/framework/winforms/advanced/media/aboutgdip03-art03a.gif "AboutGdip03_Art03A")  
  
 JPEG является схемой сжатия, а не в формат файла. Формат обмена файл JPEG (JFIF) имеет формат файла, обычно используется для хранения и передачи изображений, которые сжаты согласно схеме JPEG. Файлы JFIF, отображаемые веб-браузеры используют расширение .jpg.  
  
### <a name="exchangeable-image-file-exif"></a>Файлы файл ФОРМАТА Exif)  
 EXIF — это формат файла, используемый для хранения фотографий, полученных с помощью цифровой камеры. Файл EXIF содержит изображение, сжимается в соответствии со спецификацией JPEG. Файл EXIF также содержит данные о фотографии (дата съемки, скорость затвора, время экспонирования и т. д.) и информацию о камеры (изготовителя, модели и т. д.).  
  
### <a name="portable-network-graphics-png"></a>Формат PNG (Portable Network Graphics)  
 Формат PNG сохраняет многие преимущества формата GIF, но также предоставляет возможности, недоступные в формате GIF. Как и файлы GIF PNG-файлы сжимаются без потери данных. PNG-файлы могут содержать цвета 8, 24 или 48 бит на пиксель и оттенках серого с 1, 2, 4, 8 или 16 бит на пиксель. Напротив GIF-файлы можно использовать только 1, 2, 4 или 8 бит на пиксель. PNG-файл также можно хранить альфа-значение для каждой точки, который определяет степень слияния цвета точки с цветом фона.  
  
 Улучшает PNG сможет постепенно отображения изображения GIF (то есть, для отображения более и более подробное приближение изображения в качестве его поступает через сетевое подключение). PNG-файлы могут содержать данные о гамма коррекции и о цветовой коррекции, что позволяет точно изображение на различных устройствах отображения.  
  
### <a name="tag-image-file-format-tiff"></a>Тег Image File Format (TIFF)  
 TIFF — это гибкий и расширяемый формат, поддерживаемый разнообразных платформах и приложений для обработки изображений. TIFF-файлы можно хранить изображения с произвольным числом битов на пиксель и использовать различные алгоритмы сжатия. Несколько образов могут храниться в одном файле TIFF нескольких страниц. Сведения, связанные с изображением (модель сканера, главный компьютер, тип сжатия, ориентация, образцы на одну точку и т. д) можно в файле и упорядочены с помощью тегов. Формат TIFF может быть расширен, при необходимости, утверждения и новых тегов.
