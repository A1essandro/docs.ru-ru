## <a name="keys-and-algorithms"></a><span data-ttu-id="e707d-110">Ключи и алгоритмы</span><span class="sxs-lookup"><span data-stu-id="e707d-110">Keys and Algorithms</span></span>  
 <span data-ttu-id="e707d-111">SQL Server поддерживает несколько алгоритмов шифрования симметричным ключом, включая DES, Triple DES, RC2, RC4, 128-разрядный RC4, DESX, 128-разрядный AES, 192-разрядный AES и 256-разрядный AES.</span><span class="sxs-lookup"><span data-stu-id="e707d-111">SQL Server supports several symmetric key encryption algorithms, including DES, Triple DES, RC2, RC4, 128-bit RC4, DESX, 128-bit AES, 192-bit AES, and 256-bit AES.</span></span> <span data-ttu-id="e707d-112">Алгоритмы реализуются с помощью API-интерфейса Windows Crypto.</span><span class="sxs-lookup"><span data-stu-id="e707d-112">The algorithms are implemented using the Windows Crypto API.</span></span>  
  
 <span data-ttu-id="e707d-113">В пределах соединения с базой данных SQL Server может поддерживать несколько открытых симметричных ключей.</span><span class="sxs-lookup"><span data-stu-id="e707d-113">Within the scope of a database connection, SQL Server can maintain multiple open symmetric keys.</span></span> <span data-ttu-id="e707d-114">Открытый ключ получается из хранилища и доступен для расшифровки данных.</span><span class="sxs-lookup"><span data-stu-id="e707d-114">An open key is retrieved from the store and is available for decrypting data.</span></span> <span data-ttu-id="e707d-115">Не нужно указывать, какой симметричный ключ будет использоваться для расшифровки фрагмента данных.</span><span class="sxs-lookup"><span data-stu-id="e707d-115">When a piece of data is decrypted, there is no need to specify the symmetric key to use.</span></span> <span data-ttu-id="e707d-116">Каждое зашифрованное значение содержит идентификатор ключа (идентификатор GUID ключа), использованного для его шифрования.</span><span class="sxs-lookup"><span data-stu-id="e707d-116">Each encrypted value contains the key identifier (key GUID) of the key used to encrypt it.</span></span> <span data-ttu-id="e707d-117">Если расшифрован и открыт верный ключ, то ядро сопоставляет зашифрованный поток байтов с открытым симметричным ключом.</span><span class="sxs-lookup"><span data-stu-id="e707d-117">The engine matches the encrypted byte stream to an open symmetric key, if the correct key has been decrypted and is open.</span></span> <span data-ttu-id="e707d-118">Этот ключ затем используется для выполнения расшифровки и возвращения данных.</span><span class="sxs-lookup"><span data-stu-id="e707d-118">This key is then used to perform decryption and return the data.</span></span> <span data-ttu-id="e707d-119">Если верный ключ не открыт, возвращается значение NULL.</span><span class="sxs-lookup"><span data-stu-id="e707d-119">If the correct key is not open, NULL is returned.</span></span>  
  
 <span data-ttu-id="e707d-120">Пример, демонстрирующий способы работы с зашифрованными данными в базе данных см. в разделе [как: шифрование столбца данных](http://go.microsoft.com/fwlink/?LinkID=128559) в электронной документации по SQL Server.</span><span class="sxs-lookup"><span data-stu-id="e707d-120">For an example that shows how to work with encrypted data in a database, see [How to: Encrypt a Column of Data](http://go.microsoft.com/fwlink/?LinkID=128559) in SQL Server Books Online.</span></span>
