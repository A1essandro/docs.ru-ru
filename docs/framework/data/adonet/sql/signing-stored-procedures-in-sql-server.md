# <a name="signing-stored-procedures-in-sql-server"></a><span data-ttu-id="f8cbb-102">Подписывание хранимых процедур в SQL Server</span><span class="sxs-lookup"><span data-stu-id="f8cbb-102">Signing Stored Procedures in SQL Server</span></span>
 <span data-ttu-id="f8cbb-103">Цифровая сигнатура представляет собой хэш-код данных, зашифрованную при помощи закрытого ключа лица, ставящего свою сигнатуру.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-103">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="f8cbb-104">Закрытый ключ обеспечивает уникальность цифровой сигнатуры его владельцу.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-104">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="f8cbb-105">Можно подписывать хранимые процедуры, функции (за исключением встроенных функций возвращающих табличные значения), триггеры и сборки.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-105">You can sign stored procedures, functions (except for inline table-valued functions), triggers, and assemblies.</span></span>  
  
 <span data-ttu-id="f8cbb-106">Хранимую процедуру можно подписать сертификатом или асимметричным ключом.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-106">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="f8cbb-107">Эта возможность предназначена для сценариев, в которых разрешения не могут наследоваться по цепочке владения или если цепочка владения разорвана, например в динамическом SQL.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-107">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="f8cbb-108">Затем можно создать пользователя, сопоставленного с сертификатом, предоставление сертификата пользователя разрешений на объекты, которые требуется доступ к хранимой процедуре.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-108">You can then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  

 <span data-ttu-id="f8cbb-109">Можно также создать имя входа, сопоставленное один и тот же сертификат и затем предоставить этому имени входа, все необходимые разрешения на уровне сервера или добавьте имя входа для одного или нескольких предопределенных ролей сервера.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-109">You can also create a login mapped to the same certificate, and then grant any necessary server-level permissions to that login, or add the login to one or more of the fixed server roles.</span></span> <span data-ttu-id="f8cbb-110">Это делается во избежание включения поддержки `TRUSTWORTHY` параметр для сценариев, в которых требуется разрешений более высокого уровня базы данных.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-110">This is designed to avoid enabling the `TRUSTWORTHY` database setting for scenarios in which higher level permissions are needed.</span></span>  
  
 <span data-ttu-id="f8cbb-111">При выполнении хранимой процедуры SQL Server объединяет разрешения пользователя сертификата и/или имя входа с разрешениями вызывающего объекта.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-111">When the stored procedure is executed, SQL Server combines the permissions of the certificate user and/or login with those of the caller.</span></span> <span data-ttu-id="f8cbb-112">В отличие от `EXECUTE AS` предложения, не изменяет контекст выполнения процедуры.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-112">Unlike the `EXECUTE AS` clause, it does not change the execution context of the procedure.</span></span> <span data-ttu-id="f8cbb-113">Встроенные функции, которые возвращают имя входа и имена пользователей, возвращают имя вызывающей стороны, а не имя пользователя сертификата.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-113">Built-in functions that return login and user names return the name of the caller, not the certificate user name.</span></span>  
  
## <a name="creating-certificates"></a><span data-ttu-id="f8cbb-114">Создание сертификатов</span><span class="sxs-lookup"><span data-stu-id="f8cbb-114">Creating Certificates</span></span>  
 <span data-ttu-id="f8cbb-115">При подписании хранимой процедуры с помощью сертификата или асимметричного ключа Сводка данных, состоящая из зашифрованного хэш-кода хранимой процедуры, а также execute-имени пользователя, создается с помощью закрытого ключа.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-115">When you sign a stored procedure with a certificate or asymmetric key, a data digest consisting of the encrypted hash of the stored procedure code, along with the execute-as user, is created using the private key.</span></span> <span data-ttu-id="f8cbb-116">Во время выполнения сводка данных расшифровывается открытым ключом и сравнивается с хэш-кодом хранимой процедуры.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-116">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="f8cbb-117">Изменение execute-как пользователь делает недействительным хэш-значения, чтобы больше не соответствует цифровой подписи.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-117">Changing the execute-as user invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="f8cbb-118">Измените хранимую процедуру удаляет подпись полностью, что не позволяет тем, кто не имеет доступа к закрытому ключу изменение кода хранимой процедуры.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-118">Modifying the stored procedure drops the signature entirely, which prevents someone who does not have access to the private key from changing the stored procedure code.</span></span> <span data-ttu-id="f8cbb-119">В любом случае следует подписывать процедуру каждый раз при изменении кода или execute-имени пользователя.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-119">In either case, you must re-sign the procedure each time you change the code or the execute-as user.</span></span>  
  
 <span data-ttu-id="f8cbb-120">Подписи модуля участвуют два действия:</span><span class="sxs-lookup"><span data-stu-id="f8cbb-120">There are two required steps involved in signing a module:</span></span>  
  
1.  <span data-ttu-id="f8cbb-121">Создание сертификата с использованием инструкции Transact-SQL `CREATE CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-121">Create a certificate using the Transact-SQL `CREATE CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="f8cbb-122">Эта инструкция имеет несколько параметров для настройки даты начала и даты окончания, а также пароля.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-122">This statement has several options for setting a start and end date and a password.</span></span> <span data-ttu-id="f8cbb-123">Дата окончания срока действия по умолчанию — один год.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-123">The default expiration date is one year.</span></span>  
  
1.  <span data-ttu-id="f8cbb-124">Подписание процедуры при помощи сертификата с использованием инструкции Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-124">Sign the procedure with the certificate using the Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` statement.</span></span>  

<span data-ttu-id="f8cbb-125">После подписан модуль, одного или нескольких участников должна быть создана для хранения дополнительных разрешений, которые должны быть связаны с сертификатом.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-125">Once the module has been signed, one or more principals needs to be created in order to hold the additional permissions that should be associated with the certificate.</span></span>  

<span data-ttu-id="f8cbb-126">Если модуль нужны дополнительные разрешения уровня базы данных:</span><span class="sxs-lookup"><span data-stu-id="f8cbb-126">If the module needs additional database-level permissions:</span></span>  
  
1.  <span data-ttu-id="f8cbb-127">Создание пользователя базы данных, связанного с этим сертификатом, с использованием инструкции Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-127">Create a database user associated with that certificate using the Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="f8cbb-128">Этот пользователь существует только в базе данных и не связан с именем входа, если имя входа также должна быть создана из того же сертификата.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-128">This user exists in the database only and is not associated with a login unless a login has also been created from that same certificate.</span></span>  
  
1.  <span data-ttu-id="f8cbb-129">Предоставление пользователю сертификата необходимых разрешений уровня базы данных.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-129">Grant the certificate user the required database-level permissions.</span></span>  
  
<span data-ttu-id="f8cbb-130">Если модуль должен дополнительные разрешения на уровне сервера:</span><span class="sxs-lookup"><span data-stu-id="f8cbb-130">If the module needs additional server-level permissions:</span></span>  
  
1.  <span data-ttu-id="f8cbb-131">Скопируйте сертификат `master` базы данных.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-131">Copy the certificate to the `master` database.</span></span>  
 
1.  <span data-ttu-id="f8cbb-132">Создайте имя входа, связанный с этим сертификатом, с помощью Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` инструкции.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-132">Create a login associated with that certificate using the Transact-SQL `CREATE LOGIN [userName] FROM CERTIFICATE [certificateName]` statement.</span></span>  
  
1.  <span data-ttu-id="f8cbb-133">Предоставьте требуемые разрешения уровня сервера имя входа сертификата.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-133">Grant the certificate login the required server-level permissions.</span></span>  
  
> [!NOTE]  
>  <span data-ttu-id="f8cbb-134">Сертификат не может предоставлять разрешения пользователю, у которого разрешения были отменены инструкцией DENY.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-134">A certificate cannot grant permissions to a user that has had permissions revoked using the DENY statement.</span></span> <span data-ttu-id="f8cbb-135">DENY имеет преимущество над GRANT, предотвращая тем самым наследование вызывающей стороной разрешений, предоставленных пользователю сертификата.</span><span class="sxs-lookup"><span data-stu-id="f8cbb-135">DENY always takes precedence over GRANT, preventing the caller from inheriting permissions granted to the certificate user.</span></span>
