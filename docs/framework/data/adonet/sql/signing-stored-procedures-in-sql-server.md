<span data-ttu-id="1b386-103">Хранимую процедуру можно подписать сертификатом или асимметричным ключом.</span><span class="sxs-lookup"><span data-stu-id="1b386-103">You can sign a stored procedure with a certificate or an asymmetric key.</span></span> <span data-ttu-id="1b386-104">Эта возможность предназначена для сценариев, в которых разрешения не могут наследоваться по цепочке владения или если цепочка владения разорвана, например в динамическом SQL.</span><span class="sxs-lookup"><span data-stu-id="1b386-104">This is designed for scenarios when permissions cannot be inherited through ownership chaining or when the ownership chain is broken, such as dynamic SQL.</span></span> <span data-ttu-id="1b386-105">Можно создать пользователя, сопоставленного с сертификатом, предоставляя разрешения пользователя сертификата на объекты, доступ к которым нужен хранимым процедурам.</span><span class="sxs-lookup"><span data-stu-id="1b386-105">You then create a user mapped to the certificate, granting the certificate user permissions on the objects the stored procedure needs to access.</span></span>  
  
 <span data-ttu-id="1b386-106">При выполнении хранимой процедуры SQL Server объединяет разрешения пользователя сертификата с разрешениями вызывающей стороны.</span><span class="sxs-lookup"><span data-stu-id="1b386-106">When the stored procedure is executed, SQL Server combines the permissions of the certificate user with those of the caller.</span></span> <span data-ttu-id="1b386-107">В отличие от предложения EXECUTE AS, он не меняет контекст выполнения процедуры.</span><span class="sxs-lookup"><span data-stu-id="1b386-107">Unlike the EXECUTE AS clause, it does not change the execution context of the procedure.</span></span> <span data-ttu-id="1b386-108">Встроенные функции, которые возвращают имя входа и имена пользователей, возвращают имя вызывающей стороны, а не имя пользователя сертификата.</span><span class="sxs-lookup"><span data-stu-id="1b386-108">Built-in functions that return login and user names return the name of the caller, not the certificate user name.</span></span>  
  
 <span data-ttu-id="1b386-109">Цифровая сигнатура представляет собой сводку данных, зашифрованную при помощи закрытого ключа лица, ставящего свою сигнатуру.</span><span class="sxs-lookup"><span data-stu-id="1b386-109">A digital signature is a data digest encrypted with the private key of the signer.</span></span> <span data-ttu-id="1b386-110">Закрытый ключ обеспечивает уникальность цифровой сигнатуры его владельцу.</span><span class="sxs-lookup"><span data-stu-id="1b386-110">The private key ensures that the digital signature is unique to its bearer or owner.</span></span> <span data-ttu-id="1b386-111">Можно подписывать хранимые процедуры, функции или триггеры.</span><span class="sxs-lookup"><span data-stu-id="1b386-111">You can sign stored procedures, functions, or triggers.</span></span>  
  
> [!NOTE]

## <a name="creating-certificates"></a><span data-ttu-id="1b386-113">Создание сертификатов</span><span class="sxs-lookup"><span data-stu-id="1b386-113">Creating Certificates</span></span>  
 <span data-ttu-id="1b386-114">При подписании хранимой процедуры сертификатом создается с использованием закрытого ключа сводка данных, состоящая из зашифрованного хэш-кода хранимой процедуры.</span><span class="sxs-lookup"><span data-stu-id="1b386-114">When you sign a stored procedure with a certificate, a data digest consisting of the encrypted hash of the stored procedure code is created using the private key.</span></span> <span data-ttu-id="1b386-115">Во время выполнения сводка данных расшифровывается открытым ключом и сравнивается с хэш-значением хранимой процедуры.</span><span class="sxs-lookup"><span data-stu-id="1b386-115">At run time, the data digest is decrypted with the public key and compared with the hash value of the stored procedure.</span></span> <span data-ttu-id="1b386-116">Изменение хранимой процедуры приводит к недействительности хэш-значения, так что цифровая сигнатура больше не совпадает со сводкой данных.</span><span class="sxs-lookup"><span data-stu-id="1b386-116">Modifying the stored procedure invalidates the hash value so that the digital signature no longer matches.</span></span> <span data-ttu-id="1b386-117">Это предотвращает изменение кода хранимой процедуры лицом, не имеющим доступа к закрытому ключу.</span><span class="sxs-lookup"><span data-stu-id="1b386-117">This prevents someone who does not have access to the private key from changing the stored procedure code.</span></span> <span data-ttu-id="1b386-118">Поэтому следует подписывать процедуру при каждом ее изменении.</span><span class="sxs-lookup"><span data-stu-id="1b386-118">Therefore you must re-sign the procedure each time you modify it.</span></span>  
  
 <span data-ttu-id="1b386-119">Подписание модуля выполняется в четыре шага.</span><span class="sxs-lookup"><span data-stu-id="1b386-119">There are four steps involved in signing a module:</span></span>

2.  <span data-ttu-id="1b386-123">Создание пользователя базы данных, связанного с этим сертификатом, с использованием инструкции Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="1b386-123">Create a database user associated with that certificate using the Transact-SQL `CREATE USER [userName] FROM CERTIFICATE [certificateName]` statement.</span></span> <span data-ttu-id="1b386-124">Этот пользователь существует только в базе данных и не связан с именем входа.</span><span class="sxs-lookup"><span data-stu-id="1b386-124">This user exists in the database only and is not associated with a login.</span></span>  
  
3.  <span data-ttu-id="1b386-125">Предоставление пользователю сертификата необходимых разрешений на объекты базы данных.</span><span class="sxs-lookup"><span data-stu-id="1b386-125">Grant the certificate user the required permissions on the database objects.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="1b386-126">Сертификат не может предоставлять разрешения пользователю, у которого разрешения были отменены инструкцией DENY.</span><span class="sxs-lookup"><span data-stu-id="1b386-126">A certificate cannot grant permissions to a user that has had permissions revoked using the DENY statement.</span></span> <span data-ttu-id="1b386-127">DENY имеет преимущество над GRANT, предотвращая тем самым наследование вызывающей стороной разрешений, предоставленных пользователю сертификата.</span><span class="sxs-lookup"><span data-stu-id="1b386-127">DENY always takes precedence over GRANT, preventing the caller from inheriting permissions granted to the certificate user.</span></span>  
  
1.  <span data-ttu-id="1b386-128">Подписание процедуры при помощи сертификата с использованием инструкции Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]`.</span><span class="sxs-lookup"><span data-stu-id="1b386-128">Sign the procedure with the certificate using the Transact-SQL `ADD SIGNATURE TO [procedureName] BY CERTIFICATE [certificateName]` statement.</span></span>
