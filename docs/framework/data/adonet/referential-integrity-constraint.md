# <a name="referential-integrity-constraint"></a><span data-ttu-id="aac88-102">ограничение ссылочной целостности</span><span class="sxs-lookup"><span data-stu-id="aac88-102">referential integrity constraint</span></span>
<span data-ttu-id="aac88-103">Объект *ограничение ссылочной целостности* в модель данных сущности (EDM) похоже на ограничение ссылочной целостности в реляционной базе данных.</span><span class="sxs-lookup"><span data-stu-id="aac88-103">A *referential integrity constraint* in the Entity Data Model (EDM) is similar to a referential integrity constraint in a relational database.</span></span> <span data-ttu-id="aac88-104">Таким же образом, столбец (или столбцы) из таблицы базы данных может ссылаться первичный ключ другой таблицы [свойство](../../../../docs/framework/data/adonet/property.md) (или свойства) из [тип сущности](../../../../docs/framework/data/adonet/entity-type.md) могут ссылаться на [ключ сущности ](../../../../docs/framework/data/adonet/entity-key.md) другого типа сущности.</span><span class="sxs-lookup"><span data-stu-id="aac88-104">In the same way that a column (or columns) from a database table can reference the primary key of another table, a [property](../../../../docs/framework/data/adonet/property.md) (or properties) of an [entity type](../../../../docs/framework/data/adonet/entity-type.md) can reference the [entity key](../../../../docs/framework/data/adonet/entity-key.md) of another entity type.</span></span> <span data-ttu-id="aac88-105">Тип сущности, на который имеется ссылка называется *основной элемент* ограничения.</span><span class="sxs-lookup"><span data-stu-id="aac88-105">The entity type that is referenced is called the *principal end* of the constraint.</span></span> <span data-ttu-id="aac88-106">Тип сущности, который ссылается на основной элемент называется *зависимом* ограничения.</span><span class="sxs-lookup"><span data-stu-id="aac88-106">The entity type that references the principal end is called the *dependent end* of the constraint.</span></span>  
  
 <span data-ttu-id="aac88-107">Ограничение ссылочной целостности определяется как часть [ассоциации](../../../../docs/framework/data/adonet/association-type.md) между двумя типами сущностей.</span><span class="sxs-lookup"><span data-stu-id="aac88-107">A referential integrity constraint is defined as part of an [association](../../../../docs/framework/data/adonet/association-type.md) between two entity types.</span></span> <span data-ttu-id="aac88-108">Определение ограничения ссылочной целостности включает следующую информацию.</span><span class="sxs-lookup"><span data-stu-id="aac88-108">The definition for a referential integrity constraint specifies the following information:</span></span>  
  
-   <span data-ttu-id="aac88-109">Основной конец ограничения.</span><span class="sxs-lookup"><span data-stu-id="aac88-109">The principal end of the constraint.</span></span> <span data-ttu-id="aac88-110">(Тип сущности, ключ сущности которого имеет ссылку на зависимый конец.)</span><span class="sxs-lookup"><span data-stu-id="aac88-110">(An entity type whose entity key is referenced by the dependent end.)</span></span>  
  
-   <span data-ttu-id="aac88-111">Ключ сущности основного конца.</span><span class="sxs-lookup"><span data-stu-id="aac88-111">The entity key of the principal end.</span></span>  
  
-   <span data-ttu-id="aac88-112">Зависимый конец ограничения.</span><span class="sxs-lookup"><span data-stu-id="aac88-112">The dependent end of the constraint.</span></span> <span data-ttu-id="aac88-113">(Тип сущности, который имеет свойство или свойства, ссылающиеся на ключ сущности основного конца.)</span><span class="sxs-lookup"><span data-stu-id="aac88-113">(An entity type that has a property or properties that reference the entity key of the principal end.)</span></span>  
  
-   <span data-ttu-id="aac88-114">Ссылка на свойство или свойства зависимого конца.</span><span class="sxs-lookup"><span data-stu-id="aac88-114">The referencing property or properties of the dependent end.</span></span>  
  
 <span data-ttu-id="aac88-115">Целью ограничения ссылочной целостности в модели EDM является обеспечение постоянного существования допустимых ассоциаций.</span><span class="sxs-lookup"><span data-stu-id="aac88-115">The purpose of referential integrity constraints in the EDM is to ensure that valid associations always exist.</span></span> <span data-ttu-id="aac88-116">Дополнительные сведения см. в разделе [свойство внешнего ключа](../../../../docs/framework/data/adonet/foreign-key-property.md).</span><span class="sxs-lookup"><span data-stu-id="aac88-116">For more information, see [foreign key property](../../../../docs/framework/data/adonet/foreign-key-property.md).</span></span>
