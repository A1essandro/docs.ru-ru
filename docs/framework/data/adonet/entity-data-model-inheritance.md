data-stu-id="dc925-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="dc925-105">Как и с классами в объектно ориентированных языках, в концептуальной модели можно определить тип сущности ( *производный тип*), наследуемый от другого типа сущности ( *базовый тип*).</span><span class="sxs-lookup"><span data-stu-id="dc925-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the data-ttu-id="dc925-107">В производном типе нельзя отменить унаследованные свойства.</span><span class="sxs-lookup"><span data-stu-id="dc925-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="dc925-108">В концептуальной модели можно построить иерархию наследования, в которой один производный тип наследует свойства другого производного типа.</span><span class="sxs-lookup"><span data-stu-id="dc925-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="dc925-109">Тип в верхней части иерархии (одного типа в иерархии, который не является производным типом) называется *корневой тип*.</span><span class="sxs-lookup"><span data-stu-id="dc925-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="dc925-110">В иерархии наследования [ключ сущности](../../../../docs/framework/data/adonet/entity-key.md) должен быть определен в корневом типе.</span><span class="sxs-lookup"><span data-stu-id="dc925-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="dc925-111">Нельзя построить иерархии наследования, в которых один производный тип наследует свойства более чем одного типа.</span><span class="sxs-lookup"><span data-stu-id="dc925-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="dc925-112">Например, в концептуальной модели с типом сущности `Book` можно определить производные типы `FictionBook` и `NonFictionBook`, из которых каждый наследует свойства `Book`.</span><span class="sxs-lookup"><span data-stu-id="dc925-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="dc925-113">Тем не менее, в дальнейшем нельзя определить тип, который бы наследовал свойства обоих типов `FictionBook` и `NonFictionBook`.</span><span class="sxs-lookup"><span data-stu-id="dc925-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
